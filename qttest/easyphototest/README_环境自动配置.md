# EasyPhoto 环境自动配置说明

## 更新内容

项目已更新为**自动环境配置**模式，不再需要手动运行 `setup_embedded_python.py`。

## 工作原理

1. **程序启动时自动检测**
   - 检查应用程序目录下是否存在 `python` 文件夹
   - 检查 Python 是否可正常运行
   - 检查 rembg 模块是否已安装

2. **自动下载和安装**
   - 如果 Python 不存在，自动下载 Python 3.11.9 嵌入式版本（约 30MB）
   - 自动配置 pip
   - 自动安装 rembg 及其依赖

3. **进度显示**
   - 显示友好的图形界面
   - 实时显示下载和安装进度
   - 提供详细的日志信息

## 目录结构

```
EasyPhoto.exe              # 应用程序
python/                    # 自动下载的 Python 环境（首次运行后生成）
  ├── python.exe
  ├── python311.dll
  ├── Lib/
  └── Scripts/
      ├── pip.exe
      └── rembg.exe
```

## 使用步骤

### 方式一：直接运行（推荐）

1. 直接运行 `EasyPhoto.exe`
2. 等待环境自动配置完成（首次运行需要 3-10 分钟，取决于网络速度）
3. 配置完成后自动进入主界面

### 方式二：离线部署

如果目标机器无网络，可以预先准备：

1. 在有网络的机器上运行一次程序，让其自动下载配置
2. 将整个应用目录（包含 `python` 文件夹）复制到目标机器
3. 目标机器可直接使用，无需再次下载

## 特性

- ✅ **零依赖**：无需用户预装 Python
- ✅ **自动化**：首次运行自动配置所有环境
- ✅ **友好界面**：图形化进度显示
- ✅ **容错处理**：支持重试和跳过
- ✅ **便携性**：可整体复制到其他机器使用

## 环境配置详情

### 下载的组件

1. **Python 3.11.9 嵌入式版本**
   - 大小：约 30MB
   - 来源：https://www.python.org/ftp/python/3.11.9/python-3.11.9-embed-amd64.zip

2. **pip 包管理器**
   - 来源：https://bootstrap.pypa.io/get-pip.py

3. **rembg 及依赖**
   - 包含：rembg[cpu,cli]
   - 大小：约 150MB
   - 依赖：onnxruntime, Pillow, numpy, click 等

### 总大小

- Python 环境：~30MB
- rembg 及依赖：~150MB
- **总计：~180MB**

## 网络要求

首次运行需要网络连接以下载：
- Python 官方服务器（python.org）
- PyPI 包服务器（pypi.org）

## 故障排除

### 下载失败

如果下载失败，可能原因：
1. 网络连接问题 → 检查网络连接，点击"重试"
2. 防火墙拦截 → 临时关闭防火墙或添加白名单
3. 服务器暂时不可用 → 稍后重试

### 安装失败

如果安装失败：
1. 删除 `python` 文件夹
2. 重新运行程序
3. 如果多次失败，可以手动安装 Python 到系统，程序会自动检测使用

### 跳过环境配置

如果选择"跳过"：
- 程序会尝试使用系统已安装的 Python
- 如果系统没有 Python 或 rembg，功能将无法使用
- 不推荐跳过，建议完成环境配置

```bash
EasyPhoto.exe

```

## 技术实现

### 核心类

1. **EnvironmentManager**
   - 负责环境检测和配置
   - 管理下载和安装流程
   - 提供进度反馈

2. **SetupWindow**
   - 启动配置界面
   - 显示进度和日志
   - 处理用户交互

3. **RembgProcessor**
   - 已更新为使用本地 Python
   - 路径优先级：应用目录 > 系统环境 > 常见路径
