# 坐标系统与变换

## 概述

Qt 的 2D 绘图系统涉及**三层坐标系统**的映射关系。理解它们之间的转换是精确控制绑定输出、实现缩放/旋转/平移等效果的基础。

```
逻辑坐标（Logical）         窗口坐标（Window）         设备坐标（Device/Physical）
     │                          │                          │
     │  QPainter 变换矩阵        │  Window-Viewport 映射     │
     │  (translate/rotate/       │  (setWindow/setViewport)  │
     │   scale/shear)            │                          │
     ▼                          ▼                          ▼
你在代码中使用的坐标  ──────►  绘图设备的逻辑像素  ──────►  屏幕上的物理像素
```

日常绑定中最核心的是 **QPainter 的世界变换（World Transform）**——通过 `translate()`、`rotate()`、`scale()`、`shear()` 和 `QTransform` 矩阵来操控。

## 1. Qt 的默认坐标系

### 1.1 坐标轴方向

Qt 使用**左上角原点、Y 轴向下**的屏幕坐标系：

```
(0,0) ──────────────────► X 轴（向右为正）
  │
  │
  │
  │
  │
  ▼
Y 轴（向下为正）
```

### 1.2 像素坐标与逻辑坐标

```
逻辑坐标 (2,1)
    │
    ▼
┌───┬───┬───┬───┬───┐
│   │   │   │   │   │ 0
├───┼───┼───┼───┼───┤
│   │   │ █ │   │   │ 1  ← 像素 (2,1) 的中心在逻辑坐标 (2.5, 1.5)
├───┼───┼───┼───┼───┤
│   │   │   │   │   │ 2
└───┴───┴───┴───┴───┘
  0   1   2   3   4
```

> **重要细节**：Qt 的整数坐标指向像素的**左上角**而非中心。例如 `drawPoint(2, 1)` 绘制的像素中心实际位于 $(2.5, 1.5)$。这在使用反锯齿绑定时需要注意——用浮点坐标 $(2.5, 1.5)$ 可以获得更精准的绑定结果。

### 1.3 抗锯齿时的半像素偏移

```cpp
QPainter painter(this);
painter.setRenderHint(QPainter::Antialiasing);

// ❌ 整数坐标：线条可能模糊（落在像素边界上，被分散到两个像素）
painter.drawLine(10, 10, 10, 100);

// ✅ 半像素偏移：线条清晰（落在像素中心）
painter.drawLine(QPointF(10.5, 10.5), QPointF(10.5, 100.5));

// ✅ 对于 1 像素宽的线条，常见技巧：
painter.translate(0.5, 0.5); // 全局偏移半像素
painter.drawLine(10, 10, 10, 100); // 现在坐标自动对齐像素中心
```

图示：

```
无偏移（模糊）        半像素偏移（清晰）
  ░█░                    █
  ░█░                    █
  ░█░                    █
线条落在两个像素之间     线条精确落在像素中心
```

## 2. 四种基本变换

所有变换都通过修改 `QPainter` 的**世界变换矩阵（World Transform）**来实现。

### 2.1 平移（Translate）

将原点移动到新位置：

```cpp
QPainter painter(this);

// 在 (0,0) 绘制——左上角
painter.drawRect(0, 0, 50, 30);

// 平移原点到 (100, 80)
painter.translate(100, 80);

// 同样 drawRect(0,0)，但实际在屏幕 (100,80) 处绘制
painter.drawRect(0, 0, 50, 30);
```

**最常见用途**：以图形中心为原点绘制，简化旋转/缩放计算。

```cpp
// 在 (200, 150) 处绘制一个居中的矩形，无需手动计算偏移
painter.translate(200, 150);
painter.drawRect(-50, -30, 100, 60); // 以原点为中心的矩形
```

### 2.2 旋转（Rotate）

以当前原点为中心，顺时针旋转指定**角度**（度）：

```cpp
QPainter painter(this);
painter.setRenderHint(QPainter::Antialiasing);

// 先平移到旋转中心
painter.translate(200, 150);

// 旋转 30°（顺时针）
painter.rotate(30);

// 绘制（以旋转后的原点为中心）
painter.drawRect(-60, -20, 120, 40);
```

**围绕任意点旋转的模板**：

```cpp
void rotateAround(QPainter &painter, const QPointF &center, qreal angle) {
    painter.translate(center);   // 1. 把原点移到旋转中心
    painter.rotate(angle);        // 2. 旋转
    painter.translate(-center);   // 3. 把原点移回去
}
```

### 2.3 缩放（Scale）

沿 X/Y 轴按比例放大或缩小：

```cpp
QPainter painter(this);
painter.translate(100, 100);

// 放大 2 倍
painter.scale(2.0, 2.0);
painter.drawRect(0, 0, 50, 30);
// 实际在屏幕上绘制 100x60 的矩形

// 水平镜像
painter.scale(-1, 1);
// 垂直镜像
painter.scale(1, -1);

// 不同比例缩放（拉伸）
painter.scale(2.0, 0.5);
```

### 2.4 斜切（Shear）

沿 X/Y 方向产生倾斜效果：

```cpp
QPainter painter(this);
painter.translate(150, 100);

// 水平斜切（X 方向偏移随 Y 增长）
painter.shear(0.5, 0);
painter.drawRect(0, 0, 100, 60);
```

```
原始                  shear(0.5, 0)          shear(0, 0.5)
┌──────┐              ┌──────╱                ┌──────┐
│      │         ──► ╱      ╱           ──►  ╱      │
│      │            ╱──────╱                ╱──────╱
└──────┘                                   └──────╱
```

### 2.5 综合：变换顺序的影响

**变换的代码顺序非常重要**——后写的变换先作用于绘图坐标。

```cpp
// 方案 A：先平移后旋转
painter.translate(200, 150);
painter.rotate(45);
painter.drawRect(-30, -30, 60, 60);
// 效果：图形在 (200,150) 处就地旋转 ✅

// 方案 B：先旋转后平移
painter.rotate(45);
painter.translate(200, 150);
painter.drawRect(-30, -30, 60, 60);
// 效果：先旋转坐标系，再沿旋转后的方向平移 → 图形跑到了预期外的位置
```

图解：

```
方案 A（常用）：                     方案 B：
1. translate 移动原点到目标位置       1. rotate 旋转坐标轴
2. rotate 在目标位置就地旋转          2. translate 沿旋转后的轴移动
                                        → 位置偏移方向也跟着旋转了

      ◇ ← 在(200,150)就地旋转            ◇ ← 位置不在预期位置
     ╱ ╲
    ╱   ╲
```

## 3. QTransform —— 变换矩阵

### 3.1 概念

`QTransform` 封装了一个 3×3 仿射变换矩阵，`QPainter` 的所有变换操作本质上都是对这个矩阵的修改。

$$
\begin{bmatrix} x' \\ y' \\ 1 \end{bmatrix} = \begin{bmatrix} m_{11} & m_{21} & dx \\ m_{12} & m_{22} & dy \\ m_{13} & m_{23} & m_{33} \end{bmatrix} \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}
$$

| 矩阵元素 | 对应变换 |
|---|---|
| $dx, dy$ | 平移 |
| $m_{11}, m_{22}$ | 缩放 |
| $m_{12}, m_{21}$ | 旋转 / 斜切 |
| $m_{13}, m_{23}$ | 透视（仿射变换时为 0） |

### 3.2 基本用法

```cpp
QTransform transform;
transform.translate(200, 150);
transform.rotate(30);
transform.scale(1.5, 1.5);

QPainter painter(this);
painter.setTransform(transform);
painter.drawRect(-40, -25, 80, 50);
```

### 3.3 与 QPainter 便捷方法的等价关系

```cpp
// QPainter 便捷方法            等价的 QTransform 操作
painter.translate(dx, dy);  //  transform.translate(dx, dy);
painter.rotate(angle);      //  transform.rotate(angle);
painter.scale(sx, sy);      //  transform.scale(sx, sy);
painter.shear(sh, sv);      //  transform.shear(sh, sv);
painter.setTransform(t);    //  直接设置矩阵
painter.resetTransform();   //  恢复为单位矩阵
```

### 3.4 矩阵的组合与顺序

```cpp
QTransform t1, t2;
t1.translate(100, 0);
t2.rotate(45);

// 方式 A：t1 * t2（先旋转后平移）
QTransform combined = t1 * t2;

// 方式 B：t2 * t1（先平移后旋转）
QTransform combined2 = t2 * t1;

// 这两种组合结果不同！
```

> **矩阵乘法顺序**：`A * B` 表示先应用 B 再应用 A（从右到左）。这与代码中先调用 A 后调用 B 的效果一致。

### 3.5 逆变换（坐标转换）

将设备坐标（如鼠标位置）转换回逻辑坐标：

```cpp
QTransform transform;
transform.translate(200, 150);
transform.rotate(30);
transform.scale(2, 2);

// 正变换：逻辑 → 设备
QPointF devicePos = transform.map(QPointF(10, 20));

// 逆变换：设备 → 逻辑
QTransform inverted = transform.inverted();
QPointF logicalPos = inverted.map(QPointF(350, 280));

// 在鼠标事件中常用
void MyWidget::mousePressEvent(QMouseEvent *event) {
    QPointF scenePos = m_transform.inverted().map(QPointF(event->pos()));
    // scenePos 是变换前的逻辑坐标
}
```

### 3.6 map 系列方法

```cpp
QTransform t;
t.rotate(45);

// 变换点
QPointF pt = t.map(QPointF(100, 0));

// 变换矩形（返回多边形，因为旋转后矩形变成平行四边形）
QPolygonF poly = t.map(QRectF(0, 0, 100, 50));

// 变换路径
QPainterPath path;
path.addRect(0, 0, 100, 50);
QPainterPath transformed = t.map(path);

// 变换区域
QRegion region(QRect(0, 0, 100, 50));
QRegion transformedRegion = t.map(region);

// 只变换矩形（返回包围盒 QRectF，非精确）
QRectF boundingRect = t.mapRect(QRectF(0, 0, 100, 50));
```

### 3.7 变换类型判断

```cpp
QTransform t;
t.rotate(45);

switch (t.type()) {
case QTransform::TxNone:       qDebug() << "无变换（单位矩阵）"; break;
case QTransform::TxTranslate:  qDebug() << "仅平移"; break;
case QTransform::TxScale:      qDebug() << "包含缩放"; break;
case QTransform::TxRotate:     qDebug() << "包含旋转"; break;
case QTransform::TxShear:      qDebug() << "包含斜切"; break;
case QTransform::TxProject:    qDebug() << "包含透视"; break;
}

// 是否可逆（非退化矩阵）
bool invertible = t.isInvertible();

// 行列式
qreal det = t.determinant();
```

## 4. Window-Viewport 映射

### 4.1 概念

`QPainter` 的 Window-Viewport 系统提供了一层**独立于世界变换的坐标映射**，用于将逻辑坐标空间（Window）映射到设备坐标空间（Viewport）。

```
Window（逻辑窗口）                    Viewport（设备视口）
你定义的坐标范围                       设备上的实际像素范围

(-100,-100)  (100,-100)                (0,0)    (w,0)
    ┌────────────┐                       ┌────────────┐
    │            │                       │            │
    │    (0,0)   │  ────映射────►         │            │
    │            │                       │            │
    └────────────┘                       └────────────┘
(-100,100)   (100,100)                  (0,h)    (w,h)
```

### 4.2 基本用法

```cpp
QPainter painter(this);

// 定义逻辑坐标范围：x ∈ [-100,100], y ∈ [-100,100]
painter.setWindow(-100, -100, 200, 200);

// 定义设备输出范围（默认就是 widget 的 rect）
painter.setViewport(0, 0, width(), height());

// 现在可以用 (-100,-100) 到 (100,100) 的坐标绘制
painter.drawLine(-100, 0, 100, 0);  // 水平中线
painter.drawLine(0, -100, 0, 100);  // 垂直中线
painter.drawEllipse(-50, -50, 100, 100); // 居中圆
```

### 4.3 保持宽高比

```cpp
void MyWidget::paintEvent(QPaintEvent *) {
    QPainter painter(this);

    // 逻辑空间始终是 1000x1000
    int logicalSize = 1000;
    painter.setWindow(0, 0, logicalSize, logicalSize);

    // 视口取正方形（保持宽高比）
    int side = qMin(width(), height());
    int xOff = (width() - side) / 2;
    int yOff = (height() - side) / 2;
    painter.setViewport(xOff, yOff, side, side);

    // 在 0~1000 的逻辑坐标中绘制
    painter.drawRect(100, 100, 800, 800);
    painter.drawEllipse(200, 200, 600, 600);
    // 无论窗口怎么缩放，图形比例始终正确
}
```

### 4.4 Window-Viewport vs 世界变换

| 特性 | Window-Viewport | 世界变换（QTransform） |
|---|---|---|
| 用途 | 定义逻辑坐标空间到设备空间的全局映射 | 在逻辑空间内变换单个或一组图形 |
| 支持旋转 | ❌ 仅支持线性映射（缩放+平移） | ✅ 任意仿射变换 |
| 独立性 | 与世界变换独立，叠加作用 | 与 Window-Viewport 独立 |
| 适用场景 | 自适应大小、坐标系标准化 | 图形旋转、缩放、平移 |

## 5. 实战示例

### 5.1 可缩放平移的画布

实现鼠标滚轮缩放 + 拖拽平移的画布控件：

```cpp
class ZoomableCanvas : public QWidget
{
    Q_OBJECT
public:
    explicit ZoomableCanvas(QWidget *parent = nullptr)
        : QWidget(parent) {
        setMouseTracking(true);
    }

protected:
    void paintEvent(QPaintEvent *) override {
        QPainter painter(this);
        painter.setRenderHint(QPainter::Antialiasing);
        painter.fillRect(rect(), QColor(240, 240, 240));

        // 应用变换：先平移后缩放
        painter.translate(m_offset);
        painter.scale(m_scale, m_scale);

        // 在逻辑坐标系中绘制内容
        drawGrid(&painter);
        drawContent(&painter);
    }

    void wheelEvent(QWheelEvent *event) override {
        QPointF mousePos = event->position();
        QPointF beforeZoom = screenToLogical(mousePos);

        // 缩放因子
        double factor = event->angleDelta().y() > 0 ? 1.15 : 1.0 / 1.15;
        m_scale = qBound(0.1, m_scale * factor, 20.0);

        // 保持鼠标指向的位置不变
        QPointF afterZoom = screenToLogical(mousePos);
        m_offset += (afterZoom - beforeZoom) * m_scale;

        update();
    }

    void mousePressEvent(QMouseEvent *event) override {
        if (event->button() == Qt::MiddleButton || event->button() == Qt::LeftButton) {
            m_dragging = true;
            m_lastMousePos = event->pos();
            setCursor(Qt::ClosedHandCursor);
        }
    }

    void mouseMoveEvent(QMouseEvent *event) override {
        if (m_dragging) {
            QPointF delta = event->pos() - m_lastMousePos;
            m_offset += delta;
            m_lastMousePos = event->pos();
            update();
        }
    }

    void mouseReleaseEvent(QMouseEvent *) override {
        m_dragging = false;
        setCursor(Qt::ArrowCursor);
    }

private:
    QPointF screenToLogical(const QPointF &screenPos) const {
        return (screenPos - m_offset) / m_scale;
    }

    QPointF logicalToScreen(const QPointF &logicalPos) const {
        return logicalPos * m_scale + m_offset;
    }

    void drawGrid(QPainter *painter) {
        painter->save();
        QPen gridPen(QColor(200, 200, 200), 0); // cosmetic
        painter->setPen(gridPen);

        for (int x = -500; x <= 500; x += 50)
            painter->drawLine(x, -500, x, 500);
        for (int y = -500; y <= 500; y += 50)
            painter->drawLine(-500, y, 500, y);

        // 坐标轴
        painter->setPen(QPen(Qt::red, 0));
        painter->drawLine(-500, 0, 500, 0);
        painter->setPen(QPen(Qt::green, 0));
        painter->drawLine(0, -500, 0, 500);

        painter->restore();
    }

    void drawContent(QPainter *painter) {
        painter->save();
        painter->setBrush(QColor(60, 150, 255, 180));
        painter->setPen(QPen(Qt::white, 2));
        painter->drawRoundedRect(-60, -40, 120, 80, 10, 10);
        painter->setPen(Qt::white);
        painter->drawText(QRectF(-60, -40, 120, 80), Qt::AlignCenter, "Hello");
        painter->restore();
    }

    QPointF m_offset{200, 200};
    double m_scale = 1.0;
    bool m_dragging = false;
    QPointF m_lastMousePos;
};
```

### 5.2 时钟控件

经典的使用旋转变换绘制时钟：

```cpp
class ClockWidget : public QWidget
{
    Q_OBJECT
public:
    ClockWidget(QWidget *parent = nullptr) : QWidget(parent) {
        auto *timer = new QTimer(this);
        connect(timer, &QTimer::timeout, this, QOverload<>::of(&QWidget::update));
        timer->start(1000);
        setFixedSize(250, 250);
    }

protected:
    void paintEvent(QPaintEvent *) override {
        int side = qMin(width(), height());
        QTime time = QTime::currentTime();

        QPainter painter(this);
        painter.setRenderHint(QPainter::Antialiasing);
        painter.translate(width() / 2.0, height() / 2.0);
        painter.scale(side / 220.0, side / 220.0);

        // 表盘背景
        QRadialGradient bgGrad(0, 0, 105);
        bgGrad.setColorAt(0, QColor(50, 55, 65));
        bgGrad.setColorAt(1, QColor(30, 33, 40));
        painter.setPen(QPen(QColor(80, 85, 95), 3));
        painter.setBrush(bgGrad);
        painter.drawEllipse(QPointF(0, 0), 105, 105);

        // 刻度
        painter.setPen(Qt::NoPen);
        for (int i = 0; i < 60; ++i) {
            painter.save();
            painter.rotate(6.0 * i);  // 每刻度 6°
            if (i % 5 == 0) {
                painter.setBrush(QColor(200, 210, 220));
                painter.drawRect(-2, -98, 4, 16); // 大刻度
            } else {
                painter.setBrush(QColor(120, 130, 140));
                painter.drawRect(-1, -98, 2, 8);  // 小刻度
            }
            painter.restore();
        }

        // 时针
        painter.save();
        painter.rotate(30.0 * (time.hour() % 12) + 0.5 * time.minute());
        painter.setPen(Qt::NoPen);
        painter.setBrush(QColor(220, 225, 230));
        QPolygonF hourHand;
        hourHand << QPointF(-5, 8) << QPointF(0, -55) << QPointF(5, 8);
        painter.drawConvexPolygon(hourHand);
        painter.restore();

        // 分针
        painter.save();
        painter.rotate(6.0 * time.minute() + 0.1 * time.second());
        painter.setBrush(QColor(180, 190, 200));
        QPolygonF minuteHand;
        minuteHand << QPointF(-3, 8) << QPointF(0, -78) << QPointF(3, 8);
        painter.drawConvexPolygon(minuteHand);
        painter.restore();

        // 秒针
        painter.save();
        painter.rotate(6.0 * time.second());
        painter.setPen(QPen(QColor(255, 80, 80), 1.5));
        painter.drawLine(0, 15, 0, -90);
        painter.setPen(Qt::NoPen);
        painter.setBrush(QColor(255, 80, 80));
        painter.drawEllipse(QPointF(0, 0), 4, 4);
        painter.restore();

        // 中心小圆
        painter.setBrush(QColor(200, 205, 215));
        painter.setPen(Qt::NoPen);
        painter.drawEllipse(QPointF(0, 0), 3, 3);
    }
};
```

**核心思想**：每根指针的绘制都是——`save()` → `rotate(角度)` → 绘制竖直指针 → `restore()`。旋转 角度由当前时间计算，指针本身始终画在 12 点钟方向，由 `rotate` 转到正确位置。

### 5.3 花朵/螺旋图案（重复旋转）

```cpp
void paintEvent(QPaintEvent *) override {
    QPainter painter(this);
    painter.setRenderHint(QPainter::Antialiasing);
    painter.translate(width() / 2.0, height() / 2.0);

    int petals = 12;
    for (int i = 0; i < petals; ++i) {
        // 每个花瓣旋转 360/petals 度
        painter.rotate(360.0 / petals);

        // 渐变颜色
        QColor color = QColor::fromHsv(i * 360 / petals, 200, 230, 150);
        painter.setPen(Qt::NoPen);
        painter.setBrush(color);

        // 绘制花瓣（椭圆）
        painter.drawEllipse(QRectF(30, -15, 80, 30));
    }
}
```

```cpp
void drawCoordinateAxes(QPainter *painter, const QString &label)

    painter->drawLine(0, 0, 0, 60);
    painter->drawLine(0, 60, -5, 52);
    painter->drawLine(0, 60, 5, 52);
    painter->drawText(5, 72, "Y");

    // 原点
    painter->setPen(Qt::NoPen);
    painter->setBrush(Qt::blue);
    painter->drawEllipse(QPointF(0, 0), 3, 3);

    // 标签
    painter->setPen(Qt::black);
    painter->setFont(QFont("Arial", 9));
    painter->drawText(8, -8, label);

    painter->restore();
}

// 绘制变换前后的坐标系
void paintEvent(QPaintEvent *) override {
    QPainter painter(this);
    painter.setRenderHint(QPainter::Antialiasing);

    // 原始坐标系
    painter.save();
    painter.translate(50, 50);
    drawCoordinateAxes(&painter, "原始");
    painter.restore();

    // 平移后
    painter.save();
    painter.translate(200, 50);
    painter.translate(60, 40);   // 平移
    drawCoordinateAxes(&painter, "translate(60,40)");
    painter.restore();

    // 旋转后
    painter.save();
    painter.translate(50, 200);
    painter.rotate(30);          // 旋转
    drawCoordinateAxes(&painter, "rotate(30)");
    painter.restore();

    // 缩放后
    painter.save();
    painter.translate(200, 200);
    painter.scale(1.5, 0.8);    // 缩放
    drawCoordinateAxes(&painter, "scale(1.5,0.8)");
    painter.restore();
}
```

## 6. 高 DPI 与设备像素比

### 6.1 概念

在高 DPI 屏幕上（如 Retina/4K），一个**逻辑像素**对应多个**物理像素**。`devicePixelRatio()` 返回映射比例。

```
普通屏幕 (DPR=1)                  Retina 屏幕 (DPR=2)
1 逻辑像素 = 1 物理像素            1 逻辑像素 = 2×2 = 4 物理像素

┌─┐                               ┌──┐
│█│                               │██│
└─┘                               │██│
                                  └──┘
```

### 6.2 QPainter 中的处理

```cpp
void MyWidget::paintEvent(QPaintEvent *) {
    QPainter painter(this);

    // QPainter 的坐标始终是逻辑像素，Qt 自动处理 DPR 映射
    painter.drawRect(10, 10, 100, 100); // 始终 100x100 逻辑像素

    // 当前设备像素比
    qreal dpr = devicePixelRatioF(); // 可能是 1.0、1.25、1.5、2.0 等
}
```

### 6.3 离屏绑定的 DPR 处理

```cpp
qreal dpr = devicePixelRatioF();

// ✅ 正确创建高 DPI 离屏图片
QPixmap pixmap(QSize(200, 200) * dpr);
pixmap.setDevicePixelRatio(dpr);
pixmap.fill(Qt::transparent);

QPainter pm(&pixmap);
pm.setRenderHint(QPainter::Antialiasing);
pm.drawEllipse(0, 0, 200, 200); // 使用逻辑坐标（200x200）
pm.end();

// 绘制到 widget（大小自动匹配，高清晰度）
QPainter painter(this);
painter.drawPixmap(0, 0, pixmap); // 显示为 200x200 逻辑像素

// ❌ 若不设置 DPR
QPixmap badPixmap(200, 200); // 物理像素不够，在高 DPI 屏幕上模糊
```

### 6.4 字体大小与 DPR

```cpp
// Qt 中字体大小是逻辑大小，自动适配 DPR
QFont font("Arial", 12); // 12pt 在任何 DPR 下视觉大小一致
painter.setFont(font);
painter.drawText(10, 30, "清晰文本");
```

## 7. save() / restore() 与变换

### 7.1 在循环绘制中的典型用法

```cpp
void paintEvent(QPaintEvent *) override {
    QPainter painter(this);
    painter.setRenderHint(QPainter::Antialiasing);
    painter.translate(width() / 2.0, height() / 2.0);

    // 绘制 8 个矩形，均匀分布在圆周上
    for (int i = 0; i < 8; ++i) {
        painter.save();            // 保存当前变换
        painter.rotate(i * 45);    // 每个旋转 45°
        painter.translate(100, 0); // 沿当前 X 轴移动 100

        painter.setBrush(QColor::fromHsv(i * 45, 200, 230));
        painter.drawRect(-20, -15, 40, 30);

        painter.restore();         // 恢复到圆心的变换状态
    }
}
```

> **如果不用 save/restore**，变换会不断累积：`rotate(45)` 执行 8 次后坐标系已旋转 360°，并且每次 `translate(100, 0)` 使原点越来越远。

### 7.2 嵌套变换的层级关系

```cpp
painter.save();                 // 层级 0
    painter.translate(100, 50);
    painter.save();             // 层级 1（含 translate）
        painter.rotate(30);
        painter.save();         // 层级 2（含 translate + rotate）
            painter.scale(2, 2);
            painter.drawRect(0, 0, 30, 20);
        painter.restore();      // 恢复到层级 1
        painter.drawRect(0, 0, 30, 20); // 有 translate + rotate，无 scale
    painter.restore();          // 恢复到层级 0
    painter.drawRect(0, 0, 30, 20); // 仅有 translate
painter.restore();              // 恢复到初始
```

## 8. 实用坐标变换技巧

### 8.1 以任意中心点缩放

```cpp
void scaleAt(QPainter &painter, const QPointF &center, qreal factor) {
    painter.translate(center);
    painter.scale(factor, factor);
    painter.translate(-center);
}

// 以 (200,150) 为中心放大 2 倍
scaleAt(painter, QPointF(200, 150), 2.0);
```

### 8.2 以任意中心点旋转

```cpp
void rotateAt(QPainter &painter, const QPointF &center, qreal angle) {
    painter.translate(center);
    painter.rotate(angle);
    painter.translate(-center);
}

// 以 (300,200) 为中心旋转 45°
rotateAt(painter, QPointF(300, 200), 45);
```

### 8.3 世界坐标 ↔ 屏幕坐标互转

```cpp
class CanvasWidget : public QWidget {
    QPointF m_offset;     // 平移偏移
    qreal m_scale = 1.0;  // 缩放系数

    // 屏幕坐标 → 逻辑坐标（用于鼠标事件）
    QPointF toLogical(const QPointF &screen) const {
        return (screen - m_offset) / m_scale;
    }

    // 逻辑坐标 → 屏幕坐标（用于绘制计算）
    QPointF toScreen(const QPointF &logical) const {
        return logical * m_scale + m_offset;
    }

    // 使用 QTransform 实现同样效果
    QTransform currentTransform() const {
        QTransform t;
        t.translate(m_offset.x(), m_offset.y());
        t.scale(m_scale, m_scale);
        return t;
    }

    QPointF toLogical2(const QPointF &screen) const {
        return currentTransform().inverted().map(screen);
    }
};
```

### 8.4 坐标系翻转（数学坐标系）

Qt 默认 Y 轴向下，数学/物理通常 Y 轴向上。可以通过变换翻转：

```cpp
QPainter painter(this);

// 方案 A：scale 翻转 Y
painter.translate(0, height());
painter.scale(1, -1);
// 现在 Y 轴向上，原点在左下角

// 方案 B：Window-Viewport 翻转
painter.setWindow(0, 100, 100, -100);
// window 高度为负数实现 Y 翻转

// ⚠️ 注意：翻转后文本也会上下颠倒，需要额外处理
painter.save();
painter.scale(1, -1); // 再翻转回来
painter.drawText(x, -y, "文本"); // 注意 y 坐标取负
painter.restore();
```

## 9. API 速查

### 9.1 QPainter 变换方法

| 方法 | 说明 |
|---|---|
| `translate(dx, dy)` | 平移 |
| `rotate(angle)` | 顺时针旋转（度） |
| `scale(sx, sy)` | 缩放（负值可镜像） |
| `shear(sh, sv)` | 斜切 |
| `setTransform(t, combine)` | 设置变换矩阵，combine=true 时与现有矩阵组合 |
| `transform()` | 获取当前世界变换矩阵 |
| `worldTransform()` | 同 `transform()` |
| `resetTransform()` | 重置为单位矩阵 |
| `setWorldMatrixEnabled(bool)` | 启用/禁用世界变换 |
| `setWindow(x,y,w,h)` | 设置逻辑窗口 |
| `setViewport(x,y,w,h)` | 设置设备视口 |
| `save()` / `restore()` | 保存/恢复状态（含变换矩阵） |

### 9.2 QTransform 方法

| 方法 | 说明 |
|---|---|
| `translate(dx, dy)` | 追加平移 |
| `rotate(angle)` | 追加旋转 |
| `scale(sx, sy)` | 追加缩放 |
| `shear(sh, sv)` | 追加斜切 |
| `inverted(&invertible)` | 求逆矩阵 |
| `map(point/rect/path/region)` | 变换几何对象 |
| `mapRect(rect)` | 变换矩形（返回包围盒） |
| `type()` | 变换类型 |
| `isIdentity()` | 是否为单位矩阵 |
| `isInvertible()` | 是否可逆 |
| `determinant()` | 行列式 |
| `operator*` | 矩阵乘法（组合变换） |
| `reset()` | 重置为单位矩阵 |

## 10. 总结

| 要点 | 说明 |
|---|---|
| **坐标系** | Qt 默认左上角原点、X 向右、Y 向下 |
| **半像素偏移** | 抗锯齿模式下，1px 线条用 `+0.5` 偏移对齐像素中心 |
| **变换顺序** | 代码中后写的变换先生效（矩阵右乘），先 translate 后 rotate = 就地旋转 |
| **save/restore** | 循环绘制中必须配对使用，保护变换状态 |
| **QTransform** | 3×3 仿射矩阵，支持组合、求逆、坐标映射 |
| **逆变换** | 鼠标坐标 → 逻辑坐标必须用 `transform.inverted().map()` |
| **Window-Viewport** | 定义逻辑坐标空间到设备空间的映射，适合自适应大小的绑定 |
| **高 DPI** | QPainter 用逻辑像素；离屏 QPixmap 需乘 DPR 并设 `setDevicePixelRatio` |
| **翻转坐标系** | `scale(1,-1)` + `translate(0,h)` 可得到数学坐标系 |
