# 自定义事件

## 1. 概述

Qt 事件系统内置了大量标准事件（鼠标、键盘、绘制、定时器等），但在实际项目中，我们经常需要定义 **应用程序专属的事件类型** 来实现模块间的解耦通信。自定义事件是 Qt 事件系统的高级用法，它允许开发者：

- 在不同对象/线程之间传递自定义数据
- 实现发布-订阅模式的松耦合架构
- 跨线程安全地通知目标对象
- 替代某些场景下信号槽机制无法胜任的通信需求

**自定义事件 vs 信号槽：**

| 特性 | 自定义事件 | 信号槽 |
|------|-----------|--------|
| 耦合度 | 极低（发送者不需要知道接收者） | 需要 `connect` 建立连接 |
| 线程安全 | `postEvent` 天然线程安全 | 需 `Qt::QueuedConnection` |
| 传递方向 | 可广播到事件过滤器链 | 点对点或一对多 |
| 数据携带 | 封装在事件对象中 | 通过参数传递 |
| 优先级控制 | 支持事件优先级 | 不支持 |
| 生命周期 | 事件循环管理 | 即时调用或队列调用 |

## 2. 自定义事件的核心 API

### 2.1 QEvent::registerEventType()

Qt 提供 `QEvent::registerEventType()` 静态方法来注册自定义事件类型，保证返回的类型 ID 全局唯一：

```cpp
// 原型
static int QEvent::registerEventType(int hint = -1);
```

- `hint`：期望的事件类型值（可选）。如果该值未被占用则直接使用，否则自动分配一个新值。
- 返回值：注册成功的事件类型 ID。
- **线程安全**：该函数内部使用原子操作，可在任意线程调用。

自定义事件的类型值必须在 `QEvent::User`（1000）和 `QEvent::MaxUser`（65535）之间：

```cpp
// Qt 预定义的范围
enum QEvent::Type {
    // ...
    User = 1000,       // 自定义事件起始值
    MaxUser = 65535    // 自定义事件最大值
};
```

### 2.2 事件发送方式

Qt 提供两种发送自定义事件的方式：

#### `QCoreApplication::sendEvent()` — 同步发送

```cpp
static bool QCoreApplication::sendEvent(QObject *receiver, QEvent *event);
```

- **同步调用**：立即将事件分发给接收者，函数返回时事件已处理完毕。
- **栈上分配**：事件对象可以在栈上创建，因为发送完成后生命周期由调用者控制。
- **线程限制**：只能在接收者所在线程中调用。

#### `QCoreApplication::postEvent()` — 异步发送

```cpp
static void QCoreApplication::postEvent(QObject *receiver, QEvent *event,
                                         int priority = Qt::NormalEventPriority);
```

- **异步调用**：将事件放入接收者所在线程的事件队列，函数立即返回。
- **堆上分配**：事件对象 **必须** 在堆上创建（`new`），事件循环会自动 `delete`。
- **线程安全**：可在任何线程中调用，是跨线程通信的首选方式。
- **优先级**：支持 `Qt::HighEventPriority`、`Qt::NormalEventPriority`、`Qt::LowEventPriority`。

### 2.3 事件接收方式

接收自定义事件有两种途径：

1. **重写 `QObject::event()`**：在目标对象中判断事件类型并处理。
2. **重写 `QObject::customEvent()`**：专门用于处理用户自定义事件（`type() >= QEvent::User`），由 `event()` 默认实现调用。
3. **安装事件过滤器**：通过 `installEventFilter()` 在其他对象上拦截。

## 3. 自定义事件的实现步骤

### 3.1 基本步骤

```
1. 注册自定义事件类型 → QEvent::registerEventType()
2. 定义事件类（继承 QEvent） → 携带自定义数据
3. 发送事件 → sendEvent() 或 postEvent()
4. 接收并处理事件 → 重写 customEvent() 或 event() 或安装事件过滤器
```

### 3.2 最简示例

**定义自定义事件类：**

```cpp
// customevent.h
#pragma once
#include <QEvent>
#include <QString>

// 步骤1: 定义事件类型 ID
// 方法一：直接使用 User 范围内的值（简单但可能冲突）
// static const QEvent::Type MyEventType = static_cast<QEvent::Type>(QEvent::User + 1);

// 方法二：使用 registerEventType（推荐，保证唯一）
static const QEvent::Type MyEventType =
    static_cast<QEvent::Type>(QEvent::registerEventType());

// 步骤2: 继承 QEvent，定义事件类
class MyCustomEvent : public QEvent
{
public:
    explicit MyCustomEvent(const QString &message)
        : QEvent(MyEventType)
        , m_message(message)
    {}

    QString message() const { return m_message; }

private:
    QString m_message;
};
```

**发送事件：**

```cpp
// 同步发送（栈上创建）
MyCustomEvent event("Hello from sendEvent");
QCoreApplication::sendEvent(targetObject, &event);

// 异步发送（必须堆上创建，事件循环负责释放）
QCoreApplication::postEvent(targetObject, new MyCustomEvent("Hello from postEvent"));
```

**接收事件（方式一：重写 `customEvent`）：**

```cpp
// mywidget.h
class MyWidget : public QWidget
{
    Q_OBJECT
public:
    using QWidget::QWidget;

protected:
    void customEvent(QEvent *event) override
    {
        if (event->type() == MyEventType) {
            auto *myEvent = static_cast<MyCustomEvent *>(event);
            qDebug() << "收到自定义事件:" << myEvent->message();
            // 事件已处理，无需手动 accept/ignore
        } else {
            QWidget::customEvent(event);  // 交给基类处理
        }
    }
};
```

**接收事件（方式二：重写 `event`）：**

```cpp
bool MyWidget::event(QEvent *event)
{
    if (event->type() == MyEventType) {
        auto *myEvent = static_cast<MyCustomEvent *>(event);
        qDebug() << "收到自定义事件:" << myEvent->message();
        return true;  // 表示事件已处理
    }
    return QWidget::event(event);  // 其他事件交给基类
}
```

## 4. 进阶用法

### 4.1 携带复杂数据的自定义事件

实际项目中，自定义事件通常需要携带丰富的数据：

```cpp
#include <QEvent>
#include <QString>
#include <QVariantMap>
#include <QSharedPointer>

// 注册事件类型
static const QEvent::Type DataEventType =
    static_cast<QEvent::Type>(QEvent::registerEventType());

// 数据载体
struct TaskResult {
    int taskId;
    bool success;
    QString errorMessage;
    QByteArray payload;
};

class DataEvent : public QEvent
{
public:
    explicit DataEvent(QSharedPointer<TaskResult> result)
        : QEvent(DataEventType)
        , m_result(std::move(result))
    {}

    QSharedPointer<TaskResult> result() const { return m_result; }

private:
    QSharedPointer<TaskResult> m_result;
};
```

使用 `QSharedPointer` 的好处：
- 避免大数据拷贝
- 自动引用计数管理生命周期
- 发送方和接收方可安全共享同一份数据

### 4.2 跨线程通信

自定义事件最经典的场景之一是 **工作线程将结果通知到 UI 线程**：

```cpp
// ===== 自定义事件 =====
static const QEvent::Type ProgressEventType =
    static_cast<QEvent::Type>(QEvent::registerEventType());

class ProgressEvent : public QEvent
{
public:
    ProgressEvent(int progress, const QString &status)
        : QEvent(ProgressEventType)
        , m_progress(progress)
        , m_status(status)
    {}

    int progress() const { return m_progress; }
    QString status() const { return m_status; }

private:
    int m_progress;
    QString m_status;
};

// ===== 工作线程 =====
class Worker : public QThread
{
protected:
    void run() override
    {
        for (int i = 0; i <= 100; i += 10) {
            // 模拟耗时操作
            QThread::msleep(500);

            // 从工作线程安全地向主线程发送进度事件
            // postEvent 是线程安全的！
            QCoreApplication::postEvent(
                m_receiver,
                new ProgressEvent(i, QString("处理中... %1%").arg(i))
            );
        }
    }

public:
    void setReceiver(QObject *receiver) { m_receiver = receiver; }

private:
    QObject *m_receiver = nullptr;
};

// ===== 主窗口（UI 线程）=====
class MainWindow : public QMainWindow
{
    Q_OBJECT
public:
    MainWindow()
    {
        m_progressBar = new QProgressBar(this);
        setCentralWidget(m_progressBar);

        auto *worker = new Worker;
        worker->setReceiver(this);
        worker->start();
    }

protected:
    void customEvent(QEvent *event) override
    {
        if (event->type() == ProgressEventType) {
            auto *pe = static_cast<ProgressEvent *>(event);
            m_progressBar->setValue(pe->progress());
            statusBar()->showMessage(pe->status());
        }
    }

private:
    QProgressBar *m_progressBar;
};
```

> **关键点**：`postEvent()` 将事件投递到接收者所在线程的事件队列中，事件循环在该线程中取出事件并调用 `customEvent()`，因此 UI 更新始终在主线程中执行，天然线程安全。

### 4.3 事件优先级

`postEvent()` 支持优先级参数，可以控制事件在队列中的处理顺序：

```cpp
// 高优先级 — 插入队列前部，优先处理
QCoreApplication::postEvent(receiver,
    new MyCustomEvent("紧急通知"),
    Qt::HighEventPriority);

// 普通优先级（默认）
QCoreApplication::postEvent(receiver,
    new MyCustomEvent("常规消息"),
    Qt::NormalEventPriority);

// 低优先级 — 排在队列末尾
QCoreApplication::postEvent(receiver,
    new MyCustomEvent("后台日志"),
    Qt::LowEventPriority);
```

数值越大优先级越高。也可以使用自定义整数值：

```cpp
// 自定义优先级值（范围：INT_MAX 到 INT_MIN）
QCoreApplication::postEvent(receiver, event, 100);  // 高于 Normal(0)
QCoreApplication::postEvent(receiver, event, -100); // 低于 Normal(0)
```

### 4.4 压缩事件（Event Compression）

当同一类型的事件被频繁 `postEvent` 时（如窗口 resize），Qt 会自动压缩某些内置事件。对于自定义事件，可以通过覆盖 `QCoreApplication::compressEvent()` 实现压缩，但该方法是私有的。

实际做法是在发送前手动移除队列中已有的同类事件：

```cpp
// 先移除队列中所有同类型的旧事件，再发送新事件
QCoreApplication::removePostedEvents(receiver, MyEventType);
QCoreApplication::postEvent(receiver, new MyCustomEvent("最新状态"));
```

这在高频更新场景（如实时数据刷新）中非常实用，避免事件堆积导致延迟。

### 4.5 通过事件过滤器拦截自定义事件

可以在不修改目标对象代码的情况下，拦截其自定义事件：

```cpp
class EventMonitor : public QObject
{
    Q_OBJECT
protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == MyEventType) {
            auto *myEvent = static_cast<MyCustomEvent *>(event);
            qDebug() << "EventMonitor 拦截到事件:"
                     << watched->objectName()
                     << myEvent->message();

            // 返回 true = 拦截，目标对象不会收到该事件
            // 返回 false = 放行，目标对象继续处理
            return false;
        }
        return QObject::eventFilter(watched, event);
    }
};

// 安装事件过滤器
auto *monitor = new EventMonitor;
targetWidget->installEventFilter(monitor);
```

## 5. 完整实战示例：日志收集系统

下面是一个完整的实战示例，演示如何用自定义事件实现一个 **跨模块、跨线程的日志收集系统**：

```cpp
// ==================== logevent.h ====================
#pragma once
#include <QEvent>
#include <QString>
#include <QDateTime>

enum class LogLevel { Debug, Info, Warning, Error, Fatal };

// 注册事件类型
inline QEvent::Type logEventType()
{
    static const auto type =
        static_cast<QEvent::Type>(QEvent::registerEventType());
    return type;
}

class LogEvent : public QEvent
{
public:
    LogEvent(LogLevel level, const QString &module, const QString &message)
        : QEvent(logEventType())
        , m_level(level)
        , m_module(module)
        , m_message(message)
        , m_timestamp(QDateTime::currentDateTime())
        , m_threadId(quintptr(QThread::currentThreadId()))
    {}

    LogLevel level() const { return m_level; }
    QString module() const { return m_module; }
    QString message() const { return m_message; }
    QDateTime timestamp() const { return m_timestamp; }
    quintptr threadId() const { return m_threadId; }

    QString levelString() const
    {
        switch (m_level) {
        case LogLevel::Debug:   return "DEBUG";
        case LogLevel::Info:    return "INFO";
        case LogLevel::Warning: return "WARN";
        case LogLevel::Error:   return "ERROR";
        case LogLevel::Fatal:   return "FATAL";
        }
        return "UNKNOWN";
    }

    QString formatted() const
    {
        return QString("[%1] [%2] [%3] [Thread:%4] %5")
            .arg(m_timestamp.toString("yyyy-MM-dd hh:mm:ss.zzz"))
            .arg(levelString(), -5)
            .arg(m_module)
            .arg(m_threadId)
            .arg(m_message);
    }

private:
    LogLevel m_level;
    QString m_module;
    QString m_message;
    QDateTime m_timestamp;
    quintptr m_threadId;
};

// ==================== logcollector.h ====================
#pragma once
#include <QObject>
#include <QFile>
#include <QTextStream>
#include <QTextEdit>
#include "logevent.h"

class LogCollector : public QObject
{
    Q_OBJECT
public:
    explicit LogCollector(QTextEdit *display, QObject *parent = nullptr)
        : QObject(parent)
        , m_display(display)
        , m_file("app.log")
    {
        m_file.open(QIODevice::WriteOnly | QIODevice::Append | QIODevice::Text);
    }

    ~LogCollector()
    {
        m_file.close();
    }

protected:
    void customEvent(QEvent *event) override
    {
        if (event->type() == logEventType()) {
            auto *logEvent = static_cast<LogEvent *>(event);
            QString text = logEvent->formatted();

            // 写入文件
            QTextStream stream(&m_file);
            stream << text << "\n";
            stream.flush();

            // 显示到 UI（根据级别设置颜色）
            QString color;
            switch (logEvent->level()) {
            case LogLevel::Debug:   color = "gray"; break;
            case LogLevel::Info:    color = "black"; break;
            case LogLevel::Warning: color = "orange"; break;
            case LogLevel::Error:   color = "red"; break;
            case LogLevel::Fatal:   color = "darkred"; break;
            }
            m_display->append(
                QString("<font color='%1'>%2</font>").arg(color, text.toHtmlEscaped())
            );
        }
    }

private:
    QTextEdit *m_display;
    QFile m_file;
};

// ==================== 便捷发送宏 ====================
#define APP_LOG(collector, level, module, msg) \
```

## 6. 注意事项与最佳实践

### 6.1 内存管理

```cpp
// ✅ 正确：postEvent 使用 new，事件循环自动 delete
QCoreApplication::postEvent(obj, new MyCustomEvent("data"));

// ✅ 正确：sendEvent 可以使用栈对象
MyCustomEvent event("data");
QCoreApplication::sendEvent(obj, &event);

// ❌ 错误：postEvent 使用栈对象，事件循环 delete 栈对象会崩溃！
// MyCustomEvent event("data");
// QCoreApplication::postEvent(obj, &event);  // 未定义行为！

// ❌ 错误：postEvent 后手动 delete，双重释放！
// auto *event = new MyCustomEvent("data");
// QCoreApplication::postEvent(obj, event);
// delete event;  // 崩溃！事件循环也会 delete
```

### 6.2 事件类型 ID 的注册时机

```cpp
// ❌ 不推荐：全局静态变量直接用 User + N，多模块可能冲突
static const QEvent::Type MyType = static_cast<QEvent::Type>(QEvent::User + 1);

// ✅ 推荐：使用 registerEventType()，保证全局唯一
static const QEvent::Type MyType =
    static_cast<QEvent::Type>(QEvent::registerEventType());

// ✅ 更推荐：使用内联函数 + 局部静态变量（Meyers' Singleton 模式）
inline QEvent::Type myEventType()
{
    static const auto type =
        static_cast<QEvent::Type>(QEvent::registerEventType());
    return type;
}
```

### 6.3 避免在析构后发送事件

```cpp
// ❌ 危险：如果 receiver 已被销毁，postEvent 会导致未定义行为
QCoreApplication::postEvent(m_receiver, new MyCustomEvent("data"));

// ✅ 安全：使用 QPointer 跟踪对象生命周期
QPointer<QObject> safeReceiver = m_receiver;
if (safeReceiver) {
    QCoreApplication::postEvent(safeReceiver, new MyCustomEvent("data"));
}
```

### 6.4 `customEvent()` vs `event()` 的选择

| 场景 | 推荐方式 |
|------|---------|
| 只处理自定义事件 | `customEvent()` — 更简洁 |
| 需要处理自定义+标准事件 | `event()` — 统一入口 |
| 需要阻止事件继续传播 | `event()` — 返回 true/false 控制 |
| 不想影响基类事件处理 | `customEvent()` — 更安全 |

### 6.5 性能考量

- **避免高频 postEvent**：每次 `postEvent` 都涉及堆分配和事件队列操作。对于高频场景（如每帧更新），考虑合并数据后一次性发送，或使用 `removePostedEvents` 做事件压缩。
- **事件对象尽量轻量**：自定义事件中不要存储大量数据，使用智能指针共享重型数据。
- **sendEvent 比 postEvent 快**：同步发送省去了事件队列的入队出队开销，但只能在同线程使用。

## 7. 事件传播路径总结

```
postEvent(receiver, event)
    │
    ▼
事件进入 receiver 所在线程的事件队列
    │
    ▼
事件循环取出事件
    │
    ▼
QCoreApplication::notify(receiver, event)
    │
    ▼
检查 receiver 的事件过滤器链 (eventFilter)
    │  ├── 返回 true → 事件被拦截，不再传递
    │  └── 返回 false → 继续向下
    ▼
receiver->event(event)
    │  ├── type >= User → 调用 receiver->customEvent(event)
    │  └── 其他类型 → 调用对应的专用处理函数
    ▼
如果 event 未被 accept 且 receiver 有 parent
    │
    ▼
事件传递给 parent 对象（仅对自发事件 spontaneous）
```

> **注意**：通过 `sendEvent` / `postEvent` 发送的自定义事件 **不会** 自动传播给父对象。事件传播机制仅适用于某些特定的标准事件（如键盘事件）。如果需要自定义事件向上传播，需要手动实现。

---

## 8. 总结

| 要点 | 说明 |
|------|------|
| 事件类型范围 | `QEvent::User`(1000) ~ `QEvent::MaxUser`(65535) |
| 类型注册 | 优先使用 `QEvent::registerEventType()` 保证唯一性 |
| 同步发送 | `sendEvent()` — 立即处理，栈/堆均可，仅限同线程 |
| 异步发送 | `postEvent()` — 队列处理，必须 `new`，线程安全 |
| 接收方式 | `customEvent()` 或 `event()` 或事件过滤器 |
| 跨线程 | `postEvent()` 是跨线程通信的安全方式 |
| 事件压缩 | `removePostedEvents()` + `postEvent()` 实现手动压缩 |
| 内存管理 | `postEvent` 的事件对象由事件循环自动释放，切勿手动 delete |
