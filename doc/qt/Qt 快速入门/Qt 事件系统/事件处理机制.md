# 事件处理机制

## 1. 事件系统概述

### 1.1 什么是事件

在 Qt 中，**事件（Event）** 是描述应用程序内部或外部发生的某件事情的对象。与信号槽机制不同，事件是 Qt 底层通信的核心手段，所有用户交互（鼠标、键盘、触摸）、系统通知（窗口大小变化、绘制请求）以及自定义行为都通过事件来传递。

每个事件都是 `QEvent` 的子类实例，携带该事件的具体信息：

```cpp
// QEvent 是所有事件的基类
class QEvent
{
public:
    enum Type {
        None = 0,
        Timer = 1,                // 定时器事件
        MouseButtonPress = 2,     // 鼠标按下
        MouseButtonRelease = 3,   // 鼠标释放
        MouseMove = 5,            // 鼠标移动
        KeyPress = 6,             // 键盘按下
        KeyRelease = 7,           // 键盘释放
        FocusIn = 8,              // 获得焦点
        FocusOut = 9,             // 失去焦点
        Paint = 12,               // 绘制事件
        Resize = 14,              // 窗口大小变化
        Close = 19,               // 窗口关闭
        Show = 17,                // 窗口显示
        Hide = 18,                // 窗口隐藏
        // ... 200+ 种事件类型
        User = 1000,              // 用户自定义事件起始值
        MaxUser = 65535           // 用户自定义事件最大值
    };

    explicit QEvent(Type type);
    virtual ~QEvent();

    Type type() const;       // 获取事件类型
    bool isAccepted() const; // 事件是否被接受
    void accept();           // 接受事件（停止传播）
    void ignore();           // 忽略事件（继续向父对象传播）
    bool spontaneous() const;// 是否由系统产生（非程序发送）
};
```

### 1.2 事件 vs 信号槽

初学者常困惑事件和信号槽的关系，它们是两套互补的通信机制：

| 特性 | 事件（Event） | 信号槽（Signal & Slot） |
|------|--------------|----------------------|
| **抽象层级** | 低层 | 高层 |
| **发起者** | 系统或 `QCoreApplication` | QObject 子类 |
| **接收方式** | 重写虚函数 | 连接到槽函数 |
| **传播方式** | 沿对象树向父对象传播 | 直接调用连接的槽 |
| **可拦截** | ✅ 可接受/忽略/过滤 | ❌ 无法拦截 |
| **典型用途** | 底层交互（绘制、输入、布局） | 业务逻辑通信 |
| **关系** | 信号通常是事件处理的结果 | 信号可以触发事件发送 |

```cpp
// 典型关系示例：按钮点击
// 1. 系统产生 MouseButtonPress 事件
// 2. QPushButton::mousePressEvent() 处理该事件
// 3. QPushButton 内部判断点击有效后，发射 clicked() 信号
// 4. 业务代码的槽函数被调用
```

> 💡 简单理解：事件是 **"发生了什么"**，信号是 **"通知别人发生了什么"**。事件在底层运作，信号槽在应用层使用。

### 1.3 常用事件类型与对应的事件类

| 事件类型 | 事件类 | 说明 |
|---------|-------|------|
| `QEvent::MouseButtonPress/Release/Move` | `QMouseEvent` | 鼠标按下/释放/移动 |
| `QEvent::KeyPress/KeyRelease` | `QKeyEvent` | 键盘按下/释放 |
| `QEvent::Wheel` | `QWheelEvent` | 鼠标滚轮 |
| `QEvent::Paint` | `QPaintEvent` | 绘制请求 |
| `QEvent::Resize` | `QResizeEvent` | 窗口大小变化 |
| `QEvent::Close` | `QCloseEvent` | 窗口关闭 |
| `QEvent::Timer` | `QTimerEvent` | 定时器触发 |
| `QEvent::FocusIn/FocusOut` | `QFocusEvent` | 焦点获得/失去 |
| `QEvent::Enter/Leave` | `QEvent` | 鼠标进入/离开控件 |
| `QEvent::DragEnter/Drop` | `QDragEnterEvent`/`QDropEvent` | 拖放事件 |
| `QEvent::Show/Hide` | `QShowEvent`/`QHideEvent` | 窗口显示/隐藏 |
| `QEvent::ContextMenu` | `QContextMenuEvent` | 右键菜单 |

## 2. 事件循环（Event Loop）

### 2.1 事件循环的概念

Qt 应用程序是**事件驱动**的。程序启动后进入一个**事件循环（Event Loop）**，不断从事件队列中取出事件、分发事件、处理事件，直到应用退出：

```
┌─────────────────────────────────────────────┐
│              应用程序事件循环                  │
│                                             │
│  ┌─────────┐    ┌──────────┐    ┌────────┐  │
│  │ 事件队列 │───>│ 事件分发  │───>│事件处理 │  │
│  │         │    │          │    │        │  │
│  │Event 1  │    │dispatch()│    │handler │  │
│  │ Event 2  │    │          │    │        │  │
│  │ Event 3  │    │          │    │        │  │
│  └─────────┘    └──────────┘    └────────┘  │
│       ↑                                     │
│       │  操作系统消息 / 内部事件 / 用户发送     │
└───────┼─────────────────────────────────────┘
        │
   系统事件源（键盘、鼠标、定时器、网络...）
```

### 2.2 启动事件循环

```cpp
#include <QApplication>
#include "mainwindow.h"

int main(int argc, char *argv[])
{
    QApplication app(argc, argv);  // 创建应用对象

    MainWindow w;
    w.show();

    return app.exec();  // ← 启动事件循环，程序在此阻塞直到退出
}
```

`QCoreApplication::exec()` 内部大致等价于：

```cpp
int QCoreApplication::exec()
{
    // 伪代码
    while (!m_quit) {
        QEvent *event = waitForNextEvent();  // 从队列取事件（可能阻塞）
        dispatchEvent(event);                // 分发事件给目标对象
        delete event;
    }
    return m_returnCode;
}
```

### 2.3 嵌套事件循环

某些场景会启动**嵌套事件循环**（局部事件循环），例如模态对话框：

```cpp
void MainWindow::onOpenDialog()
{
    QDialog dlg(this);
    dlg.exec();  // ← 启动嵌套事件循环，主循环被暂停
    // exec() 返回后，主循环恢复
}
```

也可以使用 `QEventLoop` 手动创建局部事件循环：

```cpp
void MyClass::waitForSignal()
{
    QEventLoop loop;
    QTimer::singleShot(3000, &loop, &QEventLoop::quit); // 3秒后退出
    connect(worker, &Worker::finished, &loop, &QEventLoop::quit);
    loop.exec();  // 阻塞在此，直到 quit() 被调用
    // 此处继续执行
}
```

> ⚠️ 嵌套事件循环虽然方便，但容易引发**重入问题**（reentrancy）——嵌套循环期间主循环中的事件仍会被处理，可能导致意外的状态变化。生产代码中应尽量避免使用。

### 2.4 手动处理事件

在长时间运算中，可以手动调用 `processEvents()` 保持界面响应：

```cpp
void MainWindow::heavyComputation()
{
    for (int i = 0; i < 1000000; ++i) {
        // ... 耗时计算 ...

        if (i % 1000 == 0) {
            QCoreApplication::processEvents();  // 处理积压事件，刷新界面
            ui->progressBar->setValue(i / 10000);
        }
    }
}
```

> ⚠️ 更推荐的做法是将耗时任务放入**工作线程**，通过信号槽更新 UI，而非在主线程中使用 `processEvents()`。

## 3. 事件的分发机制

### 3.1 事件分发流程全景

当一个事件到达目标对象时，Qt 按以下顺序依次处理，任一环节如果"消费"了事件，后续环节将不再执行：

```
事件到达
    │
    ▼
┌─────────────────────────────────────┐
│ ① QCoreApplication::notify()       │
│    应用级分发入口（可重写）            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│ ② 安装在目标对象上的事件过滤器        │
│    obj->installEventFilter(filter)  │
│    filter->eventFilter(obj, event)  │
└──────────────┬──────────────────────┘
               │ 未被过滤
               ▼
┌─────────────────────────────────────┐
│ ③ 目标对象的 event() 函数            │
│    QObject::event(QEvent*)          │
└──────────────┬──────────────────────┘
               │ 根据事件类型分派
               ▼
┌─────────────────────────────────────┐
│ ④ 具体事件处理函数（虚函数）          │
│    mousePressEvent()                │
│    keyPressEvent()                  │
│    paintEvent()                     │
│    ...                              │
└──────────────┬──────────────────────┘
               │ 如果事件被 ignore()
               ▼
┌─────────────────────────────────────┐
│ ⑤ 传播给父对象                       │
│    parent->event(QEvent*)           │
└─────────────────────────────────────┘
```

### 3.2 第① 层：notify() — 应用级入口

所有事件最终都通过 `QCoreApplication::notify()` 发送到目标对象。重写它可以实现全局事件拦截：

```cpp
class MyApplication : public QApplication
{
public:
    using QApplication::QApplication;

    bool notify(QObject *receiver, QEvent *event) override
    {
        // 全局监控所有事件（调试用）
        if (event->type() == QEvent::KeyPress) {
            QKeyEvent *ke = static_cast<QKeyEvent*>(event);
            qDebug() << "Global key press:" << ke->key()
                     << "to" << receiver->objectName();
        }

        // 全局异常捕获（防止未处理异常导致崩溃）
        try {
            return QApplication::notify(receiver, event);
        } catch (const std::exception &e) {
            qCritical() << "Exception in event handler:" << e.what();
            return false;
        }
    }
};
```

> 💡 **使用场景**：全局快捷键、全局异常保护、性能监控。这是处理事件的最高优先级入口。

### 3.3 第② 层：eventFilter() — 事件过滤器

在不修改目标类的前提下，拦截并处理其事件：

```cpp
// 安装过滤器
lineEdit->installEventFilter(this);

// 过滤器实现
bool MainWindow::eventFilter(QObject *watched, QEvent *event)
{
    if (watched == lineEdit && event->type() == QEvent::KeyPress) {
        QKeyEvent *ke = static_cast<QKeyEvent*>(event);
        if (ke->key() == Qt::Key_Return) {
            onLineEditSubmit();
            return true;  // ← 事件已处理，不再向下传播
        }
    }
    return QMainWindow::eventFilter(watched, event);  // 交给基类处理
}
```

> 📖 关于事件过滤器的详细用法，请参考 [事件过滤器](./事件过滤器.md)。

### 3.4 第③ 层：event() — 对象级分发

`QObject::event()` 是事件到达目标对象后的**总入口**。它根据事件类型，调用对应的具体处理函数：

```cpp
// QWidget::event() 的简化实现
bool QWidget::event(QEvent *event)
{
    switch (event->type()) {
    case QEvent::MouseButtonPress:
        mousePressEvent(static_cast<QMouseEvent*>(event));
        break;
    case QEvent::KeyPress:
        keyPressEvent(static_cast<QKeyEvent*>(event));
        break;
    case QEvent::Paint:
        paintEvent(static_cast<QPaintEvent*>(event));
        break;
    case QEvent::Resize:
        resizeEvent(static_cast<QResizeEvent*>(event));
        break;
    case QEvent::Close:
        closeEvent(static_cast<QCloseEvent*>(event));
        break;
    // ... 其他事件类型
    default:
        return QObject::event(event);
    }
    return true;
}
```

重写 `event()` 可以在事件到达具体处理函数**之前**进行拦截：

```cpp
bool MyWidget::event(QEvent *event)
{
    // 拦截 Tab 键（Tab 通常在 event() 层被特殊处理，不会到 keyPressEvent）
    if (event->type() == QEvent::KeyPress) {
        QKeyEvent *ke = static_cast<QKeyEvent*>(event);
        if (ke->key() == Qt::Key_Tab) {
            // 自定义 Tab 行为
            focusNextField();
            return true;  // 事件已处理
        }
    }
    return QWidget::event(event);  // 其他事件交给基类
}
```

> ⚠️ 重写 `event()` 时务必调用基类实现，否则大量默认行为（如焦点切换、快捷键）将失效。

### 3.5 第④ 层：具体事件处理函数

这是**最常用**的事件处理方式。通过重写特定的虚函数来处理对应类型的事件：

```cpp
class MyWidget : public QWidget
{
    Q_OBJECT

protected:
    // 鼠标按下
    void mousePressEvent(QMouseEvent *event) override
    {
        if (event->button() == Qt::LeftButton) {
            m_dragStartPos = event->pos();
            m_isDragging = true;
            event->accept();  // 明确接受事件
        } else {
            QWidget::mousePressEvent(event);  // 其他按钮交给基类
        }
    }

    // 鼠标移动
    void mouseMoveEvent(QMouseEvent *event) override
    {
        if (m_isDragging) {
            QPoint delta = event->pos() - m_dragStartPos;
            move(pos() + delta);
            event->accept();
        }
    }

    // 绘制
    void paintEvent(QPaintEvent *event) override
    {
        QPainter painter(this);
        painter.setRenderHint(QPainter::Antialiasing);
        painter.setBrush(Qt::blue);
        painter.drawEllipse(rect().center(), 50, 50);
        // paintEvent 不需要调用基类实现（QWidget 基类只是填充背景）
    }

    // 窗口大小变化
    void resizeEvent(QResizeEvent *event) override
    {
        qDebug() << "Old size:" << event->oldSize()
                 << "New size:" << event->size();
        QWidget::resizeEvent(event);  // 调用基类
    }

    // 窗口关闭
    void closeEvent(QCloseEvent *event) override
    {
        if (hasUnsavedChanges()) {
            auto ret = QMessageBox::question(this, "确认",
                "有未保存的更改，确定要关闭吗？");
            if (ret == QMessageBox::No) {
                event->ignore();  // ← 拒绝关闭
                return;
            }
        }
        event->accept();  // ← 允许关闭
    }

private:
    QPoint m_dragStartPos;
    bool m_isDragging = false;
};
```

### 3.6 第⑤ 层：事件传播（Propagation）

如果目标对象没有处理某个事件（调用了 `ignore()`），该事件将**传播给父对象**。这种传播机制仅适用于继承自 `QWidget` 的类，且仅对特定事件类型生效（键盘、鼠标等输入事件）。

```cpp
// 子控件
class ChildWidget : public QWidget
{
protected:
    void keyPressEvent(QKeyEvent *event) override
    {
        if (event->key() == Qt::Key_Escape) {
            emit escapePressed();
            event->accept();    // Escape 键自己处理
        } else {
            event->ignore();    // 其他按键传播给父对象
            // 等价于调用 QWidget::keyPressEvent(event);
        }
    }
};

// 父窗口
class ParentWindow : public QWidget
{
protected:
    void keyPressEvent(QKeyEvent *event) override
    {
        // 子控件 ignore() 的键盘事件会到达这里
        qDebug() << "Parent received key:" << event->key();
        QWidget::keyPressEvent(event);
    }
};
```

**传播规则汇总：**

| 事件类型 | 是否传播给父对象 | 说明 |
|---------|---------------|------|
| 键盘事件 | ✅ 传播 | `keyPressEvent` / `keyReleaseEvent` |
| 鼠标事件 | ✅ 传播 | `mousePressEvent` 等 |
| 滚轮事件 | ✅ 传播 | `wheelEvent` |
| 绘制事件 | ❌ 不传播 | `paintEvent` 仅绘制自身 |
| 定时器事件 | ❌ 不传播 | 绑定到特定对象 |
| 关闭事件 | ❌ 不传播 | 仅针对顶层窗口 |
| 焦点事件 | ❌ 不传播 | 焦点属于特定控件 |
| 大小变化事件 | ❌ 不传播 | 仅通知自身 |

## 4. accept() 与 ignore() 详解

### 4.1 基本语义

```cpp
event->accept();   // "我处理了这个事件" → 停止传播
event->ignore();   // "我不处理这个事件" → 继续向父对象传播
```

事件默认是 `accepted` 状态（即 `isAccepted() == true`）。大多数事件处理函数中，如果你处理了事件，什么都不用做（默认就是 accept）；如果你不处理，需要显式调用 `ignore()` 或调用基类实现。

### 4.2 closeEvent 中的特殊语义

`closeEvent` 中 `accept()` 和 `ignore()` 的含义比较特殊：

```cpp
void MainWindow::closeEvent(QCloseEvent *event)
{
    // accept() = 允许窗口关闭
    // ignore() = 阻止窗口关闭
    if (maybeSave()) {
        writeSettings();
        event->accept();   // 关闭窗口
    } else {
        event->ignore();   // 阻止关闭，窗口保持打开
    }
}
```

### 4.3 调用基类 vs 显式 ignore()

以下两种写法效果相同：

```cpp
// 写法一：显式 ignore
void MyWidget::keyPressEvent(QKeyEvent *event)
{
    if (event->key() == Qt::Key_F1) {
        showHelp();
        event->accept();
    } else {
        event->ignore();  // 显式忽略，传播给父对象
    }
}

// 写法二：调用基类（推荐）
void MyWidget::keyPressEvent(QKeyEvent *event)
{
    if (event->key() == Qt::Key_F1) {
        showHelp();
        // 默认 accepted
    } else {
        QWidget::keyPressEvent(event);  // 基类会调用 ignore()
    }
}
```

> 💡 **推荐写法二**，因为基类可能有自己的处理逻辑，直接 `ignore()` 会跳过这些逻辑。

## 5. 事件的发送方式

### 5.1 三种发送方式对比

| 方式 | 函数 | 同步/异步 | 事件归属 | 适用场景 |
|------|------|----------|---------|---------|
| **sendEvent** | `QCoreApplication::sendEvent()` | 同步 | 调用者管理 | 需要立即处理、需要检查返回值 |
| **postEvent** | `QCoreApplication::postEvent()` | 异步 | Qt 接管（自动 delete） | 跨线程通信、批量事件 |
| **sendPostedEvents** | `QCoreApplication::sendPostedEvents()` | 立即处理队列 | — | 强制处理已排队事件 |

### 5.2 sendEvent — 同步发送

事件**立即**被分发给目标对象，`sendEvent()` 返回时事件已经被处理完毕：

```cpp
void MyWidget::simulateKeyPress()
{
    QKeyEvent event(QEvent::KeyPress, Qt::Key_A, Qt::NoModifier, "a");
    QCoreApplication::sendEvent(targetWidget, &event);
    // 执行到这里时，事件已经被处理
    // event 是栈上对象，由调用者负责生命周期

    if (event.isAccepted()) {
        qDebug() << "Event was accepted by target";
    }
}
```

### 5.3 postEvent — 异步发送

事件被添加到事件队列，在后续的事件循环迭代中处理：

```cpp
void MyClass::notifyLater()
{
    // 必须在堆上创建，Qt 接管所有权，自动 delete
    QKeyEvent *event = new QKeyEvent(QEvent::KeyPress, Qt::Key_B, Qt::NoModifier, "b");
    QCoreApplication::postEvent(targetWidget, event);
    // 此处事件尚未被处理，只是加入了队列
    // ❌ 不要再访问 event 指针！
}
```

> ⚠️ `postEvent()` 传入的事件必须用 `new` 在**堆上分配**，且传入后不可再访问该指针。Qt 会在处理完后自动释放。

### 5.4 事件压缩（Event Compression）

Qt 会自动合并队列中的**冗余事件**，提高性能：

- **`QEvent::Paint`**：多个绘制事件合并为一个（区域取并集）
- **`QEvent::Resize`**：只保留最后一个
- **`QEvent::LayoutRequest`**：只保留一个
- **`QEvent::Move`**：只保留最后一个

```cpp
// 假设快速连续改变窗口大小：
widget->resize(100, 100);  // 产生 Resize 事件
widget->resize(200, 200);  // 产生 Resize 事件
widget->resize(300, 300);  // 产生 Resize 事件
// 实际只会处理一次 resizeEvent，size 为 (300, 300)
```

## 6. 完整实战案例

### 6.1 案例：可拖拽的自定义控件

```cpp
// draggablelabel.h
#pragma once
#include <QLabel>
#include <QMouseEvent>
#include <QPainter>

class DraggableLabel : public QLabel
{
    Q_OBJECT

public:
    explicit DraggableLabel(const QString &text, QWidget *parent = nullptr)
        : QLabel(text, parent)
    {
        setAlignment(Qt::AlignCenter);
        setStyleSheet("background-color: #3498db; color: white; "
                       "border-radius: 8px; padding: 10px;");
        setCursor(Qt::OpenHandCursor);
    }

signals:
    void doubleClicked();
    void moved(const QPoint &newPos);

protected:
    void mousePressEvent(QMouseEvent *event) override
    {
        if (event->button() == Qt::LeftButton) {
            m_dragOffset = event->pos();
            m_isDragging = true;
            setCursor(Qt::ClosedHandCursor);
            event->accept();
        } else {
            QLabel::mousePressEvent(event);
        }
    }

    void mouseMoveEvent(QMouseEvent *event) override
    {
        if (m_isDragging && (event->buttons() & Qt::LeftButton)) {
            QPoint newPos = mapToParent(event->pos() - m_dragOffset);
            move(newPos);
            emit moved(newPos);
            event->accept();
        }
    }

    void mouseReleaseEvent(QMouseEvent *event) override
    {
        if (event->button() == Qt::LeftButton && m_isDragging) {
            m_isDragging = false;
            setCursor(Qt::OpenHandCursor);
            event->accept();
        } else {
            QLabel::mouseReleaseEvent(event);
        }
    }

    void mouseDoubleClickEvent(QMouseEvent *event) override
    {
        if (event->button() == Qt::LeftButton) {
            emit doubleClicked();
            event->accept();
        }
    }

    void enterEvent(QEvent *event) override
    {
        setStyleSheet("background-color: #2980b9; color: white; "
                       "border-radius: 8px; padding: 10px;");
        QLabel::enterEvent(event);
    }

    void leaveEvent(QEvent *event) override
    {
        setStyleSheet("background-color: #3498db; color: white; "
                       "border-radius: 8px; padding: 10px;");
        QLabel::leaveEvent(event);
    }

private:
    QPoint m_dragOffset;
    bool m_isDragging = false;
};
```

### 6.2 案例：全局快捷键管理器

```cpp
// application.h
#pragma once
#include <QApplication>
#include <QKeyEvent>
#include <QMap>
#include <functional>

class Application : public QApplication
{
    Q_OBJECT

public:
    using QApplication::QApplication;

    // 注册全局快捷键
    void registerGlobalShortcut(int key, Qt::KeyboardModifiers modifiers,
                                 std::function<void()> callback)
    {
        m_shortcuts[{key, modifiers}] = std::move(callback);
    }

protected:
    bool notify(QObject *receiver, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            auto *ke = static_cast<QKeyEvent*>(event);
            auto key = qMakePair(ke->key(), ke->modifiers());

            if (m_shortcuts.contains(key)) {
                m_shortcuts[key]();  // 执行回调
                return true;         // 吞掉事件
            }
        }
        return QApplication::notify(receiver, event);
    }

private:
    QMap<QPair<int, Qt::KeyboardModifiers>, std::function<void()>> m_shortcuts;
};

// main.cpp 中使用
int main(int argc, char *argv[])
{
    Application app(argc, argv);

    // Ctrl+Q 全局退出
    app.registerGlobalShortcut(Qt::Key_Q, Qt::ControlModifier, [&]() {
        app.quit();
    });

    // F11 全局切换全屏
    app.registerGlobalShortcut(Qt::Key_F11, Qt::NoModifier, [&]() {
        auto *w = app.activeWindow();
        if (w) {
            w->isFullScreen() ? w->showNormal() : w->showFullScreen();
        }
    });

    MainWindow w;
    w.show();
    return app.exec();
}
```

### 6.3 案例：完整的事件处理层级演示

此示例展示了事件在各层级之间的传递过程：

```cpp
#include <QApplication>
#include <QWidget>
#include <QVBoxLayout>
#include <QPushButton>
#include <QDebug>
#include <QKeyEvent>

// 自定义 Application，重写 notify
class MyApp : public QApplication
{
public:
    using QApplication::QApplication;

    bool notify(QObject *receiver, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            qDebug() << "[1] notify() ->" << receiver->metaObject()->className();
        }
        return QApplication::notify(receiver, event);
    }
};

// 子控件
class ChildButton : public QPushButton
{
public:
    explicit ChildButton(QWidget *parent = nullptr)
        : QPushButton("Press any key while focused", parent)
    {
        setFocusPolicy(Qt::StrongFocus);
        installEventFilter(parent);  // 让父对象过滤事件
    }

protected:
    bool event(QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            qDebug() << "[3] ChildButton::event()";
        }
        return QPushButton::event(event);
    }

    void keyPressEvent(QKeyEvent *event) override
    {
        qDebug() << "[4] ChildButton::keyPressEvent() key=" << event->text();
        // 不处理，传播给父对象
        event->ignore();
    }
};

// 父窗口
class ParentWidget : public QWidget
{
public:
    ParentWidget()
    {
        auto *layout = new QVBoxLayout(this);
        auto *btn = new ChildButton(this);
        layout->addWidget(btn);
        setWindowTitle("Event Propagation Demo");
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            qDebug() << "[2] eventFilter() for" << watched->metaObject()->className();
            // 返回 false 继续传递
        }
        return QWidget::eventFilter(watched, event);
    }

    void keyPressEvent(QKeyEvent *event) override
    {
        qDebug() << "[5] ParentWidget::keyPressEvent() - event propagated!";
        QWidget::keyPressEvent(event);
    }
};

int main(int argc, char *argv[])
{
    MyApp app(argc, argv);
    ParentWidget w;
    w.show();
    return app.exec();
}
```

**输出（按下 "A" 键时）：**

```
[1] notify() -> ChildButton
[2] eventFilter() for ChildButton
[3] ChildButton::event()
[4] ChildButton::keyPressEvent() key= "a"
[5] ParentWidget::keyPressEvent() - event propagated!
```

这清楚地展示了事件从 `notify()` → `eventFilter()` → `event()` → `keyPressEvent()` → 父对象 `keyPressEvent()` 的完整传播链。

## 7. 最佳实践与常见陷阱

### 7.1 推荐做法

| 场景 | 推荐方式 |
|------|---------|
| 处理特定类型的事件 | 重写对应的虚函数（`mousePressEvent` 等） |
| 拦截不属于自己的控件的事件 | 使用事件过滤器 `installEventFilter()` |
| 在事件到达具体处理函数前拦截 | 重写 `event()` |
| 全局事件监控/拦截 | 重写 `QApplication::notify()` |
| Tab/Backtab 按键处理 | 重写 `event()`（这些键不会到 `keyPressEvent`） |
| 窗口关闭确认 | 重写 `closeEvent()`，用 `ignore()` 阻止关闭 |

### 7.2 常见陷阱

**陷阱 1：忘记调用基类实现**

```cpp
// ❌ 错误：未调用基类，丢失默认行为
void MyWidget::event(QEvent *event)
{
    if (event->type() == QEvent::KeyPress) {
        // 处理按键
    }
    // 忘记 return QWidget::event(event);
    // 导致所有其他事件都不会被处理！
}

// ✅ 正确
bool MyWidget::event(QEvent *event)
{
    if (event->type() == QEvent::KeyPress) {
        // ...
    }
    return QWidget::event(event);  // 不要忘记这一行
}
```

**陷阱 2：在非 GUI 线程中操作控件**

```cpp
// ❌ 错误：在工作线程中直接操作 UI
void WorkerThread::run()
{
    // ... 计算完成
    label->setText("Done");  // 崩溃或未定义行为！
}

// ✅ 正确：通过信号槽或 postEvent 跨线程通信
void WorkerThread::run()
{
    // ... 计算完成
    emit finished("Done");  // 信号槽自动处理线程安全
}
// 或使用 postEvent
void WorkerThread::run()
{
    auto *event = new QStatusTipEvent("Done");
    QCoreApplication::postEvent(label, event);  // ✅ postEvent 是线程安全的
}
```

**陷阱 3：混淆 sendEvent 和 postEvent 的内存管理**

```cpp
// ❌ 错误：sendEvent 使用堆对象后 delete
QKeyEvent *event = new QKeyEvent(QEvent::KeyPress, Qt::Key_A, Qt::NoModifier);
QCoreApplication::sendEvent(widget, event);
delete event;  // 正确但容易遗忘 → 建议用栈对象

// ✅ 推荐：sendEvent 使用栈对象
QKeyEvent event(QEvent::KeyPress, Qt::Key_A, Qt::NoModifier);
QCoreApplication::sendEvent(widget, &event);
// 栈对象自动销毁，无泄漏风险

// ❌ 错误：postEvent 使用栈对象
QKeyEvent event(QEvent::KeyPress, Qt::Key_A, Qt::NoModifier);
QCoreApplication::postEvent(widget, &event);  // 未定义行为！event 随后被销毁

// ✅ 正确：postEvent 必须使用堆对象
QCoreApplication::postEvent(widget, new QKeyEvent(QEvent::KeyPress, Qt::Key_A, Qt::NoModifier));
```

**陷阱 4：在 paintEvent 中创建 QPixmap**

```cpp
// ❌ 低效：每次绘制都创建 QPixmap
void MyWidget::paintEvent(QPaintEvent *)
{
    QPixmap pix(":/images/background.png");  // 每帧都从磁盘加载
    QPainter painter(this);
    painter.drawPixmap(0, 0, pix);
}

// ✅ 高效：预加载资源
class MyWidget : public QWidget
{
    QPixmap m_background{":/images/background.png"};  // 构造时加载一次

    void paintEvent(QPaintEvent *) override
    {
        QPainter painter(this);
        painter.drawPixmap(0, 0, m_background);
    }
};
```

## 8. 总结

Qt 事件处理机制的核心要点：

1. **事件循环**是 Qt 应用的心脏，`exec()` 启动循环，`quit()` 退出循环
2. **事件分发**遵循五层流水线：`notify()` → `eventFilter()` → `event()` → 具体处理函数 → 父对象传播
3. **accept/ignore** 控制事件是否继续向父对象传播
4. **sendEvent** 同步发送（栈对象），**postEvent** 异步发送（堆对象，Qt 接管生命周期）
5. **重写具体事件处理函数**是最常用、最推荐的处理方式
6. 事件过滤器适合**跨对象**监控；`notify()` 适合**全局**监控
7. 永远记得在重写 `event()` 时**调用基类实现**

```
选择事件处理方式的决策树：

需要全局拦截所有事件？     → 重写 QApplication::notify()
需要监控其他对象的事件？   → installEventFilter()
需要拦截 Tab/特殊按键？    → 重写 event()
需要处理鼠标/键盘/绘制？   → 重写对应的虚函数 ✅（最常用）
业务逻辑层的通信？         → 使用信号槽
```
