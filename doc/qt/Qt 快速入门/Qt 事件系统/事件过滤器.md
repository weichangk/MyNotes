# 事件过滤器

## 1. 事件过滤器概述

### 1.1 什么是事件过滤器

**事件过滤器（Event Filter）** 是 Qt 事件系统中一种强大的事件拦截机制。它允许一个对象**监视并拦截**发送到另一个对象的事件，而不需要对目标对象进行子类化。

事件在 Qt 中的传递流程：

```
操作系统事件 → QApplication::notify()
                    ↓
            事件过滤器（最先拦截）    ← 本文重点
                    ↓
            目标对象的 event() 函数
                    ↓
            具体事件处理函数（如 mousePressEvent()）
                    ↓
            未处理则沿父对象链向上传播
```

事件过滤器处于事件分发链的**最前端**（在 `event()` 之前），这意味着它比任何其他方式都更早地看到事件，拥有最高的拦截优先级。

### 1.2 为什么需要事件过滤器

在实际开发中，经常遇到以下场景：

| 场景 | 传统方案 | 事件过滤器方案 |
|------|---------|--------------|
| 修改 `QLineEdit` 的回车键行为 | 继承 `QLineEdit`，重写 `keyPressEvent()` | 安装事件过滤器，拦截回车键 |
| 给多个控件统一添加 tooltip 延迟 | 每个控件都要子类化 | 一个过滤器统一处理 |
| 拦截第三方库控件的事件 | 无法修改源码，无从下手 | 安装事件过滤器即可 |
| 在窗口级别拦截所有子控件的按键 | 每个子控件都要处理 | 窗口安装过滤器，一处搞定 |
| 调试事件流，查看控件收到了什么事件 | 修改每个控件的代码 | 临时安装过滤器打印日志 |

> 💡 **核心价值**：事件过滤器让你**无需子类化**就能拦截和修改事件行为，是实现"非侵入式"事件处理的关键工具。

### 1.3 事件过滤器 vs 重写事件处理函数

| 特性 | 事件过滤器 | 重写事件处理函数 |
|------|-----------|----------------|
| **是否需要子类化** | ❌ 不需要 | ✅ 必须子类化 |
| **拦截时机** | `event()` 之前 | `event()` 内部或之后 |
| **可监视多个对象** | ✅ 一个过滤器可监视多个对象 | ❌ 只能处理自身事件 |
| **可监视第三方控件** | ✅ 无需源码 | ❌ 需要继承 |
| **代码组织** | 集中在一处 | 分散在各子类中 |
| **性能影响** | 有轻微开销（每个事件多一次函数调用） | 无额外开销 |
| **类型安全** | 需手动判断事件类型 | 参数已经是具体类型 |
| **适用场景** | 跨对象拦截、无法子类化时 | 深度定制单个控件行为 |

## 2. 事件过滤器的使用

### 2.1 基本步骤

使用事件过滤器只需两步：

**第一步：安装事件过滤器**

调用目标对象的 `installEventFilter()`，将监视者注册上去：

```cpp
// 被监视对象->installEventFilter(监视者)
lineEdit->installEventFilter(this);  // this 将监视 lineEdit 的所有事件
```

**第二步：重写监视者的 `eventFilter()` 函数**

```cpp
bool MyWidget::eventFilter(QObject *watched, QEvent *event)
{
    // watched: 产生事件的对象（被监视者）
    // event:   事件本身
    // 返回 true:  事件已被处理，不再传递给目标对象
    // 返回 false: 事件继续正常传递

    if (watched == lineEdit && event->type() == QEvent::KeyPress) {
        QKeyEvent *keyEvent = static_cast<QKeyEvent *>(event);
        if (keyEvent->key() == Qt::Key_Return) {
            qDebug() << "拦截到回车键！";
            return true;  // 吃掉这个事件
        }
    }
    // 其他事件交给基类处理
    return QWidget::eventFilter(watched, event);
}
```

### 2.2 完整示例：拦截 QLineEdit 的回车键

```cpp
// mainwindow.h
#pragma once
#include <QMainWindow>
#include <QLineEdit>
#include <QLabel>

class MainWindow : public QMainWindow
{
    Q_OBJECT
public:
    explicit MainWindow(QWidget *parent = nullptr);

protected:
    bool eventFilter(QObject *watched, QEvent *event) override;

private:
    QLineEdit *m_searchEdit;
    QLabel *m_statusLabel;
};
```

```cpp
// mainwindow.cpp
#include "mainwindow.h"
#include <QVBoxLayout>
#include <QKeyEvent>
#include <QDebug>

MainWindow::MainWindow(QWidget *parent) : QMainWindow(parent)
{
    auto *central = new QWidget(this);
    auto *layout = new QVBoxLayout(central);

    m_searchEdit = new QLineEdit(this);
    m_searchEdit->setPlaceholderText("输入搜索内容，按回车搜索...");

    m_statusLabel = new QLabel("等待输入...", this);

    layout->addWidget(m_searchEdit);
    layout->addWidget(m_statusLabel);
    setCentralWidget(central);

    // 关键：安装事件过滤器
    m_searchEdit->installEventFilter(this);
}

bool MainWindow::eventFilter(QObject *watched, QEvent *event)
{
    if (watched == m_searchEdit) {
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            if (keyEvent->key() == Qt::Key_Return ||
                keyEvent->key() == Qt::Key_Enter) {
                // 执行搜索逻辑
                m_statusLabel->setText("正在搜索: " + m_searchEdit->text());
                return true;  // 阻止默认的回车行为
            }
        }
        // 获得焦点时改变样式
        if (event->type() == QEvent::FocusIn) {
            m_searchEdit->setStyleSheet("border: 2px solid #4A90D9;");
        }
        // 失去焦点时恢复样式
        if (event->type() == QEvent::FocusOut) {
            m_searchEdit->setStyleSheet("");
        }
    }
    return QMainWindow::eventFilter(watched, event);
}
```

### 2.3 监视多个对象

事件过滤器的一大优势是**一个过滤器可以同时监视多个对象**：

```cpp
class FormValidator : public QWidget
{
    Q_OBJECT
public:
    FormValidator(QWidget *parent = nullptr) : QWidget(parent)
    {
        m_nameEdit = new QLineEdit(this);
        m_emailEdit = new QLineEdit(this);
        m_phoneEdit = new QLineEdit(this);

        // 同一个过滤器监视三个控件
        m_nameEdit->installEventFilter(this);
        m_emailEdit->installEventFilter(this);
        m_phoneEdit->installEventFilter(this);
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::FocusOut) {
            // 失去焦点时统一校验
            auto *edit = qobject_cast<QLineEdit *>(watched);
            if (!edit) return false;

            if (watched == m_nameEdit) {
                validateName(edit->text());
            } else if (watched == m_emailEdit) {
                validateEmail(edit->text());
            } else if (watched == m_phoneEdit) {
                validatePhone(edit->text());
            }
        }
        return QWidget::eventFilter(watched, event);
    }

private:
    void validateName(const QString &text) {
        // 名字验证逻辑
    }
    void validateEmail(const QString &text) {
        // 邮箱验证逻辑
    }
    void validatePhone(const QString &text) {
        // 电话验证逻辑
    }

    QLineEdit *m_nameEdit;
    QLineEdit *m_emailEdit;
    QLineEdit *m_phoneEdit;
};
```

### 2.4 移除事件过滤器

不再需要监视时，调用 `removeEventFilter()` 移除：

```cpp
// 移除事件过滤器
lineEdit->removeEventFilter(this);
```

> ⚠️ 当监视者对象被销毁时，Qt 会**自动移除**所有与之关联的事件过滤器，无需手动清理。但如果被监视对象先被销毁，监视者不会收到通知，需注意对象生命周期管理。

## 3. 事件过滤器的工作原理

### 3.1 底层机制

每个 `QObject` 内部维护一个事件过滤器列表 `d->extraData->eventFilters`。当事件到达目标对象时，`QCoreApplication::notify()` 会先遍历这个列表，依次调用每个过滤器的 `eventFilter()`：

```
QCoreApplication::notify(receiver, event)
    ↓
QCoreApplication::notify_helper(receiver, event)
    ↓
遍历 receiver 的 eventFilters 列表（后安装的先调用）
    ↓ 若所有过滤器都返回 false
receiver->event(event)
    ↓
具体的 xxxEvent() 处理函数
```

关键源码逻辑（简化）：

```cpp
// Qt 源码 qcoreapplication.cpp 简化
bool QCoreApplicationPrivate::notify_helper(QObject *receiver, QEvent *event)
{
    // 1. 先让事件过滤器处理
    if (receiver->d_func()->extraData) {
        auto &filters = receiver->d_func()->extraData->eventFilters;
        // 注意：从后往前遍历（后安装的优先级更高）
        for (int i = filters.size() - 1; i >= 0; --i) {
            QObject *filter = filters.at(i);
            if (filter && filter->eventFilter(receiver, event))
                return true;  // 过滤器已处理，不再传递
        }
    }

    // 2. 过滤器没拦截，交给目标对象自己处理
    return receiver->event(event);
}
```

### 3.2 多个过滤器的调用顺序

当一个对象安装了多个事件过滤器时，**后安装的先被调用**（类似栈的 LIFO 顺序）：

```cpp
lineEdit->installEventFilter(filterA);  // 先安装
lineEdit->installEventFilter(filterB);  // 后安装

// 事件到达时的调用顺序：
// 1. filterB->eventFilter()   ← 后安装的先调用
// 2. filterA->eventFilter()   ← 先安装的后调用
// 3. lineEdit->event()        ← 如果都没拦截
```

```cpp
// 验证示例
class FilterA : public QObject
{
    bool eventFilter(QObject *obj, QEvent *event) override {
        if (event->type() == QEvent::KeyPress)
            qDebug() << "FilterA 看到了按键事件";
        return false;  // 不拦截，继续传递
    }
};

class FilterB : public QObject
{
    bool eventFilter(QObject *obj, QEvent *event) override {
        if (event->type() == QEvent::KeyPress)
            qDebug() << "FilterB 看到了按键事件";
        return false;
    }
};

// 输出：
// FilterB 看到了按键事件
// FilterA 看到了按键事件
```

### 3.3 返回值的含义

`eventFilter()` 的返回值决定了事件的命运：

| 返回值 | 含义 | 后续行为 |
|-------|------|---------|
| `true` | 事件已被过滤器处理 | 事件**停止传递**，目标对象的 `event()` 不会被调用 |
| `false` | 过滤器不处理此事件 | 事件继续传递给下一个过滤器或目标对象 |

```cpp
bool MyFilter::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::MouseButtonPress) {
        qDebug() << "拦截鼠标按下！目标对象将收不到此事件";
        return true;   // 吃掉事件
    }

    if (event->type() == QEvent::MouseMove) {
        qDebug() << "看到鼠标移动，但我只是记录日志，不拦截";
        return false;  // 继续传递
    }

    return QObject::eventFilter(watched, event);
}
```

## 4. 全局事件过滤器

### 4.1 QApplication 级别的事件过滤器

可以在 `QApplication` 对象上安装事件过滤器，这样**所有对象的所有事件**都会先经过这个过滤器：

```cpp
// main.cpp
#include <QApplication>
#include "globalfilter.h"
#include "mainwindow.h"

int main(int argc, char *argv[])
{
    QApplication app(argc, argv);

    // 安装全局事件过滤器
    GlobalFilter *globalFilter = new GlobalFilter(&app);
    app.installEventFilter(globalFilter);

    MainWindow w;
    w.show();
    return app.exec();
}
```

```cpp
// globalfilter.h
#pragma once
#include <QObject>
#include <QEvent>
#include <QKeyEvent>
#include <QDebug>

class GlobalFilter : public QObject
{
    Q_OBJECT
public:
    explicit GlobalFilter(QObject *parent = nullptr) : QObject(parent) {}

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        // 全局快捷键：Ctrl+Q 退出应用
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            if (keyEvent->modifiers() == Qt::ControlModifier &&
                keyEvent->key() == Qt::Key_Q) {
                qDebug() << "全局快捷键: Ctrl+Q 退出";
                qApp->quit();
                return true;
            }
        }

        // 全局记录所有鼠标点击（不拦截）
        if (event->type() == QEvent::MouseButtonPress) {
            qDebug() << "鼠标点击对象:" << watched->objectName()
                     << "类名:" << watched->metaObject()->className();
        }

        return QObject::eventFilter(watched, event);
    }
};
```

### 4.2 重写 QApplication::notify()

比全局事件过滤器更底层的拦截方式是重写 `QApplication::notify()`，它是事件分发的**第一入口**：

```cpp
class MyApplication : public QApplication
{
public:
    MyApplication(int &argc, char **argv) : QApplication(argc, argv) {}

    bool notify(QObject *receiver, QEvent *event) override
    {
        // 这里可以看到所有事件，甚至在过滤器之前
        // 常用于全局异常捕获
        try {
            return QApplication::notify(receiver, event);
        } catch (const std::exception &e) {
            qCritical() << "事件处理异常:" << e.what()
                        << "对象:" << receiver->objectName();
            return false;
        }
    }
};
```

### 4.3 各层拦截的优先级对比

```
最高优先级
    ↓   QApplication::notify()          // 重写 notify
    ↓   QApplication 事件过滤器          // app.installEventFilter()
    ↓   目标对象的事件过滤器              // obj->installEventFilter()
    ↓   目标对象的 QObject::event()      // 重写 event()
    ↓   具体事件处理函数                  // 重写 mousePressEvent() 等
    ↓   父对象传播                        // event->ignore() 后向上传播
最低优先级
```

## 5. 实战案例

### 5.1 实现控件的悬停高亮效果

无需子类化任何控件，统一管理悬停效果：

```cpp
class HoverHighlighter : public QObject
{
    Q_OBJECT
public:
    explicit HoverHighlighter(QObject *parent = nullptr) : QObject(parent) {}

    // 便捷方法：给控件安装高亮效果
    void install(QWidget *widget)
    {
        widget->setAttribute(Qt::WA_Hover, true);  // 启用 hover 事件
        widget->installEventFilter(this);
        m_originalStyles[widget] = widget->styleSheet();
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        auto *widget = qobject_cast<QWidget *>(watched);
        if (!widget) return false;

        if (event->type() == QEvent::HoverEnter) {
            widget->setStyleSheet(
                m_originalStyles[widget] +
                " background-color: rgba(74, 144, 217, 30);"
                " border: 1px solid #4A90D9;"
            );
        } else if (event->type() == QEvent::HoverLeave) {
            widget->setStyleSheet(m_originalStyles[widget]);
        }

        return QObject::eventFilter(watched, event);
    }

private:
    QHash<QWidget *, QString> m_originalStyles;
};

// 使用方式
auto *highlighter = new HoverHighlighter(this);
highlighter->install(button1);
highlighter->install(button2);
highlighter->install(label1);
// 三个控件自动获得了悬停高亮效果，无需子类化
```

### 5.2 实现拖拽移动无边框窗口

无边框窗口（`FramelessWindowHint`）默认无法拖拽，通过事件过滤器实现：

```cpp
class FramelessDragger : public QObject
{
    Q_OBJECT
public:
    explicit FramelessDragger(QWidget *target, QObject *parent = nullptr)
        : QObject(parent), m_target(target)
    {
        target->installEventFilter(this);
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (watched != m_target) return false;

        switch (event->type()) {
        case QEvent::MouseButtonPress: {
            auto *me = static_cast<QMouseEvent *>(event);
            if (me->button() == Qt::LeftButton) {
                m_dragging = true;
                m_dragPos = me->globalPos() - m_target->frameGeometry().topLeft();
                return true;
            }
            break;
        }
        case QEvent::MouseMove: {
            auto *me = static_cast<QMouseEvent *>(event);
            if (m_dragging && (me->buttons() & Qt::LeftButton)) {
                m_target->move(me->globalPos() - m_dragPos);
                return true;
            }
            break;
        }
        case QEvent::MouseButtonRelease: {
            m_dragging = false;
            break;
        }
        default:
            break;
        }

        return QObject::eventFilter(watched, event);
    }

private:
    QWidget *m_target = nullptr;
    bool m_dragging = false;
    QPoint m_dragPos;
};

// 使用
auto *window = new QWidget();
window->setWindowFlags(Qt::FramelessWindowHint);
new FramelessDragger(window);  // 一行代码搞定拖拽
```

### 5.3 输入框字符过滤（只允许数字输入）

```cpp
class NumericFilter : public QObject
{
    Q_OBJECT
public:
    explicit NumericFilter(QObject *parent = nullptr) : QObject(parent) {}

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            int key = keyEvent->key();

            // 允许数字键、小数点、退格、删除、方向键
            bool allowed = (key >= Qt::Key_0 && key <= Qt::Key_9) ||
                           key == Qt::Key_Period ||
                           key == Qt::Key_Backspace ||
                           key == Qt::Key_Delete ||
                           key == Qt::Key_Left ||
                           key == Qt::Key_Right ||
                           key == Qt::Key_Home ||
                           key == Qt::Key_End ||
                           // 允许 Ctrl+C/V/A/X/Z
                           (keyEvent->modifiers() == Qt::ControlModifier &&
                            (key == Qt::Key_C || key == Qt::Key_V ||
                             key == Qt::Key_A || key == Qt::Key_X ||
                             key == Qt::Key_Z));

            if (!allowed) {
                return true;  // 拦截不允许的按键
            }
        }
        return QObject::eventFilter(watched, event);
    }
};

// 使用
auto *priceEdit = new QLineEdit(this);
priceEdit->installEventFilter(new NumericFilter(this));
```

### 5.4 全局键盘快捷键管理器

```cpp
class ShortcutManager : public QObject
{
    Q_OBJECT
public:
    using Callback = std::function<void()>;

    explicit ShortcutManager(QObject *parent = nullptr) : QObject(parent)
    {
        // 安装为全局过滤器
        qApp->installEventFilter(this);
    }

    void registerShortcut(QKeySequence seq, Callback callback)
    {
        m_shortcuts[seq] = std::move(callback);
    }

    void unregisterShortcut(const QKeySequence &seq)
    {
        m_shortcuts.remove(seq);
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            int keyInt = keyEvent->key();
            Qt::KeyboardModifiers modifiers = keyEvent->modifiers();

            // 排除单独的修饰键
            if (keyInt == Qt::Key_Control || keyInt == Qt::Key_Shift ||
                keyInt == Qt::Key_Alt || keyInt == Qt::Key_Meta)
                return false;

            QKeySequence seq(keyInt | modifiers);
            if (m_shortcuts.contains(seq)) {
                m_shortcuts[seq]();
                return true;  // 拦截快捷键
            }
        }
        return QObject::eventFilter(watched, event);
    }

private:
    QMap<QKeySequence, Callback> m_shortcuts;
};

// 使用
auto *sm = new ShortcutManager(this);
sm->registerShortcut(QKeySequence("Ctrl+S"), [this]() {
    qDebug() << "保存文件";
    saveFile();
});
sm->registerShortcut(QKeySequence("Ctrl+Shift+N"), [this]() {
    qDebug() << "新建窗口";
    createNewWindow();
});
```

### 5.5 事件日志调试工具

开发调试时非常有用的事件追踪器：

```cpp
class EventLogger : public QObject
{
    Q_OBJECT
public:
    explicit EventLogger(QObject *parent = nullptr) : QObject(parent) {}

    void watch(QObject *target)
    {
        target->installEventFilter(this);
    }

    void setEventTypes(const QSet<QEvent::Type> &types)
    {
        m_watchedTypes = types;
    }

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        QEvent::Type type = event->type();

        // 如果设置了过滤类型，只记录指定类型
        if (!m_watchedTypes.isEmpty() && !m_watchedTypes.contains(type)) {
            return false;
        }

        qDebug().noquote()
            << QStringLiteral("[EventLog]")
            << QStringLiteral("对象:") << watched->objectName()
            << QStringLiteral("类:") << watched->metaObject()->className()
            << QStringLiteral("事件类型:") << type
            << eventTypeName(type);

        return false;  // 永远不拦截，只记录
    }

private:
    static QString eventTypeName(QEvent::Type type)
    {
        static const QMap<QEvent::Type, QString> names = {
            {QEvent::MouseButtonPress,   "MouseButtonPress"},
            {QEvent::MouseButtonRelease, "MouseButtonRelease"},
            {QEvent::MouseMove,          "MouseMove"},
            {QEvent::KeyPress,           "KeyPress"},
            {QEvent::KeyRelease,         "KeyRelease"},
            {QEvent::FocusIn,            "FocusIn"},
            {QEvent::FocusOut,           "FocusOut"},
            {QEvent::Enter,              "Enter"},
            {QEvent::Leave,              "Leave"},
            {QEvent::Paint,              "Paint"},
            {QEvent::Resize,             "Resize"},
            {QEvent::Show,               "Show"},
            {QEvent::Hide,               "Hide"},
            {QEvent::Close,              "Close"},
            {QEvent::Wheel,              "Wheel"},
        };
        return names.value(type, QStringLiteral("Unknown(%1)").arg(type));
    }

    QSet<QEvent::Type> m_watchedTypes;
};

// 使用
auto *logger = new EventLogger(this);
logger->setEventTypes({QEvent::KeyPress, QEvent::MouseButtonPress, QEvent::FocusIn});
logger->watch(m_lineEdit);
logger->watch(m_button);
// 控制台将输出这些控件收到的键盘、鼠标和焦点事件
```

### 5.6 实现 Esc 键关闭对话框的统一处理

```cpp
class EscCloseFilter : public QObject
{
    Q_OBJECT
public:
    explicit EscCloseFilter(QObject *parent = nullptr) : QObject(parent) {}

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            if (keyEvent->key() == Qt::Key_Escape) {
                auto *widget = qobject_cast<QWidget *>(watched);
                if (widget && widget->isWindow()) {
                    widget->close();
                    return true;
                }
            }
        }
        return QObject::eventFilter(watched, event);
    }
};

// 使用：给所有对话框统一加上 Esc 关闭功能
auto *escFilter = new EscCloseFilter(this);
dialog1->installEventFilter(escFilter);
dialog2->installEventFilter(escFilter);
settingsWindow->installEventFilter(escFilter);
```

## 6. 注意事项与常见陷阱

### 6.1 不要忘记调用基类的 eventFilter()

```cpp
// ❌ 错误：只处理了自己关心的事件，其他事件直接返回 false
bool MyWidget::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::KeyPress) {
        // 处理逻辑...
        return true;
    }
    return false;  // 基类的过滤器逻辑被跳过了！
}

// ✅ 正确：将未处理的事件交给基类
bool MyWidget::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::KeyPress) {
        // 处理逻辑...
        return true;
    }
    return QWidget::eventFilter(watched, event);  // 交给基类
}
```

### 6.2 注意对象生命周期

```cpp
// ❌ 危险：被监视对象可能在过滤器之前被销毁
class Watcher : public QObject
{
    QLineEdit *m_edit;  // 可能已经被外部销毁

    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (watched == m_edit) {  // m_edit 可能是悬空指针！
            // ...
        }
        return QObject::eventFilter(watched, event);
    }
};

// ✅ 安全：使用 QPointer 跟踪对象存活状态
class Watcher : public QObject
{
    QPointer<QLineEdit> m_edit;

    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (m_edit && watched == m_edit) {  // QPointer 会自动置空
            // ...
        }
        return QObject::eventFilter(watched, event);
    }
};
```

### 6.3 避免在事件过滤器中做耗时操作

```cpp
// ❌ 不好：阻塞事件循环
bool MyFilter::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::MouseButtonPress) {
        // 网络请求、文件 IO、复杂计算都不应该在这里做
        QThread::sleep(2);          // 界面会卡顿 2 秒！
        downloadFile(url);          // 阻塞操作！
    }
    return QObject::eventFilter(watched, event);
}

// ✅ 正确：触发异步操作或投递事件
bool MyFilter::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::MouseButtonPress) {
        // 使用 QTimer 延迟执行，不阻塞事件循环
        QTimer::singleShot(0, this, [this]() {
            doHeavyWork();
        });
    }
    return QObject::eventFilter(watched, event);
}
```

### 6.4 不要在 eventFilter 中删除被监视的对象

```cpp
// ❌ 危险：在过滤器中删除对象会导致崩溃
bool MyFilter::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::Close) {
        delete watched;   // 未定义行为！Qt 还在使用这个对象
        return true;
    }
    return QObject::eventFilter(watched, event);
}

// ✅ 安全：使用 deleteLater() 延迟删除
bool MyFilter::eventFilter(QObject *watched, QEvent *event)
{
    if (event->type() == QEvent::Close) {
        watched->deleteLater();  // 安全：在事件循环空闲时删除
        return true;
    }
    return QObject::eventFilter(watched, event);
}
```

### 6.5 重复安装同一个过滤器

```cpp
// 同一个过滤器对同一个对象安装多次不会产生多条记录
lineEdit->installEventFilter(this);
lineEdit->installEventFilter(this);  // Qt 内部会检查，不会重复添加

// 但如果你创建了多个过滤器实例，每个都会被调用
auto *filter1 = new MyFilter(this);
auto *filter2 = new MyFilter(this);
lineEdit->installEventFilter(filter1);  // 会被调用
lineEdit->installEventFilter(filter2);  // 也会被调用
```

### 6.6 性能考虑

```cpp
// ❌ 不推荐：全局过滤器中做复杂判断
bool GlobalFilter::eventFilter(QObject *watched, QEvent *event)
{
    // Paint 事件、Timer 事件每秒可能触发上千次

    QString className = watched->metaObject()->className();
    if (className == "QLineEdit") {  // 字符串比较很慢
        // ...
    }
    return false;
}

// ✅ 推荐：先判断事件类型，快速过滤掉不关心的事件
bool GlobalFilter::eventFilter(QObject *watched, QEvent *event)
{
    // 只关心按键事件，其他事件直接放过
    if (event->type() != QEvent::KeyPress)
        return false;

    // 用 qobject_cast 代替字符串比较
    auto *lineEdit = qobject_cast<QLineEdit *>(watched);
    if (!lineEdit)
        return false;

    // 具体处理逻辑...
    return false;
}
```

## 7. 事件过滤器在 QML 中的应用

在 QML/C++ 混合开发中，事件过滤器可以从 C++ 端拦截 QML 控件的事件：

```cpp
// keyinterceptor.h
#pragma once
#include <QObject>
#include <QEvent>
#include <QKeyEvent>
#include <QQuickItem>

class KeyInterceptor : public QObject
{
    Q_OBJECT
    Q_PROPERTY(QObject* target READ target WRITE setTarget NOTIFY targetChanged)

public:
    explicit KeyInterceptor(QObject *parent = nullptr) : QObject(parent) {}

    QObject *target() const { return m_target; }

    void setTarget(QObject *target)
    {
        if (m_target == target) return;

        if (m_target)
            m_target->removeEventFilter(this);

        m_target = target;

        if (m_target)
            m_target->installEventFilter(this);

        emit targetChanged();
    }

signals:
    void targetChanged();
    void enterPressed();
    void escapePressed();

protected:
    bool eventFilter(QObject *watched, QEvent *event) override
    {
        if (event->type() == QEvent::KeyPress) {
            auto *keyEvent = static_cast<QKeyEvent *>(event);
            switch (keyEvent->key()) {
            case Qt::Key_Return:
            case Qt::Key_Enter:
                emit enterPressed();
                return true;
            case Qt::Key_Escape:
                emit escapePressed();
                return true;
            }
        }
        return QObject::eventFilter(watched, event);
    }

private:
    QObject *m_target = nullptr;
};
```

```cpp
// main.cpp 注册类型
qmlRegisterType<KeyInterceptor>("App.Utils", 1, 0, "KeyInterceptor");
```

```qml

    placeholderText: "搜索..."

    KeyInterceptor {
        target: searchField
        onEnterPressed: {

            performSearch(searchField.text)
        }
        onEscapePressed: {
            searchField.clear()
            searchField.focus = false
        }
    }
}
```

## 8. 总结

### 8.1 何时使用事件过滤器

✅ **适合使用事件过滤器的场景**：
- 需要监视不可子类化的对象（第三方库的控件）
- 一个逻辑需要统一监视多个控件的事件
- 临时性的事件拦截（调试、日志记录）
- 在不修改原有类的前提下扩展行为（开闭原则）
- 全局快捷键、全局事件统计
- 跨层级的事件拦截（父窗口拦截子控件的事件）

❌ **不适合使用事件过滤器的场景**：
- 只需要处理自身的事件 → 直接重写事件处理函数
- 简单的信号响应 → 使用信号槽连接
- 需要深度定制绘制行为 → 子类化并重写 `paintEvent()`
- 性能极度敏感的路径 → 避免全局过滤器的额外开销

### 8.2 事件处理的完整优先级链

```
┌────────────────────────────────────────────────────────────┐
│ 1. QApplication::notify()        最高优先级，看到所有事件   │
│ 2. QApplication 上的事件过滤器    全局拦截                  │
│ 3. 目标对象上的事件过滤器         对象级拦截                │
│ 4. QObject::event()              事件分发虚函数             │
│ 5. 具体事件处理函数               mousePressEvent() 等     │
│ 6. 父对象传播                     ignore() 后沿对象树上传  │
└────────────────────────────────────────────────────────────┘
```

### 8.3 最佳实践速查

| 实践 | 原因 |
|------|------|
| 先判断事件类型再做其他判断 | 性能优化，快速过滤掉不关心的事件 |
| `eventFilter()` 最后调用基类版本 | 确保基类的过滤逻辑不被跳过 |
| 使用 `qobject_cast` 而非字符串比较 | 类型安全且性能更好 |
| 使用 `QPointer` 跟踪被监视对象 | 避免悬空指针 |
| 不在过滤器中做阻塞操作 | 避免卡住事件循环 |
| 不在过滤器中 `delete` 对象 | 使用 `deleteLater()` 代替 |
| 只在需要时才安装全局过滤器 | 全局过滤器会影响所有事件的分发性能 |
