# 定时器

## 1. 定时器概述

### 1.1 Qt 定时器的两种使用方式

Qt 提供两种定时器机制，分别面向不同的使用场景：

| 方式 | 核心类/方法 | 特点 |
|------|-----------|------|
| **QObject 基础定时器** | `QObject::startTimer()` + `timerEvent()` | 轻量、高效，基于事件循环，需重写虚函数 |
| **QTimer 高级定时器** | `QTimer` 类 | 封装更完善，支持信号槽，使用更便捷 |

两种方式底层都依赖事件循环（`QEventLoop`），定时器到期时系统投递 `QTimerEvent`（type = `QEvent::Timer`），区别仅在上层 API 的封装程度。

### 1.2 定时器精度

Qt 5 引入了定时器精度类型 `Qt::TimerType`，可在精度和功耗之间取舍：

```cpp
enum Qt::TimerType {
    Qt::PreciseTimer   = 0,  // 精确定时器，毫秒级精度，始终准时触发
    Qt::CoarseTimer    = 1,  // 粗略定时器（默认），允许最多 5% 的误差，节省功耗
    Qt::VeryCoarseTimer = 2  // 非常粗略，精度约 1 秒，最省功耗
};
```

- **PreciseTimer**：适用于动画、音视频同步等要求严格的场景
- **CoarseTimer**：适用于大多数业务定时任务，操作系统可对齐多个定时器减少唤醒次数
- **VeryCoarseTimer**：适用于分钟级心跳检测、自动保存等场景

## 2. QObject 基础定时器

### 2.1 核心 API

`QObject` 自身内置了定时器支持，无需引入额外类：

```cpp
class QObject {
public:
    // 启动定时器，返回定时器 ID（非零表示成功）
    int startTimer(int interval, Qt::TimerType timerType = Qt::CoarseTimer);
    // Qt 5.9+ 支持 std::chrono
    int startTimer(std::chrono::milliseconds time, Qt::TimerType timerType = Qt::CoarseTimer);

    // 停止指定 ID 的定时器
    void killTimer(int id);

protected:
    // 定时器事件回调（需重写）
    virtual void timerEvent(QTimerEvent *event);
};
```

### 2.2 QTimerEvent 事件

定时器到期时，事件循环投递 `QTimerEvent`：

```cpp
class QTimerEvent : public QEvent {
public:
    explicit QTimerEvent(int timerId);
    int timerId() const;  // 返回触发此事件的定时器 ID
};
```

### 2.3 完整示例：数字时钟

```cpp
// digitalclock.h
#pragma once
#include <QLabel>

class DigitalClock : public QLabel
{
    Q_OBJECT
public:
    explicit DigitalClock(QWidget *parent = nullptr);

protected:
    void timerEvent(QTimerEvent *event) override;

private:
    void updateDisplay();
    int m_timerId = 0;
};

// digitalclock.cpp
#include "digitalclock.h"
#include <QTime>
#include <QTimerEvent>

DigitalClock::DigitalClock(QWidget *parent)
    : QLabel(parent)
{
    setAlignment(Qt::AlignCenter);
    setStyleSheet("font-size: 48px; font-family: 'Consolas';");
    updateDisplay();

    // 每秒触发一次，使用粗略定时器（默认）
    m_timerId = startTimer(1000);
}

void DigitalClock::timerEvent(QTimerEvent *event)
{
    if (event->timerId() == m_timerId) {
        updateDisplay();
    } else {
        // 不属于本对象的定时器，交给基类处理
        QLabel::timerEvent(event);
    }
}

void DigitalClock::updateDisplay()
{
    setText(QTime::currentTime().toString("HH:mm:ss"));
}
```

### 2.4 管理多个定时器

一个 `QObject` 可同时拥有多个定时器，通过 ID 区分：

```cpp
class SensorMonitor : public QObject
{
    Q_OBJECT
public:
    SensorMonitor(QObject *parent = nullptr) : QObject(parent)
    {
        // 快速采样：每 100ms
        m_fastTimerId = startTimer(100, Qt::PreciseTimer);
        // 慢速上报：每 5s
        m_slowTimerId = startTimer(5000, Qt::CoarseTimer);
        // 心跳检测：每 60s
        m_heartbeatId = startTimer(60000, Qt::VeryCoarseTimer);
    }

    ~SensorMonitor()
    {
        // 析构时自动 killTimer 所有属于该对象的定时器（Qt 内部处理）
        // 但在生命周期中间停止时需手动调用 killTimer
    }

protected:
    void timerEvent(QTimerEvent *event) override
    {
        const int id = event->timerId();
        if (id == m_fastTimerId) {
            readSensorData();
        } else if (id == m_slowTimerId) {
            reportToServer();
        } else if (id == m_heartbeatId) {
            sendHeartbeat();
        }
    }

private:
    void readSensorData();
    void reportToServer();
    void sendHeartbeat();

    int m_fastTimerId = 0;
    int m_slowTimerId = 0;
    int m_heartbeatId = 0;
};
```

> **注意**：当定时器数量较多时，`timerEvent` 中的 if-else 链会变得冗长，此时推荐改用 `QTimer` + 信号槽。

## 3. QTimer 高级定时器

### 3.1 核心 API

`QTimer` 是 `QObject` 基础定时器的高层封装，通过信号槽机制通知超时：

```cpp
class QTimer : public QObject {
    Q_OBJECT
    Q_PROPERTY(bool singleShot READ isSingleShot WRITE setSingleShot)
    Q_PROPERTY(int interval READ interval WRITE setInterval)
    Q_PROPERTY(int remainingTime READ remainingTime)
    Q_PROPERTY(Qt::TimerType timerType READ timerType WRITE setTimerType)
    Q_PROPERTY(bool active READ isActive)

public:
    explicit QTimer(QObject *parent = nullptr);

    bool isActive() const;        // 定时器是否正在运行
    int timerId() const;          // 底层定时器 ID
    int interval() const;         // 获取间隔（ms）
    void setInterval(int msec);   // 设置间隔
    int remainingTime() const;    // 距下次触发的剩余时间（ms），未运行返回 -1
    
    Qt::TimerType timerType() const;
    void setTimerType(Qt::TimerType type);

    bool isSingleShot() const;           // 是否单次触发
    void setSingleShot(bool singleShot); // 设置单次触发

    // 静态便捷方法（单次触发）
    static void singleShot(int msec, const QObject *receiver, const char *member);
    // Qt 5.4+ 函子重载
    static void singleShot(int msec, const QObject *context, Functor functor);
    static void singleShot(int msec, Qt::TimerType timerType, 
                           const QObject *context, Functor functor);

public slots:
    void start(int msec);  // 设置间隔并启动
    void start();          // 以当前 interval 启动
    void stop();           // 停止定时器

signals:
    void timeout();        // 定时器超时信号
};
```

### 3.2 基本用法

```cpp
// 方式1：构造 + 配置 + 启动
auto *timer = new QTimer(this);
timer->setInterval(1000);
timer->setTimerType(Qt::CoarseTimer);
connect(timer, &QTimer::timeout, this, &MyWidget::onUpdate);
timer->start();

// 方式2：直接 start(interval)
auto *timer = new QTimer(this);
connect(timer, &QTimer::timeout, this, &MyWidget::onUpdate);
timer->start(1000);  // 设置间隔并启动

// 方式3：Lambda
auto *timer = new QTimer(this);
connect(timer, &QTimer::timeout, this, [this]() {
    qDebug() << "Timer fired at" << QTime::currentTime();
});
timer->start(2000);
```

### 3.3 单次定时器（Single Shot）

单次定时器触发一次后自动停止，常用于延迟执行：

```cpp
// 方式1：对象方法
auto *timer = new QTimer(this);
timer->setSingleShot(true);
connect(timer, &QTimer::timeout, this, &MyWidget::delayedInit);
timer->start(500);  // 500ms 后执行一次

// 方式2：静态方法（推荐，更简洁）
QTimer::singleShot(500, this, &MyWidget::delayedInit);

// 方式3：Lambda
QTimer::singleShot(500, this, [this]() {
    statusBar()->showMessage("初始化完成", 3000);
});

// 方式4：指定精度
QTimer::singleShot(16, Qt::PreciseTimer, this, [this]() {
    renderNextFrame();
});
```

### 3.4 零超时定时器

间隔为 0 的定时器会在事件队列中所有窗口事件处理完后立即触发，常用于在空闲时执行后台任务，同时保持 UI 响应：

```cpp
class BackgroundProcessor : public QObject
{
    Q_OBJECT
public:
    void startProcessing(const QStringList &files)
    {
        m_files = files;
        m_index = 0;

        // 零超时定时器：在事件循环空闲时执行
        m_timer = new QTimer(this);
        connect(m_timer, &QTimer::timeout, this, &BackgroundProcessor::processNext);
        m_timer->start(0);  // 间隔 = 0
    }

private slots:
    void processNext()
    {
        if (m_index >= m_files.size()) {
            m_timer->stop();
            emit finished();
            return;
        }

        // 每次只处理一个文件，让出事件循环给 UI
        processFile(m_files.at(m_index));
        m_index++;

        emit progressChanged(m_index, m_files.size());
    }

signals:
    void progressChanged(int current, int total);
    void finished();

private:
    void processFile(const QString &path);
    QTimer *m_timer = nullptr;
    QStringList m_files;
    int m_index = 0;
};
```

> **原理**：零超时定时器的优先级低于窗口系统事件和已投递的事件（`QCoreApplication::sendPostedEvents`），因此不会阻塞 UI 刷新。

### 3.5 完整示例：倒计时控件

```cpp
// countdownwidget.h
#pragma once
#include <QWidget>

class QLabel;
class QPushButton;
class QTimer;

class CountdownWidget : public QWidget
{
    Q_OBJECT
public:
    explicit CountdownWidget(int totalSeconds = 60, QWidget *parent = nullptr);

signals:
    void countdownFinished();

private slots:
    void onTick();
    void onStartPause();
    void onReset();

private:
    void updateDisplay();

    QLabel *m_display;
    QPushButton *m_startPauseBtn;
    QPushButton *m_resetBtn;
    QTimer *m_timer;

    int m_totalSeconds;
    int m_remaining;
    bool m_running = false;
};

// countdownwidget.cpp
#include "countdownwidget.h"
#include <QLabel>
#include <QPushButton>
#include <QVBoxLayout>
#include <QHBoxLayout>
#include <QTimer>

CountdownWidget::CountdownWidget(int totalSeconds, QWidget *parent)
    : QWidget(parent)
    , m_totalSeconds(totalSeconds)
    , m_remaining(totalSeconds)
{
    m_display = new QLabel(this);
    m_display->setAlignment(Qt::AlignCenter);
    m_display->setStyleSheet("font-size: 64px; font-weight: bold; color: #333;");

    m_startPauseBtn = new QPushButton("开始", this);
    m_resetBtn = new QPushButton("重置", this);

    auto *btnLayout = new QHBoxLayout;
    btnLayout->addWidget(m_startPauseBtn);
    btnLayout->addWidget(m_resetBtn);

    auto *mainLayout = new QVBoxLayout(this);
    mainLayout->addWidget(m_display);
    mainLayout->addLayout(btnLayout);

    // 创建定时器但不启动
    m_timer = new QTimer(this);
    m_timer->setInterval(1000);
    m_timer->setTimerType(Qt::CoarseTimer);
    connect(m_timer, &QTimer::timeout, this, &CountdownWidget::onTick);

    connect(m_startPauseBtn, &QPushButton::clicked, this, &CountdownWidget::onStartPause);
    connect(m_resetBtn, &QPushButton::clicked, this, &CountdownWidget::onReset);

    updateDisplay();
}

void CountdownWidget::onTick()
{
    if (--m_remaining <= 0) {
        m_remaining = 0;
        m_timer->stop();
        m_running = false;
        m_startPauseBtn->setText("开始");
        m_display->setStyleSheet("font-size: 64px; font-weight: bold; color: red;");
        emit countdownFinished();
    }
    updateDisplay();
}

void CountdownWidget::onStartPause()
{
    if (m_running) {
        m_timer->stop();
        m_startPauseBtn->setText("继续");
    } else {
        if (m_remaining <= 0) return;
        m_timer->start();
        m_startPauseBtn->setText("暂停");
    }
    m_running = !m_running;
}

void CountdownWidget::onReset()
{
    m_timer->stop();
    m_running = false;
    m_remaining = m_totalSeconds;
    m_startPauseBtn->setText("开始");
    m_display->setStyleSheet("font-size: 64px; font-weight: bold; color: #333;");
    updateDisplay();
}

void CountdownWidget::updateDisplay()
{
    int min = m_remaining / 60;
    int sec = m_remaining % 60;
    m_display->setText(QString("%1:%2")
        .arg(min, 2, 10, QChar('0'))
        .arg(sec, 2, 10, QChar('0')));
}
```

## 4. QBasicTimer 轻量级封装

`QBasicTimer` 是介于裸 `startTimer()` 和 `QTimer` 之间的轻量值类型封装，无需继承 `QObject`，没有信号槽开销：

```cpp
class QBasicTimer {
public:
    QBasicTimer();
    ~QBasicTimer();                         // 析构时自动 stop

    bool isActive() const;                   // 是否在运行
    int timerId() const;                     // 底层 ID

    void start(int msec, QObject *obj);      // 绑定到目标对象
    void start(int msec, Qt::TimerType timerType, QObject *obj);
    void stop();                             // 停止
};
```

### 4.1 使用示例

```cpp
class AnimationWidget : public QWidget
{
    Q_OBJECT
public:
    AnimationWidget(QWidget *parent = nullptr) : QWidget(parent)
    {
        m_timer.start(16, Qt::PreciseTimer, this);  // ~60 FPS
    }

protected:
    void timerEvent(QTimerEvent *event) override
    {
        if (event->timerId() == m_timer.timerId()) {
            m_angle += 2.0;
            if (m_angle >= 360.0) m_angle -= 360.0;
            update();  // 触发 paintEvent
        }
    }

    void paintEvent(QPaintEvent *) override
    {
        QPainter painter(this);
        painter.setRenderHint(QPainter::Antialiasing);
        painter.translate(width() / 2, height() / 2);
        painter.rotate(m_angle);
        painter.drawRect(-50, -50, 100, 100);
    }

private:
    QBasicTimer m_timer;
    qreal m_angle = 0.0;
};
```

### 4.2 三种方式对比

| 特性 | `startTimer()` | `QBasicTimer` | `QTimer` |
|------|----------------|---------------|----------|
| **需要继承 QObject** | 是 | 否（但目标需是 QObject） | 否 |
| **通知方式** | 重写 `timerEvent` | 重写 `timerEvent` | 信号 `timeout()` |
| **自动停止** | 对象销毁时 | 析构时 `stop()` | 对象销毁时 |
| **内存开销** | 无额外对象 | 仅一个 int | 一个完整 QObject |
| **信号槽支持** | ❌ | ❌ | ✅ |
| **单次触发** | 需手动 `killTimer` | 需手动 `stop` | `setSingleShot(true)` |
| **适用场景** | 极致性能 | 性能敏感的重复动画 | 通用业务逻辑 |

## 5. 实战场景与技巧

### 5.1 防抖（Debounce）

搜索框输入时，用户停止输入一段时间后才执行搜索，避免频繁请求：

```cpp
class SearchBox : public QLineEdit
{
    Q_OBJECT
public:
    SearchBox(QWidget *parent = nullptr)
        : QLineEdit(parent)
    {
        m_debounceTimer = new QTimer(this);
        m_debounceTimer->setSingleShot(true);
        m_debounceTimer->setInterval(300);  // 300ms 防抖

        connect(this, &QLineEdit::textChanged, this, [this]() {
            // 每次输入都重新计时
            m_debounceTimer->start();
        });

        connect(m_debounceTimer, &QTimer::timeout, this, [this]() {
            emit searchRequested(text());
        });
    }

signals:
    void searchRequested(const QString &keyword);

private:
    QTimer *m_debounceTimer;
};
```

### 5.2 节流（Throttle）

限制高频事件的处理频率，如窗口 resize 时避免过多布局计算：

```cpp
class ThrottledWidget : public QWidget
{
    Q_OBJECT
public:
    ThrottledWidget(QWidget *parent = nullptr)
        : QWidget(parent)
    {
        m_throttleTimer = new QTimer(this);
        m_throttleTimer->setSingleShot(true);
        m_throttleTimer->setInterval(100);  // 最多 100ms 执行一次

        connect(m_throttleTimer, &QTimer::timeout, this, [this]() {
            doExpensiveLayout();
        });
    }

protected:
    void resizeEvent(QResizeEvent *event) override
    {
        QWidget::resizeEvent(event);
        // 如果定时器未运行，启动它；如果已在运行，忽略本次触发
        if (!m_throttleTimer->isActive()) {
            m_throttleTimer->start();
        }
    }

private:
    void doExpensiveLayout();
    QTimer *m_throttleTimer;
};
```

### 5.3 自动保存

定期自动保存文档：

```cpp
class Document : public QObject
{
    Q_OBJECT
public:
    Document(QObject *parent = nullptr) : QObject(parent)
    {
        m_autoSaveTimer = new QTimer(this);
        m_autoSaveTimer->setInterval(30000);  // 30 秒
        m_autoSaveTimer->setTimerType(Qt::VeryCoarseTimer);  // 无需高精度

        connect(m_autoSaveTimer, &QTimer::timeout, this, [this]() {
            if (m_modified) {
                save();
                m_modified = false;
            }
        });

        m_autoSaveTimer->start();
    }

    void setContent(const QString &content)
    {
        if (m_content != content) {
            m_content = content;
            m_modified = true;
        }
    }

private:
    void save();
    QTimer *m_autoSaveTimer;
    QString m_content;
    bool m_modified = false;
};
```

### 5.4 超时机制

网络请求或长时间操作的超时控制：

```cpp
class NetworkRequest : public QObject
{
    Q_OBJECT
public:
    void sendRequest(const QUrl &url, int timeoutMs = 10000)
    {
        auto *reply = m_manager.get(QNetworkRequest(url));

        // 设置超时
        QTimer::singleShot(timeoutMs, this, [reply, this]() {
            if (reply->isRunning()) {
                reply->abort();
                emit requestTimeout();
            }
        });

        connect(reply, &QNetworkReply::finished, this, [reply, this]() {
            if (reply->error() == QNetworkReply::NoError) {
                emit requestFinished(reply->readAll());
            } else if (reply->error() != QNetworkReply::OperationCanceledError) {
                emit requestError(reply->errorString());
            }
            reply->deleteLater();
        });
    }

signals:
    void requestFinished(const QByteArray &data);
    void requestError(const QString &error);
    void requestTimeout();

private:
    QNetworkAccessManager m_manager;
};
```

### 5.5 帧率控制的游戏循环

```cpp
class GameEngine : public QObject
{
    Q_OBJECT
public:
    GameEngine(QObject *parent = nullptr) : QObject(parent)
    {
        m_timer = new QTimer(this);
        m_timer->setTimerType(Qt::PreciseTimer);
        connect(m_timer, &QTimer::timeout, this, &GameEngine::gameLoop);
    }

    {

private:

        renderFrame();
    }

    void updateLogic(qreal dt);
    void renderFrame();

    QTimer *m_timer;
    QElapsedTimer m_elapsed;
    qreal m_targetInterval = 16.67;
};
```

## 6. 线程与定时器

### 6.1 线程亲和性规则

**关键规则**：定时器只能在创建它的线程中启动和停止。`QTimer` 的 `start()` 和 `stop()` 必须在其所属线程的事件循环中调用。

```cpp
// ❌ 错误：在主线程创建 QTimer，在子线程启动
QTimer *timer = new QTimer(this);  // 主线程
std::thread([timer]() {
    timer->start(1000);  // 错误！跨线程调用
}).detach();

// ✅ 正确：通过信号槽跨线程控制
// 方式1：使用 QMetaObject::invokeMethod
QMetaObject::invokeMethod(timer, "start", Qt::QueuedConnection, Q_ARG(int, 1000));

// 方式2：在工作线程内创建和管理
class Worker : public QObject
{
    Q_OBJECT
public slots:
    void start()
    {
        // 在 Worker 所在线程的事件循环中创建和启动
        auto *timer = new QTimer(this);
        connect(timer, &QTimer::timeout, this, &Worker::doWork);
        timer->start(1000);
    }
    void doWork() { /* ... */ }
};

// 使用：
auto *thread = new QThread;
auto *worker = new Worker;
worker->moveToThread(thread);
connect(thread, &QThread::started, worker, &Worker::start);
thread->start();
```

### 6.2 子线程中使用定时器

```cpp
class TimerThread : public QThread
{
    Q_OBJECT
protected:
    void run() override
    {
        // 子线程必须有自己的事件循环才能使用 QTimer
        QTimer timer;
        connect(&timer, &QTimer::timeout, this, [this]() {
            // 在子线程中执行
            emit dataReady(collectData());
        });
        timer.start(500);

        exec();  // 启动事件循环，定时器才能工作
    }

signals:
    void dataReady(const QByteArray &data);
};
```

> **要点**：没有事件循环，定时器就无法工作。`QThread::run()` 默认调用 `exec()` 进入事件循环，但如果你重写了 `run()` 而没调用 `exec()`，则定时器不会触发。

## 7. 注意事项与常见陷阱

### 7.1 定时器的精度不是保证

```
设置 interval = 100ms
实际触发间隔可能为：98ms、102ms、105ms、97ms ...
```

操作系统调度、事件队列积压、CPU 负载都会影响精度。**不要用定时器累加来计时**，应使用 `QElapsedTimer` 测量真实经过的时间：

```cpp
// ❌ 错误：用定时器次数计算时间
int m_count = 0;
// timeout 槽中：
m_count++;
qreal elapsed = m_count * 0.1;  // 不准确！会累积误差

// ✅ 正确：用 QElapsedTimer 测量真实时间
QElapsedTimer m_elapsed;
m_elapsed.start();
// timeout 槽中：
qreal elapsed = m_elapsed.elapsed() / 1000.0;  // 精确
```

### 7.2 避免在 timerEvent 中做耗时操作

定时器回调在主线程的事件循环中执行。如果回调耗时过长，会阻塞 UI：

```cpp
// ❌ 耗时操作阻塞 UI
void MyWidget::timerEvent(QTimerEvent *event) {
    heavyComputation();  // 耗时 500ms，界面冻结！
}

// ✅ 将耗时任务放到工作线程
void MyWidget::onTimeout() {
    QtConcurrent::run([this]() {
        auto result = heavyComputation();
        QMetaObject::invokeMethod(this, [this, result]() {
            updateUI(result);  // 回到主线程更新 UI
        });
    });
}
```

### 7.3 重复调用 start() 会重置定时器

```cpp
QTimer *timer = new QTimer(this);
timer->start(1000);
// ... 一段时间后 ...
timer->start(1000);  // 重新开始计时！不是无操作！
timer->start(2000);  // 同时也会更新 interval
```

如果已在运行的定时器再次调用 `start()`，它会先隐式 `stop()` 再重新启动。这在防抖场景中是期望行为，但在其他场景中可能导致意外重置。

### 7.4 定时器 ID 的生命周期

```cpp
int id = startTimer(1000);
killTimer(id);
// id 现在是无效的，不要再使用
// Qt 可能会将此 ID 分配给后续的新定时器

killTimer(id);  // ⚠️ 重复 kill 同一个 ID，行为未定义
```

### 7.5 QTimer 与 QObject 生命周期

```cpp
// QTimer 作为子对象，父对象销毁时自动销毁
auto *timer = new QTimer(this);  // this 是父对象

// ⚠️ 注意：Lambda 捕获中的悬垂指针
QTimer::singleShot(5000, this, [widget]() {
    widget->update();  // 如果 widget 已销毁，崩溃！
});

// ✅ 安全做法：传入 context 对象（第二个参数）
// 当 context 对象销毁时，Lambda 不会被调用
QTimer::singleShot(5000, widget, [widget]() {
    widget->update();  // 安全：widget 销毁后不会执行
});
```

## 8. QTimer 内部实现原理

### 8.1 事件循环驱动

```
QTimer::start()
  └─→ QObject::startTimer()
        └─→ QAbstractEventDispatcher::registerTimer()
              └─→ 平台原生定时器（Windows: SetTimer/timeSetEvent, Linux: timerfd）

定时器到期：
  OS 通知 → QAbstractEventDispatcher → 生成 QTimerEvent → 放入事件队列
  → QCoreApplication::processEvents() → QObject::timerEvent()
  → QTimer 内部拦截 → 发射 timeout() 信号
```

### 8.2 QTimer 的 timerEvent 实现

`QTimer` 内部通过重写 `timerEvent` 来发射 `timeout()` 信号：

```cpp
// Qt 源码简化版
void QTimer::timerEvent(QTimerEvent *e)
{
    if (e->timerId() == d->id) {
        if (d->single)
            stop();  // 单次定时器自动停止
        emit timeout();
    }
}
```

这就是为什么 `QTimer` 既是 QObject 定时器的封装，又能提供信号槽接口。

## 9. 总结速查表

| 需求 | 推荐方案 | 示例 |
|------|---------|------|
| 简单延迟执行 | `QTimer::singleShot()` | `QTimer::singleShot(200, this, &X::init)` |
| 周期性任务 | `QTimer` + `timeout` 信号 | `timer->start(1000)` |
| 高性能动画/渲染 | `QBasicTimer` + `PreciseTimer` | `m_timer.start(16, Qt::PreciseTimer, this)` |
| 大量并发定时器 | `QObject::startTimer()` | 避免大量 QTimer 对象开销 |
| 防抖 | `QTimer` + `setSingleShot(true)` | 每次输入重新 `start()` |
| 节流 | `QTimer` + `isActive()` 检查 | 仅在未运行时 `start()` |
| 空闲处理 | `QTimer` + `interval(0)` | 零超时定时器 |
| 子线程定时任务 | `QTimer` + `QThread::exec()` | 确保有事件循环 |
| 精确耗时测量 | `QElapsedTimer`（非定时器） | `elapsed.nsecsElapsed()` |
