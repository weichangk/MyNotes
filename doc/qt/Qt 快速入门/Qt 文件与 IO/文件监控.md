# 文件监控

`QFileSystemWatcher` 是 Qt 提供的跨平台文件系统监控类，能够实时检测文件和目录的变化（内容修改、创建、删除、重命名等）。底层利用操作系统原生通知机制（Windows 的 `ReadDirectoryChangesW`、Linux 的 `inotify`、macOS 的 `kqueue`），无需轮询，效率极高。

---

## 1. 核心 API

### 1.1 类概览

```cpp
#include <QFileSystemWatcher>

class QFileSystemWatcher : public QObject {
    Q_OBJECT
public:
    QFileSystemWatcher(QObject *parent = nullptr);
    QFileSystemWatcher(const QStringList &paths, QObject *parent = nullptr);

    // 添加监控
    bool addPath(const QString &path);
    QStringList addPaths(const QStringList &paths);

    // 移除监控
    bool removePath(const QString &path);
    QStringList removePaths(const QStringList &paths);

    // 查询当前监控列表
    QStringList files() const;        // 被监控的文件列表
    QStringList directories() const;  // 被监控的目录列表

signals:
    void fileChanged(const QString &path);       // 文件被修改或删除
    void directoryChanged(const QString &path);  // 目录内容发生变化
};
```

### 1.2 两个信号的区别

| 信号 | 触发条件 | 监控粒度 |
|---|---|---|
| `fileChanged(path)` | 被监控的**文件**内容修改、属性变更、被删除 | 精确到单个文件 |
| `directoryChanged(path)` | 被监控的**目录**中有文件新增、删除、重命名 | 只通知目录变化，不具体说明哪个文件 |

> **注意**：`directoryChanged` 不会告诉你**哪个文件**变了，只告诉你**哪个目录**有变化。需要自行对比前后的目录内容来确定具体差异。

## 2. 基本用法

### 2.1 监控单个文件

```cpp
#include <QFileSystemWatcher>
#include <QDebug>

class ConfigMonitor : public QObject {
    Q_OBJECT
public:
    ConfigMonitor(const QString &configPath, QObject *parent = nullptr)
        : QObject(parent), m_path(configPath) {

        m_watcher = new QFileSystemWatcher(this);

        if (!m_watcher->addPath(configPath)) {
            qWarning() << "Failed to watch:" << configPath;
            return;
        }

        connect(m_watcher, &QFileSystemWatcher::fileChanged,
                this, &ConfigMonitor::onFileChanged);

        qDebug() << "Watching:" << configPath;
    }

private slots:
    void onFileChanged(const QString &path) {
        qDebug() << "File changed:" << path;

        // 重新加载配置
        loadConfig(path);

        // ⚠ 关键：某些编辑器保存时会先删除再创建文件
        // 此时 watcher 会自动移除对该文件的监控
        // 需要重新添加
        if (!m_watcher->files().contains(path)) {
            // 文件可能正在被重建，稍作延迟
            QTimer::singleShot(100, this, [this, path]() {
                if (QFile::exists(path)) {
                    m_watcher->addPath(path);
                    qDebug() << "Re-added watch for:" << path;
                }
            });
        }
    }

private:
    void loadConfig(const QString &path) {
        QFile file(path);
        if (file.open(QIODevice::ReadOnly | QIODevice::Text)) {
            QString content = QTextStream(&file).readAll();
            qDebug() << "Config content:" << content.left(100);
            emit configReloaded();
        }
    }

    QFileSystemWatcher *m_watcher;
    QString m_path;

signals:
    void configReloaded();
};
```

### 2.2 监控目录

```cpp
class DirectoryMonitor : public QObject {
    Q_OBJECT
public:
    DirectoryMonitor(const QString &dirPath, QObject *parent = nullptr)
        : QObject(parent), m_dirPath(dirPath) {

        m_watcher = new QFileSystemWatcher({dirPath}, this);

        // 记录初始文件列表
        m_knownFiles = QDir(dirPath).entryList(QDir::Files | QDir::NoDotAndDotDot);

        connect(m_watcher, &QFileSystemWatcher::directoryChanged,
                this, &DirectoryMonitor::onDirectoryChanged);
    }

private slots:
    void onDirectoryChanged(const QString &path) {
        QDir dir(path);
        QStringList currentFiles = dir.entryList(QDir::Files | QDir::NoDotAndDotDot);

        // 检测新增文件
        for (const QString &file : currentFiles) {
            if (!m_knownFiles.contains(file)) {
                qDebug() << "File added:" << dir.filePath(file);
                emit fileAdded(dir.filePath(file));
            }
        }

        // 检测删除文件
        for (const QString &file : m_knownFiles) {
            if (!currentFiles.contains(file)) {
                qDebug() << "File removed:" << dir.filePath(file);
                emit fileRemoved(dir.filePath(file));
            }
        }

        // 更新已知列表
        m_knownFiles = currentFiles;
    }

private:
    QFileSystemWatcher *m_watcher;
    QString m_dirPath;
    QStringList m_knownFiles;

signals:
    void fileAdded(const QString &path);
    void fileRemoved(const QString &path);
};
```

### 2.3 同时监控文件和目录

```cpp
// 添加多个路径（文件和目录可以混合）
QStringList paths = {

qDebug() << "Watched dirs:" << watcher->directories();
```

## 3. 平台行为差异

### 3.1 各平台底层机制

| 平台 | 底层 API | 特点 |
|---|---|---|
| **Linux** | `inotify` | 高效，有 watch 数量限制（默认 8192，可通过 `fs.inotify.max_user_watches` 调整） |
| **macOS** | `kqueue` + `FSEvents` | 稳定，但目录监控可能有短暂延迟 |
| **Windows** | `ReadDirectoryChangesW` | 快速，但"保存并替换"模式可能丢失监控 |

### 3.2 文件删除/重建问题（最常见坑）

许多文本编辑器（Vim、VS Code、Notepad++ 等）保存文件时并非直接覆写，而是：

```
1. 写入临时文件 config.ini.tmp
2. 删除原文件 config.ini
3. 将临时文件重命名为 config.ini
```

这导致：
- 步骤 2 触发 `fileChanged` 信号
- `QFileSystemWatcher` 自动**移除**对已删除文件的监控
- 步骤 3 后文件恢复，但**不再被监控**

**解决方案：**

```cpp
connect(watcher, &QFileSystemWatcher::fileChanged,
        this, [this](const QString &path) {
    // 处理变化
    processFileChange(path);

    // 重新添加监控（延迟确保文件已重建完毕）
    QTimer::singleShot(200, this, [this, path]() {
        if (QFile::exists(path) && !m_watcher->files().contains(path)) {
            m_watcher->addPath(path);
        }
    });
});
```

### 3.3 信号触发次数

同一次保存操作可能触发**多次** `fileChanged` 信号（例如先修改内容再修改属性/时间戳）。应做**防抖处理**：

```cpp
class DebouncedWatcher : public QObject {
    Q_OBJECT
public:
    DebouncedWatcher(QObject *parent = nullptr) : QObject(parent) {
        m_watcher = new QFileSystemWatcher(this);
        m_debounceTimer = new QTimer(this);
        m_debounceTimer->setSingleShot(true);
        m_debounceTimer->setInterval(300);  // 300ms 防抖

        connect(m_watcher, &QFileSystemWatcher::fileChanged,
                this, &DebouncedWatcher::onRawFileChanged);
        connect(m_debounceTimer, &QTimer::timeout,
                this, &DebouncedWatcher::processPendingChanges);
    }

    void addPath(const QString &path) {
        m_watcher->addPath(path);
    }

signals:
    void fileChanged(const QString &path);  // 防抖后的信号

private slots:
    void onRawFileChanged(const QString &path) {
        m_pendingPaths.insert(path);
        m_debounceTimer->start();  // 重置定时器

        // 重新添加监控
        if (!m_watcher->files().contains(path)) {
            QTimer::singleShot(100, this, [this, path]() {
                if (QFile::exists(path))
                    m_watcher->addPath(path);
            });
        }
    }

    void processPendingChanges() {
        for (const QString &path : m_pendingPaths) {
            emit fileChanged(path);
        }
        m_pendingPaths.clear();
    }

private:
    QFileSystemWatcher *m_watcher;
    QTimer *m_debounceTimer;
    QSet<QString> m_pendingPaths;
};
```

## 4. 递归目录监控

`QFileSystemWatcher` **不支持递归监控**。需要手动添加所有子目录，并在子目录创建/删除时动态维护。

```cpp
class RecursiveWatcher : public QObject {
    Q_OBJECT
public:
    RecursiveWatcher(const QString &rootPath,
                     const QStringList &nameFilters = {},
                     QObject *parent = nullptr)
        : QObject(parent)
        , m_rootPath(rootPath)
        , m_nameFilters(nameFilters) {

        m_watcher = new QFileSystemWatcher(this);
        connect(m_watcher, &QFileSystemWatcher::directoryChanged,
                this, &RecursiveWatcher::onDirectoryChanged);
        connect(m_watcher, &QFileSystemWatcher::fileChanged,
                this, &RecursiveWatcher::onFileChanged);

        // 递归添加所有子目录
        addDirectoryRecursive(rootPath);
    }

signals:
    void fileModified(const QString &path);
    void fileAdded(const QString &path);
    void fileRemoved(const QString &path);

private:
    void addDirectoryRecursive(const QString &path) {
        QDir dir(path);
        if (!dir.exists()) return;

        m_watcher->addPath(path);

        // 记录当前目录内容
        m_dirContents[path] = dir.entryList(
            QDir::Files | QDir::Dirs | QDir::NoDotAndDotDot);

        // 递归子目录
        for (const QFileInfo &fi : dir.entryInfoList(
                 QDir::Dirs | QDir::NoDotAndDotDot | QDir::NoSymLinks)) {
            addDirectoryRecursive(fi.absoluteFilePath());
        }

        // 如果有文件过滤器，监控匹配的文件
        if (!m_nameFilters.isEmpty()) {
            for (const QFileInfo &fi : dir.entryInfoList(
                     m_nameFilters, QDir::Files)) {
                m_watcher->addPath(fi.absoluteFilePath());
            }
        }
    }

    void onDirectoryChanged(const QString &dirPath) {
        QDir dir(dirPath);
        QStringList current = dir.entryList(
            QDir::Files | QDir::Dirs | QDir::NoDotAndDotDot);
        QStringList previous = m_dirContents.value(dirPath);

        // 新增
        for (const QString &entry : current) {
            if (!previous.contains(entry)) {
                QString fullPath = dir.filePath(entry);
                QFileInfo fi(fullPath);
                if (fi.isDir()) {
                    // 新目录：递归添加监控
                    addDirectoryRecursive(fullPath);
                } else {
                    emit fileAdded(fullPath);
                    if (!m_nameFilters.isEmpty()) {
                        for (const QString &filter : m_nameFilters) {
                            QRegExp rx(filter, Qt::CaseInsensitive, QRegExp::Wildcard);
                            if (rx.exactMatch(fi.fileName())) {
                                m_watcher->addPath(fullPath);
                                break;
                            }
                        }
                    }
                }
            }
        }

        // 删除
        for (const QString &entry : previous) {
            if (!current.contains(entry)) {
                QString fullPath = dir.filePath(entry);
                emit fileRemoved(fullPath);
                // watcher 会自动移除已删除的路径
            }
        }

        m_dirContents[dirPath] = current;
    }

    void onFileChanged(const QString &path) {
        emit fileModified(path);
        // 重新添加监控（应对删除-重建）
        if (!m_watcher->files().contains(path)) {
            QTimer::singleShot(200, this, [this, path]() {
                if (QFile::exists(path))
                    m_watcher->addPath(path);
            });
        }
    }

    QFileSystemWatcher *m_watcher;
    QString m_rootPath;
    QStringList m_nameFilters;
    QMap<QString, QStringList> m_dirContents;
};

// 使用
auto *watcher = new RecursiveWatcher("C:/project/src", {"*.cpp", "*.h"});
connect(watcher, &RecursiveWatcher::fileModified, [](const QString &path) {
    qDebug() << "Modified:" << path;
});
connect(watcher, &RecursiveWatcher::fileAdded, [](const QString &path) {
    qDebug() << "Added:" << path;
});
connect(watcher, &RecursiveWatcher::fileRemoved, [](const QString &path) {
    qDebug() << "Removed:" << path;
});
```

## 5. 实战案例

### 5.1 配置文件热重载

```cpp
class HotReloadConfig : public QObject {
    Q_OBJECT
public:
    HotReloadConfig(const QString &configFile, QObject *parent = nullptr)
        : QObject(parent), m_configFile(configFile) {

        loadConfig();

        m_watcher = new QFileSystemWatcher({configFile}, this);
        connect(m_watcher, &QFileSystemWatcher::fileChanged,
                this, &HotReloadConfig::onChanged);

        m_debounce = new QTimer(this);
        m_debounce->setSingleShot(true);
        m_debounce->setInterval(500);
        connect(m_debounce, &QTimer::timeout, this, [this]() {
            loadConfig();
            emit configChanged();
        });
    }

    QString value(const QString &key, const QString &defaultVal = {}) const {
        return m_config.value(key, defaultVal);
    }

signals:
    void configChanged();

private slots:
    void onChanged(const QString &path) {
        m_debounce->start();

        // 重新添加监控
        if (!m_watcher->files().contains(path)) {
            QTimer::singleShot(200, this, [this, path]() {
                if (QFile::exists(path))
                    m_watcher->addPath(path);
            });
        }
    }

private:
    void loadConfig() {
        QFile file(m_configFile);
        if (!file.open(QIODevice::ReadOnly | QIODevice::Text)) return;

        m_config.clear();
        QJsonDocument doc = QJsonDocument::fromJson(file.readAll());
        QJsonObject obj = doc.object();
        for (auto it = obj.begin(); it != obj.end(); ++it) {
            m_config[it.key()] = it.value().toVariant().toString();
        }
        qDebug() << "Config reloaded, keys:" << m_config.keys();
    }

    QFileSystemWatcher *m_watcher;
    QTimer *m_debounce;
    QString m_configFile;
    QMap<QString, QString> m_config;
};

// 使用
auto *config = new HotReloadConfig("settings.json");
connect(config, &HotReloadConfig::configChanged, [config]() {
    QString theme = config->value("theme", "light");
    applyTheme(theme);
});
```

### 5.2 自动编译触发器

```cpp
class AutoBuilder : public QObject {
    Q_OBJECT
public:
    AutoBuilder(const QString &srcDir, const QString &buildCmd, QObject *parent = nullptr)
        : QObject(parent), m_buildCmd(buildCmd) {

        m_watcher = new RecursiveWatcher(srcDir, {"*.cpp", "*.h", "*.cmake"}, this);

        m_buildTimer = new QTimer(this);
        m_buildTimer->setSingleShot(true);
        m_buildTimer->setInterval(1000);  // 1 秒无改动后触发构建

        connect(m_watcher, &RecursiveWatcher::fileModified,
                this, [this](const QString &path) {
            qDebug() << "Source changed:" << path;
            m_buildTimer->start();
        });

        connect(m_buildTimer, &QTimer::timeout, this, &AutoBuilder::build);
    }

private slots:
    void build() {
        qDebug() << "Starting build...";
        QProcess *proc = new QProcess(this);
        proc->setProcessChannelMode(QProcess::MergedChannels);

        connect(proc, QOverload<int, QProcess::ExitStatus>::of(&QProcess::finished),
                this, [proc](int exitCode, QProcess::ExitStatus) {
            qDebug().noquote() << proc->readAll();
            qDebug() << "Build finished with code:" << exitCode;
            proc->deleteLater();
        });

        proc->start("cmd", {"/c", m_buildCmd});
    }

private:
    RecursiveWatcher *m_watcher;
    QTimer *m_buildTimer;
    QString m_buildCmd;
};
```

### 5.3 日志文件实时跟踪（类似 tail -f）

```cpp
class LogTailer : public QObject {
    Q_OBJECT
public:
    LogTailer(const QString &logFile, QObject *parent = nullptr)
        : QObject(parent), m_logFile(logFile) {

        m_file.setFileName(logFile);
        if (m_file.open(QIODevice::ReadOnly | QIODevice::Text)) {
            m_file.seek(m_file.size());  // 跳到末尾，只看新内容
            m_lastPos = m_file.pos();
        }

        m_watcher = new QFileSystemWatcher({logFile}, this);
        connect(m_watcher, &QFileSystemWatcher::fileChanged,
                this, &LogTailer::onFileChanged);
    }

signals:
    void newLine(const QString &line);

private slots:
    void onFileChanged(const QString &path) {
        // 文件可能被截断（日志轮转）
        if (m_file.size() < m_lastPos) {
            m_file.seek(0);  // 文件被截断，从头开始
        }

        // 读取新增内容
        while (!m_file.atEnd()) {
            QString line = QString::fromUtf8(m_file.readLine()).trimmed();
            if (!line.isEmpty()) {
                emit newLine(line);
            }
        }
        m_lastPos = m_file.pos();

        // 重新添加监控
        if (!m_watcher->files().contains(path)) {
            QTimer::singleShot(100, this, [this, path]() {
                if (QFile::exists(path))
                    m_watcher->addPath(path);
            });
        }
    }

private:
    QFileSystemWatcher *m_watcher;
    QFile m_file;
    QString m_logFile;
    qint64 m_lastPos = 0;
};

// 使用
auto *tailer = new LogTailer("/var/log/app.log");
connect(tailer, &LogTailer::newLine, [](const QString &line) {
    if (line.contains("ERROR")) {
        qWarning() << "!! ERROR detected:" << line;
    }
});
```

### 5.4 资源文件同步（开发工具场景）

```cpp
class AssetSync : public QObject {
    Q_OBJECT
public:
    AssetSync(const QString &srcDir, const QString &dstDir, QObject *parent = nullptr)
        : QObject(parent), m_srcDir(srcDir), m_dstDir(dstDir) {

        QDir().mkpath(dstDir);

        m_watcher = new RecursiveWatcher(srcDir, {"*.png", "*.jpg", "*.qss", "*.qml"}, this);

        connect(m_watcher, &RecursiveWatcher::fileModified,
                this, &AssetSync::syncFile);
        connect(m_watcher, &RecursiveWatcher::fileAdded,
                this, &AssetSync::syncFile);
        connect(m_watcher, &RecursiveWatcher::fileRemoved,
                this, &AssetSync::removeFile);
    }

private:
    void syncFile(const QString &srcPath) {
        QDir src(m_srcDir);
        QString relativePath = src.relativeFilePath(srcPath);
        QString dstPath = m_dstDir + "/" + relativePath;

        // 确保目标目录存在
        QDir().mkpath(QFileInfo(dstPath).absolutePath());

        // 复制（覆盖）
        if (QFile::exists(dstPath)) QFile::remove(dstPath);
        if (QFile::copy(srcPath, dstPath)) {
            qDebug() << "Synced:" << relativePath;
        }
    }

    void removeFile(const QString &srcPath) {
        QDir src(m_srcDir);
        QString relativePath = src.relativeFilePath(srcPath);
        QString dstPath = m_dstDir + "/" + relativePath;

        if (QFile::remove(dstPath)) {
            qDebug() << "Removed:" << relativePath;
        }
    }

    RecursiveWatcher *m_watcher;
    QString m_srcDir, m_dstDir;
};
```

## 6. 轮询方案（QFileSystemWatcher 的补充）

在某些场景下 `QFileSystemWatcher` 可能不可靠（网络文件系统 NFS/SMB、虚拟机共享文件夹等）。此时需要轮询作为补充或替代：

```cpp
class PollingWatcher : public QObject {
    Q_OBJECT
public:
    PollingWatcher(int intervalMs = 1000, QObject *parent = nullptr)
        : QObject(parent) {
        m_timer = new QTimer(this);
        m_timer->setInterval(intervalMs);
        connect(m_timer, &QTimer::timeout, this, &PollingWatcher::checkFiles);
        m_timer->start();
    }

    void addFile(const QString &path) {
        QFileInfo fi(path);
        if (fi.exists()) {
            m_files[path] = {fi.lastModified(), fi.size()};
        }
    }

signals:
    void fileChanged(const QString &path);

private slots:
    void checkFiles() {
        for (auto it = m_files.begin(); it != m_files.end(); ++it) {
            QFileInfo fi(it.key());
            if (!fi.exists()) {
                emit fileChanged(it.key());
                continue;
            }
            if (fi.lastModified() != it.value().modified ||
                fi.size() != it.value().size) {
                it.value() = {fi.lastModified(), fi.size()};
                emit fileChanged(it.key());
            }
        }
    }

private:
    struct FileState {
        QDateTime modified;
        qint64 size;
    };
    QTimer *m_timer;
    QMap<QString, FileState> m_files;
};
```

### 轮询 vs 原生监控对比

| 特性 | QFileSystemWatcher | 轮询 |
|---|---|---|
| CPU 占用 | 极低（事件驱动） | 与文件数和频率成正比 |
| 响应延迟 | 几乎即时 | 取决于轮询间隔 |
| 网络文件系统 | 可能不工作 | 可靠 |
| 监控数量限制 | Linux inotify 有上限 | 无内核限制 |
| 实现复杂度 | 低（Qt 封装好） | 中（需自行管理状态） |

## 7. 资源限制与优化

### 7.1 Linux inotify 限制

```bash
# 查看当前限制
cat /proc/sys/fs/inotify/max_user_watches    # 默认 8192
cat /proc/sys/fs/inotify/max_user_instances  # 默认 128

# 临时调大
sudo sysctl fs.inotify.max_user_watches=65536

# 永久修改
echo "fs.inotify.max_user_watches=65536" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

```cpp
    watcher->addPath(it.next());  // 目录数量远少于文件数量
}
// 配合 directoryChanged + 前后对比确定具体变化的文件
```

### 7.3 排除不需要的目录

```cpp
void addDirectoryRecursive(const QString &path) {
    static const QSet<QString> ignored = {
        ".git", ".svn", "node_modules", "build", "__pycache__", ".cache"
    };

    QDir dir(path);
    if (ignored.contains(dir.dirName()))
        return;  // 跳过

    m_watcher->addPath(path);

    for (const QFileInfo &fi : dir.entryInfoList(
             QDir::Dirs | QDir::NoDotAndDotDot | QDir::NoSymLinks)) {
        addDirectoryRecursive(fi.absoluteFilePath());
    }
}
```

## 8. 常见问题

**问题 1：addPath 返回 false**

```cpp
bool ok = watcher->addPath("missing.txt");
// false：文件/目录不存在

bool ok = watcher->addPath("already_watched.txt");
// false：已在监控列表中（重复添加）

// 解决：先检查存在性
if (QFile::exists(path) && !watcher->files().contains(path)) {
    watcher->addPath(path);
}
```

**问题 2：文件修改后信号不再触发**

```cpp
// 原因：编辑器"删除+重建"模式导致 watcher 自动移除
// 解决：在 fileChanged 回调中重新 addPath（见 §2.1）
```

**问题 3：短时间内收到多次信号**

```cpp
// 原因：一次保存可能触发多次系统事件（内容变更 + 属性变更 + 时间戳更新）
// 解决：用防抖定时器（见 §3.3 DebouncedWatcher）
```

**问题 4：网络文件系统（NFS/SMB）上不工作**

```cpp
// 原因：inotify/kqueue 不跨文件系统边界，网络文件系统变更不通知本机
// 解决：改用轮询方案（见 §6 PollingWatcher）
```

**问题 5：监控大量目录导致程序启动缓慢**

```cpp
// 原因：addPath 是同步操作，逐个添加数千个目录阻塞主线程
// 解决：在工作线程中批量添加
QtConcurrent::run([this, paths]() {
    for (const auto &path : paths) {
        QMetaObject::invokeMethod(m_watcher, [this, path]() {
            m_watcher->addPath(path);
        }, Qt::QueuedConnection);
    }
});
```

## 9. 最佳实践

| 实践 | 说明 |
|---|---|
| 始终处理文件删除-重建 | `fileChanged` 回调中检查并重新 `addPath` |
| 信号防抖 | 用 `QTimer::singleShot` 或单次定时器合并短时间内的多次通知 |
| 目录优先于文件 | 监控目录数量远少于文件，通过 `directoryChanged` + 前后对比确定变化 |
| 排除无关目录 | `.git`/`build`/`node_modules` 等不需要监控的目录 |
| 网络文件系统用轮询 | NFS/SMB 等场景用定时器轮询替代 |
| 关注 Linux inotify 限制 | 大项目可能需要调大 `max_user_watches` |
| 延迟处理 | 文件可能正在被写入，延迟 100-500ms 后再读取内容 |
| 监控路径用绝对路径 | 避免工作目录变化导致路径解析错误 |
