# 信号与槽机制

信号与槽（Signals & Slots）是 Qt 框架的核心通信机制，用于对象之间的松耦合通信。它取代了传统的回调函数模式，提供了类型安全、灵活且可跨线程的事件通知方式。

---

## 1. 基本概念

### 1.1 什么是信号（Signal）

信号是对象在某个特定事件发生时发出的通知。信号只有**声明**，没有**实现**——它的实现由 `moc`（元对象编译器）自动生成。

**关键特性：**
- 信号是一个函数声明，没有函数体
- 信号的访问权限始终是 `public`（即使写在 `protected` 或 `private` 区域）
- 信号可以有参数，用于传递数据
- 信号可以被重载
- 使用 `emit` 关键字发射信号

### 1.2 什么是槽（Slot）

槽是一个普通的成员函数，用于响应信号。当与之连接的信号被发射时，槽函数会被自动调用。

**关键特性：**
- 槽是普通的 C++ 成员函数，可以被直接调用
- 槽可以有 `public`、`protected`、`private` 访问权限
- 槽的参数必须与信号匹配（数量可以少于信号，但类型必须对应）
- 槽可以是虚函数

### 1.3 概念图

```
┌──────────────┐          connect()          ┌──────────────┐
│   Object A   │ ──────────────────────────▸ │   Object B   │
│              │                              │              │
│  signal:     │    emit clicked()            │  slot:       │
│  clicked()   │ ═══════════════════════════▸ │  onClicked() │
└──────────────┘                              └──────────────┘
```

## 2. 声明信号与槽

### 2.1 基本声明

使用信号与槽的类必须满足两个条件：
1. 继承自 `QObject`（直接或间接）
2. 在类声明中包含 `Q_OBJECT` 宏

```cpp
#include <QObject>

class Counter : public QObject
{
    Q_OBJECT  // 必须在类的私有区域声明此宏

public:
    explicit Counter(QObject *parent = nullptr);

    int value() const { return m_value; }

public slots:
    void setValue(int value);

signals:
    void valueChanged(int newValue);

private:
    int m_value = 0;
};
```

### 2.2 实现槽函数

```cpp
Counter::Counter(QObject *parent)
    : QObject(parent)
{
}

void Counter::setValue(int value)
{
    if (m_value != value) {
        m_value = value;
        emit valueChanged(m_value);  // 发射信号
    }
}
```

> **最佳实践**：在发射信号前检查值是否真正改变，避免无限递归和不必要的通知。

## 3. 连接信号与槽

### 3.1 Qt5 新语法（推荐）

Qt5 引入了基于函数指针的连接语法，提供**编译时类型检查**：

```cpp
Counter a, b;

// 基本连接：a 的值改变时，自动设置 b 的值
QObject::connect(&a, &Counter::valueChanged,
                 &b, &Counter::setValue);

a.setValue(42);   // b.value() 现在也是 42
a.setValue(42);   // 值没变，不会发射信号，b 不会收到通知
a.setValue(100);  // b.value() 现在是 100
```

### 3.2 Qt4 旧语法（基于字符串）

```cpp
// 使用 SIGNAL() 和 SLOT() 宏 —— 运行时检查，易出错
QObject::connect(&a, SIGNAL(valueChanged(int)),
                 &b, SLOT(setValue(int)));
```

> **注意**：旧语法只在**运行时**检查连接是否有效，如果信号或槽名拼写错误，编译不会报错，仅在运行时输出警告。**新项目应始终使用 Qt5 新语法**。

### 3.3 两种语法对比

| 特性 | Qt5 新语法 | Qt4 旧语法 |
|------|-----------|-----------|
| 类型检查 | 编译时 | 运行时 |
| 重载支持 | 需要 `qOverload` | 通过参数列表区分 |
| 连接 Lambda | ✅ 支持 | ❌ 不支持 |
| 连接普通函数 | ✅ 支持 | ❌ 不支持 |
| 性能 | 更快（直接函数调用） | 稍慢（字符串匹配） |
| 错误提示 | 编译时报错 | 运行时 qWarning |

## 4. 连接方式详解

### 4.1 连接到成员函数

```cpp
// 信号 → 槽
connect(sender, &Sender::signalName, receiver, &Receiver::slotName);

// 信号 → 信号（信号转发）
connect(sender, &Sender::signal1, receiver, &Receiver::signal2);
```

### 4.2 连接到 Lambda 表达式

这是 Qt5 新语法最强大的特性之一：

```cpp
auto *button = new QPushButton("Click Me");

// 简单 Lambda
connect(button, &QPushButton::clicked, []() {
    qDebug() << "Button clicked!";
});

// 带捕获的 Lambda
connect(button, &QPushButton::clicked, this, [this]() {
    m_clickCount++;
    updateUI();
});

// 带参数的 Lambda
auto *slider = new QSlider;
auto *label = new QLabel;
connect(slider, &QSlider::valueChanged, label, [label](int value) {
    label->setText(QString::number(value));
});
```

> **重要**：Lambda 连接时推荐传入接收者对象（第3个参数），这样当接收者被销毁时，连接会自动断开，避免悬空引用导致崩溃。

```cpp
// ❌ 危险：如果 this 先于 button 被销毁，Lambda 中访问 this 会崩溃
connect(button, &QPushButton::clicked, [this]() {
    this->doSomething();
});

// ✅ 安全：传入 this 作为上下文对象，this 销毁时连接自动断开
connect(button, &QPushButton::clicked, this, [this]() {
    this->doSomething();
});
```

### 4.3 连接到普通函数 / 静态函数

```cpp
// 连接到全局函数
void onButtonClicked() {
    qDebug() << "clicked";
}
connect(button, &QPushButton::clicked, &onButtonClicked);

// 连接到静态成员函数
connect(button, &QPushButton::clicked, &MyClass::staticHandler);
```

### 4.4 连接到函数对象（Functor）

```cpp
struct MyFunctor {
    void operator()() const {
        qDebug() << "Functor called!";
    }
};

connect(button, &QPushButton::clicked, MyFunctor());
```

## 5. 处理重载信号与槽

当信号或槽存在重载时，Qt5 新语法需要显式指定使用哪个重载版本。

### 5.1 使用 qOverload（Qt 5.7+，推荐）

```cpp
// QSpinBox 有两个 valueChanged 重载：
//   void valueChanged(int)
//   void valueChanged(const QString &)

// 连接 int 版本
connect(spinBox, qOverload<int>(&QSpinBox::valueChanged),
        this, &MyWidget::onValueChanged);

// 连接 QString 版本
connect(spinBox, qOverload<const QString &>(&QSpinBox::valueChanged),
        this, &MyWidget::onTextChanged);
```

### 5.2 使用 static_cast（通用方法）

```cpp
connect(spinBox,
        static_cast<void (QSpinBox::*)(int)>(&QSpinBox::valueChanged),
        this, &MyWidget::onValueChanged);
```

### 5.3 使用 QOverload 模板类（Qt 5.7+）

```cpp
connect(spinBox, QOverload<int>::of(&QSpinBox::valueChanged),
        this, &MyWidget::onValueChanged);
```

## 6. 断开连接

### 6.1 使用 disconnect

```cpp
// 断开特定连接
QObject::disconnect(&a, &Counter::valueChanged,
                    &b, &Counter::setValue);

// 断开对象 a 的所有信号与 b 之间的连接
QObject::disconnect(&a, nullptr, &b, nullptr);

// 断开信号的所有连接
QObject::disconnect(&a, &Counter::valueChanged, nullptr, nullptr);

// 断开对象 a 发出的所有信号与所有接收者
QObject::disconnect(&a, nullptr, nullptr, nullptr);
// 等价于
a.disconnect();
```

### 6.2 使用 QMetaObject::Connection

`connect` 函数返回一个 `QMetaObject::Connection` 对象，可以用于断开特定的连接：

```cpp
QMetaObject::Connection conn = connect(button, &QPushButton::clicked,
                                        this, &MyWidget::onClicked);

// 稍后断开这个特定连接
QObject::disconnect(conn);
```

这在 Lambda 连接中特别有用，因为 Lambda 没有地址，无法通过函数指针断开。

### 6.3 一次性连接（Qt 6.0+）

```cpp
// 信号发射一次后自动断开
connect(button, &QPushButton::clicked,
        this, &MyWidget::onFirstClick,
        Qt::SingleShotConnection);
```

在 Qt5 中可以手动模拟：

```cpp
auto conn = std::make_shared<QMetaObject::Connection>();
*conn = connect(button, &QPushButton::clicked, this, [this, conn]() {
    QObject::disconnect(*conn);
    onFirstClick();
});
```

## 7. 连接类型（ConnectionType）

`connect` 的第 5 个参数指定连接类型，控制槽函数的调用时机和线程：

```cpp
QObject::connect(sender, &Sender::signal,
                 receiver, &Receiver::slot,
                 Qt::ConnectionType);
```

### 7.1 五种连接类型

| 类型 | 值 | 行为 |
|------|---|------|
| `Qt::AutoConnection` | 0 | **默认值**。自动判断：同线程用 Direct，跨线程用 Queued |
| `Qt::DirectConnection` | 1 | 立即在**发送者线程**中调用槽函数（类似直接函数调用） |
| `Qt::QueuedConnection` | 2 | 将调用投递到**接收者线程**的事件队列，异步执行 |
| `Qt::BlockingQueuedConnection` | 3 | 类似 Queued，但发送者线程**阻塞等待**槽函数执行完毕 |
| `Qt::UniqueConnection` | 0x80 | 标志位，可与上面的类型按位或组合。确保同一连接不会重复建立 |

### 7.2 AutoConnection 的判定逻辑

```
emit signal()
    │
    ▼
  sender 线程 == receiver 线程?
    │                │
   YES              NO
    │                │
    ▼                ▼
  Direct          Queued
  (同步调用)       (投递到事件队列)
```

### 7.3 跨线程通信示例

```cpp
// 工作线程
class Worker : public QObject
{
    Q_OBJECT
public slots:
    void doHeavyWork() {
        // 在工作线程中执行耗时操作
        QThread::sleep(3);
        emit workFinished("Done!");
    }

signals:
    void workFinished(const QString &result);
};

// 主线程（UI 线程）
class MainWindow : public QMainWindow
{
    Q_OBJECT
public:
    MainWindow() {
        auto *thread = new QThread(this);
        auto *worker = new Worker;
        worker->moveToThread(thread);

        // 跨线程连接：信号在主线程发射，槽在工作线程执行
        connect(this, &MainWindow::startWork,
                worker, &Worker::doHeavyWork);  // AutoConnection → Queued

        // 跨线程连接：信号在工作线程发射，槽在主线程执行
        connect(worker, &Worker::workFinished,
                this, &MainWindow::onWorkFinished);  // AutoConnection → Queued

        // 线程结束时清理
        connect(thread, &QThread::finished, worker, &QObject::deleteLater);

        thread->start();
    }

signals:
    void startWork();

private slots:
    void onWorkFinished(const QString &result) {
        // 安全地更新 UI（在主线程执行）
        statusBar()->showMessage(result);
    }
};
```

### 7.4 BlockingQueuedConnection 注意事项

```cpp
// ⚠️ 危险：如果 sender 和 receiver 在同一线程，会造成死锁！
connect(sender, &Sender::sig, receiver, &Receiver::slot,
        Qt::BlockingQueuedConnection);
```

`BlockingQueuedConnection` 只能用于跨线程场景，且需要确保接收者线程正在运行事件循环。

### 7.5 UniqueConnection 的使用

```cpp
// 防止重复连接（例如在多次调用的初始化函数中）
connect(button, &QPushButton::clicked,
        this, &MyWidget::onClicked,
        Qt::UniqueConnection);

// 可以与其他类型组合
connect(sender, &Sender::sig, receiver, &Receiver::slot,
        Qt::QueuedConnection | Qt::UniqueConnection);
```

## 8. 信号与槽的参数规则

### 8.1 参数匹配规则

- 槽的参数数量可以**少于或等于**信号的参数数量
- 对应位置的参数类型必须**完全匹配**
- 多余的信号参数会被忽略

```cpp
signals:
    void dataReady(int id, const QString &name, double score);

public slots:
    void handleAll(int id, const QString &name, double score);  // ✅ 完全匹配
    void handlePartial(int id, const QString &name);             // ✅ 忽略 score
    void handleIdOnly(int id);                                    // ✅ 只接收 id
    void handleNone();                                            // ✅ 忽略所有参数
    // void handleWrong(const QString &name, int id);             // ❌ 类型顺序不匹配
```

### 8.2 参数类型要求

**对于 DirectConnection（同线程）：**
- 参数可以是任意 C++ 类型，包括引用和指针

**对于 QueuedConnection（跨线程）：**
- 参数类型必须是 Qt 元类型系统已注册的类型
- 内置类型和大多数 Qt 类型已预注册
- 自定义类型需要使用 `qRegisterMetaType` 注册

```cpp
// 自定义类型注册
struct MyData {
    int id;
    QString name;
};
Q_DECLARE_METATYPE(MyData)

// 在使用前注册（通常在 main 函数或构造函数中）
qRegisterMetaType<MyData>("MyData");
```

### 8.3 使用默认参数

```cpp
signals:
    void messageReady(const QString &msg, int priority = 0);

// ❌ Qt5 新语法不支持区分默认参数的重载
// 如需使用，应定义两个信号或使用 Qt4 旧语法
```

## 9. 返回值

- 信号的返回类型通常是 `void`
- 虽然技术上信号可以有返回值，但在实际使用中**几乎不使用**
- 当信号连接了多个槽时，返回值行为是不确定的
- **最佳实践**：信号始终使用 `void` 返回类型

```cpp
// ✅ 推荐
signals:
    void clicked();
    void valueChanged(int newValue);

// ❌ 不推荐（虽然合法）
signals:
    int requestValue();
```

## 10. 信号与槽的高级用法

### 10.1 信号转发（Signal Forwarding）

一个信号可以连接到另一个信号，形成信号链：

```cpp
class Button : public QObject {
    Q_OBJECT
signals:
    void clicked();
    void pressed();
};

class Panel : public QObject {
    Q_OBJECT
public:
    Panel() {
        m_button = new Button(this);
        // 按钮的 clicked 信号转发为面板的 buttonClicked 信号
        connect(m_button, &Button::clicked, this, &Panel::buttonClicked);
    }
signals:
    void buttonClicked();
private:
    Button *m_button;
};
```

### 10.2 一个信号连接多个槽

```cpp
connect(button, &QPushButton::clicked, this, &MyWidget::logClick);
connect(button, &QPushButton::clicked, this, &MyWidget::updateUI);
connect(button, &QPushButton::clicked, this, &MyWidget::playSound);

// 槽的调用顺序与连接顺序一致（但不应依赖此行为）
```

### 10.3 多个信号连接同一个槽

```cpp
connect(buttonA, &QPushButton::clicked, this, &MyWidget::onAnyButtonClicked);
connect(buttonB, &QPushButton::clicked, this, &MyWidget::onAnyButtonClicked);
connect(buttonC, &QPushButton::clicked, this, &MyWidget::onAnyButtonClicked);

void MyWidget::onAnyButtonClicked()
{
    // 使用 sender() 获取发送信号的对象
    auto *button = qobject_cast<QPushButton*>(sender());
    if (button) {
        qDebug() << "Clicked button:" << button->text();
    }
}
```

> **注意**：`sender()` 在 `DirectConnection` 下返回发送者指针，在 `QueuedConnection` 下也可用，但应谨慎使用——它违反了面向对象的封装原则，使槽函数依赖于具体的信号来源。

### 10.4 使用 QSignalMapper（Qt4 风格，已过时）

```cpp
// Qt4 风格（不推荐）
auto *mapper = new QSignalMapper(this);
for (int i = 0; i < 10; ++i) {
    auto *btn = new QPushButton(QString::number(i));
    connect(btn, SIGNAL(clicked()), mapper, SLOT(map()));
    mapper->setMapping(btn, i);
}
connect(mapper, SIGNAL(mapped(int)), this, SLOT(onButtonClicked(int)));
```

```cpp
// Qt5 风格（推荐，使用 Lambda 替代 QSignalMapper）
for (int i = 0; i < 10; ++i) {
    auto *btn = new QPushButton(QString::number(i), this);
    connect(btn, &QPushButton::clicked, this, [this, i]() {
        onButtonClicked(i);
    });
}
```

### 10.5 使用 QSignalBlocker 临时屏蔽信号

```cpp
void MyWidget::initializeUI()
{
    // 防止初始化过程中触发不必要的信号
    const QSignalBlocker blocker(m_spinBox);  // RAII 方式
    m_spinBox->setRange(0, 100);
    m_spinBox->setValue(50);
    // blocker 析构时自动取消屏蔽
}

// 等价的手动方式
m_spinBox->blockSignals(true);
m_spinBox->setValue(50);
m_spinBox->blockSignals(false);
```

## 11. 信号与槽的工作原理

### 11.1 moc 的作用

Qt 的信号与槽机制依赖**元对象编译器（moc）**。moc 扫描头文件中的 `Q_OBJECT` 宏、`signals:`、`slots:` 关键字，为每个类生成一个 `moc_*.cpp` 文件。

**moc 生成的内容包括：**
- 信号的实现代码（即 `emit` 信号时实际调用的函数）
- 元对象信息表（`QMetaObject`）：包含类名、信号列表、槽列表、属性列表等
- 字符串表：存储信号和槽的名称
- 连接索引映射

### 11.2 信号的真正实现

当你写 `emit valueChanged(42)` 时，moc 生成的代码大致如下：

```cpp
// moc 生成的信号实现（简化）
void Counter::valueChanged(int _t1)
{
    void *_a[] = {
        nullptr,                           // 返回值（void）
        const_cast<void*>(reinterpret_cast<const void*>(&_t1))  // 参数
    };
    QMetaObject::activate(this,                    // 发送者
                          &staticMetaObject,       // 元对象
                          0,                       // 信号索引
                          _a);                     // 参数数组
}
```

### 11.3 连接存储结构

```
QObject (sender)
  └─ d_ptr->connectionLists
       └─ ConnectionList[signalIndex]
            ├─ Connection { receiver, slotIndex, type, ... }
            ├─ Connection { receiver, slotIndex, type, ... }
            └─ ...
```

每个 `QObject` 内部维护一个连接列表，按信号索引组织。`activate()` 函数遍历对应信号的连接列表，依次调用每个连接的槽函数。

### 11.4 性能分析

- **DirectConnection**：约等于一次虚函数调用的开销（非常快）
- **QueuedConnection**：涉及参数拷贝、事件分配、事件循环处理——开销明显更大
- **与回调函数对比**：信号与槽比直接函数调用慢约 10 倍，但仍然非常快（纳秒级），在实际应用中几乎无感知

## 12. 实战示例

### 12.1 自定义进度通知

```cpp
// filedownloader.h
class FileDownloader : public QObject
{
    Q_OBJECT
public:
    explicit FileDownloader(QObject *parent = nullptr);

    void startDownload(const QUrl &url);
    void cancel();

signals:
    void progressChanged(int percent);
    void downloadFinished(const QString &filePath);
    void errorOccurred(const QString &errorMsg);

private slots:
    void onNetworkReply();

private:
    QNetworkAccessManager *m_manager;
    QNetworkReply *m_reply = nullptr;
};

// main.cpp
int main(int argc, char *argv[])
{
    QApplication app(argc, argv);

    FileDownloader downloader;
    QProgressBar progressBar;
    QLabel statusLabel;

    // 进度更新
    QObject::connect(&downloader, &FileDownloader::progressChanged,
                     &progressBar, &QProgressBar::setValue);

    // 下载完成
    QObject::connect(&downloader, &FileDownloader::downloadFinished,
                     &statusLabel, [&statusLabel](const QString &path) {
        statusLabel.setText("Downloaded: " + path);
    });

    // 错误处理
    QObject::connect(&downloader, &FileDownloader::errorOccurred,
                     &statusLabel, [&statusLabel](const QString &error) {
        statusLabel.setText("Error: " + error);
        statusLabel.setStyleSheet("color: red;");
    });

    downloader.startDownload(QUrl("https://example.com/file.zip"));

    return app.exec();
}
```

### 12.2 观察者模式实现

```cpp
// 使用信号与槽实现松耦合的观察者模式
class DataModel : public QObject
{
    Q_OBJECT
    Q_PROPERTY(QString name READ name WRITE setName NOTIFY nameChanged)
    Q_PROPERTY(int age READ age WRITE setAge NOTIFY ageChanged)

public:
    QString name() const { return m_name; }
    int age() const { return m_age; }

public slots:
    void setName(const QString &name) {
        if (m_name != name) {
            m_name = name;
            emit nameChanged(m_name);
        }
    }

    void setAge(int age) {
        if (m_age != age) {
            m_age = age;
            emit ageChanged(m_age);
        }
    }

signals:
    void nameChanged(const QString &name);
    void ageChanged(int age);

private:
    QString m_name;
    int m_age = 0;
};

// 视图1：表格视图
class TableView : public QTableWidget
{
    Q_OBJECT
public:
    void bindModel(DataModel *model) {
        connect(model, &DataModel::nameChanged, this, [this](const QString &name) {
            item(0, 1)->setText(name);
        });
        connect(model, &DataModel::ageChanged, this, [this](int age) {
            item(1, 1)->setText(QString::number(age));
        });
    }
};

// 视图2：标签视图
class LabelView : public QLabel
{
    Q_OBJECT
public:
    void bindModel(DataModel *model) {
        connect(model, &DataModel::nameChanged, this, &LabelView::updateDisplay);
        connect(model, &DataModel::ageChanged, this, &LabelView::updateDisplay);
        m_model = model;
    }
private:
    void updateDisplay() {
        setText(QString("%1, Age: %2").arg(m_model->name()).arg(m_model->age()));
    }
    DataModel *m_model = nullptr;
};
```

### 12.3 线程安全的生产者-消费者

```cpp
class Producer : public QObject
{
    Q_OBJECT
public slots:
    void produce() {
        for (int i = 0; i < 100; ++i) {
            QThread::msleep(50);
            emit itemProduced(i, QString("Item_%1").arg(i));
        }
        emit productionFinished();
    }

signals:
    void itemProduced(int id, const QString &data);
    void productionFinished();
};

class Consumer : public QObject
{
    Q_OBJECT
public slots:
    void consume(int id, const QString &data) {
        qDebug() << "Consuming:" << id << data
                 << "in thread:" << QThread::currentThread();
    }

    void onFinished() {
        qDebug() << "All items consumed.";
    }
};

// 使用
void setupProducerConsumer()
{
    auto *producerThread = new QThread;
    auto *consumerThread = new QThread;
    auto *producer = new Producer;
    auto *consumer = new Consumer;

    producer->moveToThread(producerThread);
    consumer->moveToThread(consumerThread);

    // 跨线程连接（AutoConnection → QueuedConnection）
    connect(producer, &Producer::itemProduced,
            consumer, &Consumer::consume);
    connect(producer, &Producer::productionFinished,
            consumer, &Consumer::onFinished);

    // 启动生产
    connect(producerThread, &QThread::started,
            producer, &Producer::produce);

    // 清理
    connect(producer, &Producer::productionFinished,
            producerThread, &QThread::quit);
    connect(producerThread, &QThread::finished,
            producer, &QObject::deleteLater);
    connect(consumerThread, &QThread::finished,
            consumer, &QObject::deleteLater);

    consumerThread->start();
    producerThread->start();
}
```

## 13. 常见错误与调试

### 13.1 常见错误清单

| 错误 | 症状 | 解决方法 |
|------|------|---------|
| 忘记 `Q_OBJECT` 宏 | 编译错误 / 连接无效 | 在类声明中添加 `Q_OBJECT` |
| 信号写了实现 | 链接错误（重复定义） | 删除信号的实现代码，moc 会自动生成 |
| 参数类型不匹配 | 编译错误（新语法）/ 运行时警告（旧语法） | 检查并对齐参数类型 |
| 跨线程传递未注册类型 | 运行时警告 | 用 `qRegisterMetaType` 注册类型 |
| Lambda 中捕获已销毁对象 | 崩溃 | 传入上下文对象作为 connect 第3个参数 |
| 在构造函数中 `emit` | 接收者可能尚未连接 | 延迟发射或在初始化完成后发射 |
| `sender()` 返回 nullptr | 槽不是通过信号调用的 | 检查调用场景，避免直接调用时使用 `sender()` |
| 同线程使用 BlockingQueued | 死锁 | 改用 DirectConnection 或 AutoConnection |

### 13.2 调试技巧

**1. 启用 Qt 连接调试输出：**

```bash
# 设置环境变量查看连接/断开信息
QT_LOGGING_RULES="qt.core.qobject.connect=true"
```

**2. 检查 connect 返回值：**

```cpp
bool ok = connect(sender, &Sender::sig, receiver, &Receiver::slot);
Q_ASSERT(ok);  // 连接失败时断言
```

**3. 使用 QObject::dumpObjectInfo()：**

```cpp
myObject->dumpObjectInfo();
// 输出对象的信号/槽连接信息到调试输出
```

**4. 检查对象的线程归属：**

```cpp
qDebug() << "Object thread:" << obj->thread();
qDebug() << "Current thread:" << QThread::currentThread();
```

## 14. 最佳实践总结

1. **始终使用 Qt5 新语法**——获得编译时类型检查，减少运行时错误
2. **信号命名使用过去式或进行时**——如 `valueChanged`、`downloadFinished`、`loading`
3. **槽命名使用动词**——如 `setValue`、`handleClick`、`updateUI`
4. **Lambda 连接时传入上下文对象**——确保生命周期安全
5. **避免长链式信号转发**——超过 3 层时考虑重构
6. **在设值前检查值是否改变**——避免无限循环
7. **使用 `QSignalBlocker` 而非 `blockSignals`**——RAII 更安全
8. **跨线程通信优先使用信号与槽**——不要直接调用其他线程对象的方法
9. **不要在信号中传递大对象**——使用 `QSharedPointer` 或引用
10. **避免过度使用 `sender()`**——它破坏封装性，优先使用 Lambda 捕获或 `QSignalMapper` 替代

## 15. 与其他机制的关系

| 机制 | 与信号槽的关系 |
|------|--------------|
| **元对象系统** | 信号槽的基础设施，moc 生成元对象信息供 `activate()` 使用 |
| **属性系统** | `Q_PROPERTY` 的 `NOTIFY` 参数指定属性变化时发射的信号 |
| **事件系统** | 事件是低层机制，`QueuedConnection` 的信号调用实际通过事件系统投递 |
| **对象树** | 父对象销毁时子对象随之销毁，相关连接也自动断开 |
| **QML** | QML 可以直接连接 C++ 对象的信号，是 Qt Quick 的数据绑定基础 |
