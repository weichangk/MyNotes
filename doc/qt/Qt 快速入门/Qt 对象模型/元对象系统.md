# 元对象系统（Meta-Object System）

## 1. 概述

Qt 的元对象系统（Meta-Object System）是 Qt 框架的核心基础设施之一，它为 C++ 语言扩展了**运行时类型信息（RTTI）**、**信号与槽机制**、**动态属性系统**等能力。标准 C++ 的 RTTI 功能十分有限（仅有 `typeid` 和 `dynamic_cast`），而 Qt 的元对象系统在编译阶段通过 **MOC（Meta-Object Compiler）** 代码生成器，为每个 `QObject` 子类生成额外的元数据代码，从而在运行时实现丰富的反射和内省能力。

元对象系统由以下三大要素构成：

1. **`QObject` 基类** — 所有需要使用元对象特性的类必须继承自 `QObject`
2. **`Q_OBJECT` 宏** — 在类声明的 private 区域声明，启用元对象特性
3. **MOC（Meta-Object Compiler）** — Qt 的元对象编译器，在编译前扫描头文件并生成 `moc_*.cpp` 文件

```
┌─────────────────────────────────────────────────────┐
│                   编译流程                           │
│                                                     │
│  myclass.h ──→ MOC ──→ moc_myclass.cpp              │
│       │                       │                     │
│       └───────┬───────────────┘                     │
│               ▼                                     │
│         C++ 编译器 ──→ myclass.o ──→ 链接 ──→ 可执行文件│
└─────────────────────────────────────────────────────┘
```

## 2. Q_OBJECT 宏

### 2.1 作用

`Q_OBJECT` 宏是启用元对象系统的关键。它在类中声明了若干虚函数和静态成员，这些由 MOC 生成的代码来实现。

```cpp
class MyWidget : public QWidget {
    Q_OBJECT  // 必须放在类声明的最开头（private 区域）

public:
    explicit MyWidget(QWidget *parent = nullptr);

signals:
    void valueChanged(int newValue);

public slots:
    void setValue(int value);
};
```

### 2.2 Q_OBJECT 宏展开内容

`Q_OBJECT` 宏大致展开为：

```cpp
#define Q_OBJECT \
public: \
    QT_WARNING_PUSH \
    Q_OBJECT_NO_OVERRIDE_WARNING \
    static const QMetaObject staticMetaObject; \
    virtual const QMetaObject *metaObject() const; \
    virtual void *qt_metacast(const char *); \
    virtual int qt_metacall(QMetaObject::Call, int, void **); \
    QT_TR_FUNCTIONS \
private: \
    Q_OBJECT_NO_ATTRIBUTES_WARNING \
    Q_DECL_HIDDEN_STATIC_METACALL static void qt_static_metacall(QObject *, QMetaObject::Call, int, void **); \
    QT_WARNING_POP \
    struct QPrivateSignal {}; \
    QT_ANNOTATE_CLASS(qt_qobject, "")
```

其中：
- `staticMetaObject`：静态元对象实例，存储类的元数据
- `metaObject()`：返回当前对象的元对象指针（支持多态）
- `qt_metacast()`：实现动态类型转换
- `qt_metacall()`：实现信号槽的动态调用分发
- `qt_static_metacall()`：静态成员函数，执行具体的方法调用
- `QPrivateSignal`：私有结构体，确保信号只能由定义它的类发出

### 2.3 不声明 Q_OBJECT 的后果

如果一个 `QObject` 子类没有声明 `Q_OBJECT` 宏：
- **信号和槽无法使用**（编译期可能不报错，但运行时连接失败）
- `metaObject()` 返回父类的元对象，导致类型判断错误
- `qobject_cast<>()` 失败
- 动态属性系统不可用
- `tr()` 翻译函数无法正确识别上下文

## 3. MOC（Meta-Object Compiler）

### 3.1 MOC 的工作原理

MOC 是一个**代码生成工具**，而非真正的编译器。它在 C++ 编译之前运行，扫描头文件中的 Qt 特殊宏（`Q_OBJECT`、`Q_PROPERTY`、`signals`、`slots` 等），然后生成对应的 C++ 源文件。

对于头文件 `mywidget.h`，MOC 会生成 `moc_mywidget.cpp`，其中包含：
- `QMetaObject` 的静态实例及其元数据表
- 信号的实现函数（信号是由 MOC 自动实现的）
- 字符串表（类名、信号名、槽名、属性名等）
- 方法索引映射表

### 3.2 MOC 生成代码示例

假设有如下类定义：

```cpp
// counter.h
class Counter : public QObject {
    Q_OBJECT
    Q_PROPERTY(int value READ value WRITE setValue NOTIFY valueChanged)

public:
    explicit Counter(QObject *parent = nullptr) : QObject(parent), m_value(0) {}
    int value() const { return m_value; }

public slots:
    void setValue(int val) {
        if (m_value != val) {
            m_value = val;
            emit valueChanged(m_value);
        }
    }

signals:
    void valueChanged(int newValue);

private:
    int m_value;
};
```

MOC 生成的 `moc_counter.cpp`（简化版）核心内容如下：

```cpp
// MOC 生成的字符串数据（Qt 5 使用 QByteArrayData 数组）
static const uint qt_meta_data_Counter[] = {
    // content:
    8,        // revision
    0,        // classname (index into string table)
    0, 0,     // classinfo
    2, 14,    // methods (信号+槽的数量=2，起始偏移=14)
    1, 30,    // properties (属性数量=1，起始偏移=30)
    ...
};

// 信号 valueChanged 的实现——由 MOC 自动生成
void Counter::valueChanged(int _t1)
{
    void *_a[] = { nullptr, const_cast<void*>(reinterpret_cast<const void*>(&_t1)) };
    QMetaObject::activate(this, &staticMetaObject, 0 /*signal index*/, _a);
}
```

**关键发现：信号函数的实现不是程序员写的，而是 MOC 自动生成的。** 这就是为什么在头文件中声明信号时不需要提供实现。

### 3.3 MOC 的局限性

1. **不支持模板类**：`Q_OBJECT` 不能用于模板类
   ```cpp
   // ❌ 错误：MOC 无法处理模板
   template <typename T>
   class MyContainer : public QObject {
       Q_OBJECT  // 编译失败
   };
   ```

2. **不支持多重继承中多个 QObject**：
   ```cpp
   // ❌ 错误：不能从多个 QObject 派生
   class MyClass : public QObject, public AnotherQObject {
       Q_OBJECT
   };

   // ✅ 正确：只有第一个基类可以是 QObject
   class MyClass : public QObject, public SomeInterface {
       Q_OBJECT
   };
   ```

3. **宏和预处理器指令**：MOC 不完全理解 C++ 预处理器，条件编译中的信号/槽声明可能导致问题
   ```cpp
   // ⚠️ 可能有问题
   #ifdef FEATURE_X
   signals:
       void featureSignal();
   #endif
   ```

### 3.4 CMake 中的 MOC 集成

在 CMake 项目中，通过 `AUTOMOC` 自动调用 MOC：

```cmake
set(CMAKE_AUTOMOC ON)

add_executable(myapp main.cpp mywidget.cpp)
target_link_libraries(myapp Qt5::Widgets)
```

也可以手动调用：

```cmake
qt5_wrap_cpp(MOC_FILES mywidget.h)
add_executable(myapp main.cpp mywidget.cpp ${MOC_FILES})
```

## 4. QMetaObject — 元对象

### 4.1 QMetaObject 结构

每个声明了 `Q_OBJECT` 的类都拥有一个 `QMetaObject` 静态实例 `staticMetaObject`，它记录了类在运行时的完整元信息。

`QMetaObject` 内部存储以下信息：

| 信息类型 | 说明 | 访问方式 |
|---------|------|---------|
| 类名 | 类的名称字符串 | `className()` |
| 父类元对象 | 指向父类的 QMetaObject | `superClass()` |
| 类信息 | `Q_CLASSINFO` 声明的键值对 | `classInfo(int index)` |
| 构造函数 | 用 `Q_INVOKABLE` 标记的构造函数 | `constructor(int index)` |
| 方法 | 信号、槽、可调用方法 | `method(int index)` |
| 枚举 | `Q_ENUM`/`Q_FLAG` 声明的枚举 | `enumerator(int index)` |
| 属性 | `Q_PROPERTY` 声明的属性 | `property(int index)` |

### 4.2 运行时类型内省

```cpp
#include <QDebug>
#include <QMetaObject>
#include <QMetaMethod>
#include <QMetaProperty>
#include <QPushButton>

void inspectObject(QObject *obj) {
    const QMetaObject *meta = obj->metaObject();

    // 1. 获取类名
    qDebug() << "Class:" << meta->className();

    // 2. 遍历继承链
    const QMetaObject *super = meta->superClass();
    while (super) {
        qDebug() << "  inherits:" << super->className();
        super = super->superClass();
    }

    // 3. 列出所有方法（信号 + 槽 + 可调用方法）
    qDebug() << "Methods:";
    for (int i = meta->methodOffset(); i < meta->methodCount(); ++i) {
        QMetaMethod method = meta->method(i);
        qDebug() << "  " << method.methodType() << method.methodSignature();
    }

    // 4. 列出所有属性
    qDebug() << "Properties:";
    for (int i = meta->propertyOffset(); i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        qDebug() << "  " << prop.name() << ":" << prop.typeName()
                 << "readable:" << prop.isReadable()
                 << "writable:" << prop.isWritable();
    }
}

// 使用示例
QPushButton btn("Hello");
inspectObject(&btn);
```

输出（部分）：

```
Class: QPushButton
  inherits: QAbstractButton
  inherits: QWidget
  inherits: QObject
Methods:
  Signal "clicked(bool)"
  ...
Properties:
  "text" : "QString" readable: true writable: true
  "flat" : "bool" readable: true writable: true
  ...
```

### 4.3 methodOffset() 与 propertyOffset()

`methodOffset()` 返回本类新增方法的起始索引（排除父类方法）。遍历时从 `offset` 开始可以只查看当前类自身声明的方法和属性，不包含继承的。若从 `0` 开始则包含所有继承的方法。

```cpp
// 只遍历本类方法
for (int i = meta->methodOffset(); i < meta->methodCount(); ++i) { ... }

// 遍历所有方法（包含继承）
for (int i = 0; i < meta->methodCount(); ++i) { ... }
```

## 5. 动态类型转换：qobject_cast

### 5.1 基本用法

`qobject_cast<>()` 是 Qt 版的 `dynamic_cast<>()`，但它不依赖 C++ RTTI，而是基于元对象系统实现，效率更高。

```cpp
QObject *obj = new QTimer(this);

// 安全的向下转型
QTimer *timer = qobject_cast<QTimer *>(obj);
if (timer) {
    timer->start(1000);
}

// 转型失败返回 nullptr
QLabel *label = qobject_cast<QLabel *>(obj);
Q_ASSERT(label == nullptr);  // obj 不是 QLabel
```

### 5.2 qobject_cast vs dynamic_cast

| 特性 | `qobject_cast` | `dynamic_cast` |
|------|---------------|----------------|
| 依赖 | Qt 元对象系统 | C++ RTTI |
| 要求 | 目标类必须有 `Q_OBJECT` | 类必须有虚函数 |
| 性能 | 更快（字符串比较链） | 依赖编译器实现 |
| 跨 DLL | 安全可靠 | 某些平台可能失败 |
| 适用范围 | 仅 `QObject` 子类 | 所有多态类型 |

### 5.3 实现原理

`qobject_cast` 内部调用 `QMetaObject::cast()`，通过遍历元对象的继承链来判断类型兼容性：

```cpp
// 简化版实现逻辑
template <class T>
inline T qobject_cast(QObject *object) {
    // 在编译期检查 T 是 QObject 子类
    // 运行时通过 qt_metacast 沿继承链查找
    return static_cast<T>(
        reinterpret_cast<T>(0)->staticMetaObject.cast(object)
    );
}
```

## 6. Q_INVOKABLE 与动态方法调用

### 6.1 Q_INVOKABLE 宏

`Q_INVOKABLE` 标记的方法可以通过元对象系统在运行时动态调用，这在 QML 集成和插件系统中非常有用。

```cpp
class Calculator : public QObject {
    Q_OBJECT
public:
    Q_INVOKABLE int add(int a, int b) const { return a + b; }
    Q_INVOKABLE QString greet(const QString &name) const {
        return QString("Hello, %1!").arg(name);
    }
};
```

### 6.2 QMetaObject::invokeMethod()

通过字符串名称在运行时调用方法：

```cpp
Calculator calc;
int result = 0;

// 方式一：同步调用（直接连接）
QMetaObject::invokeMethod(&calc, "add",
    Qt::DirectConnection,
    Q_RETURN_ARG(int, result),
    Q_ARG(int, 3),
    Q_ARG(int, 5));
qDebug() << "3 + 5 =" << result;  // 输出：3 + 5 = 8

// 方式二：异步调用（队列连接，常用于跨线程）
QMetaObject::invokeMethod(&calc, "greet",
    Qt::QueuedConnection,
    Q_ARG(QString, "Qt"));

// 方式三：Qt 5.10+ 的 lambda 写法
QMetaObject::invokeMethod(&calc, [&calc]() {
    qDebug() << calc.add(10, 20);
});
```

### 6.3 连接类型

| 连接类型 | 说明 |
|---------|------|
| `Qt::DirectConnection` | 在当前线程立即调用（同步） |
| `Qt::QueuedConnection` | 投递到目标对象所在线程的事件循环（异步） |
| `Qt::BlockingQueuedConnection` | 投递到目标线程并阻塞等待完成（跨线程同步） |
| `Qt::AutoConnection`（默认）| 同线程用 Direct，跨线程用 Queued |

## 7. Q_ENUM 与 Q_FLAG — 枚举注册

### 7.1 Q_ENUM

将枚举注册到元对象系统，使其可以在调试输出、属性系统和 QML 中使用。

```cpp
class TrafficLight : public QObject {
    Q_OBJECT
public:
    enum Color {
        Red,
        Yellow,
        Green
    };
    Q_ENUM(Color)  // 注册枚举到元对象系统

    void setColor(Color c) {
        m_color = c;
        qDebug() << "Color changed to:" << c;  // 自动输出枚举名
    }

private:
    Color m_color = Red;
};
```

注册后的效果：
- `qDebug()` 输出枚举名而非数字：`Color changed to: TrafficLight::Green`
- 可用于 `QVariant` 转换
- 可通过 `QMetaEnum` 实现字符串与枚举值的互转

### 7.2 QMetaEnum 使用

```cpp
// 枚举值 → 字符串
QMetaEnum metaEnum = QMetaEnum::fromType<TrafficLight::Color>();
const char *name = metaEnum.valueToKey(TrafficLight::Green);
qDebug() << name;  // "Green"

// 字符串 → 枚举值
int value = metaEnum.keyToValue("Yellow");
qDebug() << value;  // 1

// 遍历所有枚举值
for (int i = 0; i < metaEnum.keyCount(); ++i) {
    qDebug() << metaEnum.key(i) << "=" << metaEnum.value(i);
}
// 输出：
// "Red" = 0
// "Yellow" = 1
// "Green" = 2
```

### 7.3 Q_FLAG

用于位标志枚举（可按位组合的枚举），使 QMetaEnum 支持解析组合值：

```cpp
class FileAccess : public QObject {
    Q_OBJECT
public:
    enum Permission {
        NoPermission = 0x0,
        Read         = 0x1,
        Write        = 0x2,
        Execute      = 0x4
    };
    Q_DECLARE_FLAGS(Permissions, Permission)
    Q_FLAG(Permissions)
};
Q_DECLARE_OPERATORS_FOR_FLAGS(FileAccess::Permissions)

// 使用
FileAccess::Permissions perms = FileAccess::Read | FileAccess::Write;
qDebug() << perms;  // 输出 "Read|Write" 而非 "3"
```

## 8. Q_CLASSINFO — 类信息

`Q_CLASSINFO` 为类附加键值对形式的元数据，常用于 D-Bus 接口声明、QML 注册和文档标注。

```cpp
class MyPlugin : public QObject {
    Q_OBJECT
    Q_CLASSINFO("Author", "Zhang San")
    Q_CLASSINFO("Version", "1.2.0")
    Q_CLASSINFO("Description", "A sample plugin")
    Q_CLASSINFO("D-Bus Interface", "com.example.MyPlugin")

public:
    explicit MyPlugin(QObject *parent = nullptr);
};

// 运行时读取类信息
const QMetaObject *meta = MyPlugin::staticMetaObject;
for (int i = meta->classInfoOffset(); i < meta->classInfoCount(); ++i) {
    QMetaClassInfo info = meta->classInfo(i);
    qDebug() << info.name() << "=" << info.value();
}
// 输出：
// "Author" = "Zhang San"
// "Version" = "1.2.0"
// "Description" = "A sample plugin"
// "D-Bus Interface" = "com.example.MyPlugin"
```

## 9. Q_INTERFACES — 接口声明

用于声明类实现了某个接口，配合 `Q_DECLARE_INTERFACE` 使 `qobject_cast` 能够跨接口转换。这在**插件系统**中非常重要。

```cpp
// 1. 定义接口
class IMediaPlayer {
public:
    virtual ~IMediaPlayer() = default;
    virtual void play(const QString &file) = 0;
    virtual void stop() = 0;
    virtual bool isPlaying() const = 0;
};
// 注册接口标识符
Q_DECLARE_INTERFACE(IMediaPlayer, "com.example.IMediaPlayer/1.0")

// 2. 实现接口
class VideoPlayer : public QObject, public IMediaPlayer {
    Q_OBJECT
    Q_INTERFACES(IMediaPlayer)  // 声明实现了 IMediaPlayer 接口

public:
    void play(const QString &file) override {
        m_playing = true;
        qDebug() << "Playing:" << file;
    }
    void stop() override { m_playing = false; }
    bool isPlaying() const override { return m_playing; }

private:
    bool m_playing = false;
};

// 3. 使用 qobject_cast 进行接口转换
QObject *obj = new VideoPlayer();
IMediaPlayer *player = qobject_cast<IMediaPlayer *>(obj);
if (player) {
    player->play("video.mp4");  // 成功转换
}
```

## 10. Q_GADGET — 轻量级元对象

`Q_GADGET` 为不继承 `QObject` 的值类型（如结构体）提供部分元对象能力，支持 `Q_ENUM`、`Q_PROPERTY` 和 `Q_INVOKABLE`，但**不支持信号与槽**。

```cpp
class Color {
    Q_GADGET
    Q_PROPERTY(int red MEMBER m_red)
    Q_PROPERTY(int green MEMBER m_green)
    Q_PROPERTY(int blue MEMBER m_blue)

public:
    enum Format { RGB, HSL, HSV };
    Q_ENUM(Format)

    Q_INVOKABLE QString toString() const {
        return QString("rgb(%1, %2, %3)").arg(m_red).arg(m_green).arg(m_blue);
    }

private:
    int m_red = 0;
    int m_green = 0;
    int m_blue = 0;
};
```

**Q_GADGET vs Q_OBJECT 对比：**

| 特性 | Q_OBJECT | Q_GADGET |
|------|----------|----------|
| 需要继承 QObject | ✅ 是 | ❌ 否 |
| 信号与槽 | ✅ 支持 | ❌ 不支持 |
| Q_PROPERTY | ✅ 支持 | ✅ 支持 |
| Q_ENUM / Q_FLAG | ✅ 支持 | ✅ 支持 |
| Q_INVOKABLE | ✅ 支持 | ✅ 支持 |
| qobject_cast | ✅ 支持 | ❌ 不支持 |
| 可作为值类型使用 | ❌ 不可（无拷贝） | ✅ 可以 |
| QML 中使用 | ✅ 完全支持 | ✅ 部分支持 |

## 11. 元对象系统的实际应用场景

### 11.1 反射式序列化/反序列化

利用属性系统实现通用的 JSON 序列化：

```cpp
#include <QJsonObject>
#include <QMetaProperty>

QJsonObject serialize(const QObject *obj) {
    QJsonObject json;
    const QMetaObject *meta = obj->metaObject();
    for (int i = meta->propertyOffset(); i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        json[prop.name()] = QJsonValue::fromVariant(obj->property(prop.name()));
    }
    return json;
}

void deserialize(QObject *obj, const QJsonObject &json) {
    const QMetaObject *meta = obj->metaObject();
    for (int i = meta->propertyOffset(); i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        if (json.contains(prop.name()) && prop.isWritable()) {
            obj->setProperty(prop.name(), json[prop.name()].toVariant());
        }
    }
}

// 使用示例
class UserProfile : public QObject {
    Q_OBJECT
    Q_PROPERTY(QString name READ name WRITE setName)
    Q_PROPERTY(int age READ age WRITE setAge)
    Q_PROPERTY(QString email READ email WRITE setEmail)
    // ... getter/setter 省略
};

UserProfile user;
user.setName("Alice");
user.setAge(28);
user.setEmail("alice@example.com");

QJsonObject json = serialize(&user);
// {"name": "Alice", "age": 28, "email": "alice@example.com"}
```

### 11.2 插件工厂模式

利用 `QMetaObject::newInstance()` 实现动态对象创建：

```cpp
class Shape : public QObject {
    Q_OBJECT
public:
    Q_INVOKABLE explicit Shape(QObject *parent = nullptr) : QObject(parent) {}
    virtual void draw() = 0;
};

class Circle : public Shape {
    Q_OBJECT
public:
    Q_INVOKABLE explicit Circle(QObject *parent = nullptr) : Shape(parent) {}
    void draw() override { qDebug() << "Drawing circle"; }
};

class Rectangle : public Shape {
    Q_OBJECT
public:
    Q_INVOKABLE explicit Rectangle(QObject *parent = nullptr) : Shape(parent) {}
    void draw() override { qDebug() << "Drawing rectangle"; }
};

// 工厂：根据类名动态创建对象
class ShapeFactory {
public:
    void registerShape(const QMetaObject *metaObj) {
        m_shapes[metaObj->className()] = metaObj;
    }

    Shape* create(const QString &className, QObject *parent = nullptr) {
        auto it = m_shapes.find(className);
        if (it != m_shapes.end()) {
            QObject *obj = it.value()->newInstance(Q_ARG(QObject*, parent));
            return qobject_cast<Shape*>(obj);
        }
        return nullptr;
    }

private:
    QMap<QString, const QMetaObject*> m_shapes;
};

// 使用
ShapeFactory factory;
factory.registerShape(&Circle::staticMetaObject);
factory.registerShape(&Rectangle::staticMetaObject);

Shape *shape = factory.create("Circle");
shape->draw();  // "Drawing circle"
```

### 11.3 跨线程方法调用

```cpp
class Worker : public QObject {
    Q_OBJECT
public:
    Q_INVOKABLE void heavyComputation(const QString &data) {
        qDebug() << "Computing on thread:" << QThread::currentThread();
        // ... 耗时操作
    }
};

// 在主线程中，安全地在工作线程中执行方法
Worker *worker = new Worker;
QThread *thread = new QThread;
worker->moveToThread(thread);
thread->start();

// 异步投递到 worker 所在线程执行
QMetaObject::invokeMethod(worker, "heavyComputation",
    Qt::QueuedConnection,
    Q_ARG(QString, "some data"));
```

## 12. Qt 5 vs Qt 6 元对象系统的变化

| 特性 | Qt 5 | Qt 6 |
|------|-------|-------|
| 字符串数据存储 | `QByteArrayData` 数组 | 改用偏移量表 + UTF-8 字符串池 |
| 属性系统 | `Q_PROPERTY` 需 READ/WRITE | 新增 `BINDABLE` 支持属性绑定 |
| 信号槽连接 | 编译期 + 运行时检查 | 更严格的编译期检查 |
| `QMetaType` | 需 `Q_DECLARE_METATYPE` | 更多类型自动注册 |
| MOC 输出格式 | revision 67 | revision 68+，更紧凑 |
| 枚举序列化 | 数字或字符串 | 默认使用字符串表示 |

### Qt 6 属性绑定示例

```cpp
// Qt 6 新增的 Bindable 属性
class Slider : public QObject {
    Q_OBJECT
    Q_PROPERTY(int value READ value WRITE setValue NOTIFY valueChanged BINDABLE bindableValue)

public:
    int value() const { return m_value.value(); }
    void setValue(int v) { m_value.setValue(v); }
    QBindable<int> bindableValue() { return &m_value; }

signals:
    void valueChanged();

private:
    Q_OBJECT_BINDABLE_PROPERTY(Slider, int, m_value, &Slider::valueChanged)
};

// 使用属性绑定（无需手动连接信号槽）
Slider slider1, slider2;
slider2.bindableValue().setBinding([&]() { return slider1.value() * 2; });
slider1.setValue(5);
qDebug() << slider2.value();  // 自动变为 10
```

## 14. 常见问题与最佳实践

### 14.1 常见错误

**1. 忘记声明 Q_OBJECT**
```
// 错误信息："undefined reference to vtable for MyClass"
// 解决：在类中添加 Q_OBJECT 宏，重新运行 CMake/qmake
```

**2. MOC 未处理文件**
```
// 现象：信号槽连接编译通过但运行时不工作
// 原因：头文件未被 MOC 扫描
// 解决（CMake）：确保 AUTOMOC=ON，且类声明在头文件中
//              如果类在 .cpp 中定义，需在末尾添加 #include "moc_xxx.cpp"
```

**3. Q_OBJECT 在 .cpp 文件中定义的类**
```cpp
// mywidget.cpp
class PrivateHelper : public QObject {
    Q_OBJECT
    // ...
};

// 必须在文件末尾添加：
#include "moc_mywidget.cpp"  // 或 "mywidget.moc"（取决于构建系统）
```

### 14.2 最佳实践

1. **始终在 QObject 子类中声明 Q_OBJECT**，即使当前不使用信号/槽，为将来扩展预留。

2. **优先使用 `qobject_cast` 而非 `dynamic_cast`**，它更高效且跨动态库安全。

3. **值类型用 `Q_GADGET`**，避免不必要地继承 `QObject`（`QObject` 不可拷贝）。

4. **在 QML 暴露的类中使用 `Q_ENUM`**，使枚举在 QML 端直接可读。

5. **构造函数标记 `Q_INVOKABLE`** 如果需要从 QML 创建对象或使用 `QMetaObject::newInstance()`。

6. **修改 Q_OBJECT 相关声明后**，清理构建目录并重新编译（CMake: 删除 build 目录或只删除 moc 缓存）。
