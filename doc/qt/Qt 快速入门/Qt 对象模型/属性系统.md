# Qt 属性系统（Property System）

Qt 属性系统是建立在**元对象系统（Meta-Object System）**之上的一套机制，允许在运行时对 QObject 派生类的成员进行**动态查询、读取和设置**。它是 Qt Designer、QML 引擎、样式表系统和序列化框架的核心基础设施。

---

## 1. 属性系统概述

### 1.1 为什么需要属性系统

传统 C++ 的成员变量在编译后就失去了名称信息，无法在运行时按名字访问。Qt 属性系统解决了以下场景的需求：

- **Qt Designer**：在可视化编辑器中按名称列出和修改控件属性
- **QML 绑定**：QML 引擎通过属性名访问 C++ 对象的数据
- **样式表**：`qproperty-<name>` 语法直接设置属性值
- **序列化**：按属性名遍历对象状态，实现持久化
- **脚本引擎**：QtScript / JavaScript 通过属性名操作 C++ 对象

### 1.2 属性系统的组成

| 组件 | 作用 |
|------|------|
| `Q_PROPERTY` 宏 | 在类中声明属性的元信息 |
| `QMetaProperty` | 运行时属性的描述对象 |
| `QObject::property()` / `setProperty()` | 运行时按名称读写属性 |
| `QObject::dynamicPropertyNames()` | 查询动态属性列表 |
| `moc`（元对象编译器） | 解析 `Q_PROPERTY` 并生成反射代码 |

## 2. Q_PROPERTY 宏详解

### 2.1 完整语法

```cpp
Q_PROPERTY(type name
    READ getter
    [WRITE setter]
    [RESET resetter]
    [NOTIFY notifySignal]
    [REVISION int]
    [DESIGNABLE bool]
    [SCRIPTABLE bool]
    [STORED bool]
    [USER bool]
    [CONSTANT]
    [FINAL]
    [MEMBER memberVariable]
)
```

### 2.2 各关键字含义

| 关键字 | 必须 | 说明 |
|--------|------|------|
| `READ` | 是* | 读取函数，返回属性值。若使用 `MEMBER` 则可省略 |
| `WRITE` | 否 | 设置函数，接收属性值。省略则为只读属性 |
| `RESET` | 否 | 重置函数，将属性恢复为默认值，无参数 |
| `NOTIFY` | 否 | 值变化时发出的信号，**QML 绑定必须** |
| `MEMBER` | 否 | 直接绑定成员变量，moc 自动生成 getter/setter |
| `REVISION` | 否 | 用于 QML 版本控制，标注属性所属的修订号 |
| `DESIGNABLE` | 否 | 是否在 Qt Designer 中显示（默认 `true`） |
| `SCRIPTABLE` | 否 | 是否对脚本引擎可见（默认 `true`） |
| `STORED` | 否 | 是否参与序列化（默认 `true`） |
| `USER` | 否 | 是否标记为"用户属性"，每个类只有一个（如 QLineEdit 的 `text`） |
| `CONSTANT` | 否 | 属性值不会变化，与 `WRITE`/`NOTIFY` 互斥 |
| `FINAL` | 否 | 属性不会被子类覆盖，允许优化 |

> **注意**：`NOTIFY` 信号必须是该类中已声明的信号，且**至多一个参数**（新值）或无参数。

### 2.3 基础示例

```cpp
#include <QObject>

class Person : public QObject
{
    Q_OBJECT
    // 标准写法：READ + WRITE + NOTIFY
    Q_PROPERTY(QString name READ name WRITE setName NOTIFY nameChanged)
    // 只读属性
    Q_PROPERTY(int age READ age NOTIFY ageChanged)
    // 带 RESET 的属性
    Q_PROPERTY(QColor color READ color WRITE setColor RESET resetColor NOTIFY colorChanged)

public:
    explicit Person(QObject *parent = nullptr) : QObject(parent), m_age(0) {}

    QString name() const { return m_name; }
    void setName(const QString &name) {
        if (m_name != name) {
            m_name = name;
            emit nameChanged(m_name);
        }
    }

    int age() const { return m_age; }

    QColor color() const { return m_color; }
    void setColor(const QColor &c) {
        if (m_color != c) {
            m_color = c;
            emit colorChanged();
        }
    }
    void resetColor() { setColor(QColor(Qt::white)); }

signals:
    void nameChanged(const QString &name);
    void ageChanged(int age);
    void colorChanged();

private:
    QString m_name;
    int m_age;
    QColor m_color;
};
```

## 3. MEMBER 关键字（Qt 5.1+）

`MEMBER` 允许直接将属性绑定到成员变量，**无需手动编写 getter/setter**，moc 会自动生成访问代码。

### 3.1 基本用法

```cpp
class Settings : public QObject
{
    Q_OBJECT
    Q_PROPERTY(int volume MEMBER m_volume NOTIFY volumeChanged)
    Q_PROPERTY(bool muted MEMBER m_muted NOTIFY mutedChanged)
    Q_PROPERTY(QString theme MEMBER m_theme NOTIFY themeChanged)

signals:
    void volumeChanged();
    void mutedChanged();
    void themeChanged();

private:
    int m_volume = 50;
    bool m_muted = false;
    QString m_theme = "default";
};
```

### 3.2 MEMBER 与 READ/WRITE 混用

可以只提供 `MEMBER` + 自定义 `WRITE`，在 setter 中加入验证逻辑：

```cpp
Q_PROPERTY(int volume MEMBER m_volume WRITE setVolume NOTIFY volumeChanged)

void setVolume(int v) {
    v = qBound(0, v, 100);  // 限制范围
    if (m_volume != v) {
        m_volume = v;
        emit volumeChanged();
    }
}
```

### 3.3 MEMBER 的局限

- moc 生成的 setter 不包含验证逻辑，直接赋值
- 若需要在写入时执行副作用操作，应使用显式的 `WRITE`
- `MEMBER` 生成的 getter 返回值的**副本**，对引用类型需注意

## 4. 运行时读写属性

### 4.1 property() 和 setProperty()

```cpp
Person *person = new Person;

// 设置属性（通过属性名字符串）
person->setProperty("name", "Alice");
person->setProperty("age", 25);  // 注意：age 是只读的，此操作会失败并返回 false

// 读取属性
QVariant value = person->property("name");
qDebug() << value.toString();  // "Alice"
```

### 4.2 QVariant 类型转换

属性值通过 `QVariant` 传递，Qt 会**自动进行类型转换**：

```cpp
// 设置 int 属性为 QString，Qt 会自动转换
person->setProperty("volume", "75");

// 读取时转换
int vol = person->property("volume").toInt();
QString volStr = person->property("volume").toString();
```

### 4.3 判断属性是否存在

```cpp
// property() 不存在时返回无效 QVariant
QVariant v = obj->property("nonExistent");
if (!v.isValid()) {
    qDebug() << "属性不存在";
}
```

## 5. 动态属性

Qt 允许在运行时为**任意 QObject 实例**添加新属性，这些属性不在 `Q_PROPERTY` 中声明，称为**动态属性**。

### 5.1 添加和移除动态属性

```cpp
QPushButton *btn = new QPushButton("OK");

// 添加动态属性
btn->setProperty("customId", 42);
btn->setProperty("category", "action");

// 读取
qDebug() << btn->property("customId").toInt();  // 42

// 移除（设置为无效 QVariant）
btn->setProperty("customId", QVariant());
```

### 5.2 查询所有动态属性

```cpp
QList<QByteArray> dynProps = btn->dynamicPropertyNames();
for (const QByteArray &name : dynProps) {
    qDebug() << name << "=" << btn->property(name);
}
```

### 5.3 动态属性的典型应用

#### 场景 1：配合样式表实现状态切换

```cpp
// C++ 中设置动态属性
btn->setProperty("urgency", "high");
// 样式表中使用属性选择器
// QPushButton[urgency="high"] { background: red; }

// 属性变化后需要刷新样式
btn->style()->unpolish(btn);
btn->style()->polish(btn);
```

对应 QSS：

```css
QPushButton[urgency="high"] {
    background-color: #e74c3c;
    color: white;
}

QPushButton[urgency="low"] {
    background-color: #2ecc71;
}
```

#### 场景 2：在模型/视图中传递额外数据

```cpp
// 在控件上附加业务数据
listWidget->item(0)->setData(Qt::UserRole, QVariant::fromValue(recordId));

// 或者用动态属性标记
widget->setProperty("dataSource", "remote");
```

#### 场景 3：动态属性变化通知

动态属性没有 `NOTIFY` 信号，但可以通过重写 `QDynamicPropertyChangeEvent` 来监听：

```cpp
bool MyWidget::event(QEvent *e) {
    if (e->type() == QEvent::DynamicPropertyChange) {
        auto *pe = static_cast<QDynamicPropertyChangeEvent*>(e);
        qDebug() << "动态属性变化:" << pe->propertyName();
    }
    return QWidget::event(e);
}
```

## 6. 通过 QMetaProperty 反射属性

`QMetaProperty` 提供了属性的完整元信息，可用于通用的属性遍历、序列化等场景。

### 6.1 遍历所有属性

```cpp
void dumpProperties(const QObject *obj) {
    const QMetaObject *meta = obj->metaObject();

    qDebug() << "=== 类名:" << meta->className() << "===";

    // 从 QObject::staticMetaObject.propertyOffset() 开始跳过父类属性
    // 或从 0 开始获取包含继承的全部属性
    for (int i = 0; i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        QVariant value = obj->property(prop.name());

        qDebug() << QString("  [%1] %2 (%3) = %4")
                        .arg(i)
                        .arg(prop.name())
                        .arg(prop.typeName())
                        .arg(value.toString());
    }
}
```

输出示例：

```
=== 类名: Person ===
  [0] objectName (QString) =
  [1] name (QString) = Alice
  [2] age (int) = 25
  [3] color (QColor) = #ffffff
```

### 6.2 QMetaProperty 常用 API

```cpp
QMetaProperty prop = meta->property(index);

prop.name();           // 属性名（const char*）
prop.typeName();       // 类型名字符串
prop.type();           // QVariant::Type 枚举
prop.userType();       // 用户类型 ID（QMetaType）

prop.isReadable();     // 是否可读
prop.isWritable();     // 是否可写
prop.isResettable();   // 是否有 RESET
prop.hasNotifySignal();// 是否有 NOTIFY
prop.notifySignal();   // 返回 QMetaMethod

prop.isDesignable();   // DESIGNABLE
prop.isScriptable();   // SCRIPTABLE
prop.isStored();       // STORED
prop.isUser();         // USER
prop.isConstant();     // CONSTANT
prop.isFinal();        // FINAL

prop.read(obj);        // 读取值（等价于 obj->property(name)）
prop.write(obj, val);  // 写入值
prop.reset(obj);       // 重置值

prop.isEnumType();     // 是否为枚举属性
prop.enumerator();     // 返回 QMetaEnum
```

### 6.3 只获取自身类（不含继承）的属性

```cpp
const QMetaObject *meta = obj->metaObject();
int offset = meta->propertyOffset();  // 本类属性起始索引

for (int i = offset; i < meta->propertyCount(); ++i) {
    QMetaProperty prop = meta->property(i);
    qDebug() << prop.name();
}
```

> `propertyOffset()` 返回**第一个属于本类自身的属性索引**，之前的属性都属于父类。

## 7. 枚举属性（Q_ENUM / Q_FLAG）

当属性类型为枚举时，需要使用 `Q_ENUM` 或 `Q_FLAG` 注册，使其参与属性系统的字符串—枚举值转换。

### 7.1 Q_ENUM 示例

```cpp
class TrafficLight : public QObject
{
    Q_OBJECT
public:
    enum State { Red, Yellow, Green };
    Q_ENUM(State)

    Q_PROPERTY(State currentState READ currentState WRITE setCurrentState NOTIFY stateChanged)

    State currentState() const { return m_state; }
    void setCurrentState(State s) {
        if (m_state != s) {
            m_state = s;
            emit stateChanged();
        }
    }

signals:
    void stateChanged();

private:
    State m_state = Red;
};
```

使用方式：

```cpp
TrafficLight light;

// 可以用枚举值设置
light.setProperty("currentState", TrafficLight::Green);

// 也可以用字符串（通过 QMetaEnum）
light.setProperty("currentState", "Green");  // ⚠ 需通过 QMetaEnum 手动转换

// 查看枚举元信息
QMetaEnum me = QMetaEnum::fromType<TrafficLight::State>();
qDebug() << me.valueToKey(TrafficLight::Green);  // "Green"
qDebug() << me.keyToValue("Yellow");              // 1
qDebug() << me.keyCount();                        // 3
```

### 7.2 Q_FLAG 示例（位掩码）

```cpp
class TextEditor : public QObject
{
    Q_OBJECT
public:
    enum Option {
        NoOption    = 0x0,
        WordWrap    = 0x1,
        LineNumbers = 0x2,
        HighlightCurrentLine = 0x4
    };
    Q_DECLARE_FLAGS(Options, Option)
    Q_FLAG(Options)

    Q_PROPERTY(Options options READ options WRITE setOptions NOTIFY optionsChanged)

    Options options() const { return m_options; }
    void setOptions(Options opts) {
        if (m_options != opts) {
            m_options = opts;
            emit optionsChanged();
        }
    }

signals:
    void optionsChanged();

private:
    Options m_options = NoOption;
};

Q_DECLARE_OPERATORS_FOR_FLAGS(TextEditor::Options)
```

## 8. 属性与 QML 的集成

### 8.1 在 QML 中使用 C++ 属性

```cpp
// C++ 端注册
qmlRegisterType<Person>("com.example", 1, 0, "Person");
```

```qml
// QML 端使用
import com.example 1.0

Person {
    id: person
    name: "Bob"  // 调用 setName()

    onNameChanged: console.log("名字变了:", name)
}

Text {
    text: person.name  // 自动绑定，name 变化时 Text 自动刷新
}
```

### 8.2 NOTIFY 信号对 QML 的重要性

**QML 属性绑定依赖 NOTIFY 信号**。如果一个属性没有声明 `NOTIFY`：

- QML 会发出警告：`NON-NOTIFYABLE property "xxx" used in binding`
- 绑定只在初始化时读取一次，之后不再更新
- 这是 QML 开发中**最常见的 bug 来源之一**

### 8.3 REVISION 版本控制

```cpp
Q_PROPERTY(QString email READ email WRITE setEmail NOTIFY emailChanged REVISION 2)
```

```qml
// QML 中只有 import 2.0+ 才能访问 email 属性
import com.example 2.0
```

## 9. 属性与样式表

Qt 样式表支持通过 `qproperty-<name>` 语法直接设置属性值：

```css
/* 直接设置 QLabel 的属性 */
QLabel {
    qproperty-alignment: AlignCenter;
    qproperty-text: "Hello";
}

/* 设置自定义属性 */
MyWidget {
    qproperty-borderRadius: 10;
    qproperty-backgroundColor: #3498db;
}
```

**限制**：
- 只能设置简单类型（int、QString、QColor 等）
- 属性必须有 `WRITE` 方法
- 复杂类型无法用样式表设置

## 10. 完整实战：可序列化的配置对象

以下示例展示如何利用属性系统实现通用的**对象序列化/反序列化**：

### 10.1 配置类定义

```cpp
class AppConfig : public QObject
{
    Q_OBJECT
    Q_PROPERTY(QString language READ language WRITE setLanguage NOTIFY languageChanged)
    Q_PROPERTY(int fontSize READ fontSize WRITE setFontSize NOTIFY fontSizeChanged)
    Q_PROPERTY(bool darkMode READ darkMode WRITE setDarkMode NOTIFY darkModeChanged)
    Q_PROPERTY(QStringList recentFiles READ recentFiles WRITE setRecentFiles NOTIFY recentFilesChanged)

public:
    explicit AppConfig(QObject *parent = nullptr);

    QString language() const { return m_language; }
    void setLanguage(const QString &lang) {
        if (m_language != lang) { m_language = lang; emit languageChanged(); }
    }

    int fontSize() const { return m_fontSize; }
    void setFontSize(int size) {
        size = qBound(8, size, 72);
        if (m_fontSize != size) { m_fontSize = size; emit fontSizeChanged(); }
    }

    bool darkMode() const { return m_darkMode; }
    void setDarkMode(bool on) {
        if (m_darkMode != on) { m_darkMode = on; emit darkModeChanged(); }
    }

    QStringList recentFiles() const { return m_recentFiles; }
    void setRecentFiles(const QStringList &files) {
        if (m_recentFiles != files) { m_recentFiles = files; emit recentFilesChanged(); }
    }

signals:
    void languageChanged();
    void fontSizeChanged();
    void darkModeChanged();
    void recentFilesChanged();

private:
    QString m_language = "zh-CN";
    int m_fontSize = 14;
    bool m_darkMode = false;
    QStringList m_recentFiles;
};
```

### 10.2 通用序列化工具

```cpp
#include <QJsonObject>
#include <QJsonDocument>
#include <QMetaProperty>
#include <QFile>

// 将 QObject 的所有可存储属性导出为 JSON
QJsonObject serializeObject(const QObject *obj)
{
    QJsonObject json;
    const QMetaObject *meta = obj->metaObject();

    for (int i = meta->propertyOffset(); i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        if (!prop.isStored()) continue;

        QVariant value = prop.read(obj);
        json[prop.name()] = QJsonValue::fromVariant(value);
    }
    return json;
}

// 从 JSON 恢复属性值
void deserializeObject(QObject *obj, const QJsonObject &json)
{
    const QMetaObject *meta = obj->metaObject();

    for (int i = meta->propertyOffset(); i < meta->propertyCount(); ++i) {
        QMetaProperty prop = meta->property(i);
        if (!prop.isWritable()) continue;

        const QString key = prop.name();
        if (json.contains(key)) {
            QVariant value = json[key].toVariant();
            prop.write(obj, value);
        }
    }
}

// 保存到文件
void saveConfig(const QObject *obj, const QString &filePath)
{
    QJsonDocument doc(serializeObject(obj));
    QFile file(filePath);
    if (file.open(QIODevice::WriteOnly)) {
        file.write(doc.toJson(QJsonDocument::Indented));
    }
}

// 从文件加载
void loadConfig(QObject *obj, const QString &filePath)
{
    QFile file(filePath);
    if (file.open(QIODevice::ReadOnly)) {
        QJsonDocument doc = QJsonDocument::fromJson(file.readAll());
        deserializeObject(obj, doc.object());
    }
}
```

### 10.3 使用示例

```cpp
AppConfig config;
config.setLanguage("en-US");
config.setFontSize(16);
config.setDarkMode(true);
config.setRecentFiles({"a.mp4", "b.png"});

// 序列化保存
saveConfig(&config, "config.json");

// 反序列化加载
AppConfig newConfig;
loadConfig(&newConfig, "config.json");

qDebug() << newConfig.language();     // "en-US"
qDebug() << newConfig.fontSize();     // 16
qDebug() << newConfig.darkMode();     // true
qDebug() << newConfig.recentFiles();  // ("a.mp4", "b.png")
```

生成的 JSON 文件：

```json
{
    "language": "en-US",
    "fontSize": 16,
    "darkMode": true,
    "recentFiles": ["a.mp4", "b.png"]
}
```

## 11. 属性系统内部原理

### 11.1 moc 生成的代码

当 moc 处理 `Q_PROPERTY` 声明时，会在 `moc_*.cpp` 文件中生成：

```cpp
// 1. 属性元数据表（静态数组）
static const qt_meta_stringdata_Person_t qt_meta_stringdata_Person = { ... };
static const uint qt_meta_data_Person[] = { ... };

// 2. qt_metacall 中的属性处理
int Person::qt_metacall(QMetaObject::Call call, int id, void **arguments)
{
    id = QObject::qt_metacall(call, id, arguments);
    if (id < 0) return id;

    if (call == QMetaObject::ReadProperty) {
        switch (id) {
        case 0: *reinterpret_cast<QString*>(arguments[0]) = m_name; break;
        case 1: *reinterpret_cast<int*>(arguments[0]) = m_age; break;
        // ...
        }
    } else if (call == QMetaObject::WriteProperty) {
        switch (id) {
        case 0: setName(*reinterpret_cast<QString*>(arguments[0])); break;
        // ...
        }
    } else if (call == QMetaObject::ResetProperty) {
        switch (id) {
        case 2: resetColor(); break;
        }
    }
    return id;
}
```

### 11.2 property() 的调用链

```
QObject::property("name")
  → QMetaObject::property(index)      // 查找属性索引
  → QMetaObject::ReadProperty         // 构造调用类型
  → qt_metacall(ReadProperty, id, args)  // 虚函数调用到具体类
  → 执行 READ getter → 返回 QVariant
```

### 11.3 动态属性的存储

动态属性存储在 `QObjectPrivate` 内部的 `QList<PropertyData>` 中：

```
QObject
  └── QObjectPrivate (d_ptr)
        └── extraData
              └── propertyNames: QList<QByteArray>
              └── propertyValues: QList<QVariant>
```

每次调用 `setProperty()` 时，若属性名不在 `Q_PROPERTY` 中声明，就作为动态属性存储。

## 12. 常见陷阱与最佳实践

### 12.1 常见陷阱

#### ❌ NOTIFY 信号中忘记判断值是否真的变化

```cpp
// 错误：每次调用都发射信号，可能导致无限循环
void setName(const QString &name) {
    m_name = name;
    emit nameChanged();  // 即使值没变也发射
}

// 正确：先判断再发射
void setName(const QString &name) {
    if (m_name != name) {
        m_name = name;
        emit nameChanged();
    }
}
```

#### ❌ CONSTANT 与 NOTIFY/WRITE 同时使用

```cpp
// 编译警告：CONSTANT 属性不应有 NOTIFY 或 WRITE
Q_PROPERTY(int maxSize READ maxSize CONSTANT NOTIFY maxSizeChanged)  // 矛盾
```

#### ❌ 属性类型未注册

```cpp
// 自定义类型必须注册
struct MyData { int x; };
Q_DECLARE_METATYPE(MyData)

// 在 main() 中或构造函数中调用
qRegisterMetaType<MyData>("MyData");
```

#### ❌ 动态属性改变后样式不刷新

```cpp
btn->setProperty("state", "error");
// 样式不会自动更新，需要手动触发
btn->style()->unpolish(btn);
btn->style()->polish(btn);
btn->update();
```

### 12.2 最佳实践

| 实践 | 说明 |
|------|------|
| **始终提供 NOTIFY 信号** | 除非属性是 `CONSTANT`，否则一定声明 NOTIFY |
| **setter 中做值变化检查** | 避免不必要的信号发射和重复刷新 |
| **为 QML 属性加 NOTIFY** | QML 绑定必须有 NOTIFY 才能自动更新 |
| **getter 标记 const** | `Type property() const` 是正确的函数签名 |
| **用 FINAL 优化** | 如果属性不需要被覆盖，标记 FINAL 可让 QML 引擎优化 |
| **属性名避免下划线开头** | 遵循 Qt 命名规范：`color`、`fontSize`，而非 `_color` |
| **优先使用显式 READ/WRITE** | MEMBER 虽简洁，但缺少验证和副作用处理的机会 |

## 13. Q_PROPERTY 与 Qt 6 的变化

### 13.1 Qt 6 中的 QProperty（绑定属性）

Qt 6 引入了全新的 `QProperty<T>` 模板类，提供**声明式属性绑定**功能：

```cpp
#include <QProperty>

class Rect : public QObject
{
    Q_OBJECT
    Q_BINDABLE_PROPERTY(Rect, int, width, &Rect::widthChanged)
    Q_BINDABLE_PROPERTY(Rect, int, height, &Rect::heightChanged)

public:
    QProperty<int> width{0};
    QProperty<int> height{0};
    QProperty<int> area;

    Rect() {
        // area 自动跟随 width 和 height 的变化
        area.setBinding([this]() { return width.value() * height.value(); });
    }

signals:
    void widthChanged();
    void heightChanged();
};
```

```cpp
Rect rect;
rect.width = 10;
rect.height = 5;
qDebug() << rect.area.value();  // 50

rect.width = 20;
qDebug() << rect.area.value();  // 100（自动更新）
```

### 13.2 兼容性

| 特性 | Qt 5 | Qt 6 |
|------|-------|-------|
| `Q_PROPERTY` | ✅ | ✅ |
| `MEMBER` | ✅（5.1+） | ✅ |
| `QProperty<T>` | ❌ | ✅ |
| `Q_BINDABLE_PROPERTY` | ❌ | ✅ |
| 属性绑定链 | 需手动信号连接 | 自动传播 |

> Qt 6 的 `Q_PROPERTY` 完全兼容 Qt 5 的写法，`QProperty` 是额外的增强，不是替代。

## 14. 总结

| 核心概念 | 一句话 |
|----------|--------|
| `Q_PROPERTY` | 在类中声明属性的元信息，moc 生成反射代码 |
| `READ` / `WRITE` | 属性的 getter 和 setter |
| `NOTIFY` | 值变化通知信号，QML 绑定的基础 |
| `MEMBER` | 简化写法，直接绑定成员变量 |
| `property()` / `setProperty()` | 运行时按名称读写属性 |
| 动态属性 | 运行时临时添加的属性，不需要预先声明 |
| `QMetaProperty` | 属性的反射信息，用于通用遍历 |
| `Q_ENUM` / `Q_FLAG` | 枚举注册，让枚举参与属性系统 |
| `CONSTANT`/`FINAL` | 标记不变性，允许引擎优化 |

属性系统是 Qt 元对象能力中最实用的部分之一，理解它是掌握 QML 开发、Qt Designer 自定义控件和高级框架设计的关键。
