# 国际化与翻译

Qt 的国际化（i18n，Internationalization）与本地化（l10n，Localization）体系是工业级的，覆盖文字翻译、数字/日期/货币格式、文字方向（RTL）、字体等各维度。核心工作流：**用 `tr()` 标记字符串 → `lupdate` 提取 → Qt Linguist 翻译 → `lrelease` 编译 → 运行时加载 `.qm` 文件**。

---

## 1. 核心概念与架构

### 整体流程

```
源代码中 tr("Hello")
        ↓  lupdate（提取）
    app_zh_CN.ts（XML 翻译源文件，给翻译人员编辑）
        ↓  Qt Linguist / 手动编辑
    app_zh_CN.ts（已填写译文）
        ↓  lrelease（编译）
    app_zh_CN.qm（二进制，体积小，运行时加载）
        ↓  QTranslator::load() + QApplication::installTranslator()
    运行时 tr("Hello") → "你好"
```

### 关键组件

| 组件 | 说明 |
|---|---|
| `tr()` / `QObject::tr()` | 标记可翻译字符串，运行时查找翻译 |
| `QCoreApplication::translate()` | 非 QObject 类中标记翻译的底层函数 |
| `.ts` 文件 | 翻译源文件（XML），供翻译人员使用 |
| `.qm` 文件 | 编译后的二进制翻译文件，运行时加载 |
| `lupdate` | 扫描源码，生成/更新 `.ts` 文件 |
| `lrelease` | 将 `.ts` 编译为 `.qm` |
| `Qt Linguist` | 图形化翻译编辑器 |
| `QTranslator` | 运行时翻译对象，管理一个 `.qm` 文件 |
| `QLocale` | 本地化格式处理（数字、日期、货币等） |

## 2. 标记可翻译字符串

### 2.1 QObject 派生类中使用 tr()

```cpp
class MainWindow : public QMainWindow {
    Q_OBJECT   // 必须有 Q_OBJECT，tr() 才能绑定正确的翻译上下文
public:
    void setupUi() {
        setWindowTitle(tr("My Application"));          // 最简用法
        saveBtn->setText(tr("Save"));
        openBtn->setText(tr("Open File..."));
        statusBar()->showMessage(tr("Ready"));
    }
};
```

**tr() 的三个参数：**

```cpp
QString tr(const char *sourceText,
           const char *disambiguation = nullptr,
           int n = -1);
```

| 参数 | 用途 |
|---|---|
| `sourceText` | 原始英文字符串（必填） |
| `disambiguation` | 消歧注释，相同原文不同语境时区分（可选） |
| `n` | 复数形式判断依据（≥0 时启用复数逻辑）（可选） |

### 2.2 非 QObject 类中使用 QCoreApplication::translate()

```cpp
// 普通 struct / 全局函数中
QString text = QCoreApplication::translate("MyContext", "Submit");

// 或使用宏（等价，更简洁）
#include <QCoreApplication>
QString text = qApp->translate("MyContext", "Submit");
```

### 2.3 静态字符串：QT_TR_NOOP / QT_TRANSLATE_NOOP

静态存储的字符串（如枚举描述数组）在定义时无法调用 `tr()`，使用这两个宏仅标记，不翻译，运行时再用 `tr()` 取翻译：

```cpp
// QT_TR_NOOP：在当前类上下文中标记（不翻译）
static const char * const days[] = {
    QT_TR_NOOP("Monday"),
    QT_TR_NOOP("Tuesday"),
    QT_TR_NOOP("Wednesday"),
    // ...
};

void MyWidget::showDay(int index) {
    label->setText(tr(days[index]));  // 运行时翻译
}
```

```cpp
// QT_TRANSLATE_NOOP：指定上下文（用于非 QObject 类）
static const char * const errors[] = {
    QT_TRANSLATE_NOOP("NetworkError", "Connection refused"),
    QT_TRANSLATE_NOOP("NetworkError", "Timeout"),
};

void handleError(int code) {
    QString msg = QCoreApplication::translate("NetworkError", errors[code]);
}
```

### 2.4 消歧（disambiguation）：同一原文不同含义

英文中同一单词在不同语境含义不同，翻译到其他语言时需区分：

```cpp
// "Open" 在不同上下文含义不同，翻译为中文可能不一样
openFileAction->setText(tr("Open", "menu action: open file"));   // 打开
openDoorBtn->setText(tr("Open", "door status: currently open")); // 开启状态

// 二义性消除后，lupdate 会生成两条独立翻译条目
```

### 2.5 复数形式处理

不同语言复数规则不同（英文 1 个用单数，其余复数；波兰语更复杂）。使用第三个参数 `n`：

```cpp
// n 传入实际数量，tr() 根据语言规则返回正确形式
int count = files.size();
QString msg = tr("%n file(s) selected", "", count);
// 英文：n=1 → "1 file selected"，n=5 → "5 files selected"
// 中文：始终使用同一形式，翻译为 "已选 %n 个文件"

label->setText(tr("%n item(s)", "", itemCount));
```

在 `.ts` 文件中，翻译人员会看到 `<numerusform>` 标签来填写各复数形式：

```xml
<message numerus="yes">
    <source>%n file(s) selected</source>
    <translation>
        <numerusform>已选 %n 个文件</numerusform>
    </translation>
</message>
```

## 3. .ts 文件详解

### 3.1 文件结构

`.ts` 是标准 XML，人类可读，纳入版本控制：

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="zh_CN" sourcelanguage="en_US">
<context>
    <name>MainWindow</name>              <!-- 翻译上下文（类名） -->
    <message>
        <location filename="../mainwindow.cpp" line="42"/>
        <source>Save</source>            <!-- 原文 -->
        <translation>保存</translation>  <!-- 译文 -->
    </message>
    <message>
        <location filename="../mainwindow.cpp" line="45"/>
        <source>Open File...</source>
        <translation>打开文件…</translation>
    </message>
    <message>
        <location filename="../mainwindow.cpp" line="60"/>
        <source>%n file(s) selected</source>
        <translation type="unfinished">
            <numerusform>已选 %n 个文件</numerusform>
        </translation>
    </message>
</context>
<context>
    <name>SettingsDialog</name>
    <message>
        <location filename="../settingsdialog.cpp" line="30"/>
        <source>Language</source>
        <translation>语言</translation>
    </message>
</context>
</TS>
```

**`translation` 的 `type` 属性：**

| type 值 | 含义 |
|---|---|
| （无） | 已完成的翻译 |
| `unfinished` | 尚未完成，`lrelease` 默认不包含（除非加 `-nounfinished` 的反选） |
| `obsolete` | 源码中已删除的字符串，`lupdate` 自动标记 |
| `vanished` | Qt 5.x 新增，表示暂时消失但不删除（保留备用） |

### 3.2 生成 .ts 文件（lupdate）

```bash
# 扫描单个源文件
lupdate main.cpp mainwindow.cpp -ts app_zh_CN.ts

# 扫描整个项目（推荐：通过 .pro 文件）
lupdate myapp.pro

# 指定源语言（默认 en_US）
lupdate -source-language en_US src/ -ts translations/app_zh_CN.ts app_fr_FR.ts

# 同时更新多个语言文件
lupdate src/ -ts translations/app_zh_CN.ts translations/app_ja_JP.ts translations/app_de_DE.ts
```

**常用选项：**

| 选项 | 说明 |
|---|---|
| `-no-obsolete` | 删除 `.ts` 中废弃（obsolete）的条目 |
| `-locations none` | 不写入源码位置信息（减少 diff 噪音） |
| `-tr-function-alias` | 支持自定义翻译函数名 |
| `-extensions cpp,h,ui` | 指定要扫描的文件扩展名 |

### 3.3 编译 .ts 为 .qm（lrelease）

```bash
# 编译单个文件
lrelease translations/app_zh_CN.ts

# 编译输出到指定目录
lrelease translations/app_zh_CN.ts -qm resources/i18n/app_zh_CN.qm

# 批量编译（通过 .pro 文件）
lrelease myapp.pro

# 包含未完成的翻译（unfinished），回退到原文
lrelease -nounfinished translations/app_zh_CN.ts
```

## 4. QTranslator：运行时加载翻译

### 4.1 基本用法

```cpp
int main(int argc, char *argv[]) {
    QApplication app(argc, argv);

    // 加载翻译文件
    QTranslator translator;
    if (translator.load(":/i18n/app_zh_CN.qm")) {
        app.installTranslator(&translator);
    }

    MainWindow w;
    w.show();
    return app.exec();
}
```

### 4.2 根据系统语言自动选择

```cpp
int main(int argc, char *argv[]) {
    QApplication app(argc, argv);

    // 获取系统 UI 语言（如 "zh_CN"、"en_US"）
    QString locale = QLocale::system().name();  // e.g. "zh_CN"

    QTranslator translator;
    // 尝试加载对应语言，找不到则静默失败（保持原文）
    if (translator.load(QString(":/i18n/app_%1.qm").arg(locale))) {
        app.installTranslator(&translator);
    } else {
        // 尝试只匹配语言部分（zh_CN → zh）
        QString lang = locale.section('_', 0, 0);
        translator.load(QString(":/i18n/app_%1.qm").arg(lang));
        app.installTranslator(&translator);
    }

    // 同时加载 Qt 自身的翻译（按钮文字、对话框等系统字符串）
    QTranslator qtTranslator;
    qtTranslator.load("qt_" + locale, QLibraryInfo::location(QLibraryInfo::TranslationsPath));
    app.installTranslator(&qtTranslator);

    MainWindow w;
    w.show();
    return app.exec();
}
```

### 4.3 多 QTranslator 叠加

可以安装多个 `QTranslator`，Qt 按**后安装先查找**的顺序依次查找：

```cpp
QTranslator baseTranslator;   // 基础翻译（公共模块）
baseTranslator.load(":/i18n/base_zh_CN.qm");
app.installTranslator(&baseTranslator);

QTranslator appTranslator;    // 应用层翻译（覆盖基础翻译）
appTranslator.load(":/i18n/app_zh_CN.qm");
app.installTranslator(&appTranslator);
// appTranslator 中有的优先用，没有的再查 baseTranslator
```

### 4.4 QTranslator::load() 的路径搜索机制

```cpp
// 带搜索路径的 load() 重载（更灵活）
bool QTranslator::load(const QLocale &locale,
                       const QString &filename,       // 文件名前缀
                       const QString &prefix = "_",   // 分隔符
                       const QString &directory = "", // 搜索目录
                       const QString &suffix = ".qm");// 后缀

// 示例：自动搜索 app_zh_CN.qm、app_zh.qm、app.qm
translator.load(QLocale::system(), "app", "_", ":/i18n/", ".qm");
```

Qt 会按 `zh_CN → zh → (空)` 的顺序依次尝试，实现优雅的回退（fallback）。

## 5. 动态语言切换

应用运行中切换语言需要：**卸载旧翻译 → 安装新翻译 → 重新翻译 UI**。

### 5.1 实现方案

```cpp
// languagemanager.h
class LanguageManager : public QObject {
    Q_OBJECT
public:
    static LanguageManager *instance();

    void switchLanguage(const QString &locale);  // e.g. "zh_CN"

signals:
    void languageChanged();  // 通知所有窗口重新翻译

private:
    QTranslator m_appTranslator;
    QTranslator m_qtTranslator;
};
```

```cpp
// languagemanager.cpp
void LanguageManager::switchLanguage(const QString &locale) {
    // 先卸载旧翻译
    qApp->removeTranslator(&m_appTranslator);
    qApp->removeTranslator(&m_qtTranslator);

    // 加载新翻译
    if (m_appTranslator.load(QLocale(locale), "app", "_", ":/i18n/", ".qm")) {
        qApp->installTranslator(&m_appTranslator);
    }
    if (m_qtTranslator.load("qt_" + locale,
                            QLibraryInfo::location(QLibraryInfo::TranslationsPath))) {
        qApp->installTranslator(&m_qtTranslator);
    }

    emit languageChanged();
}
```

### 5.2 窗口响应语言切换

Qt 5.3+ 中，安装新翻译器后会自动发出 `QEvent::LanguageChange` 事件。重写 `changeEvent()` 响应：

```cpp
// mainwindow.cpp
void MainWindow::changeEvent(QEvent *event) {
    if (event->type() == QEvent::LanguageChange) {
        retranslateUi();  // 重新设置所有文字
    }
    QMainWindow::changeEvent(event);
}

void MainWindow::retranslateUi() {
    setWindowTitle(tr("My Application"));
    saveBtn->setText(tr("Save"));
    openBtn->setText(tr("Open File..."));
    menuFile->setTitle(tr("&File"));
    // ... 重新设置所有 UI 上的文字
}
```

**或者用 Qt Designer 生成的 `ui_*.h` 中的 `retranslateUi()` 方法：**

```cpp
void MainWindow::changeEvent(QEvent *event) {
    if (event->type() == QEvent::LanguageChange) {
        ui->retranslateUi(this);  // Designer 自动生成
    }
    QMainWindow::changeEvent(event);
}
```

### 5.3 完整示例：语言切换下拉框

```cpp
// 在设置对话框中
void SettingsDialog::setupLanguageCombo() {
    struct LangItem { QString locale; QString displayName; };
    static const LangItem langs[] = {
        {"en_US", "English"},
        {"zh_CN", "简体中文"},
        {"ja_JP", "日本語"},
        {"de_DE", "Deutsch"},
    };
    for (auto &lang : langs) {
        langCombo->addItem(lang.displayName, lang.locale);
    }
    connect(langCombo, &QComboBox::currentIndexChanged, this, [this](int idx) {
        QString locale = langCombo->itemData(idx).toString();
        LanguageManager::instance()->switchLanguage(locale);
    });
}
```

## 6. CMake 集成

### 6.1 Qt 6 方式（推荐）

Qt 6 提供 `qt_add_translations()` 宏，自动管理 `lupdate` / `lrelease`：

```cmake
find_package(Qt6 REQUIRED COMPONENTS Core Widgets LinguistTools)

qt_add_executable(myapp
    main.cpp
    mainwindow.cpp
)

# 声明翻译文件（自动 lupdate + lrelease + 嵌入资源）
qt_add_translations(myapp
    TS_FILES
        translations/app_zh_CN.ts
        translations/app_en_US.ts
        translations/app_ja_JP.ts
    QM_FILES_OUTPUT_VARIABLE qm_files   # 可选：获取 .qm 文件列表
)
```

构建时会自动：
1. 运行 `lupdate` 更新 `.ts` 文件
2. 运行 `lrelease` 编译 `.qm`
3. 将 `.qm` 嵌入 `:/i18n/` 资源路径

### 6.2 Qt 5 方式

```cmake
find_package(Qt5 REQUIRED COMPONENTS Core Widgets LinguistTools)

set(TS_FILES
    translations/app_zh_CN.ts
    translations/app_en_US.ts
)

# 处理翻译：生成 .qm 并嵌入资源
qt5_create_translation(QM_FILES ${CMAKE_SOURCE_DIR} ${TS_FILES})
# 或只编译（不更新 .ts）：
# qt5_add_translation(QM_FILES ${TS_FILES})

# 将 .qm 文件打包进资源
configure_file(translations.qrc.in translations.qrc)
qt5_add_resources(RESOURCES ${CMAKE_CURRENT_BINARY_DIR}/translations.qrc)

add_executable(myapp main.cpp mainwindow.cpp ${RESOURCES} ${QM_FILES})
```

`translations.qrc.in` 模板：

```xml
<!DOCTYPE RCC>
<RCC version="1.0">
    <qresource prefix="/i18n">
        <file>@CMAKE_CURRENT_BINARY_DIR@/app_zh_CN.qm</file>
        <file>@CMAKE_CURRENT_BINARY_DIR@/app_en_US.qm</file>
    </qresource>
</RCC>
```

### 6.3 .pro 文件方式（qmake）

```pro
TRANSLATIONS = \
    translations/app_zh_CN.ts \
    translations/app_en_US.ts \
    translations/app_ja_JP.ts

# 自动把 .qm 文件嵌入资源（Qt 5.15+）
CONFIG += lrelease embed_translations
```

## 7. Qt Linguist 工具详解

### 7.1 界面布局

```
┌─────────────────────────────────────────────────────────┐
│  上下文列表（左上）  │ 字符串列表（右上）                  │
│  - MainWindow       │ ● Save → 保存                      │
│  - SettingsDialog   │ ● Open → 打开                      │
├───────────────────────────────────────────────────────── │
│  翻译编辑区（中）：填写译文                                │
├─────────────────────────────────────────────────────────│
│  注释 / 源文位置 / 翻译建议（下）                         │
└─────────────────────────────────────────────────────────┘
```

### 7.2 翻译状态

| 图标颜色 | 状态 | 说明 |
|---|---|---|
| 绿色 ✓ | 已接受 | 译文完成并确认 |
| 黄色 ? | 需审阅 | 有译文但未确认（仅警告） |
| 红色 ✗ | 未完成 | 无译文 |
| 灰色 | 废弃 | obsolete，原文已从源码删除 |

**快捷键：**

| 快捷键 | 功能 |
|---|---|
| `Ctrl+Return` | 接受并跳转下一条 |
| `Ctrl+L` | 跳转第一条未完成 |
| `Alt+↓` | 下一条 |
| `Ctrl+F` | 查找 |

### 7.3 短语书（Phrase Book）

可创建 `.qph` 短语书，积累常用翻译对，Linguist 会自动提示：

```bash
# 创建短语书
File → New Phrase Book → common_zh_CN.qph
# 添加词条后，翻译其他文件时 Linguist 自动提示匹配的译文
```

## 8. QLocale：数字、日期、货币格式

国际化不只是翻译文字，数字/日期的格式也因地区而异。

### 8.1 数字格式

```cpp
QLocale zhCN(QLocale::Chinese, QLocale::China);
QLocale enUS(QLocale::English, QLocale::UnitedStates);
QLocale deDE(QLocale::German, QLocale::Germany);

double value = 1234567.89;
qDebug() << zhCN.toString(value, 'f', 2);  // "1,234,567.89"（中文用逗号分组）
qDebug() << deDE.toString(value, 'f', 2);  // "1.234.567,89"（德语点分组，逗号小数）
qDebug() << enUS.toString(value, 'f', 2);  // "1,234,567.89"

// 解析（字符串 → 数字）
bool ok;
double d = deDE.toDouble("1.234,56", &ok);  // ok=true, d=1234.56
```

### 8.2 日期时间格式

```cpp
QLocale zhCN(QLocale::Chinese, QLocale::China);
QDate date(2024, 3, 15);
QTime time(14, 30, 0);
QDateTime dt(date, time);

qDebug() << zhCN.toString(date, QLocale::LongFormat);   // "2024年3月15日"
qDebug() << zhCN.toString(date, QLocale::ShortFormat);  // "2024/3/15"
qDebug() << zhCN.toString(time, QLocale::ShortFormat);  // "14:30"
qDebug() << zhCN.toString(dt,   QLocale::LongFormat);   // "2024年3月15日 14:30:00 中国标准时间"

// 自定义格式（跨语言一致，不受 locale 影响）
qDebug() << date.toString("yyyy-MM-dd");  // "2024-03-15"（ISO 8601）
```

### 8.3 货币格式

```cpp
// QLocale 没有原生货币格式方法，通常自行拼接
QLocale zhCN(QLocale::Chinese, QLocale::China);
double price = 99.9;
QString priceStr = QString("¥%1").arg(zhCN.toString(price, 'f', 2));
// "¥99.90"

// Qt 5.12+ QLocale::currencySymbol()
qDebug() << zhCN.currencySymbol();                          // "¥"（默认）
qDebug() << zhCN.currencySymbol(QLocale::CurrencyIsoCode);  // "CNY"
qDebug() << zhCN.currencySymbol(QLocale::CurrencyDisplayName); // "人民币"
qDebug() << zhCN.toCurrencyString(1234.56);                 // "¥1,234.56"
```

### 8.4 设置应用的全局 Locale

```cpp
// 强制用特定 locale（覆盖系统设置）
QLocale::setDefault(QLocale(QLocale::Chinese, QLocale::China));

// 之后 QLocale() 默认构造即为 zh_CN
QLocale loc;
qDebug() << loc.toString(3.14);  // "3.14"（中文 locale 用点作小数点）
```

## 9. 文字方向（RTL 支持）

阿拉伯语、希伯来语等从右到左（RTL）的语言，Qt 原生支持布局镜像。

```cpp
// 设置应用为 RTL 布局
qApp->setLayoutDirection(Qt::RightToLeft);

// 根据语言自动设置
if (QLocale::system().textDirection() == Qt::RightToLeft) {
    qApp->setLayoutDirection(Qt::RightToLeft);
}
```

**Qt 在 RTL 模式下自动处理：**
- 所有 `QLayout` 中的控件顺序镜像
- `QAbstractItemView` 列顺序镜像
- 文本对齐自动翻转
- 图标（如箭头）可通过 `QIcon::setIsMask()` 自动镜像

```cpp
// 镜像图标（RTL 下左右箭头需对调）
QIcon backIcon = QIcon::fromTheme("go-previous");
// Qt 在 RTL 模式下会自动翻转，无需手动处理
```

## 10. 翻译中的 %1 占位符与动态字符串

### 10.1 使用 arg() 插入动态内容

```cpp
// ❌ 错误：用 + 拼接，翻译无法调整词序
QString msg = tr("File: ") + filename + tr(" saved.");

// ✅ 正确：用 %1 占位符，翻译可自由调整词序
QString msg = tr("File \"%1\" saved successfully.").arg(filename);
// 日文译文可能是："「%1」を保存しました。"（词序不同）

// 多个占位符（%1 %2 %3 ...）
QString msg = tr("Copied %1 files to \"%2\".").arg(count).arg(destPath);
// 翻译可以写："%2 に %1 個のファイルをコピーしました。"（词序可调换）
```

### 10.2 富文本中的翻译

```cpp
// HTML 标签不应被翻译，但文字内容要翻译
label->setText(QString("<b>%1</b>").arg(tr("Warning")));
// ❌ 不要：tr("<b>Warning</b>")  —— 让翻译人员处理 HTML 容易出错

// 更安全的写法
label->setText(tr("<b>Warning:</b> %1").arg(warningMsg));
// .ts 中的原文为 "<b>Warning:</b> %1"，翻译人员需保留标签
```

## 11. UI 文件（.ui）的翻译

Qt Designer 的 `.ui` 文件中的文字属性（如 `text`、`windowTitle`、`toolTip`）会被 `lupdate` 自动提取，无需额外处理。

```xml
<!-- mainwindow.ui 片段 -->
<widget class="QPushButton" name="saveButton">
    <property name="text">
        <string>Save</string>          <!-- 被 lupdate 提取 -->
    </property>
    <property name="toolTip">
        <string>Save the current file</string>
    </property>
</widget>
```

**不可翻译字符串：**有些属性（如对象名）不需要翻译，在 Designer 中可取消勾选 "Translatable"：

```xml
<property name="text">
    <string notr="true">debugLabel</string>  <!-- notr="true" 不提取 -->
</property>
```

## 12. 实战：完整多语言项目示例

### 项目结构

```
myapp/
├── CMakeLists.txt
├── main.cpp
├── mainwindow.h / .cpp
├── settingsdialog.h / .cpp
├── languagemanager.h / .cpp
└── translations/
    ├── app_en_US.ts
    ├── app_zh_CN.ts
    └── app_ja_JP.ts
```

### main.cpp

```cpp
#include <QApplication>
#include <QTranslator>
#include <QLocale>
#include <QLibraryInfo>
#include "mainwindow.h"

int main(int argc, char *argv[]) {
    QApplication app(argc, argv);
    app.setApplicationName("MyApp");
    app.setOrganizationName("MyCompany");

    // 1. 加载 Qt 自身翻译（标准对话框、按钮等）
    QTranslator qtTrans;
    QString locale = QLocale::system().name();  // "zh_CN"
    qtTrans.load(QLocale::system(), "qt", "_",
                 QLibraryInfo::location(QLibraryInfo::TranslationsPath));
    app.installTranslator(&qtTrans);

    // 2. 加载应用翻译（带回退：zh_CN → zh → 原文）
    QTranslator appTrans;
    appTrans.load(QLocale::system(), "app", "_", ":/translations/", ".qm");
    app.installTranslator(&appTrans);

    MainWindow w;
    w.show();
    return app.exec();
}
```

### languagemanager.cpp（运行时切换）

```cpp
#include "languagemanager.h"
#include <QApplication>
#include <QLibraryInfo>
#include <QSettings>
#include <QEvent>

LanguageManager *LanguageManager::instance() {
    static LanguageManager mgr;
    return &mgr;
}

QString LanguageManager::currentLocale() const {
    return QSettings().value("language", QLocale::system().name()).toString();
}

void LanguageManager::switchLanguage(const QString &locale) {
    if (locale == m_currentLocale) return;
    m_currentLocale = locale;

    qApp->removeTranslator(&m_appTrans);
    qApp->removeTranslator(&m_qtTrans);

    QLocale qlocale(locale);
    QLocale::setDefault(qlocale);

    if (m_qtTrans.load(qlocale, "qt", "_",
                       QLibraryInfo::location(QLibraryInfo::TranslationsPath))) {
        qApp->installTranslator(&m_qtTrans);
    }
    if (m_appTrans.load(qlocale, "app", "_", ":/translations/", ".qm")) {
        qApp->installTranslator(&m_appTrans);
    }

    // 持久化用户选择
    QSettings().setValue("language", locale);

    // Qt 会自动向所有 QWidget 发送 LanguageChange 事件
}
```

### CMakeLists.txt（Qt 6）

```cmake
cmake_minimum_required(VERSION 3.19)
project(myapp)

find_package(Qt6 REQUIRED COMPONENTS Core Widgets LinguistTools)

qt_add_executable(myapp
    main.cpp

    settingsdialog.cpp
    languagemanager.cpp
)
target_link_libraries(myapp PRIVATE Qt6::Core Qt6::Widgets)

qt_add_translations(myapp

)
```

## 13. 常见问题与最佳实践

### 13.1 常见问题

**问题 1：tr() 不起作用（返回原文）**

```cpp
// ❌ 原因：类没有 Q_OBJECT 宏，tr() 找不到正确上下文
class MyWidget : public QWidget {
    // 缺少 Q_OBJECT
};

// ✅ 修复
class MyWidget : public QWidget {
    Q_OBJECT
    // ...
};
```

**问题 2：字符串在 lupdate 提取后不翻译**

```cpp
// ❌ 问题：tr() 的参数不能是变量，lupdate 是静态分析工具
void show(const char* key) {
    label->setText(tr(key));  // lupdate 无法提取变量内容
}

// ✅ 修复：用 QT_TR_NOOP 在定义处标记，tr() 在使用处翻译
static const char * const keys[] = {
    QT_TR_NOOP("Apple"),
    QT_TR_NOOP("Banana"),
};
label->setText(tr(keys[index]));
```

**问题 3：数字字符串被错误地本地化**

```cpp
// ❌ 问题：阿拉伯语 locale 下，QString::number() 可能输出阿拉伯数字
QString id = QString::number(userId);  // 可能是 "١٢٣" 而非 "123"

// ✅ 修复：用 QLocale::c() 强制用 C locale（ASCII 数字）
QString id = QLocale::c().toString(userId);
// 或
QString id = QString::number(userId);  // 配合 Qt::AA_DisableWindowContextHelpButton 等
```

**问题 4：切换语言后界面没有更新**

```cpp
// ❌ 构造函数中的 tr() 在切换语言后不会重新执行
MyWidget::MyWidget() {
    label->setText(tr("Hello"));  // 初始化时设置，之后不更新
}

// ✅ 修复：重写 changeEvent，在 LanguageChange 事件时更新
void MyWidget::changeEvent(QEvent *e) {
    if (e->type() == QEvent::LanguageChange) {
        label->setText(tr("Hello"));  // 重新翻译
    }
    QWidget::changeEvent(e);
}
```

### 13.2 最佳实践

| 实践 | 说明 |
|---|---|
| 源语言用英文 | 英文最通用，翻译工具支持最好 |
| 避免在 tr() 中拼接 | 始终用 `%1` 占位符，让译者可调整词序 |
| 为 UI 设计留白 | 中文字短，译为德文/芬兰文可能长 2-3 倍 |
| .ts 文件纳入版本控制 | 与源码一起提交，避免翻译丢失 |
| .qm 文件不提交 | 由 CI/CD 构建时生成 |
| 及时 lupdate | 每次迭代后及时同步 .ts，避免翻译人员积压 |
| 测试 RTL 布局 | 即使不针对 RTL 语言，检查布局是否支持镜像 |
| 使用 `QLocale::c()` 处理内部数据 | 文件格式、协议中的数字不受 locale 影响 |
| Qt 自身字符串也要翻译 | 加载 `qt_xx.qm` 否则标准按钮仍是英文 |

## 14. 高级话题

### 14.1 自定义翻译函数（lupdate alias）

有时需要用自定义函数名包装 `tr()`，通过 `-tr-function-alias` 告知 `lupdate`：

```cpp
// 自定义包装（e.g. 添加日志）
inline QString T(const char *key) {
    return QCoreApplication::translate("Global", key);
}

// 使用
label->setText(T("Hello"));
```

```bash
# 告知 lupdate T() 等同于 translate()
lupdate -tr-function-alias translate+=T src/ -ts app_zh_CN.ts
```

### 14.2 翻译字体（中英文使用不同字体）

```cpp
void MyWidget::changeEvent(QEvent *e) {
    if (e->type() == QEvent::LanguageChange) {
        // 根据当前语言切换字体
        if (QLocale().language() == QLocale::Chinese) {
            qApp->setFont(QFont("Noto Sans SC", 10));
        } else {
            qApp->setFont(QFont("Segoe UI", 10));
        }
    }
    QWidget::changeEvent(e);
}
```

### 14.4 i18n 测试技巧：伪翻译（Pseudo-translation）

在开发阶段使用伪翻译快速验证 UI 是否可翻译、布局是否有足够空间：

```python
# 简单的 Python 脚本：把所有译文改为加方括号 + 重复字符
# [Saaave] 这样的格式，模拟长字符串，便于发现截断问题
import xml.etree.ElementTree as ET
import sys

tree = ET.parse(sys.argv[1])
for msg in tree.findall('.//message'):
    source = msg.find('source').text or ''
    trans = msg.find('translation')
    if trans is not None:
        # 伪翻译：加长 50% 并用括号包裹
        trans.text = f"[{source * 2}]"
        trans.attrib.pop('type', None)
tree.write(sys.argv[2], encoding='utf-8', xml_declaration=True)
```
