# 标准模型

Qt 提供了多种**开箱即用**的标准模型类，覆盖了从简单字符串列表到文件系统、数据库查询等常见场景。掌握这些标准模型可以大幅减少重复开发，只有在标准模型无法满足需求时才需要自定义模型。

---

## 1. 标准模型一览

| 模型类 | 数据结构 | 数据来源 | 可编辑 | 典型用途 |
|--------|----------|----------|--------|----------|
| `QStringListModel` | 列表 | 内存 `QStringList` | ✅ | 简单字符串列表 |
| `QStandardItemModel` | 列表/表格/树 | 内存 `QStandardItem` | ✅ | 通用结构化数据 |
| `QFileSystemModel` | 树 | 文件系统 | 部分 | 文件浏览器 |
| `QDirModel` | 树 | 文件系统 | 部分 | 已废弃，用 QFileSystemModel |
| `QSqlTableModel` | 表格 | 数据库表 | ✅ | 数据库 CRUD |
| `QSqlQueryModel` | 表格 | SQL 查询结果 | ❌ | 只读数据库查询 |
| `QSqlRelationalTableModel` | 表格 | 多表关联 | ✅ | 带外键关系的数据库 |
| `QConcatenateTablesProxyModel` | 表格 | 多个模型 | ❌ | 合并多个模型 |

## 2. QStringListModel — 字符串列表模型

最简单的标准模型，用于管理一个 `QStringList`，天然适配 `QListView` 和 `QComboBox`。

### 2.1 基本用法

```cpp
#include <QStringListModel>
#include <QListView>

// 创建模型
QStringList languages = {"C++", "Python", "Java", "Go", "Rust"};
QStringListModel *model = new QStringListModel(languages, this);

// 绑定到视图
QListView *view = new QListView(this);
view->setModel(model);

// 默认可编辑（双击即可修改）
```

### 2.2 动态操作数据

```cpp
// 获取当前字符串列表
QStringList list = model->stringList();

// 整体替换
list << "JavaScript" << "TypeScript";
model->setStringList(list);  // 替换全部数据（触发 modelReset）

// 插入行
model->insertRows(2, 1);                                // 在第 2 行插入 1 行
model->setData(model->index(2), "Kotlin");               // 设置数据

// 删除行
model->removeRows(0, 1);                                 // 删除第 0 行起的 1 行

// 通过 index 修改
model->setData(model->index(3), "Swift", Qt::EditRole);
```

### 2.3 配合 QComboBox 使用

```cpp
QComboBox *combo = new QComboBox(this);
combo->setModel(model);
combo->setEditable(true);  // 允许用户输入新值

// 获取选中值
QString selected = combo->currentText();
```

### 2.4 配合 QCompleter 实现自动补全

```cpp
QLineEdit *lineEdit = new QLineEdit(this);
QCompleter *completer = new QCompleter(model, this);
completer->setCaseSensitivity(Qt::CaseInsensitive);
lineEdit->setCompleter(completer);
// 输入 "py" 会自动弹出 "Python" 提示
```

### 2.5 局限性

- 只有一列，只支持 `Qt::DisplayRole` 和 `Qt::EditRole`
- 不支持图标、字体、颜色等装饰角色
- 不支持层级结构

> **适用场景：** 纯文本列表、标签列表、下拉框选项、自动补全数据源。

## 3. QStandardItemModel — 通用标准模型

最灵活的标准模型，支持列表、表格、树形三种结构，每个数据项是一个 `QStandardItem` 对象，可携带多种角色数据。

### 3.1 核心类关系

```
QStandardItemModel
├── 管理 QStandardItem 的集合
├── 支持行列索引与树形层级
└── 内部维护 invisibleRootItem（隐形根节点）

QStandardItem
├── 存储多种角色数据（text, icon, font, color, checkState...）
├── 可设置 flags（可选、可拖拽、可编辑等）
├── 可拥有子项（appendRow / insertRow）→ 构成树
└── 通过 setData(value, role) 设定任意角色
```

### 3.2 构建表格

```cpp
#include <QStandardItemModel>
#include <QTableView>

// 创建 4 行 3 列的空模型
QStandardItemModel *model = new QStandardItemModel(4, 3, this);

// 设置表头
model->setHorizontalHeaderLabels({"姓名", "年龄", "城市"});

// 逐项填充（方式一：通过 setItem）
model->setItem(0, 0, new QStandardItem("张三"));
model->setItem(0, 1, new QStandardItem("25"));
model->setItem(0, 2, new QStandardItem("北京"));

// 填充（方式二：通过 setData + index）
model->setData(model->index(1, 0), "李四");
model->setData(model->index(1, 1), "30");
model->setData(model->index(1, 2), "上海");

// 绑定视图
QTableView *view = new QTableView(this);
view->setModel(model);
view->resizeColumnsToContents();
```

### 3.3 构建树形结构

```cpp
QStandardItemModel *model = new QStandardItemModel(this);
model->setHorizontalHeaderLabels({"项目", "大小", "修改时间"});

// 创建顶层节点
QStandardItem *srcFolder = new QStandardItem(QIcon(":/icons/folder.png"), "src");
QStandardItem *srcSize   = new QStandardItem("—");
QStandardItem *srcTime   = new QStandardItem("2026-03-27");

// 创建子节点
QStandardItem *mainCpp  = new QStandardItem(QIcon(":/icons/cpp.png"), "main.cpp");
QStandardItem *mainSize = new QStandardItem("2.4 KB");
QStandardItem *mainTime = new QStandardItem("2026-03-27");

QStandardItem *utilCpp  = new QStandardItem(QIcon(":/icons/cpp.png"), "util.cpp");
QStandardItem *utilSize = new QStandardItem("1.1 KB");
QStandardItem *utilTime = new QStandardItem("2026-03-26");

// 挂载子行（每行多列，用 QList<QStandardItem*>）
srcFolder->appendRow({mainCpp, mainSize, mainTime});
srcFolder->appendRow({utilCpp, utilSize, utilTime});

// 挂载到模型（顶层也是多列）
model->appendRow({srcFolder, srcSize, srcTime});

// 绑定 QTreeView
QTreeView *tree = new QTreeView(this);
tree->setModel(model);
tree->expandAll();
```

### 3.4 QStandardItem 的丰富角色

```cpp
QStandardItem *item = new QStandardItem();

// 文本与图标
item->setText("重要文件");
item->setIcon(QIcon(":/icons/important.png"));

// 工具提示
item->setToolTip("这是一个重要的配置文件，请勿删除");

// 字体
QFont font;
font.setBold(true);
font.setPointSize(12);
item->setFont(font);

// 前景色 / 背景色
item->setForeground(QBrush(Qt::darkRed));
item->setBackground(QBrush(QColor(255, 255, 200)));  // 浅黄色背景

// 文本对齐
item->setTextAlignment(Qt::AlignCenter);

// 复选框
item->setCheckable(true);
item->setCheckState(Qt::Checked);

// 自定义数据（不显示，可存任意 QVariant）
item->setData(QVariant::fromValue(myObject), Qt::UserRole);
item->setData("/path/to/file", Qt::UserRole + 1);

// 编辑控制
item->setEditable(false);       // 禁止编辑
item->setSelectable(true);      // 允许选中
item->setDragEnabled(true);     // 允许拖拽
item->setDropEnabled(false);    // 禁止放置
```

### 3.5 动态增删行列

```cpp
// --- 行操作 ---

// 追加一行（表格模式）
QList<QStandardItem *> row;
row << new QStandardItem("王五")
    << new QStandardItem("28")
    << new QStandardItem("深圳");
model->appendRow(row);

// 在第 1 行前插入
model->insertRow(1, {new QStandardItem("赵六"),
                     new QStandardItem("35"),
                     new QStandardItem("广州")});

// 删除第 2 行
model->removeRow(2);

// --- 列操作 ---

// 追加一列
model->appendColumn({new QStandardItem("A"),
                     new QStandardItem("B"),
                     new QStandardItem("C")});

// 删除最后一列
model->removeColumn(model->columnCount() - 1);
```

### 3.6 查找与遍历

```cpp
// 查找包含 "张三" 的项（全模型搜索）
QList<QStandardItem *> results = model->findItems("张三",
    Qt::MatchContains | Qt::MatchRecursive, 0);  // 第 0 列递归查找

for (QStandardItem *item : results) {
    qDebug() << "找到:" << item->text()
             << "行:" << item->row()
             << "父:" << (item->parent() ? item->parent()->text() : "root");
}

// 遍历所有顶层项
for (int row = 0; row < model->rowCount(); ++row) {
    QStandardItem *item = model->item(row, 0);
    qDebug() << item->text();

    // 遍历子项（树形）
    for (int childRow = 0; childRow < item->rowCount(); ++childRow) {
        QStandardItem *child = item->child(childRow, 0);
        qDebug() << "  ├──" << child->text();
    }
}

// 通过 invisibleRootItem 访问（等价于顶层操作）
QStandardItem *root = model->invisibleRootItem();
qDebug() << "顶层行数:" << root->rowCount();
```

### 3.7 序列化（保存 / 加载）

`QStandardItemModel` 本身没有直接的序列化接口，但可以利用 `QDataStream` 配合遍历实现：

```cpp
// 保存到文件
void saveModel(QStandardItemModel *model, const QString &filePath) {
    QFile file(filePath);
    if (!file.open(QIODevice::WriteOnly)) return;
    QDataStream out(&file);
    out << model->rowCount() << model->columnCount();
    for (int r = 0; r < model->rowCount(); ++r) {
        for (int c = 0; c < model->columnCount(); ++c) {
            out << model->data(model->index(r, c)).toString();
        }
    }
}

// 从文件加载
void loadModel(QStandardItemModel *model, const QString &filePath) {
    QFile file(filePath);
    if (!file.open(QIODevice::ReadOnly)) return;
    QDataStream in(&file);
    int rows, cols;
    in >> rows >> cols;
    model->setRowCount(rows);
    model->setColumnCount(cols);
    for (int r = 0; r < rows; ++r) {
        for (int c = 0; c < cols; ++c) {
            QString text;
            in >> text;
            model->setItem(r, c, new QStandardItem(text));
        }
    }
}
```

### 3.8 性能注意

- `QStandardItem` 是一个**胖对象**，每个实例约占 200-300 字节
- 10 万行 × 5 列 = 50 万个 `QStandardItem`，内存约 100-150 MB
- **数据量超过 1 万行时**，建议使用自定义 `QAbstractTableModel` 代替

## 4. QFileSystemModel — 文件系统模型

异步加载本地文件系统目录结构，天然支持 `QTreeView`，是构建文件浏览器的首选。

### 4.1 基本用法

```cpp
#include <QFileSystemModel>
#include <QTreeView>

QFileSystemModel *model = new QFileSystemModel(this);
model->setRootPath(QDir::homePath());  // 设置监控根路径（异步加载）

QTreeView *tree = new QTreeView(this);
tree->setModel(model);
tree->setRootIndex(model->index(QDir::homePath()));  // 设置视图根节点

// 默认显示 4 列：名称、大小、类型、修改日期
tree->setColumnWidth(0, 250);
tree->setSortingEnabled(true);
```

### 4.2 过滤设置

```cpp
// 只显示特定文件类型
model->setNameFilters({"*.cpp", "*.h", "*.txt"});
model->setNameFilterDisables(false);  // false=隐藏不匹配项；true=灰显

// 显示/隐藏过滤
model->setFilter(QDir::AllDirs | QDir::Files | QDir::NoDotAndDotDot);

// 显示隐藏文件
model->setFilter(model->filter() | QDir::Hidden);
```

### 4.3 获取文件信息

```cpp
// 通过索引获取文件详情
connect(tree, &QTreeView::clicked, [model](const QModelIndex &index) {
    qDebug() << "文件名:" << model->fileName(index);
    qDebug() << "路径:"   << model->filePath(index);
    qDebug() << "大小:"   << model->size(index) << "bytes";
    qDebug() << "类型:"   << model->type(index);
    qDebug() << "是目录:" << model->isDir(index);

    // 获取 QFileInfo
    QFileInfo info = model->fileInfo(index);
    qDebug() << "后缀:" << info.suffix();
    qDebug() << "权限:" << info.permissions();
});
```

### 4.4 设置只读 / 可重命名

```cpp
// 只读模式（默认即只读）
model->setReadOnly(true);

// 允许重命名（删除和创建需手动实现）
model->setReadOnly(false);
```

### 4.5 监听文件系统变化

`QFileSystemModel` 内置 `QFileSystemWatcher`，会**自动监听**目录变化并更新视图：

```cpp
// 文件新增 / 删除 / 重命名 → 视图自动刷新
// 无需手动处理

// 但可以监听 directoryLoaded 信号（异步加载完成时触发）
connect(model, &QFileSystemModel::directoryLoaded,
    [](const QString &path) {
        qDebug() << "目录加载完成:" << path;
    });

// rootPathChanged 信号
connect(model, &QFileSystemModel::rootPathChanged,
    [](const QString &newPath) {
        qDebug() << "根路径变更为:" << newPath;
    });
```

### 4.6 自定义图标

```cpp
// 继承 QFileSystemModel 重写 data()
class MyFileSystemModel : public QFileSystemModel {
public:
    using QFileSystemModel::QFileSystemModel;

    QVariant data(const QModelIndex &index, int role) const override {
        if (role == Qt::DecorationRole && index.column() == 0) {
            QFileInfo info = fileInfo(index);
            if (info.isDir())
                return QIcon(":/icons/folder_custom.png");
            if (info.suffix() == "cpp")
                return QIcon(":/icons/cpp.png");
            if (info.suffix() == "h")
                return QIcon(":/icons/header.png");
        }
        return QFileSystemModel::data(index, role);
    }
};
```

### 4.7 配合 QListView 实现图标视图

```cpp
QFileSystemModel *model = new QFileSystemModel(this);
model->setRootPath("C:/Users/ws/Pictures");
model->setFilter(QDir::Files | QDir::NoDotAndDotDot);
model->setNameFilters({"*.png", "*.jpg", "*.bmp"});
model->setNameFilterDisables(false);

QListView *iconView = new QListView(this);
iconView->setModel(model);
iconView->setRootIndex(model->index("C:/Users/ws/Pictures"));
iconView->setViewMode(QListView::IconMode);     // 图标模式
iconView->setIconSize(QSize(64, 64));
iconView->setGridSize(QSize(100, 100));
iconView->setResizeMode(QListView::Adjust);     // 窗口缩放时重排
iconView->setWrapping(true);
```

### 4.8 注意事项

- `setRootPath()` 触发**异步加载**，调用后不能立即访问子节点
- 列固定为 4 列（Name、Size、Type、Date Modified），不可自定义
- 不支持远程/虚拟文件系统，仅限本地文件

## 5. QSqlQueryModel — SQL 查询只读模型

将 SQL 查询结果映射为只读表格模型，适合纯展示场景。

### 5.1 前置：建立数据库连接

```cpp
#include <QSqlDatabase>
#include <QSqlQuery>
#include <QSqlError>

// 创建 SQLite 连接
QSqlDatabase db = QSqlDatabase::addDatabase("QSQLITE");
db.setDatabaseName("contacts.db");
if (!db.open()) {
    qCritical() << "数据库打开失败:" << db.lastError().text();
    return;
}

// 建表并插入示例数据
QSqlQuery query;
query.exec("CREATE TABLE IF NOT EXISTS contacts ("
           "id INTEGER PRIMARY KEY AUTOINCREMENT,"
           "name TEXT NOT NULL,"
           "phone TEXT,"
           "email TEXT,"
           "city TEXT)");
query.exec("INSERT INTO contacts (name, phone, email, city) "
           "VALUES ('张三', '138-0001', 'zs@test.com', '北京')");
query.exec("INSERT INTO contacts (name, phone, email, city) "
           "VALUES ('李四', '139-0002', 'ls@test.com', '上海')");
```

### 5.2 基本用法

```cpp
#include <QSqlQueryModel>
#include <QTableView>

QSqlQueryModel *model = new QSqlQueryModel(this);
model->setQuery("SELECT name, phone, email, city FROM contacts");

// 检查错误
if (model->lastError().isValid()) {
    qCritical() << "查询错误:" << model->lastError().text();
}

// 设置表头（默认使用列名）
model->setHeaderData(0, Qt::Horizontal, tr("姓名"));
model->setHeaderData(1, Qt::Horizontal, tr("电话"));
model->setHeaderData(2, Qt::Horizontal, tr("邮箱"));
model->setHeaderData(3, Qt::Horizontal, tr("城市"));

// 绑定视图
QTableView *view = new QTableView(this);
view->setModel(model);
view->resizeColumnsToContents();
```

### 5.3 刷新数据

```cpp
// 重新执行查询以刷新
model->setQuery("SELECT name, phone, email, city FROM contacts");

// 带条件查询
void searchByCity(QSqlQueryModel *model, const QString &city) {
    QSqlQuery query;
    query.prepare("SELECT name, phone, email, city FROM contacts WHERE city = ?");
    query.addBindValue(city);
    query.exec();
    model->setQuery(std::move(query));
}
```

### 5.4 使其可编辑（继承方式）

```cpp
class EditableSqlModel : public QSqlQueryModel {
    Q_OBJECT
public:
    Qt::ItemFlags flags(const QModelIndex &index) const override {
        Qt::ItemFlags f = QSqlQueryModel::flags(index);
        if (index.column() == 0 || index.column() == 3)  // 姓名和城市可编辑
            f |= Qt::ItemIsEditable;
        return f;
    }

    bool setData(const QModelIndex &index, const QVariant &value,
                 int role = Qt::EditRole) override {
        if (role != Qt::EditRole) return false;

        // 获取 id（假设第一列的隐藏 id）
        int id = QSqlQueryModel::data(this->index(index.row(), 0)).toInt();

        QSqlQuery query;
        QString columnName;
        switch (index.column()) {
        case 0: columnName = "name"; break;
        case 3: columnName = "city"; break;
        default: return false;
        }
        query.prepare(QString("UPDATE contacts SET %1 = ? WHERE id = ?").arg(columnName));
        query.addBindValue(value);
        query.addBindValue(id);

        if (query.exec()) {
            refresh();  // 重新查询以刷新
            return true;
        }
        return false;
    }

    void refresh() {
        setQuery("SELECT name, phone, email, city FROM contacts");
    }
};
```

## 6. QSqlTableModel — SQL 表格可编辑模型

直接映射一张数据库表，支持完整的 CRUD 操作。

### 6.1 基本使用

```cpp
#include <QSqlTableModel>

QSqlTableModel *model = new QSqlTableModel(this);
model->setTable("contacts");         // 指定表名
model->setEditStrategy(QSqlTableModel::OnFieldChange);  // 编辑策略
model->select();                     // 执行 SELECT 查询

// 设置表头
model->setHeaderData(0, Qt::Horizontal, tr("ID"));
model->setHeaderData(1, Qt::Horizontal, tr("姓名"));
model->setHeaderData(2, Qt::Horizontal, tr("电话"));

// 绑定视图（自动可编辑）
QTableView *view = new QTableView(this);
view->setModel(model);
view->hideColumn(0);  // 隐藏 ID 列
view->setSelectionBehavior(QAbstractItemView::SelectRows);
```

### 6.2 三种编辑策略

| 策略枚举 | 行为 | 适用场景 |
|----------|------|----------|
| `OnFieldChange` | 每修改一个字段立即写入数据库 | 实时保存，字段级即时响应 |
| `OnRowChange` | 切换到其他行时写入 | 行级编辑，兼顾性能与即时性 |
| `OnManualSubmit` | 手动调用 `submitAll()` 才写入 | 批量编辑，支持撤销全部修改 |

```cpp
// 手动提交策略
model->setEditStrategy(QSqlTableModel::OnManualSubmit);

// 保存按钮
connect(saveBtn, &QPushButton::clicked, [model]() {
    if (!model->submitAll()) {
        qWarning() << "提交失败:" << model->lastError().text();
        model->revertAll();  // 回滚所有未提交的修改
    }
});

// 撤销按钮
connect(revertBtn, &QPushButton::clicked, [model]() {
    model->revertAll();
});
```

### 6.3 增删改查

```cpp
// --- 查 ---
model->setFilter("city = '北京' AND age > 20");  // WHERE 条件
model->setSort(1, Qt::AscendingOrder);            // ORDER BY 第 1 列 ASC
model->select();                                   // 执行查询

// --- 增 ---
int row = model->rowCount();
model->insertRow(row);
model->setData(model->index(row, 1), "王五");
model->setData(model->index(row, 2), "137-0003");
model->setData(model->index(row, 3), "ww@test.com");
model->setData(model->index(row, 4), "深圳");
model->submitAll();  // OnManualSubmit 策略下需要手动提交

// --- 改 ---
QModelIndex idx = model->index(0, 1);   // 第 0 行第 1 列
model->setData(idx, "张三丰");
model->submitAll();

// --- 删 ---
model->removeRow(2);     // 删除第 2 行
model->submitAll();

// 删除选中行（配合视图）
QModelIndexList selected = view->selectionModel()->selectedRows();
for (int i = selected.size() - 1; i >= 0; --i) {  // 倒序删除避免索引偏移
    model->removeRow(selected[i].row());
}
model->submitAll();
```

### 6.4 与 QDataWidgetMapper 配合做表单

`QDataWidgetMapper` 将模型的一行数据映射到多个表单控件：

```cpp
#include <QDataWidgetMapper>

QLineEdit *nameEdit  = new QLineEdit;
QLineEdit *phoneEdit = new QLineEdit;
QLineEdit *emailEdit = new QLineEdit;
QLineEdit *cityEdit  = new QLineEdit;

QDataWidgetMapper *mapper = new QDataWidgetMapper(this);
mapper->setModel(model);
mapper->addMapping(nameEdit,  1);  // 第 1 列 → nameEdit
mapper->addMapping(phoneEdit, 2);  // 第 2 列 → phoneEdit
mapper->addMapping(emailEdit, 3);
mapper->addMapping(cityEdit,  4);
mapper->toFirst();                 // 定位到第一行

// 上一条 / 下一条
connect(prevBtn, &QPushButton::clicked, mapper, &QDataWidgetMapper::toPrevious);
connect(nextBtn, &QPushButton::clicked, mapper, &QDataWidgetMapper::toNext);

// 提交修改
mapper->setSubmitPolicy(QDataWidgetMapper::ManualSubmit);
connect(submitBtn, &QPushButton::clicked, mapper, &QDataWidgetMapper::submit);

// 与视图联动（点击表格行 → 表单更新）
connect(view->selectionModel(), &QItemSelectionModel::currentRowChanged,
    [mapper](const QModelIndex &current) {
        mapper->setCurrentModelIndex(current);
    });
```

## 7. QSqlRelationalTableModel — 关联表模型

继承 `QSqlTableModel`，额外支持**外键关系**，将外键 ID 自动显示为关联表的可读文字。

### 7.1 场景示例

```sql
-- 部门表
CREATE TABLE departments (id INTEGER PRIMARY KEY, name TEXT);
INSERT INTO departments VALUES (1, '研发部'), (2, '市场部'), (3, '人事部');

-- 员工表（dept_id 是外键）
CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    name TEXT,
    dept_id INTEGER REFERENCES departments(id),
    salary REAL
);
INSERT INTO employees VALUES (1, '张三', 1, 15000);
INSERT INTO employees VALUES (2, '李四', 2, 12000);
```

### 7.2 使用方式

```cpp
#include <QSqlRelationalTableModel>
#include <QSqlRelationalDelegate>

QSqlRelationalTableModel *model = new QSqlRelationalTableModel(this);
model->setTable("employees");

// 第 2 列 (dept_id) 关联到 departments 表的 id 列，显示 name 列
model->setRelation(2, QSqlRelation("departments", "id", "name"));

model->setHeaderData(0, Qt::Horizontal, tr("ID"));
model->setHeaderData(1, Qt::Horizontal, tr("姓名"));
model->setHeaderData(2, Qt::Horizontal, tr("部门"));    // 显示部门名而非 ID
model->setHeaderData(3, Qt::Horizontal, tr("薪资"));. 
model->select();

QTableView *view = new QTableView(this);
view->setModel(model);

// 使用 QSqlRelationalDelegate → 部门列自动变成下拉框
view->setItemDelegate(new QSqlRelationalDelegate(view));
view->hideColumn(0);
```

效果：部门列直接显示"研发部"、"市场部"，编辑时弹出下拉框可选择部门。

## 8. QConcatenateTablesProxyModel — 合并模型

```cpp
#include <QConcatenateTablesProxyModel>

QStringListModel *fruits = new QStringListModel({"苹果", "香蕉", "樱桃"});
```

## 9. 实战示例：文件浏览器

综合运用 `QFileSystemModel` + `QTreeView` + `QTableView` 构建双栏文件浏览器：

```cpp
#include <QApplication>
#include <QSplitter>
#include <QTreeView>
#include <QTableView>
#include <QFileSystemModel>
#include <QHeaderView>

int main(int argc, char *argv[]) {
    QApplication app(argc, argv);

    // 目录树模型（只显示目录）
    QFileSystemModel *dirModel = new QFileSystemModel;
    dirModel->setRootPath(QDir::rootPath());
    dirModel->setFilter(QDir::Dirs | QDir::NoDotAndDotDot);

    // 文件列表模型（显示目录 + 文件）
    QFileSystemModel *fileModel = new QFileSystemModel;
    fileModel->setRootPath(QDir::homePath());
    fileModel->setFilter(QDir::AllEntries | QDir::NoDotAndDotDot);

    // 目录树视图
    QTreeView *dirTree = new QTreeView;
    dirTree->setModel(dirModel);
    dirTree->setRootIndex(dirModel->index(QDir::rootPath()));
    dirTree->hideColumn(1);  // 隐藏大小
    dirTree->hideColumn(2);  // 隐藏类型
    dirTree->hideColumn(3);  // 隐藏日期
    dirTree->setHeaderHidden(true);

    // 文件列表视图
    QTableView *fileTable = new QTableView;
    fileTable->setModel(fileModel);
    fileTable->setRootIndex(fileModel->index(QDir::homePath()));
    fileTable->setSortingEnabled(true);
    fileTable->setSelectionBehavior(QAbstractItemView::SelectRows);
    fileTable->horizontalHeader()->setStretchLastSection(true);
    fileTable->verticalHeader()->hide();

    // 点击目录树 → 更新文件列表
    QObject::connect(dirTree, &QTreeView::clicked,
        [fileModel, fileTable, dirModel](const QModelIndex &index) {
            QString path = dirModel->filePath(index);
            fileTable->setRootIndex(fileModel->setRootPath(path));
        });

    // 双栏布局
    QSplitter *splitter = new QSplitter;
    splitter->addWidget(dirTree);
    splitter->addWidget(fileTable);
    splitter->setStretchFactor(0, 1);
    splitter->setStretchFactor(1, 3);
    splitter->setWindowTitle("文件浏览器");
    splitter->resize(900, 600);
    splitter->show();

    return app.exec();
}
```

## 10. 标准模型选型指南

```
需要文件浏览？ ──→ QFileSystemModel
       │
       ↓ 否
数据在数据库？ ──→ 只读？ ──→ QSqlQueryModel
       │              │
       ↓ 否           ↓ 否
       │         有外键关系？ ──→ QSqlRelationalTableModel
       │              │
       ↓              ↓ 否
       │         QSqlTableModel
       │
纯字符串列表？ ──→ QStringListModel
       │
       ↓ 否
数据量 < 1万行？ ──→ QStandardItemModel
       │
       ↓ 否
自定义 QAbstractItemModel（性能最优）
```

**核心原则：**

1. **能用标准模型就不自定义** — 标准模型经过充分测试和优化
2. **只读优先** — 不需要编辑就用只读模型，更简单更安全
3. **大数据量用自定义** — `QStandardItem` 对象开销大，万行以上考虑自定义
4. **数据库场景用 SQL 模型** — 不要手动查询再填充 `QStandardItemModel`
5. **文件系统用 `QFileSystemModel`** — 内置异步加载和文件监控，不要手动遍历
