# 排序与过滤

在 Model/View 架构中，**排序与过滤**是最常见的需求之一。Qt 通过 `QSortFilterProxyModel` 提供了一种优雅的代理模型方案——不修改源数据，在源模型和视图之间插入一个"中间层"，对数据进行排序、过滤和索引映射。

```
┌──────────────┐    数据     ┌──────────────────────┐    排序/过滤后数据     ┌──────────┐
│  Source Model │ ────────→ │ QSortFilterProxyModel │ ──────────────────→ │   View   │
│  (源模型)      │ ←──────── │      (代理模型)        │ ←────────────────── │  (视图)   │
└──────────────┘   索引映射   └──────────────────────┘      用户交互         └──────────┘
```

---

## 1. QSortFilterProxyModel 基础

### 1.1 设计哲学

`QSortFilterProxyModel` 继承自 `QAbstractProxyModel`，核心设计原则：

| 原则 | 说明 |
|------|------|
| **不修改源数据** | 代理模型只维护索引映射表，源模型的数据保持不变 |
| **可链式叠加** | 多个代理模型可以串联，实现"先过滤再排序"或"多重过滤" |
| **透明传递** | 编辑操作自动透传到源模型，视图感知不到代理层的存在 |
| **自动同步** | 源模型数据变化时，代理模型自动重新排序/过滤 |

### 1.2 继承关系

```
QAbstractItemModel
└── QAbstractProxyModel        ← 代理模型基类
    └── QSortFilterProxyModel  ← 排序/过滤代理模型
        └── 你的自定义代理模型    ← 重写 filterAcceptsRow() / lessThan()
```

### 1.3 基本用法

```cpp
#include <QSortFilterProxyModel>
#include <QStandardItemModel>
#include <QTableView>

// 1. 创建源模型
auto *sourceModel = new QStandardItemModel(this);
sourceModel->setHorizontalHeaderLabels({"姓名", "年龄", "城市"});

// 填充数据
QList<QList<QStandardItem*>> rows = {
    {new QStandardItem("张三"), new QStandardItem("28"), new QStandardItem("北京")},
    {new QStandardItem("李四"), new QStandardItem("35"), new QStandardItem("上海")},
    {new QStandardItem("王五"), new QStandardItem("22"), new QStandardItem("广州")},
    {new QStandardItem("赵六"), new QStandardItem("31"), new QStandardItem("北京")},
};
for (auto &row : rows)
    sourceModel->appendRow(row);

// 2. 创建代理模型
auto *proxyModel = new QSortFilterProxyModel(this);
proxyModel->setSourceModel(sourceModel);

// 3. 视图绑定代理模型（不是源模型！）
auto *tableView = new QTableView(this);
tableView->setModel(proxyModel);

// 4. 启用点击表头排序
tableView->setSortingEnabled(true);
```

> **关键点：** 视图绑定的是 `proxyModel`，而非 `sourceModel`。这样视图展示的是排序/过滤后的数据。

### 1.4 索引映射

代理模型维护了源模型索引与代理模型索引之间的双向映射：

```cpp
// 代理索引 → 源索引
QModelIndex sourceIndex = proxyModel->mapToSource(proxyIndex);

// 源索引 → 代理索引
QModelIndex proxyIndex = proxyModel->mapFromSource(sourceIndex);

// 获取源模型
QAbstractItemModel *source = proxyModel->sourceModel();
```

**使用场景：** 当用户在视图中选中一行后，需要操作源模型的数据时：

```cpp
// 用户在视图中选中一行
QModelIndex proxyIndex = tableView->currentIndex();

// 转换到源模型索引，才能正确地修改源数据
QModelIndex srcIndex = proxyModel->mapToSource(proxyIndex);
QString name = sourceModel->data(srcIndex).toString();
```

## 2. 排序（Sorting）

### 2.1 内置排序

`QSortFilterProxyModel` 默认使用 `QString::localeAwareCompare()` 对 `Qt::DisplayRole` 数据进行字符串排序。

```cpp
// 按第 0 列升序排序
proxyModel->sort(0, Qt::AscendingOrder);

// 按第 1 列降序排序
proxyModel->sort(1, Qt::DescendingOrder);
```

**通过视图触发排序：**

```cpp
tableView->setSortingEnabled(true);  // 允许点击表头排序
// 用户点击表头后，视图自动调用 proxyModel->sort(column, order)
```

### 2.2 大小写敏感性

```cpp
// 排序时忽略大小写（默认为 Qt::CaseSensitive）
proxyModel->setSortCaseSensitivity(Qt::CaseInsensitive);
```

### 2.3 排序角色

默认排序基于 `Qt::DisplayRole`，可以切换为其他角色：

```cpp
// 使用 UserRole 的数据进行排序（数值型等）
proxyModel->setSortRole(Qt::UserRole);
```

**典型场景：** 显示为 "28 岁" 但排序应按数值 28 排序：

```cpp
// 源模型中同时设置两种角色
item->setData("28 岁", Qt::DisplayRole);   // 视图显示
item->setData(28, Qt::UserRole);            // 排序依据

// 代理模型按 UserRole 排序
proxyModel->setSortRole(Qt::UserRole);
```

### 2.4 自定义排序：重写 lessThan()

当默认的字符串比较无法满足需求时，重写 `lessThan()` 实现自定义排序逻辑：

```cpp
class CustomSortProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    using QSortFilterProxyModel::QSortFilterProxyModel;

protected:
    bool lessThan(const QModelIndex &left, const QModelIndex &right) const override
    {
        QVariant leftData = sourceModel()->data(left);
        QVariant rightData = sourceModel()->data(right);

        // 根据列选择不同的比较策略
        switch (left.column()) {
        case 0: // 姓名列：按拼音排序
            return leftData.toString().localeAwareCompare(rightData.toString()) < 0;

        case 1: // 年龄列：按数值排序
            return leftData.toInt() < rightData.toInt();

        case 2: // 日期列：按日期排序
            return leftData.toDate() < rightData.toDate();

        case 3: // IP 地址列：按数值段排序
            return compareIP(leftData.toString(), rightData.toString());

        default:
            return QSortFilterProxyModel::lessThan(left, right);
        }
    }

private:
    bool compareIP(const QString &left, const QString &right) const
    {
        auto toNum = [](const QString &ip) -> quint32 {
            quint32 result = 0;
            const auto parts = ip.split('.');
            for (const auto &part : parts)
                result = (result << 8) | part.toUInt();
            return result;
        };
        return toNum(left) < toNum(right);
    }
};
```

### 2.5 多列排序

`QSortFilterProxyModel` 原生只支持单列排序。实现多列排序需要在 `lessThan()` 中手动处理：

```cpp
class MultiColumnSortProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    // 设置次要排序列
    void setSecondarySortColumn(int column, Qt::SortOrder order = Qt::AscendingOrder)
    {
        m_secondaryColumn = column;
        m_secondaryOrder = order;
        invalidate();  // 触发重新排序
    }

protected:
    bool lessThan(const QModelIndex &left, const QModelIndex &right) const override
    {
        // 主排序列比较
        QVariant leftData = sourceModel()->data(left);
        QVariant rightData = sourceModel()->data(right);

        if (leftData != rightData)
            return leftData.toString() < rightData.toString();

        // 主排序列相等时，按次要排序列排序
        if (m_secondaryColumn >= 0) {
            QModelIndex leftSecondary = sourceModel()->index(left.row(), m_secondaryColumn, left.parent());
            QModelIndex rightSecondary = sourceModel()->index(right.row(), m_secondaryColumn, right.parent());
            QVariant leftSec = sourceModel()->data(leftSecondary);
            QVariant rightSec = sourceModel()->data(rightSecondary);

            bool result = leftSec.toString() < rightSec.toString();
            return m_secondaryOrder == Qt::AscendingOrder ? result : !result;
        }

        return false;  // 完全相等
    }

private:
    int m_secondaryColumn = -1;
    Qt::SortOrder m_secondaryOrder = Qt::AscendingOrder;
};
```

### 2.6 动态排序

```cpp
// 启用动态排序：源模型数据变化时自动重新排序
proxyModel->setDynamicSortFilter(true);   // 默认值为 true

// 禁用动态排序：适用于频繁批量更新源模型的场景
proxyModel->setDynamicSortFilter(false);

// 手动触发重新排序/过滤
proxyModel->invalidate();
```

> **性能提示：** 批量更新数据前，先设置 `setDynamicSortFilter(false)`，更新完成后再设回 `true` 并调用 `invalidate()`，避免每次数据变化都触发重排。

## 3. 过滤（Filtering）

### 3.1 基本过滤

过滤通过正则表达式或字符串匹配来决定哪些行应该显示。

```cpp
// 设置过滤的列（-1 表示搜索所有列）
proxyModel->setFilterKeyColumn(0);     // 只在第 0 列过滤
proxyModel->setFilterKeyColumn(-1);    // 在所有列中过滤

// 使用固定字符串过滤
proxyModel->setFilterFixedString("北京");

// 使用通配符过滤
proxyModel->setFilterWildcard("张*");

// 使用正则表达式过滤
proxyModel->setFilterRegularExpression("^[张李].*");

// 大小写不敏感
proxyModel->setFilterCaseSensitivity(Qt::CaseInsensitive);
```

### 3.2 过滤角色

默认过滤基于 `Qt::DisplayRole`，可切换为其他角色：

```cpp
proxyModel->setFilterRole(Qt::UserRole);  // 基于 UserRole 数据过滤
```

### 3.3 搜索框联动过滤

一个非常经典的应用场景——搜索框输入实时过滤列表：


### 3.4 自定义过滤：重写 filterAcceptsRow()

`filterAcceptsRow()` 是过滤的核心虚函数。返回 `true` 表示该行可见，`false` 表示被过滤掉。

```cpp
class AdvancedFilterProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    using QSortFilterProxyModel::QSortFilterProxyModel;

    void setMinAge(int age) { m_minAge = age; invalidateFilter(); }
    void setMaxAge(int age) { m_maxAge = age; invalidateFilter(); }
    void setCity(const QString &city) { m_city = city; invalidateFilter(); }

protected:
    bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
    {
        QAbstractItemModel *model = sourceModel();

        // 获取该行各列的数据
        QString name = model->index(sourceRow, 0, sourceParent).data().toString();
        int age = model->index(sourceRow, 1, sourceParent).data().toInt();
        QString city = model->index(sourceRow, 2, sourceParent).data().toString();

        // 多条件组合过滤（AND 逻辑）
        if (age < m_minAge || age > m_maxAge)
            return false;

        if (!m_city.isEmpty() && city != m_city)
            return false;

        // 还要执行基类的文本过滤（搜索框关键字）
        return QSortFilterProxyModel::filterAcceptsRow(sourceRow, sourceParent);
    }

private:
    int m_minAge = 0;
    int m_maxAge = 200;
    QString m_city;
};
```

**使用方式：**

```cpp
auto *proxy = new AdvancedFilterProxy(this);
proxy->setSourceModel(sourceModel);
proxy->setMinAge(25);
proxy->setMaxAge(40);
proxy->setCity("北京");

tableView->setModel(proxy);
```

### 3.5 自定义过滤：重写 filterAcceptsColumn()

除了行过滤，还可以过滤列（隐藏不需要的列）：

```cpp
class ColumnFilterProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    void setVisibleColumns(const QSet<int> &columns)
    {
        m_visibleColumns = columns;
        invalidateFilter();
    }

protected:
    bool filterAcceptsColumn(int sourceColumn, const QModelIndex &sourceParent) const override
    {
        Q_UNUSED(sourceParent);
        if (m_visibleColumns.isEmpty())
            return true;  // 未设置时显示所有列
        return m_visibleColumns.contains(sourceColumn);
    }

private:
    QSet<int> m_visibleColumns;
};

// 使用：只显示第 0 列和第 2 列
auto *proxy = new ColumnFilterProxy(this);
proxy->setSourceModel(sourceModel);
proxy->setVisibleColumns({0, 2});
```

### 3.6 树形模型的过滤

对树形模型过滤时需要特别注意：**默认情况下，如果父节点被过滤掉，其所有子节点也会被隐藏**。Qt 5.10+ 提供了递归过滤：

```cpp
// 启用递归过滤：如果子节点满足条件，父节点也会保留
proxyModel->setRecursiveFilteringEnabled(true);  // Qt 5.10+
```

**手动实现递归过滤（兼容 Qt 5.10 以下）：**

```cpp
class TreeFilterProxy : public QSortFilterProxyModel
{
protected:
    bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
    {
        // 当前节点匹配则直接显示
        if (matchesFilter(sourceRow, sourceParent))
            return true;

        // 递归检查子节点：任何子节点匹配，则父节点也显示
        QModelIndex index = sourceModel()->index(sourceRow, 0, sourceParent);
        int childCount = sourceModel()->rowCount(index);
        for (int i = 0; i < childCount; ++i) {
            if (filterAcceptsRow(i, index))
                return true;
        }

        return false;
    }

private:
    bool matchesFilter(int sourceRow, const QModelIndex &sourceParent) const
    {
        QModelIndex index = sourceModel()->index(sourceRow, filterKeyColumn(), sourceParent);
        QString data = sourceModel()->data(index).toString();
        return data.contains(filterRegularExpression());
    }
};
```

## 4. 代理模型链

多个代理模型可以链式串联，每层实现不同的职责：

```
┌──────────┐   ┌─────────────┐   ┌─────────────┐   ┌──────────┐
│  Source   │→  │ FilterProxy │→  │  SortProxy  │→  │   View   │
│  Model    │   │ (过滤)       │   │ (排序)       │   │          │
└──────────┘   └─────────────┘   └─────────────┘   └──────────┘
```

```cpp
// 源模型
auto *sourceModel = new QStandardItemModel(this);

// 第一层：过滤
auto *filterProxy = new QSortFilterProxyModel(this);
filterProxy->setSourceModel(sourceModel);
filterProxy->setFilterKeyColumn(2);
filterProxy->setFilterFixedString("北京");

// 第二层：排序（基于已过滤的数据）
auto *sortProxy = new QSortFilterProxyModel(this);
sortProxy->setSourceModel(filterProxy);   // 注意：源是 filterProxy

// 视图绑定最终的代理
tableView->setModel(sortProxy);

// 索引多层映射（代理索引 → 源模型索引）
QModelIndex viewIndex = tableView->currentIndex();
QModelIndex filterIndex = sortProxy->mapToSource(viewIndex);
QModelIndex sourceIndex = filterProxy->mapToSource(filterIndex);
```

> **注意：** 链式代理会增加索引映射开销。大多数场景下单个 `QSortFilterProxyModel` 同时处理排序和过滤就足够了，只有当过滤和排序逻辑需要独立管理时才使用链式代理。

## 5. 完整实战案例：员工管理系统

### 5.1 效果说明

一个支持多条件过滤 + 排序的员工表格：

- 下拉框过滤部门
- 滑块过滤薪资范围
- 点击表头排序

### 5.2 数据结构

```cpp

#include <QString>
#include <QDate>

struct Employee {
    QString name;
    QString department;
    int salary;
    QDate hireDate;
```

### 5.3 自定义代理模型

```cpp
// employeefilterproxy.h
#pragma once
#include <QSortFilterProxyModel>
#include <QDate>

class EmployeeFilterProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    explicit EmployeeFilterProxy(QObject *parent = nullptr)
        : QSortFilterProxyModel(parent) {}

    // --- 过滤条件设置 ---
    void setNameFilter(const QString &name)
    {
        m_nameFilter = name;
        invalidateFilter();
    }

    void setDepartmentFilter(const QString &dept)
    {
        m_deptFilter = dept;
        invalidateFilter();
    }

    void setSalaryRange(int min, int max)
    {
        m_minSalary = min;
        m_maxSalary = max;
        invalidateFilter();
    }

    void setHireDateRange(const QDate &from, const QDate &to)
    {
        m_hireDateFrom = from;
        m_hireDateTo = to;
        invalidateFilter();
    }

protected:
    // --- 自定义过滤逻辑 ---
    bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
    {
        auto *model = sourceModel();

        // 姓名过滤（模糊匹配）
        QString name = model->index(sourceRow, 0, sourceParent).data().toString();
        if (!m_nameFilter.isEmpty() && !name.contains(m_nameFilter, Qt::CaseInsensitive))
            return false;

        // 部门过滤（精确匹配）
        QString dept = model->index(sourceRow, 1, sourceParent).data().toString();
        if (!m_deptFilter.isEmpty() && dept != m_deptFilter)
            return false;

        // 薪资范围过滤
        int salary = model->index(sourceRow, 2, sourceParent).data().toInt();
        if (salary < m_minSalary || salary > m_maxSalary)
            return false;

        // 入职日期范围过滤
        QDate hireDate = model->index(sourceRow, 3, sourceParent).data().toDate();
        if (m_hireDateFrom.isValid() && hireDate < m_hireDateFrom)
            return false;
        if (m_hireDateTo.isValid() && hireDate > m_hireDateTo)
            return false;

        return true;
    }

    // --- 自定义排序逻辑 ---
    bool lessThan(const QModelIndex &left, const QModelIndex &right) const override
    {
        QVariant leftData = sourceModel()->data(left, Qt::UserRole);
        QVariant rightData = sourceModel()->data(right, Qt::UserRole);

        // 优先使用 UserRole 数据排序（包含原始类型数据）
        if (leftData.isValid() && rightData.isValid()) {
            // 根据类型选择比较方式
            if (leftData.type() == QVariant::Int)
                return leftData.toInt() < rightData.toInt();
            if (leftData.type() == QVariant::Date)
                return leftData.toDate() < rightData.toDate();
            if (leftData.type() == QVariant::Double)
                return leftData.toDouble() < rightData.toDouble();
        }

        // 回退到字符串比较
        return QSortFilterProxyModel::lessThan(left, right);
    }

private:
    QString m_nameFilter;
    QString m_deptFilter;
    int m_minSalary = 0;
    int m_maxSalary = INT_MAX;
    QDate m_hireDateFrom;
    QDate m_hireDateTo;
};
```

### 5.4 员工模型

```cpp
// employeemodel.h
#pragma once
#include <QAbstractTableModel>
#include <QVector>
#include "employee.h"

class EmployeeModel : public QAbstractTableModel
{
    Q_OBJECT
public:
    explicit EmployeeModel(QObject *parent = nullptr) : QAbstractTableModel(parent) {}

    void setEmployees(const QVector<Employee> &employees)
    {
        beginResetModel();
        m_employees = employees;
        endResetModel();
    }

    int rowCount(const QModelIndex &parent = QModelIndex()) const override
    {
        return parent.isValid() ? 0 : m_employees.size();
    }

    int columnCount(const QModelIndex &parent = QModelIndex()) const override
    {
        return parent.isValid() ? 0 : 4;
    }

    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override
    {
        if (!index.isValid() || index.row() >= m_employees.size())
            return {};

        const Employee &emp = m_employees[index.row()];

        if (role == Qt::DisplayRole) {
            switch (index.column()) {
            case 0: return emp.name;
            case 1: return emp.department;
            case 2: return QString("%1 元").arg(emp.salary);  // 格式化显示
            case 3: return emp.hireDate.toString("yyyy-MM-dd");
            }
        }

        // UserRole 存储原始数据（用于排序）
        if (role == Qt::UserRole) {
            switch (index.column()) {
            case 0: return emp.name;
            case 1: return emp.department;
            case 2: return emp.salary;           // int 原始值
            case 3: return emp.hireDate;         // QDate 原始值
            }
        }

        return {};
    }

    QVariant headerData(int section, Qt::Orientation orientation, int role) const override
    {
        if (role != Qt::DisplayRole || orientation != Qt::Horizontal)
            return {};
        switch (section) {
        case 0: return "姓名";
        case 1: return "部门";
        case 2: return "薪资";
        case 3: return "入职日期";
        }
        return {};
    }

private:
    QVector<Employee> m_employees;
};
```

## 6. 性能优化

### 6.1 invalidateFilter() vs invalidate()

| 方法 | 作用 | 性能 |
|------|------|------|
| `invalidateFilter()` | 只重新执行过滤 | 较快 |
| `invalidate()` | 重新排序 + 重新过滤 | 较慢 |

> **规则：** 只改了过滤条件用 `invalidateFilter()`，改了排序条件或两者都改了用 `invalidate()`。

### 6.2 批量更新优化

```cpp
// ❌ 错误：每次设置都触发一次过滤
proxy->setNameFilter("张");       // 触发 invalidateFilter
proxy->setDepartmentFilter("技术部"); // 触发 invalidateFilter
proxy->setSalaryRange(10000, 20000);  // 触发 invalidateFilter
// 共触发 3 次过滤，浪费

// ✅ 正确：暂停动态过滤，批量更新后一次性生效
proxy->setDynamicSortFilter(false);
proxy->setNameFilter("张");
proxy->setDepartmentFilter("技术部");
proxy->setSalaryRange(10000, 20000);
proxy->setDynamicSortFilter(true);
proxy->invalidate();  // 一次性重新排序 + 过滤
```

**或者设计一个 `beginFilter` / `endFilter` 机制：**

```cpp
class BatchFilterProxy : public QSortFilterProxyModel
{
    Q_OBJECT
public:
    void beginFilterUpdate() { m_updating = true; }

    void endFilterUpdate()
    {
        m_updating = false;
        invalidateFilter();
    }

    void setNameFilter(const QString &name)
    {
        m_name = name;
        if (!m_updating) invalidateFilter();
    }

    void setAgeRange(int min, int max)
    {
        m_minAge = min;
        m_maxAge = max;
        if (!m_updating) invalidateFilter();
    }

protected:
    bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
    {
        // ... 过滤逻辑
        return true;
    }

private:
    bool m_updating = false;
    QString m_name;
    int m_minAge = 0;
    int m_maxAge = 200;
};

// 使用
proxy->beginFilterUpdate();
proxy->setNameFilter("张");
proxy->setAgeRange(20, 30);
proxy->endFilterUpdate();  // 只触发一次过滤
```

### 6.3 延迟过滤（Debounce）

搜索框输入时，避免每个字符都触发过滤：

```cpp
// 使用 QTimer 实现防抖
auto *debounceTimer = new QTimer(this);
debounceTimer->setSingleShot(true);
debounceTimer->setInterval(300);  // 300ms 防抖

connect(searchEdit, &QLineEdit::textChanged, this, [=](const QString &text) {
    m_pendingFilter = text;
    debounceTimer->start();
});

connect(debounceTimer, &QTimer::timeout, this, [=]() {
    proxyModel->setFilterFixedString(m_pendingFilter);
});
```

### 6.4 大数据量下的注意事项

| 数据规模 | 建议 |
|----------|------|
| < 1,000 行 | 直接使用 `QSortFilterProxyModel`，无需优化 |
| 1,000 ~ 10,000 行 | 使用延迟过滤（debounce），禁用动态排序进行批量更新 |
| 10,000 ~ 100,000 行 | 考虑在自定义模型的数据层做预排序/预索引 |
| > 100,000 行 | 考虑虚拟化（只加载可见区域数据）或数据库端排序/过滤 |

## 7. 常见陷阱与最佳实践

### 7.1 索引混淆

```cpp
// ❌ 错误：用代理索引访问源模型
QModelIndex proxyIdx = tableView->currentIndex();
QString name = sourceModel->data(proxyIdx).toString();  // 行号对不上！

// ✅ 正确：先转换索引
QModelIndex proxyIdx = tableView->currentIndex();
QModelIndex sourceIdx = proxyModel->mapToSource(proxyIdx);
QString name = sourceModel->data(sourceIdx).toString();
```

### 7.2 过滤后选择丢失

过滤条件变化后，之前选中的行可能被过滤掉。需要在过滤前保存选择：

```cpp
// 保存当前选中项的源模型索引
QModelIndex proxyIdx = tableView->currentIndex();
QPersistentModelIndex sourceIdx;
if (proxyIdx.isValid())
    sourceIdx = proxyModel->mapToSource(proxyIdx);

// 执行过滤
proxyModel->setFilterFixedString("新条件");

// 恢复选择（如果该行仍然可见）
if (sourceIdx.isValid()) {
    QModelIndex newProxyIdx = proxyModel->mapFromSource(sourceIdx);
    if (newProxyIdx.isValid())
        tableView->setCurrentIndex(newProxyIdx);
}
```

### 7.3 排序列数据类型不一致

```cpp
// ❌ 年龄存储为字符串，排序不正确（"9" > "10" 按字符串比较）
item->setData("9", Qt::DisplayRole);

// ✅ 分离显示和排序数据
item->setData("9 岁", Qt::DisplayRole);      // 显示用
item->setData(9, Qt::UserRole);               // 排序用
proxyModel->setSortRole(Qt::UserRole);
```

### 7.4 filterAcceptsRow 中避免昂贵操作

```cpp
// ❌ 每次判断都做正则编译
bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
{
    QRegularExpression re(m_pattern);  // 每行都编译，巨慢！
    // ...
}

// ✅ 预编译正则，只在 pattern 变化时更新
void setPattern(const QString &pattern)
{
    m_regex = QRegularExpression(pattern, QRegularExpression::CaseInsensitiveOption);
    invalidateFilter();
}

bool filterAcceptsRow(int sourceRow, const QModelIndex &sourceParent) const override
{
    QString data = sourceModel()->index(sourceRow, 0, sourceParent).data().toString();
    return m_regex.match(data).hasMatch();  // 直接使用预编译对象
}
```

### 8.2 可重写的虚函数

| 虚函数 | 用途 |
|--------|------|
| `filterAcceptsRow(int row, QModelIndex parent)` | 判断某行是否通过过滤。返回 `true` 显示该行 |

## 9. 总结

```
                     QSortFilterProxyModel 知识图谱
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│  排序 (Sorting)                    过滤 (Filtering)             │
│  ├── sort(column, order)          ├── setFilterFixedString()   │
│  ├── setSortRole()                ├── setFilterWildcard()      │
│  ├── setSortCaseSensitivity()     ├── setFilterRegularExpression()│
│  ├── setSortLocaleAware()         ├── setFilterKeyColumn()     │
│  └── 重写 lessThan()              ├── setFilterRole()          │
│      ├── 多类型比较                ├── setFilterCaseSensitivity()│
│      └── 多列排序                  ├── setRecursiveFilteringEnabled()│
│                                   └── 重写 filterAcceptsRow()  │
│  性能优化                              └── 多条件组合           │
│  ├── invalidate vs invalidateFilter                            │
│  ├── 批量更新（暂停动态过滤）                                    │
│  ├── 延迟过滤（debounce）                                       │
│  └── 预编译正则                                                 │
│                                                                │
│  代理链：Source → FilterProxy → SortProxy → View               │
│  索引映射：mapToSource() / mapFromSource()                      │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**核心要点回顾：**

1. `QSortFilterProxyModel` 是 **代理模型**，不修改源数据，插在源模型和视图之间
2. 视图必须绑定 **代理模型** 而非源模型
3. 操作源数据时要用 `mapToSource()` 转换索引
4. 自定义排序重写 `lessThan()`，自定义过滤重写 `filterAcceptsRow()`
5. 性能敏感场景注意 **批量更新** 和 **延迟过滤**
6. 分离 `DisplayRole`（显示）和 `UserRole`（排序/过滤）是常见最佳实践
