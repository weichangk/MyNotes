# 自定义模型

当 Qt 提供的标准模型（`QStringListModel`、`QStandardItemModel` 等）无法满足需求时，就需要继承抽象基类来实现自定义模型。自定义模型可以直接对接任意数据源（内存容器、数据库、网络、文件等），避免数据拷贝，获得最佳性能和最大灵活性。

---

## 1. 何时需要自定义模型

| 场景 | 标准模型的问题 | 自定义模型的优势 |
|------|---------------|-----------------|
| 数据量巨大（百万行+） | `QStandardItemModel` 为每个单元格创建 `QStandardItem` 对象，内存开销大 | 直接从底层数据源读取，零拷贝 |
| 数据源是已有数据结构 | 需要把数据搬运到 `QStandardItem` 中，维护两份数据 | 直接包装已有数据结构 |
| 需要复杂的层级关系 | `QStandardItemModel` 的树结构操作较笨重 | 完全控制树节点的组织方式 |
| 数据来自外部（网络/数据库） | 需要先查询再填充到标准模型 | 按需加载（lazy loading） |
| 需要特殊的编辑/验证逻辑 | `QStandardItem` 只提供通用的 `setData` | 在 `setData()` 中实现业务校验 |

> **经验法则：** 原型阶段用 `QStandardItemModel` 快速验证交互，性能或灵活性不满足后再切换为自定义模型。

---

## 2. 抽象基类选择

Qt 提供三个抽象基类，选择取决于数据结构的维度：

```
QAbstractItemModel              ← 最通用（列表/表格/树 皆可）
├── QAbstractListModel          ← 一维列表（columnCount 固定为 1）
└── QAbstractTableModel         ← 二维表格（无层级，parent 固定为无效索引）
```

### 2.1 选择指南

| 数据形态 | 推荐基类 | 原因 |
|----------|---------|------|
| 一维列表 | `QAbstractListModel` | 已实现 `index()`、`parent()`、`columnCount()`，只需实现 `rowCount()` + `data()` |
| 二维表格 | `QAbstractTableModel` | 已实现 `index()`、`parent()`，只需实现 `rowCount()` + `columnCount()` + `data()` |
| 树形结构 | `QAbstractItemModel` | 需要自己实现 `index()`、`parent()`，完全控制层级关系 |

### 2.2 各基类已实现 vs 需实现的方法对比

```
                          QAbstractItemModel  QAbstractListModel  QAbstractTableModel
─────────────────────────────────────────────────────────────────────────────────────
rowCount()                   需实现 ✍          需实现 ✍            需实现 ✍
columnCount()                需实现 ✍          已实现 ✅ (=1)       需实现 ✍
data()                       需实现 ✍          需实现 ✍            需实现 ✍
index()                      需实现 ✍          已实现 ✅            已实现 ✅
parent()                     需实现 ✍          已实现 ✅ (=无效)     已实现 ✅ (=无效)
headerData()                 可选 📌           可选 📌             可选 📌
flags()                      可选 📌           可选 📌             可选 📌
setData()                    可选（编辑）📌     可选（编辑）📌       可选（编辑）📌
insertRows/removeRows()      可选（增删）📌     可选（增删）📌       可选（增删）📌
```

## 3. 只读列表模型

最简单的自定义模型——将一个 `QVector` 包装为只读列表。

### 3.1 完整示例：联系人列表

**数据结构：**

```cpp
// contact.h
struct Contact {
    QString name;
    QString phone;
    QString email;
};
```

**模型实现：**

```cpp
// contactlistmodel.h
#pragma once
#include <QAbstractListModel>
#include <QVector>
#include "contact.h"

class ContactListModel : public QAbstractListModel
{
    Q_OBJECT
public:
    // 自定义角色枚举（用于 QML 或多角色场景）
    enum ContactRoles {
        NameRole = Qt::UserRole + 1,
        PhoneRole,
        EmailRole
    };

    explicit ContactListModel(QObject *parent = nullptr);

    // ===== 只读模型必须实现 =====
    int rowCount(const QModelIndex &parent = QModelIndex()) const override;
    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override;

    // ===== 可选：表头 =====
    QVariant headerData(int section, Qt::Orientation orientation,
                        int role = Qt::DisplayRole) const override;

    // ===== 可选：角色名映射（QML 访问需要） =====
    QHash<int, QByteArray> roleNames() const override;

    // ===== 业务方法 =====
    void setContacts(const QVector<Contact> &contacts);
    void addContact(const Contact &contact);
    void removeContact(int row);

private:
    QVector<Contact> m_contacts;
};
```

```cpp
// contactlistmodel.cpp
#include "contactlistmodel.h"

ContactListModel::ContactListModel(QObject *parent)
    : QAbstractListModel(parent)
{
}

int ContactListModel::rowCount(const QModelIndex &parent) const
{
    // 列表模型：parent 有效时返回 0（列表没有子节点）
    if (parent.isValid())
        return 0;
    return m_contacts.size();
}

QVariant ContactListModel::data(const QModelIndex &index, int role) const
{
    if (!index.isValid() || index.row() >= m_contacts.size())
        return QVariant();

    const Contact &contact = m_contacts.at(index.row());

    switch (role) {
    case Qt::DisplayRole:
    case NameRole:
        return contact.name;
    case PhoneRole:
        return contact.phone;
    case EmailRole:
        return contact.email;
    case Qt::ToolTipRole:
        return QString("%1 (%2)").arg(contact.name, contact.email);
    default:
        return QVariant();
    }
}

QVariant ContactListModel::headerData(int section, Qt::Orientation orientation, int role) const
{
    if (role == Qt::DisplayRole && orientation == Qt::Horizontal && section == 0)
        return tr("联系人");
    return QVariant();
}

QHash<int, QByteArray> ContactListModel::roleNames() const
{
    QHash<int, QByteArray> roles;
    roles[NameRole]  = "name";
    roles[PhoneRole] = "phone";
    roles[EmailRole] = "email";
    return roles;
}

void ContactListModel::setContacts(const QVector<Contact> &contacts)
{
    beginResetModel();       // 通知视图：模型即将重置
    m_contacts = contacts;
    endResetModel();         // 通知视图：重置完成，请刷新
}

void ContactListModel::addContact(const Contact &contact)
{
    const int row = m_contacts.size();
    beginInsertRows(QModelIndex(), row, row);  // 通知视图：即将在 row 位置插入 1 行
    m_contacts.append(contact);
    endInsertRows();                            // 通知视图：插入完成
}

void ContactListModel::removeContact(int row)
{
    if (row < 0 || row >= m_contacts.size())
        return;
    beginRemoveRows(QModelIndex(), row, row);   // 通知视图：即将删除 row 行
    m_contacts.removeAt(row);
    endRemoveRows();                             // 通知视图：删除完成
}
```

**使用方式：**

```cpp
// 在 Widget 中使用
ContactListModel *model = new ContactListModel(this);
model->setContacts({
    {"张三", "13800138000", "zhangsan@example.com"},
    {"李四", "13900139000", "lisi@example.com"},
    {"王五", "13700137000", "wangwu@example.com"}
});

QListView *listView = new QListView(this);
listView->setModel(model);
```

## 4. 可编辑表格模型

表格模型比列表模型多了**列维度**和**编辑支持**，需要额外实现 `setData()`、`flags()` 等方法。

### 4.1 完整示例：员工信息表

```cpp
// employee.h
struct Employee {
    int     id;
    QString name;
    QString department;
    double  salary;
    bool    active;
};
```

```cpp
// employeetablemodel.h
#pragma once
#include <QAbstractTableModel>
#include <QVector>
#include "employee.h"

class EmployeeTableModel : public QAbstractTableModel
{
    Q_OBJECT
public:
    enum Column {
        ColId = 0,
        ColName,
        ColDepartment,
        ColSalary,
        ColActive,
        ColCount  // 列数哨兵
    };

    explicit EmployeeTableModel(QObject *parent = nullptr);

    // ===== 只读接口（必须实现） =====
    int rowCount(const QModelIndex &parent = QModelIndex()) const override;
    int columnCount(const QModelIndex &parent = QModelIndex()) const override;
    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override;
    QVariant headerData(int section, Qt::Orientation orientation,
                        int role = Qt::DisplayRole) const override;

    // ===== 编辑接口 =====
    bool setData(const QModelIndex &index, const QVariant &value,
                 int role = Qt::EditRole) override;
    Qt::ItemFlags flags(const QModelIndex &index) const override;

    // ===== 行增删接口 =====
    bool insertRows(int row, int count, const QModelIndex &parent = QModelIndex()) override;
    bool removeRows(int row, int count, const QModelIndex &parent = QModelIndex()) override;

    // ===== 业务方法 =====
    void setEmployees(const QVector<Employee> &employees);
    const QVector<Employee> &employees() const { return m_employees; }

private:
    QVector<Employee> m_employees;
};
```

```cpp
// employeetablemodel.cpp
#include "employeetablemodel.h"
#include <QBrush>
#include <QFont>

EmployeeTableModel::EmployeeTableModel(QObject *parent)
    : QAbstractTableModel(parent)
{
}

int EmployeeTableModel::rowCount(const QModelIndex &parent) const
{
    if (parent.isValid())  // 表格模型没有子节点
        return 0;
    return m_employees.size();
}

int EmployeeTableModel::columnCount(const QModelIndex &parent) const
{
    if (parent.isValid())
        return 0;
    return ColCount;
}

QVariant EmployeeTableModel::data(const QModelIndex &index, int role) const
{
    if (!index.isValid() || index.row() >= m_employees.size())
        return QVariant();

    const Employee &emp = m_employees.at(index.row());

    // ---------- 显示角色 ----------
    if (role == Qt::DisplayRole || role == Qt::EditRole) {
        switch (index.column()) {
        case ColId:         return emp.id;
        case ColName:       return emp.name;
        case ColDepartment: return emp.department;
        case ColSalary:
            // 编辑时返回原始数值，显示时格式化
            return (role == Qt::EditRole)
                       ? QVariant(emp.salary)
                       : QVariant(QString("¥%1").arg(emp.salary, 0, 'f', 2));
        case ColActive:
            if (role == Qt::DisplayRole)
                return emp.active ? tr("在职") : tr("离职");
            return emp.active;  // EditRole 返回 bool
        }
    }

    // ---------- 勾选角色（Active 列显示复选框） ----------
    if (role == Qt::CheckStateRole && index.column() == ColActive) {
        return emp.active ? Qt::Checked : Qt::Unchecked;
    }

    // ---------- 文本对齐 ----------
    if (role == Qt::TextAlignmentRole) {
        if (index.column() == ColSalary)
            return QVariant(Qt::AlignRight | Qt::AlignVCenter);
        return QVariant(Qt::AlignLeft | Qt::AlignVCenter);
    }

    // ---------- 前/背景色 ----------
    if (role == Qt::ForegroundRole) {
        if (index.column() == ColActive && !emp.active)
            return QBrush(Qt::red);   // 离职人员标红
    }

    if (role == Qt::BackgroundRole) {
        if (index.row() % 2 == 1)
            return QBrush(QColor(245, 245, 250));  // 隔行变色
    }

    return QVariant();
}

QVariant EmployeeTableModel::headerData(int section, Qt::Orientation orientation, int role) const
{
    if (role != Qt::DisplayRole)
        return QVariant();

    if (orientation == Qt::Horizontal) {
        switch (section) {
        case ColId:         return tr("工号");
        case ColName:       return tr("姓名");
        case ColDepartment: return tr("部门");
        case ColSalary:     return tr("薪资");
        case ColActive:     return tr("状态");
        }
    } else {
        // 垂直表头显示行号（从 1 开始）
        return section + 1;
    }
    return QVariant();
}

bool EmployeeTableModel::setData(const QModelIndex &index, const QVariant &value, int role)
{
    if (!index.isValid() || index.row() >= m_employees.size())
        return false;

    Employee &emp = m_employees[index.row()];

    if (role == Qt::EditRole) {
        switch (index.column()) {
        case ColId:
            emp.id = value.toInt();
            break;
        case ColName:
            if (value.toString().trimmed().isEmpty())
                return false;  // 姓名不能为空 —— 业务校验
            emp.name = value.toString().trimmed();
            break;
        case ColDepartment:
            emp.department = value.toString();
            break;
        case ColSalary:
            if (value.toDouble() < 0)
                return false;  // 薪资不能为负
            emp.salary = value.toDouble();
            break;
        case ColActive:
            emp.active = value.toBool();
            break;
        default:
            return false;
        }
    } else if (role == Qt::CheckStateRole && index.column() == ColActive) {
        emp.active = (value.toInt() == Qt::Checked);
    } else {
        return false;
    }

    // 关键：通知视图该单元格数据已变化
    emit dataChanged(index, index, {role});
    return true;
}

Qt::ItemFlags EmployeeTableModel::flags(const QModelIndex &index) const
{
    if (!index.isValid())
        return Qt::NoItemFlags;

    Qt::ItemFlags flags = Qt::ItemIsEnabled | Qt::ItemIsSelectable;

    // 工号列只读
    if (index.column() != ColId)
        flags |= Qt::ItemIsEditable;

    // Active 列支持复选框
    if (index.column() == ColActive)
        flags |= Qt::ItemIsUserCheckable;

    return flags;
}

bool EmployeeTableModel::insertRows(int row, int count, const QModelIndex &parent)
{
    if (parent.isValid())
        return false;

    beginInsertRows(parent, row, row + count - 1);
    for (int i = 0; i < count; ++i) {
        m_employees.insert(row, Employee{0, tr("新员工"), "", 0.0, true});
    }
    endInsertRows();
    return true;
}

bool EmployeeTableModel::removeRows(int row, int count, const QModelIndex &parent)
{
    if (parent.isValid())
        return false;
    if (row < 0 || row + count > m_employees.size())
        return false;

    beginRemoveRows(parent, row, row + count - 1);
    m_employees.remove(row, count);
    endRemoveRows();
    return true;
}

void EmployeeTableModel::setEmployees(const QVector<Employee> &employees)
{
    beginResetModel();
    m_employees = employees;
    endResetModel();
}
```

**使用方式：**

```cpp
EmployeeTableModel *model = new EmployeeTableModel(this);
model->setEmployees({
    {1001, "张三", "研发部", 15000.0, true},
    {1002, "李四", "市场部", 12000.0, true},
    {1003, "王五", "人事部",  9500.0, false}
});

QTableView *tableView = new QTableView(this);
tableView->setModel(model);
tableView->setSelectionBehavior(QAbstractItemView::SelectRows);
tableView->horizontalHeader()->setStretchLastSection(true);
tableView->resizeColumnsToContents();

// 添加一行
model->insertRows(model->rowCount(), 1);

// 删除选中行
QModelIndexList selected = tableView->selectionModel()->selectedRows();
if (!selected.isEmpty())
    model->removeRows(selected.first().row(), 1);
```

## 5. 树形模型

树形模型是最复杂的自定义模型，需要自己维护节点层级关系、实现 `index()` 和 `parent()`。核心思想是用**内部指针（`internalPointer()`）**将 `QModelIndex` 与自定义树节点关联。

### 5.1 树节点类设计

```cpp
// treeitem.h
#pragma once
#include <QVariant>
#include <QVector>

class TreeItem
{
public:
    explicit TreeItem(const QVector<QVariant> &data, TreeItem *parent = nullptr)
        : m_itemData(data), m_parentItem(parent)
    {
    }

    ~TreeItem()
    {
        qDeleteAll(m_childItems);  // 递归释放所有子节点
    }

    // ---------- 子节点管理 ----------
    void appendChild(TreeItem *child)
    {
        m_childItems.append(child);
    }

    void insertChild(int row, TreeItem *child)
    {
        m_childItems.insert(row, child);
    }

    void removeChild(int row)
    {
        delete m_childItems.takeAt(row);
    }

    TreeItem *child(int row) const
    {
        if (row < 0 || row >= m_childItems.size())
            return nullptr;
        return m_childItems.at(row);
    }

    int childCount() const { return m_childItems.size(); }

    // ---------- 数据访问 ----------
    int columnCount() const { return m_itemData.size(); }

    QVariant data(int column) const
    {
        if (column < 0 || column >= m_itemData.size())
            return QVariant();
        return m_itemData.at(column);
    }

    void setData(int column, const QVariant &value)
    {
        if (column >= 0 && column < m_itemData.size())
            m_itemData[column] = value;
    }

    // ---------- 层级关系 ----------
    TreeItem *parentItem() const { return m_parentItem; }

    int row() const
    {
        // 返回自己在父节点中的行号
        if (m_parentItem)
            return m_parentItem->m_childItems.indexOf(const_cast<TreeItem *>(this));
        return 0;
    }

private:
    QVector<TreeItem *>  m_childItems;   // 子节点列表
    QVector<QVariant>    m_itemData;     // 本节点各列数据
    TreeItem            *m_parentItem;   // 父节点指针
};
```

### 5.2 树形模型实现

```cpp
// treemodel.h
#pragma once
#include <QAbstractItemModel>
#include "treeitem.h"

class TreeModel : public QAbstractItemModel
{
    Q_OBJECT
public:
    explicit TreeModel(QObject *parent = nullptr);
    ~TreeModel() override;

    // ===== 必须实现的 5 个方法 =====
    QModelIndex index(int row, int column,
                      const QModelIndex &parent = QModelIndex()) const override;
    QModelIndex parent(const QModelIndex &index) const override;
    int rowCount(const QModelIndex &parent = QModelIndex()) const override;
    int columnCount(const QModelIndex &parent = QModelIndex()) const override;
    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override;

    // ===== 可选 =====
    QVariant headerData(int section, Qt::Orientation orientation,
                        int role = Qt::DisplayRole) const override;
    Qt::ItemFlags flags(const QModelIndex &index) const override;
    bool setData(const QModelIndex &index, const QVariant &value,
                 int role = Qt::EditRole) override;

    // ===== 业务方法 =====
    void setupModelData();  // 构建测试数据

private:
    TreeItem *getItem(const QModelIndex &index) const;
    TreeItem *m_rootItem;  // 虚拟根节点（不显示）
};
```

```cpp
// treemodel.cpp
#include "treemodel.h"

TreeModel::TreeModel(QObject *parent)
    : QAbstractItemModel(parent)
{
    // 虚拟根节点，定义列头
    m_rootItem = new TreeItem({"名称", "类型", "大小"});
    setupModelData();
}

TreeModel::~TreeModel()
{
    delete m_rootItem;  // 递归释放整棵树
}

// ========== index() — 创建模型索引 ==========
// 核心思想：将 TreeItem 指针嵌入 QModelIndex 的 internalPointer
QModelIndex TreeModel::index(int row, int column, const QModelIndex &parent) const
{
    if (!hasIndex(row, column, parent))
        return QModelIndex();

    TreeItem *parentItem = getItem(parent);
    TreeItem *childItem = parentItem->child(row);

    if (childItem)
        return createIndex(row, column, childItem);  // 关键：绑定指针
    return QModelIndex();
}

// ========== parent() — 返回父索引 ==========
QModelIndex TreeModel::parent(const QModelIndex &index) const
{
    if (!index.isValid())
        return QModelIndex();

    TreeItem *childItem = static_cast<TreeItem *>(index.internalPointer());
    TreeItem *parentItem = childItem->parentItem();

    if (parentItem == m_rootItem)
        return QModelIndex();  // 根节点的父索引为无效索引

    return createIndex(parentItem->row(), 0, parentItem);
}

// ========== rowCount() ==========
int TreeModel::rowCount(const QModelIndex &parent) const
{
    TreeItem *parentItem = getItem(parent);
    return parentItem->childCount();
}

// ========== columnCount() ==========
int TreeModel::columnCount(const QModelIndex &parent) const
{
    Q_UNUSED(parent)
    return m_rootItem->columnCount();
}

// ========== data() ==========
QVariant TreeModel::data(const QModelIndex &index, int role) const
{
    if (!index.isValid())
        return QVariant();

    if (role == Qt::DisplayRole || role == Qt::EditRole) {
        TreeItem *item = static_cast<TreeItem *>(index.internalPointer());
        return item->data(index.column());
    }
    return QVariant();
}

QVariant TreeModel::headerData(int section, Qt::Orientation orientation, int role) const
{
    if (orientation == Qt::Horizontal && role == Qt::DisplayRole)
        return m_rootItem->data(section);
    return QVariant();
}

Qt::ItemFlags TreeModel::flags(const QModelIndex &index) const
{
    if (!index.isValid())
        return Qt::NoItemFlags;
    return Qt::ItemIsEnabled | Qt::ItemIsSelectable | Qt::ItemIsEditable;
}

bool TreeModel::setData(const QModelIndex &index, const QVariant &value, int role)
{
    if (!index.isValid() || role != Qt::EditRole)
        return false;

    TreeItem *item = static_cast<TreeItem *>(index.internalPointer());
    item->setData(index.column(), value);

    emit dataChanged(index, index, {role});
    return true;
}

// ========== 辅助方法 ==========
TreeItem *TreeModel::getItem(const QModelIndex &index) const
{
    if (index.isValid()) {
        auto *item = static_cast<TreeItem *>(index.internalPointer());
        if (item)
            return item;
    }
    return m_rootItem;
}

void TreeModel::setupModelData()
{
    // 构建示例：文件系统结构
    //
    // 项目根目录
    // ├── src/
    // │   ├── main.cpp        C++ 源文件    1.2 KB
    // │   └── widget.cpp      C++ 源文件    3.4 KB
    // ├── include/
    // │   └── widget.h        C++ 头文件    0.8 KB
    // └── CMakeLists.txt      CMake 文件    0.5 KB

    auto *srcDir = new TreeItem({"src", "文件夹", ""}, m_rootItem);
    m_rootItem->appendChild(srcDir);
    srcDir->appendChild(new TreeItem({"main.cpp",   "C++ 源文件", "1.2 KB"}, srcDir));
    srcDir->appendChild(new TreeItem({"widget.cpp", "C++ 源文件", "3.4 KB"}, srcDir));

    auto *incDir = new TreeItem({"include", "文件夹", ""}, m_rootItem);
    m_rootItem->appendChild(incDir);
    incDir->appendChild(new TreeItem({"widget.h", "C++ 头文件", "0.8 KB"}, incDir));

    m_rootItem->appendChild(new TreeItem({"CMakeLists.txt", "CMake 文件", "0.5 KB"}, m_rootItem));
}
```

**使用方式：**

```cpp
TreeModel *model = new TreeModel(this);

QTreeView *treeView = new QTreeView(this);
treeView->setModel(model);
treeView->expandAll();
treeView->resizeColumnToContents(0);
```

## 6. 数据角色（Role）深入理解

`data()` 的 `role` 参数决定返回何种类型的数据，视图会用不同角色多次调用 `data()`：

```
视图绘制一个单元格时的调用序列：
  data(index, Qt::DisplayRole)       → 显示文本
  data(index, Qt::DecorationRole)    → 图标
  data(index, Qt::FontRole)          → 字体
  data(index, Qt::TextAlignmentRole) → 对齐方式
  data(index, Qt::BackgroundRole)    → 背景色
  data(index, Qt::ForegroundRole)    → 前景色
  data(index, Qt::ToolTipRole)       → 悬停提示
  data(index, Qt::CheckStateRole)    → 复选框状态
  ...
```

### 6.1 常用内置角色

| 角色 | 值 | 返回类型 | 用途 |
|------|-----|---------|------|
| `Qt::DisplayRole` | 0 | `QString` | 显示文本 |
| `Qt::EditRole` | 2 | `QVariant` | 编辑器中的数据 |
| `Qt::DecorationRole` | 1 | `QIcon` / `QPixmap` / `QColor` | 左侧图标/色块 |
| `Qt::ToolTipRole` | 3 | `QString` | 鼠标悬停提示 |
| `Qt::StatusTipRole` | 4 | `QString` | 状态栏提示 |
| `Qt::WhatsThisRole` | 5 | `QString` | What's This 帮助文本 |
| `Qt::FontRole` | 6 | `QFont` | 字体 |
| `Qt::TextAlignmentRole` | 7 | `Qt::Alignment` | 文本对齐 |
| `Qt::BackgroundRole` | 8 | `QBrush` | 背景 |
| `Qt::ForegroundRole` | 9 | `QBrush` | 前景（文本颜色） |
| `Qt::CheckStateRole` | 10 | `Qt::CheckState` | 复选框 |
| `Qt::SizeHintRole` | 13 | `QSize` | 建议尺寸 |
| `Qt::UserRole` | 0x0100 | 任意 | 自定义数据起始值 |

### 6.2 自定义角色

```cpp
enum MyRoles {
    // 从 Qt::UserRole 开始编号，避开内置角色
    ProgressRole = Qt::UserRole + 1,   // 存储进度值 0~100
    PriorityRole,                      // 存储优先级
    TimestampRole,                     // 存储时间戳
    RawDataRole                        // 存储原始数据指针
};

QVariant data(const QModelIndex &index, int role) const override
{
    if (role == ProgressRole)
        return m_tasks[index.row()].progress;
    if (role == PriorityRole)
        return m_tasks[index.row()].priority;
    // ...
}
```

> **注意：** `Qt::DisplayRole` 和 `Qt::EditRole` 通常返回相同的数据，但有时编辑时需要返回未格式化的原始值（如货币显示 "¥15,000.00"，编辑时返回 `15000.0`）。

## 7. 模型变更通知机制

自定义模型**必须**在修改数据前后正确发射通知信号，否则视图不会更新甚至崩溃。这是自定义模型最容易出错的地方。

### 7.1 通知 API 一览

| 操作类型 | 开始通知 | 结束通知 | 说明 |
|---------|---------|---------|------|
| 修改单元格数据 | — | `emit dataChanged(topLeft, bottomRight, roles)` | 无需 begin/end 配对 |
| 插入行 | `beginInsertRows(parent, first, last)` | `endInsertRows()` | **必须配对** |
| 删除行 | `beginRemoveRows(parent, first, last)` | `endRemoveRows()` | **必须配对** |
| 插入列 | `beginInsertColumns(parent, first, last)` | `endInsertColumns()` | **必须配对** |
| 删除列 | `beginRemoveColumns(parent, first, last)` | `endRemoveColumns()` | **必须配对** |
| 移动行 | `beginMoveRows(srcParent, srcFirst, srcLast, destParent, destRow)` | `endMoveRows()` | **必须配对** |
| 整体重置 | `beginResetModel()` | `endResetModel()` | 视图丢弃所有状态 |
| 布局变化 | `emit layoutAboutToBeChanged()` | `emit layoutChanged()` | 排序、过滤场景 |

### 7.2 关键规则

```
┌──────────────────────────────────────────────────────────────────────┐
│ 规则 1：先 begin，再改数据，最后 end                                  │
│                                                                      │
│   beginInsertRows(parent, 3, 5);  // 通知：即将在 3~5 行插入           │
│   m_data.insert(3, newItem1);     // 实际修改数据                      │
│   m_data.insert(4, newItem2);                                        │
│   m_data.insert(5, newItem3);                                        │
│   endInsertRows();                // 通知：插入完成                     │
│                                                                      │
│ 规则 2：begin/end 必须严格配对，不能嵌套不同类型                        │
│                                                                      │
│   ✅ beginInsertRows → endInsertRows                                 │
│   ❌ beginInsertRows → endRemoveRows                                 │
│   ❌ beginInsertRows → beginInsertRows → endInsertRows → endInsertRows│
│                                                                      │
│ 规则 3：dataChanged 的范围要准确                                      │
│                                                                      │
│   ✅ emit dataChanged(index(2,1), index(2,1));  // 精确到一个单元格     │
│   ⚠️ emit dataChanged(index(0,0), index(rowCount()-1, colCount()-1)); │
│      // 全表刷新 — 功能正确但性能差                                     │
│                                                                      │
│ 规则 4：大批量修改优先用 beginResetModel/endResetModel                 │
│                                                                      │
│   // 替换全部数据时不要逐行 insert/remove                              │
│   beginResetModel();                                                 │
│   m_data = newData;                                                  │
│   endResetModel();                                                   │
└──────────────────────────────────────────────────────────────────────┘
```

### 8.2 序列化拖动数据

```cpp
QStringList MyModel::mimeTypes() const
{
    return {"application/x-mymodel-items"};  // 自定义 MIME 类型
}

QMimeData *MyModel::mimeData(const QModelIndexList &indexes) const
{
    QMimeData *mimeData = new QMimeData();
    QByteArray encodedData;
    QDataStream stream(&encodedData, QIODevice::WriteOnly);

    for (const QModelIndex &index : indexes) {
        if (index.isValid()) {
            stream << index.row() << index.column()
                   << data(index, Qt::DisplayRole).toString();
        }
    }

    mimeData->setData("application/x-mymodel-items", encodedData);
    return mimeData;
}
```

### 8.3 接收放置数据

```cpp
bool MyModel::dropMimeData(const QMimeData *data, Qt::DropAction action,
                           int row, int column, const QModelIndex &parent)
{
    if (action == Qt::IgnoreAction)
        return true;

    if (!data->hasFormat("application/x-mymodel-items"))
        return false;

    // 确定插入位置
    int targetRow = (row != -1) ? row : rowCount(parent);

    QByteArray encodedData = data->data("application/x-mymodel-items");
    QDataStream stream(&encodedData, QIODevice::ReadOnly);

    while (!stream.atEnd()) {
        int srcRow, srcCol;
        QString text;
        stream >> srcRow >> srcCol >> text;

        // 插入新行并设置数据
        beginInsertRows(parent, targetRow, targetRow);
        // ... 插入数据逻辑
        endInsertRows();
        ++targetRow;
    }

    return true;
}
```

## 9. 性能优化

### 9.1 大数据量场景

```cpp
// ❌ 错误做法：逐行通知
for (int i = 0; i < 10000; ++i) {
    beginInsertRows(QModelIndex(), i, i);
    m_data.append(newItem);
    endInsertRows();  // 每次都触发视图更新，极其缓慢
}

// ✅ 正确做法一：批量通知

for (int i = 0; i < 10000; ++i) {
    m_data.append(newItem);
}
endInsertRows();  // 只触发一次视图更新

// ✅ 正确做法二：整体重置
beginResetModel();
m_data = generateLargeDataSet();
endResetModel();
```

### 9.2 `canFetchMore()` / `fetchMore()` — 延迟加载

对于数据量极大的场景（如数据库、文件系统），可以实现按需加载：

```cpp
class LazyModel : public QAbstractListModel
{
    static constexpr int BatchSize = 100;

public:
    int rowCount(const QModelIndex &parent) const override
    {
        return parent.isValid() ? 0 : m_loadedCount;
    }

    // 视图询问：还有更多数据吗？
    bool canFetchMore(const QModelIndex &parent) const override
    {
        if (parent.isValid())
            return false;
        return m_loadedCount < m_totalCount;
    }

    // 视图请求：请加载更多数据
    void fetchMore(const QModelIndex &parent) override
    {
        if (parent.isValid())
            return;

        int remainder = m_totalCount - m_loadedCount;
        int itemsToFetch = qMin(BatchSize, remainder);

        beginInsertRows(QModelIndex(), m_loadedCount, m_loadedCount + itemsToFetch - 1);
        m_loadedCount += itemsToFetch;
        endInsertRows();
    }

private:
    int m_loadedCount = 0;
    int m_totalCount = 1000000;  // 假设有一百万条记录
};
```

当用户滚动 `QListView` 到底部时，视图会自动调用 `canFetchMore()` 和 `fetchMore()` 来加载更多数据——实现流畅的无限滚动效果。

### 9.3 `data()` 的性能注意事项

```cpp
QVariant data(const QModelIndex &index, int role) const override
{

        role != Qt::ToolTipRole && role != Qt::BackgroundRole)
```

## 11. 实战：任务管理器模型

综合运用前面的知识，实现一个支持多种角色、编辑、排序的完整模型。

### 11.1 数据结构

```cpp
// task.h
#pragma once
#include <QString>
#include <QDateTime>

struct Task {
    int       id;
    QString   title;
    QString   description;
    int       priority;    // 1=低, 2=中, 3=高
    int       progress;    // 0~100
    bool      completed;
    QDateTime createdAt;
    QDateTime dueDate;
};
```

### 11.2 模型实现

```cpp
// taskmodel.h
#pragma once
#include <QAbstractTableModel>
#include <QVector>
#include <QColor>
#include "task.h"

class TaskModel : public QAbstractTableModel
{
    Q_OBJECT
public:
    enum Column { ColTitle, ColPriority, ColProgress, ColDueDate, ColStatus, ColCount };
    enum TaskRoles {
        TaskIdRole = Qt::UserRole + 1,
        DescriptionRole,
        ProgressRole,
        PriorityRole,
        CompletedRole,
        CreatedAtRole
    };

    explicit TaskModel(QObject *parent = nullptr) : QAbstractTableModel(parent) {}

    int rowCount(const QModelIndex &parent = {}) const override
    {
        return parent.isValid() ? 0 : m_tasks.size();
    }

    int columnCount(const QModelIndex &parent = {}) const override
    {
        return parent.isValid() ? 0 : ColCount;
    }

    QVariant data(const QModelIndex &index, int role) const override
    {
        if (!index.isValid() || index.row() >= m_tasks.size())
            return {};

        const Task &task = m_tasks[index.row()];

        switch (role) {
        case Qt::DisplayRole:
            switch (index.column()) {
            case ColTitle:    return task.title;
            case ColPriority: return priorityText(task.priority);
            case ColProgress: return QString("%1%").arg(task.progress);
            case ColDueDate:  return task.dueDate.toString("yyyy-MM-dd");
            case ColStatus:   return task.completed ? tr("已完成") : tr("进行中");
            }
            break;

        case Qt::DecorationRole:
            if (index.column() == ColPriority)
                return priorityColor(task.priority);
            break;

        case Qt::BackgroundRole:
            if (task.completed)
                return QColor(240, 255, 240);  // 浅绿色背景
            if (task.dueDate < QDateTime::currentDateTime() && !task.completed)
                return QColor(255, 240, 240);  // 逾期标红
            break;

        case Qt::ForegroundRole:
            if (task.completed)
                return QColor(150, 150, 150);  // 已完成条目灰色
            break;

        case Qt::FontRole:
            if (task.priority == 3) {
                QFont font;
                font.setBold(true);  // 高优先级加粗
                return font;
            }
            break;

        case Qt::TextAlignmentRole:
            if (index.column() == ColProgress || index.column() == ColPriority)
                return QVariant(Qt::AlignCenter);
            break;

        case Qt::ToolTipRole:
            return QString("%1\n\n%2\n\n创建时间: %3")
                .arg(task.title, task.description,
                     task.createdAt.toString("yyyy-MM-dd hh:mm"));

        // 自定义角色
        case TaskIdRole:     return task.id;
        case DescriptionRole:return task.description;
        case ProgressRole:   return task.progress;
        case PriorityRole:   return task.priority;
        case CompletedRole:  return task.completed;
        case CreatedAtRole:  return task.createdAt;
        }

        return {};
    }

    QVariant headerData(int s, Qt::Orientation o, int role) const override
    {
        if (o != Qt::Horizontal || role != Qt::DisplayRole)
            return {};
        static const QString headers[] = {
            tr("任务"), tr("优先级"), tr("进度"), tr("截止日期"), tr("状态")
        };
        return (s >= 0 && s < ColCount) ? headers[s] : QVariant();
    }

    Qt::ItemFlags flags(const QModelIndex &index) const override
    {
        if (!index.isValid()) return Qt::NoItemFlags;
        auto f = Qt::ItemIsEnabled | Qt::ItemIsSelectable;
        if (index.column() == ColTitle || index.column() == ColStatus)
            f |= Qt::ItemIsEditable;
        return f;
    }

    bool setData(const QModelIndex &index, const QVariant &value, int role) override
    {
        if (!index.isValid()) return false;
        Task &task = m_tasks[index.row()];

        if (role == Qt::EditRole) {
            if (index.column() == ColTitle) {
                task.title = value.toString();
            } else {
                return false;
            }
        } else if (role == CompletedRole) {
            task.completed = value.toBool();
            if (task.completed)
                task.progress = 100;
        } else if (role == ProgressRole) {
            task.progress = qBound(0, value.toInt(), 100);
            task.completed = (task.progress == 100);
        } else {
            return false;
        }

        // 通知整行更新（因为 completed 可能联动 progress）
        emit dataChanged(this->index(index.row(), 0),
                         this->index(index.row(), ColCount - 1));
        return true;
    }

    // ===== 业务方法 =====
    void addTask(const Task &task)
    {
        int row = m_tasks.size();
        beginInsertRows({}, row, row);
        m_tasks.append(task);
        endInsertRows();
    }

    void removeTask(int row)
    {
        if (row < 0 || row >= m_tasks.size()) return;
        beginRemoveRows({}, row, row);
        m_tasks.removeAt(row);
        endRemoveRows();
    }

    void toggleComplete(int row)
    {
        if (row < 0 || row >= m_tasks.size()) return;
        setData(index(row, ColStatus), !m_tasks[row].completed, CompletedRole);
    }

    void updateProgress(int row, int progress)
    {
        if (row < 0 || row >= m_tasks.size()) return;
        setData(index(row, ColProgress), progress, ProgressRole);
    }

private:
    static QString priorityText(int p)
    {
        switch (p) {
        case 1:  return "●  低";
        case 2:  return "●● 中";
        case 3:  return "●●●高";
        default: return "?";
        }
    }

    static QColor priorityColor(int p)
    {
        switch (p) {
        case 1:  return QColor(100, 180, 100);  // 绿色
        case 2:  return QColor(220, 180, 50);   // 黄色
        case 3:  return QColor(220, 80, 80);    // 红色
        default: return QColor(150, 150, 150);
        }
    }

    QVector<Task> m_tasks;
};
```

## 12. 常见错误与排查

### 12.1 典型错误清单

| 错误 | 症状 | 解决方案 |
|------|------|---------|
| 忘记 `begin/end` 配对 | 视图不更新或崩溃 | 每次增删都包裹 `beginXxx()` / `endXxx()` |
| `begin/end` 参数错误 | 视图显示错乱：空行、重复行 | 确保 first/last 参数与实际修改范围一致 |
| `rowCount` 返回错误值 | 越界访问导致崩溃 | `parent.isValid()` 时列表/表格模型返回 0 |
| `index()` 中使用了无效指针 | 段错误 | 确保 `child()` 返回非空后再 `createIndex()` |
| `data()` 未处理所有请求角色 | 显示异常（无文本/无图标） | 不处理的角色返回默认 `QVariant()` |
| `setData()` 忘记发射 `dataChanged` | 修改后视图不刷新 | `setData()` 成功后必须 `emit dataChanged()` |
| 在 `begin/end` 外修改数据 | 视图内部索引错乱 | 所有数据修改都在 `begin/end` 之间执行 |
| 递归调用 `dataChanged` | 无限循环 | 检查信号槽连接，避免 `setData` 触发循环 |

### 12.2 调试技巧

```cpp
// 1. 在模型的 data() 中加日志
QVariant data(const QModelIndex &index, int role) const override
{
    qDebug() << "data() called: row=" << index.row()
             << "col=" << index.column() << "role=" << role;
    // ...
}

// 2. 使用 QAbstractItemModelTester（Qt Test 模块）

```

> **`QAbstractItemModelTester` 是验证自定义模型正确性的利器。** 它会在模型的每次变化时自动检查一致性约束（如：`index()` 和 `parent()` 互逆、`rowCount` 与实际行数匹配等）。开发阶段务必使用。

## 13. 总结：自定义模型检查清单

实现自定义模型时，按以下清单逐项检查：

```
□ 选择正确的基类（List / Table / Item）
□ 实现 rowCount()，列表/表格模型在 parent.isValid() 时返回 0
□ 实现 columnCount()（表格/树模型）
□ 实现 data()，处理 DisplayRole + 其他需要的角色
□ 不处理的角色返回默认 QVariant()
□ 实现 headerData()（如果需要列头/行头）
□ 如需编辑：实现 setData() + flags() 返回 ItemIsEditable
□ setData() 成功后发射 dataChanged 信号
□ 如需增删行：实现 insertRows() / removeRows()
□ 增删操作用 beginXxx() / endXxx() 包裹
```
