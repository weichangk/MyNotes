**多线程问题（死锁 / 竞争 / 卡顿 / 崩溃）没有“单一神器”，通常是“调试器 + 线程分析工具 + 性能采样工具”组合使用。**

下面按实战来分。

------

# 一、Windows / Visual Studio（最常用）

## 1. Visual Studio Threads 窗口（核心）

调试时：

```
Debug -> Windows -> Threads
```

可以看到：

- 所有线程列表
- 当前线程
- 调用栈
- 线程 ID
- 当前执行位置

👉 用来定位：

- 卡在哪个线程
- 哪个线程没动
- 死锁卡住的现场

------

## 2. Parallel Stacks（并行堆栈）

```
Debug -> Windows -> Parallel Stacks
```

可以看到：

- 所有线程的调用关系图
- 哪些线程在等锁
- 哪些线程互相阻塞

👉 非常适合：

- 死锁分析
- 线程卡死分析

------

## 3. Concurrency Visualizer（强力）

VS 插件：

```
Diagnostic Tools -> Concurrency
```

能看到：

- CPU 利用率
- 线程时间线
- 阻塞时间
- 锁竞争
- I/O等待

👉 本质是：

```
线程时间轴 + 调度分析
```



## 4. VTune（非常强）

VTune 是一个基于采样的性能分析工具，可以从 CPU 热点、线程调度和锁竞争三个维度分析程序性能瓶颈，特别适合分析 C++/Qt 多线程和高性能应用。

适合：

- CPU 调度分析
- 线程等待
- 锁竞争

可以看到：

- 哪个 mutex 最慢
- 哪个线程被阻塞
- cache miss

👉 工业级工具
