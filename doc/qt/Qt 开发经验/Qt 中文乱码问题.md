### 编译报错：error C2001: 常量中有换行符

通常是由于 **字符串常量中包含非法的换行符**（未正确转义）造成的，而你提到是“有中文导致的”，这很可能是由于：

- **源文件编码不是 UTF-8 with BOM**，而 MSVC 默认按 ANSI 或 UTF-8 无 BOM 解析。
- **中文字符在字符串字面量中出现编码不一致或断行的问题**。

✅ 解决方案一：确保文件编码为 UTF-8 with BOM

✅ 解决方案二：检查字符串中是否存在非法换行

比如你有这样的代码：

```cpp
QString str = "这是一段中文
字符串";
```

这会报错，因为 C++ 不允许字符串跨行写，除非使用反斜杠 `\` 或多段拼接。

正确写法应该是：

```cpp
QString str = "这是一段中文\n字符串";  // 使用 \n
```

或：

```cpp
QString str = "这是一段中文"
              "字符串";  // 多段字符串自动拼接
```

或者：

```cpp
QString str = QStringLiteral("这是一段中文字符串");
```

✅ 解决方案三：使用 `QStringLiteral` 或 `u8""` 字符串

如果你的字符串中包含中文，建议用：

```cpp
QString str = QStringLiteral("你好，世界");
```

或者用 UTF-8 字面量（C++11 及以上）：

```cpp
QString str = u8"你好，世界";
```

🔍 编译器编码行为因环境不同而异，MSVC 在处理源文件时，会根据以下优先级判断编码：

| 编码方式优先级  | 是否支持中文         | 备注                                     |
| --------------- | -------------------- | ---------------------------------------- |
| UTF-8 with BOM  | ✅ 安全稳定           | 推荐                                     |
| UTF-16 with BOM | ✅ 不推荐             | 会导致识别失败或乱码                     |
| UTF-8 无 BOM    | ⚠️ 有风险             | 某些情况下会当作 ANSI 处理，特别是含中文 |
| ANSI（默认）    | ❌ 中文容易乱码或出错 | 编码与系统语言环境强相关                 |

> 所以即使文件内容完全一致，不同电脑：
>
> - 如果系统语言、VS 设置、默认代码页不一样，
> - 就会对 **同一个 UTF-8 无 BOM 文件** 产生不同解释，**中文可能被拆成非法字符，造成“字符串常量中有换行符”错误**。

## 中文乱码的常见原因及解决方案

| 原因                                        | 表现                              | 解决方法                                                     |
| ------------------------------------------- | --------------------------------- | ------------------------------------------------------------ |
| ① 源代码中文字符串编码不正确                | 界面控件显示乱码，如 "ÎÒÊÇÖÐ¹úÈË" | 确保 `.cpp/.h` 文件保存为 **UTF-8 with BOM**，或用 `QStringLiteral()` 包裹中文 |
| ② 程序运行时未设置正确的编码                | 显示为问号 "???" 或乱码           | 在程序入口设置编码（特别是 Qt5）                             |
| ③ 使用 `char*` 而不是 `QString`             | 控件显示乱码，尤其是多字节中文    | 用 `QString::fromUtf8()` 或 `QStringLiteral()` 替代 `char*`  |
| ④ UI 字体不支持中文                         | 界面上不显示或乱码                | 设置支持中文的字体，如 `"Microsoft YaHei"`                   |
| ⑤ 控件未启用 Unicode 字符集（Windows 项目） | 仅显示英文，中文乱码              | 确保 VS 项目设置中字符集为“使用 Unicode 字符集”              |
| ⑥ 翻译文件未加载或未设置本地化              | 国际化时中文未生效                | 检查 `.qm` 文件是否加载成功（Qt）                            |
