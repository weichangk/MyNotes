## 7.1 Qt 的内存管理机制是什么？

### 答案

Qt 通过**对象树（Parent-Child）**和**智能指针**实现自动内存管理。

### 核心原则

1. **有父对象的 QObject** → 父对象析构时自动删除
2. **无父对象的 QObject** → 需要手动 delete
3. **栈对象** → 离开作用域自动析构

### 示例

```cpp
// 自动管理
QWidget* parent = new QWidget();
QLabel* label = new QLabel(parent);  // label 由 parent 管理
delete parent;  // label 自动删除

// 手动管理
QObject* obj = new QObject();  // 无父对象
delete obj;  // 必须手动删除
```

---

## 7.2 如何避免内存泄漏？

### 答案

### 1. 使用对象树

```cpp
QWidget* widget = new QWidget(parent);  // parent 管理 widget
```

### 2. 使用智能指针

```cpp
QScopedPointer<QObject> obj(new QObject());
QSharedPointer<Data> data = QSharedPointer<Data>::create();
```

### 3. 使用 deleteLater()

```cpp
obj->deleteLater();  // 在事件循环中安全删除
```

### 4. 使用 Qt 容器

```cpp
QList<QObject*> list;  // 不拥有所有权，需手动管理
```

## 7.3 什么时候使用 delete，什么时候使用 deleteLater()？

### 答案

| 场景 | 使用 |
|------|------|
| 明确知道对象不再使用 | delete |
| 在信号槽处理中删除发送者 | deleteLater() |
| 在事件处理中删除对象 | deleteLater() |
| 在对象自己的槽函数中删除自己 | deleteLater() |

### 错误示例

```cpp
connect(button, &QPushButton::clicked, [button]() {
    delete button;  // ❌ 崩溃！按钮还在处理 clicked 事件
});
```

### 正确示例

```cpp
connect(button, &QPushButton::clicked, [button]() {
    button->deleteLater();  // ✅ 安全
});
```
