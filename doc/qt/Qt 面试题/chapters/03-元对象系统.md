# 第三章：元对象系统

[返回目录](../README.md) | [上一章](./02-信号与槽.md) | [下一章](./04-控件与界面.md)

Qt 的**元对象系统（Meta-Object System）**是 Qt 框架中非常核心的一套机制，它在 C++ 静态类型系统之上“额外加了一层运行时能力”，让 Qt 具备类似“反射 + 动态调用 + 信号槽”的能力。

可以把它理解为一句话：

> **Qt 用 moc（Meta-Object Compiler）给 C++ 增加了一层“运行时类型信息对象模型”，让 Qt 具备“信号槽 + 动态调用 + 属性系统 + 反射”的能力。**

核心组件包括：

- `moc`（Meta-Object Compiler）
- `QMetaObject`
- `QMetaMethod`
- `QMetaProperty`
- `QMetaType`

它解决的问题是：

> **让 C++ 拥有“类似反射”的能力**

比如：

- 运行时调用函数（invoke）
- 查找信号/槽
- 读取属性（property）
- 动态连接信号槽
- 类型字符串化

---

## 3.1 什么是 MOC（Meta-Object Compiler）？它的作用是什么？

### 答案

**MOC（Meta-Object Compiler）** 是 Qt 的预处理工具，用于生成元对象代码。

### 工作流程

```
MyClass.h (含 Q_OBJECT)
         ↓
      [MOC 处理]
         ↓
    moc_MyClass.cpp (生成的元对象代码)
         ↓
      [C++ 编译器]
         ↓
    最终可执行文件
```

### MOC 的作用

1. **信号与槽机制** - 生成信号的实现代码、连接表
2. **元对象信息** - 生成类的元数据（QMetaObject）
3. **运行时类型信息** - 支持 `qobject_cast<T>()`、`metaObject()->className()`
4. **属性系统** - Q_PROPERTY 的读写函数映射
5. **动态调用** - `QMetaObject::invokeMethod()` 动态调用方法

### Q_OBJECT 宏

```cpp
#define Q_OBJECT \
public: \
    static const QMetaObject staticMetaObject; \
    virtual const QMetaObject *metaObject() const; \
    virtual void *qt_metacast(const char *); \
    virtual int qt_metacall(QMetaObject::Call, int, void **); \
private:
```

### 何时需要运行 MOC

- 类中声明了 `Q_OBJECT` 宏
- 定义了信号（`signals`）
- 定义了槽（`slots`）
- 使用了 `Q_PROPERTY`
- 使用了 `Q_ENUM` / `Q_FLAG`

## 3.2 Q_PROPERTY 宏是什么？如何使用？

### 答案

`Q_PROPERTY` 用于定义可读写、可绑定、可通知的属性。

### 基本语法

```cpp
class MyClass : public QObject {
    Q_OBJECT
    Q_PROPERTY(int value READ value WRITE setValue NOTIFY valueChanged)
    
public:
    int value() const { return m_value; }
    
public slots:
    void setValue(int value) {
        if (m_value != value) {
            m_value = value;
            emit valueChanged(value);
        }
    }
    
signals:
    void valueChanged(int value);
    
private:
    int m_value;
};
```

### 属性关键字

| 关键字 | 说明 |
|--------|------|
| READ | 读取函数（必须）|
| WRITE | 写入函数（可选）|
| NOTIFY | 变化通知信号（可选，强烈推荐）|
| RESET | 重置函数（可选）|
| CONSTANT | 常量属性（不可变）|
| DESIGNABLE | 是否在 Qt Designer 中可见 |

### 属性使用

#### 1. 通用属性访问（QObject API）

```cpp
MyWidget* widget = new MyWidget();

// 设置属性
widget->setProperty("title", "Hello");

// 读取属性
QString title = widget->property("title").toString();

// 列出所有属性
const QMetaObject* meta = widget->metaObject();
for (int i = 0; i < meta->propertyCount(); ++i) {
    QMetaProperty prop = meta->property(i);
    qDebug() << prop.name() << widget->property(prop.name());
}
```

#### 2. QML 集成

```qml
MyWidget {
    title: "Hello QML"
    opacity: 80
    
    onTitleChanged: {
        console.log("Title changed:", title)
    }
}
```

## 3.3 什么是动态属性？如何使用？

### 答案

**动态属性** 是运行时添加到 QObject 的属性，无需类定义声明。

### 基本用法

```cpp
QObject* obj = new QObject();

// 设置动态属性
obj->setProperty("customId", 12345);
obj->setProperty("userName", "Alice");

// 读取动态属性
int id = obj->property("customId").toInt();

// 删除动态属性
obj->setProperty("customId", QVariant());
```

### 应用场景

#### 1. UI 控件关联数据

```cpp
QPushButton* button = new QPushButton("User 1");
button->setProperty("userId", 1001);

connect(button, &QPushButton::clicked, [button]() {
    int userId = button->property("userId").toInt();
    qDebug() << "userId:" << userId;
});
```

#### 2. QSS 条件样式

```cpp
QPushButton* button = new QPushButton();
button->setProperty("level", "danger");

button->setStyleSheet(
    "QPushButton { background: green; }"
    "QPushButton[level=\"danger\"] { background: red; }"
);
```

#### 3. 状态机标记

```cpp
QWidget* widget = new QWidget();
widget->setProperty("state", "loading");

if (widget->property("state") == "loading") {
    showLoadingSpinner();
}
```

### 动态属性 vs Q_PROPERTY

| 特性 | 动态属性 | Q_PROPERTY |
|------|---------|-----------|
| 声明方式 | 运行时动态添加 | 编译期类定义 |
| 类型检查 | QVariant（运行时）| 强类型（编译期）|
| 性能 | 稍慢 | 快 |
| NOTIFY 信号 | 无 | 有 |
| QML 绑定 | 不支持 | 支持 |

## 3.4 如何实现运行时类型信息查询？

### 答案

### 1. 获取类名

```cpp
QObject* obj = new QPushButton();
QString className = obj->metaObject()->className();
qDebug() << className;  // "QPushButton"
```

### 2. 检查继承关系

```cpp
if (obj->inherits("QWidget")) {
    qDebug() << "obj is a QWidget";
}
```

### 3. qobject_cast 动态类型转换

```cpp
QObject* obj = new QPushButton();

QWidget* widget = qobject_cast<QWidget*>(obj);
if (widget) {
    widget->show();
}
```

### 4. 查询属性列表

```cpp
const QMetaObject* meta = obj->metaObject();
for (int i = 0; i < meta->propertyCount(); ++i) {
    QMetaProperty prop = meta->property(i);
    qDebug() << prop.name() << prop.read(obj);
}
```

### 5. 查询方法列表

```cpp
for (int i = 0; i < meta->methodCount(); ++i) {
    QMetaMethod method = meta->method(i);
    qDebug() << method.methodSignature();
}
```

### 6. 动态方法调用

```cpp
// 调用无参方法
QMetaObject::invokeMethod(button, "click");

// 调用带参方法
QMetaObject::invokeMethod(button, "setText", 
                          Q_ARG(QString, "New Text"));

// 获取返回值
QString text;
QMetaObject::invokeMethod(button, "text", 
                          Q_RETURN_ARG(QString, text));
```
