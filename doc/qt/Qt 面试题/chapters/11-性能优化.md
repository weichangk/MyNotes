## 11.1 如何优化 Qt 应用的启动速度？

### 答案

启动速度优化需要从**模块加载、资源加载、初始化**三个方面入手。

### 优化策略对比

| 策略 | 效果 | 难度 |
|------|------|------|
| 延迟加载模块 | ⭐⭐⭐ | 低 |
| 优化资源文件 | ⭐⭐ | 低 |
| 异步初始化 | ⭐⭐⭐ | 中 |
| 预编译头文件 | ⭐⭐ | 低 |
| 减少全局对象 | ⭐ | 中 |

---

### 1. 延迟加载模块

```cpp
// ❌ 错误：启动时加载所有模块
int main(int argc, char* argv[]) {
    QApplication app(argc, argv);
    
    // 启动时就初始化所有模块
    DatabaseManager::instance();
    NetworkManager::instance();
    PluginManager::instance();
    
    MainWindow window;
    window.show();
    return app.exec();
}

// ✅ 正确：按需延迟加载
int main(int argc, char* argv[]) {
    QApplication app(argc, argv);
    
    // 只创建主窗口
    MainWindow window;
    window.show();
    
    // 延迟初始化（使用定时器或线程）
    QTimer::singleShot(100, []() {
        DatabaseManager::instance();  // 100ms 后初始化
    });
    
    QTimer::singleShot(200, []() {
        PluginManager::instance();    // 200ms 后初始化
    });
    
    return app.exec();
}
```

```cpp
// ❌ 错误：所有资源打包在一起
```

### 3. 异步初始化

```cpp
class AsyncInitializer : public QObject {
    Q_OBJECT
public:
    static void initialize() {
        // 创建工作线程
        QThread* thread = new QThread();
        AsyncInitializer* worker = new AsyncInitializer();
        worker->moveToThread(thread);
        
        connect(thread, &QThread::started, worker, &AsyncInitializer::doInit);
        connect(worker, &AsyncInitializer::finished, thread, &QThread::quit);
        connect(thread, &QThread::finished, thread, &QThread::deleteLater);
        
        thread->start();
    }
    
private slots:
    void doInit() {
        // 耗时初始化操作
        loadConfiguration();
        initDatabase();
        loadPlugins();
        
        emit finished();
    }
    
signals:
    void finished();
};

// 使用
int main(int argc, char* argv[]) {
    QApplication app(argc, argv);
    
    MainWindow window;
    window.show();
    
    // 异步初始化
    AsyncInitializer::initialize();
    
    return app.exec();
}
```

### 5. 减少全局对象构造


## 11.2 如何优化界面渲染性能？

### 答案

界面渲染优化需要从**重绘频率、绘制复杂度、布局计算**三个方面优化。

---

### 1. 减少重绘次数

```cpp
// ❌ 错误：频繁调用 update() 导致重绘
void updateData() {
    for (int i = 0; i < 100; ++i) {
        m_data[i] = newValue;
        update();  // ❌ 每次都重绘，非常慢
    }
}

// ✅ 正确：批量更新后统一重绘
void updateData() {
    for (int i = 0; i < 100; ++i) {
        m_data[i] = newValue;
    }
    update();  // ✅ 只重绘一次
}
```

---

### 2. update() vs repaint()

```cpp
// update()：异步重绘（推荐）
void onDataChanged() {
    update();  // 事件循环中重绘，自动合并多次 update()
}

// repaint()：立即重绘（慎用）
void onUrgentUpdate() {
    repaint();  // 立即绘制，阻塞当前线程
}
```

| 方法 | 执行时机 | 合并重绘 | 性能 |
|------|---------|---------|------|
| update() | 事件循环 | ✅ | 高 |
| repaint() | 立即执行 | ❌ | 低 |

---

### 3. 局部刷新

```cpp
void paintEvent(QPaintEvent* event) {
    QPainter painter(this);
    
    // ❌ 错误：每次都绘制全部内容
    painter.fillRect(rect(), Qt::white);
    drawAllItems();
    
    // ✅ 正确：只绘制需要更新的区域
    QRect updateRect = event->rect();  // 获取需要重绘的区域
    painter.setClipRect(updateRect);
    
    // 只绘制在 updateRect 内的元素
    for (const Item& item : m_items) {
        if (item.rect.intersects(updateRect)) {
            drawItem(item);
        }
    }
}

// 只刷新特定区域
void onItemChanged(const QRect& itemRect) {
    update(itemRect);  // 只刷新这个矩形区域
}
```

### 4. 使用缓存

```cpp
class CachedWidget : public QWidget {
public:
    CachedWidget() {
        // 使用 QPixmap 缓存
        m_cache = QPixmap(size());
        m_cacheValid = false;
    }
    
protected:
    void paintEvent(QPaintEvent*) override {
        // 缓存失效时重新绘制
        if (!m_cacheValid) {
            m_cache = QPixmap(size());
            QPainter cachePainter(&m_cache);
            drawContent(&cachePainter);  // 绘制到缓存
            m_cacheValid = true;
        }
        
        // 从缓存绘制（非常快）
        QPainter painter(this);
        painter.drawPixmap(0, 0, m_cache);
    }
    
    void resizeEvent(QResizeEvent*) override {
        m_cacheValid = false;  // 大小改变，缓存失效
    }
    
    void dataChanged() {
        m_cacheValid = false;  // 数据改变，缓存失效
        update();
    }
    
private:
    void drawContent(QPainter* painter) {
        // 复杂的绘制逻辑
        painter->fillRect(rect(), Qt::white);
        // ...
    }
    
    QPixmap m_cache;
    bool m_cacheValid;
};
```

### 5. 避免复杂样式表

```cpp
// ❌ 错误：样式表使用渐变、阴影（性能差）
button->setStyleSheet(R"(
    QPushButton {
        background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
                                   stop:0 #6fa8dc, stop:1 #3d85c6);
        border-radius: 5px;
        padding: 10px;
    }
    QPushButton:hover {
        background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
                                   stop:0 #8fc1e3, stop:1 #5fa3d9);
    }
)");

// ✅ 正确：使用简单样式或自绘
button->setStyleSheet(R"(
    QPushButton {
        background-color: #6fa8dc;
        border-radius: 5px;
        padding: 10px;
    }
    QPushButton:hover {
        background-color: #8fc1e3;
    }
)");

// 或者自绘控件（更高效）
class FastButton : public QPushButton {
protected:
    void paintEvent(QPaintEvent*) override {
        QPainter painter(this);
        painter.setRenderHint(QPainter::Antialiasing);
        
        QColor color = isDown() ? QColor(61, 133, 198)
                                : QColor(111, 168, 220);
        painter.setBrush(color);
        painter.setPen(Qt::NoPen);
        painter.drawRoundedRect(rect(), 5, 5);
        
        painter.setPen(Qt::white);
        painter.drawText(rect(), Qt::AlignCenter, text());
    }
};
```

## 11.3 Qt 容器 vs STL 容器，如何选择？

### 答案

Qt 容器和 STL 容器**各有优势**，根据场景选择。

### 对比表

| 特性 | Qt 容器 | STL 容器 |
|------|---------|---------|
| 内存管理 | 隐式共享（COW） | 标准分配器 |
| 线程安全 | 读安全（隐式共享） | 不保证 |
| Qt 集成 | 原生支持 | 需要转换 |
| 性能 | 略低（COW 开销） | 略高 |
| foreach 支持 | ✅ | ❌（需 range-for） |

---

### Qt 容器的隐式共享

```cpp
// 隐式共享（Copy-On-Write）
QVector<int> v1 = {1, 2, 3};
QVector<int> v2 = v1;  // 不复制数据，共享内存

v2.append(4);  // 此时才复制（写时复制）

// STL 容器（立即复制）
std::vector<int> v3 = {1, 2, 3};
std::vector<int> v4 = v3;  // 立即深拷贝
```

---

### 常用容器对比

| Qt 容器 | STL 容器 | 说明 |
|---------|---------|------|
| QVector | std::vector | 动态数组 |
| QList | std::list | 双向链表 |
| QMap | std::map | 红黑树映射 |
| QHash | std::unordered_map | 哈希表 |
| QSet | std::unordered_set | 哈希集合 |

---

### 选择建议

#### 使用 Qt 容器的场景

```cpp
// 1. 与 Qt API 交互
QStringList items = model->stringList();  // Qt API 返回 QStringList
for (const QString& item : items) {
    // ...
}

// 2. 需要隐式共享（减少复制）
void processData(const QVector<int>& data) {
    // data 是引用，不复制，很高效
}

// 3. 使用 foreach（Qt 风格）
QMap<QString, int> map;
foreach (const QString& key, map.keys()) {
    qDebug() << key << map[key];
}
```

#### 使用 STL 容器的场景

```cpp
// 1. 纯算法代码（不涉及 Qt）
std::vector<int> numbers = {1, 2, 3, 4, 5};
std::sort(numbers.begin(), numbers.end());

// 2. 性能关键路径
std::vector<Point> points;  // 避免 COW 开销
points.reserve(10000);
for (int i = 0; i < 10000; ++i) {
    points.push_back({i, i});
}

// 3. 使用 STL 算法
std::vector<int> v = {1, 2, 3, 4, 5};
auto it = std::find(v.begin(), v.end(), 3);
```

### 性能测试对比

```cpp
#include <QDebug>
#include <QElapsedTimer>
#include <QVector>
#include <vector>

void benchmark() {
    const int N = 1000000;
    
    // Qt 容器
    QElapsedTimer timer;
    timer.start();
    
    QVector<int> qvec;
    for (int i = 0; i < N; ++i) {
        qvec.append(i);
    }
    
    qint64 qtTime = timer.elapsed();
    qDebug() << "QVector:" << qtTime << "ms";
    
    // STL 容器
    timer.restart();
    
    std::vector<int> svec;
    for (int i = 0; i < N; ++i) {
        svec.push_back(i);
    }
    
    qint64 stlTime = timer.elapsed();
    qDebug() << "std::vector:" << stlTime << "ms";
}
```

## 11.4 如何进行性能分析？

### 答案

使用 **Qt Creator Profiler** 或外部工具进行性能分析。

---

### 1. Qt Creator 性能分析器

```
菜单 → 分析 → QML Profiler（QML 应用）
      → 性能分析器（C++ 应用）
```

**支持的分析类型**：
- CPU 使用率
- 内存分配
- 函数调用次数
- 函数耗时

---

### 2. 手动计时

```cpp
#include <QElapsedTimer>
#include <QDebug>

void functionToProfile() {
    QElapsedTimer timer;
    timer.start();
    
    // 要分析的代码
    for (int i = 0; i < 1000000; ++i) {
        // ...
    }
    
    qDebug() << "耗时:" << timer.elapsed() << "ms";
}
```

---

### 3. 宏定义计时器

```cpp
#define PROFILE_SCOPE(name) \
    QElapsedTimer timer_##name; \
    timer_##name.start(); \
    auto profiler_##name = qScopeGuard([&timer_##name]() { \
        qDebug() << #name << "耗时:" << timer_##name.elapsed() << "ms"; \
    });

// 使用
void myFunction() {
    PROFILE_SCOPE(myFunction);
    
    // 函数代码
    // ...
    
    // 离开作用域时自动输出耗时
}
```

---

### 4. Valgrind（Linux）

```bash
# 内存泄漏检测
valgrind --leak-check=full ./myapp

# 性能分析
valgrind --tool=callgrind ./myapp
kcachegrind callgrind.out.12345
```

---

### 5. Visual Studio Profiler（Windows）

```
菜单 → 调试 → 性能探查器
    → CPU 使用率
    → 内存使用率
```

## 本章小结

### 核心知识点

1. **启动速度优化**
   - 延迟加载模块
   - 优化资源文件
   - 异步初始化
   - 预编译头文件

2. **渲染性能优化**
   - 减少重绘（update 合并）
   - 局部刷新（update(rect)）
   - 使用缓存（QPixmap）
   - 避免复杂样式表
   - OpenGL 加速

3. **容器选择**
   - Qt 容器：Qt API 交互、隐式共享
   - STL 容器：纯算法、性能关键

4. **性能分析**
   - Qt Creator Profiler
   - QElapsedTimer 手动计时
   - Valgrind / VS Profiler

### 优化原则

- ✅ 先测量，再优化（避免过早优化）
- ✅ 优化热点代码（80/20 原则）
- ✅ 权衡开发效率和运行效率
- ✅ 使用缓存减少重复计算
- ✅ 异步处理耗时操作
