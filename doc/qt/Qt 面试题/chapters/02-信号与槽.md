## 2.1 什么是信号与槽？有什么优势？

### 答案

**信号与槽（Signals and Slots）** 是 Qt 的核心机制，用于对象间的通信。

### 基本概念

- **信号（Signal）**：事件发生时发出的通知（声明但不实现）
- **槽（Slot）**：响应信号的函数（普通成员函数）
- **连接（Connection）**：将信号与槽关联起来

### 示例

```cpp
class Counter : public QObject {
    Q_OBJECT
    
public:
    Counter() : m_value(0) {}
    
    int value() const { return m_value; }
    
public slots:
    void setValue(int value) {
        if (m_value != value) {
            m_value = value;
            emit valueChanged(value);  // 发射信号
        }
    }
    
signals:
    void valueChanged(int newValue);  // 信号声明（无需实现）
    
private:
    int m_value;
};

// 使用
Counter counter;
QLabel* label = new QLabel();

// 连接信号与槽
QObject::connect(&counter, &Counter::valueChanged,
                 label, [label](int value) {
    label->setText(QString::number(value));
});

counter.setValue(42);  // 触发信号，label 自动更新
```

### 优势

1. **松耦合**：信号发送者不需要知道接收者是谁
2. **类型安全**：编译期检查信号与槽的参数类型
3. **灵活连接**：一对多、多对一、信号连接信号
4. **跨线程安全**：自动处理跨线程信号传递
5. **自动断开**：对象销毁时自动断开所有连接

## 2.2 信号与槽有哪几种连接方式？各有什么特点？

### 答案

Qt 提供了 5 种连接类型（`Qt::ConnectionType`）：

### 连接类型对比

| 连接类型 | 执行线程 | 是否阻塞 | 跨线程安全 | 使用场景 |
|---------|---------|---------|-----------|---------|
| AutoConnection | 自动判断 | 同线程阻塞 | 是 | 默认推荐 |
| DirectConnection | 发送者线程 | 阻塞 | 否 | 同线程高性能 |
| QueuedConnection | 接收者线程 | 不阻塞 | 是 | 跨线程异步 |
| BlockingQueuedConnection | 接收者线程 | 阻塞 | 是（同线程死锁）| 跨线程同步 |
| UniqueConnection | - | - | - | 防止重复连接 |

### 详细说明

#### 1. Qt::AutoConnection（默认）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot);
// 自动判断：同线程→DirectConnection，跨线程→QueuedConnection
```

#### 2. Qt::DirectConnection（直接连接）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot, Qt::DirectConnection);
```

- 信号发射时**立即同步调用**槽函数
- 槽函数在发送者线程中执行
- **跨线程不安全**

#### 3. Qt::QueuedConnection（队列连接）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot, Qt::QueuedConnection);
```

- 信号加入事件队列，异步执行
- 槽函数在接收者线程中执行
- **线程安全**，跨线程通信的标准方式

#### 4. Qt::BlockingQueuedConnection（阻塞队列连接）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot, Qt::BlockingQueuedConnection);
```

- 发送者线程**阻塞等待**槽函数执行完毕
- 槽函数在接收者线程中执行
- **同线程使用会死锁**

#### 5. Qt::UniqueConnection（唯一连接）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot, Qt::UniqueConnection);
```

- 防止重复连接
- 可与其他类型组合（`Qt::AutoConnection | Qt::UniqueConnection`）

## 2.3 信号与槽的连接语法有哪几种？各有什么优缺点？

### 答案

### 1. 函数指针语法（Qt 5，推荐）

```cpp
connect(sender, &Sender::signal, receiver, &Receiver::slot);
```

**优点**：
- **编译期类型检查**
- 支持 IDE 自动补全
- 代码清晰易维护

**缺点**：
- 重载函数需要类型转换

### 2. Lambda 表达式（Qt 5+）

```cpp
connect(button, &QPushButton::clicked, [this]() {
    qDebug() << "Button clicked";
    label->setText("Clicked!");
});

// 带上下文对象（推荐）
connect(button, &QPushButton::clicked, this, [this]() {
    label->setText("Clicked!");
});
```

**优点**：
- 无需定义槽函数
- 可捕获外部变量

**缺点**：
- 复杂逻辑可读性差
- 捕获 `this` 可能导致悬空指针

### 3. 字符串语法（Qt 4，已废弃）

```cpp
connect(sender, SIGNAL(signal()), receiver, SLOT(slot()));
```

**缺点**：
- **无编译期检查**
- 不支持重载
- 不支持 Lambda

### 语法对比总结

| 语法 | 类型检查 | 重载支持 | Lambda | 推荐度 |
|------|---------|---------|--------|--------|
| 函数指针 | ✅ 编译期 | ✅ 需转换 | ❌ | ⭐⭐⭐⭐⭐ |
| Lambda | ✅ 编译期 | ✅ | ✅ | ⭐⭐⭐⭐ |
| SIGNAL/SLOT 宏 | ❌ 运行时 | ❌ | ❌ | ⭐（已废弃）|

## 2.4 如何断开信号与槽的连接？

### 答案

### 断开连接的方法

#### 1. 断开特定连接

```cpp
disconnect(sender, &Sender::signal, receiver, &Receiver::slot);
```

#### 2. 使用连接对象

```cpp
QMetaObject::Connection conn = connect(sender, &Sender::signal, 
                                       receiver, &Receiver::slot);
disconnect(conn);
```

#### 3. 断开所有连接

```cpp
// 断开 sender 的所有连接
disconnect(sender, nullptr, nullptr, nullptr);

// 断开 sender 的特定信号
disconnect(sender, &Sender::signal, nullptr, nullptr);
```

#### 4. 自动断开

```cpp
{
    QObject sender;
    QObject receiver;
    connect(&sender, &QObject::destroyed, &receiver, &QObject::deleteLater);
}  // sender 和 receiver 析构时自动断开所有连接
```

### 实用技巧

#### RAII 式连接管理

```cpp
class ScopedConnection {
public:
    ScopedConnection(QMetaObject::Connection conn) : m_conn(conn) {}
    ~ScopedConnection() {
        if (m_conn) {
            QObject::disconnect(m_conn);
        }
    }
private:
    QMetaObject::Connection m_conn;
};
```

## 2.5 信号可以有返回值吗？多个槽的返回值如何处理？

### 答案

**信号可以有返回值，但不推荐使用**。

### 基本用法

```cpp
class MyClass : public QObject {
    Q_OBJECT
signals:
    int getValue();  // 信号可以有返回值
};

// 多个槽连接
int result = emit sender.getValue();
qDebug() << result;  // 只返回最后一个槽的返回值
```

### 推荐替代方案

#### 方案1：通过信号参数传递结果

```cpp
class MyClass : public QObject {
    Q_OBJECT
signals:
    void requestValue(int* result);
    
public:
    int getValue() {
        int result = 0;
        emit requestValue(&result);
        return result;
    }
};
```

#### 方案2：使用 QSharedPointer

```cpp
class Result {
public:
    QList<int> values;
};

signals:
    void requestValues(QSharedPointer<Result> result);

QList<int> getValues() {
    auto result = QSharedPointer<Result>::create();
    emit requestValues(result);
    return result->values;
}
```
