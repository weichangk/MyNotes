## 一、Qt 基础 (1-15)

**1. Qt 的对象模型是什么？**

- **简答：** 基于 QObject，支持信号槽、事件机制、元对象系统、对象树内存管理。
- **扩展：** 与标准 C++ 对象模型不同，Qt 增加了运行时类型识别、动态属性、事件分发、父子树析构机制，避免手动 delete。

------

**2. Q_OBJECT 宏的作用？**

- **简答：** 启用 MOC 生成的元对象信息，支持信号槽、属性、动态反射。
- **扩展：** 没有 Q_OBJECT，类不会生成 `moc_xxx.cpp`，就无法使用信号槽、`metaObject()`、`qobject_cast()`。

------

**3. 信号和槽的实现机制？**

- **简答：** MOC 生成 metaObject，信号通过 `QMetaObject::activate` 分发到槽。
- **扩展：** 信号本质是类的函数声明，MOC 生成元数据表，连接关系存储在 `QObjectPrivate::ConnectionList`，调用时遍历触发。

------

**4. Qt 如何实现跨平台？**

- **简答：** 使用 QPA（Qt Platform Abstraction）封装不同平台 API。
- **扩展：** Windows 用 Win32，Linux 用 XCB/Wayland，macOS 用 Cocoa。Qt 控件和事件经过 QPA 抽象，上层 API 保持一致。

------

**5. QWidget 与 QWindow 区别？**

- **简答：** QWidget 是传统控件，QWindow 抽象底层窗口。
- **扩展：** QWidget 基于 CPU 渲染（QPainter），QWindow 更接近原生窗口，QML/Quick UI 基于 QWindow。

------

**6. Qt 的事件机制？**

- **简答：** 事件由 QEventLoop 分发，event() 调用具体事件处理函数。
- **扩展：** 系统事件被封装为 QEvent，投递到对象的事件队列。可以通过 `installEventFilter` 进行事件拦截。

------

**7. Qt 内存管理机制？**

- **简答：** 父子对象树 + 隐式共享（COW）+ 智能指针。
- **扩展：** QObject 派生类禁止拷贝，依赖父子树管理；Qt 容器（QList、QString）使用隐式共享提高性能。

------

**8. Qt 常见容器类？**

- **简答：** QVector、QList、QMap、QHash。
- **扩展：** Qt 容器对 Qt 类型优化（支持隐式共享），对 POD 类型性能略低于 STL。

------

**9. Qt 字符串类有哪些？**

- **简答：** QString（UTF-16），QByteArray（字节数组，常用 UTF-8）。
- **扩展：** QString 使用隐式共享，常用于 UI 和国际化，QByteArray 更适合网络、文件传输。

------

**10. QVariant 的作用？**

- **简答：** 类型安全的通用数据封装。
- **扩展：** QVariant 可以存储 Qt 基础类型，结合 QMetaType 实现动态类型系统。

------

**11. QPalette 的作用？**

- **简答：** 控件颜色管理。
- **扩展：** 支持不同状态（Normal、Disabled、Active）设置不同颜色，常与 QStyle 一起决定控件最终显示效果。

------

**12. QStyle 的作用？**

- **简答：** 控件外观统一抽象，支持跨平台主题。
- **扩展：** QStyle 封装了按钮、菜单等绘制方法，Windows 用 WinVistaStyle，macOS 用 MacStyle。

------

**13. Qt 的绘图设备有哪些？**

- **简答：** QWidget、QPixmap、QImage、QPrinter。
- **扩展：** QWidget 用于控件绘制，QPixmap 用于显示优化，QImage 适合图像处理，QPrinter 用于打印。

------

**14. Qt 的国际化机制？**

- **简答：** `tr()` 标记文本 → TS 文件 → 翻译 → QM 文件 → QTranslator 加载。
- **扩展：** 支持动态切换语言，TS 文件为 XML 格式，QM 文件为二进制翻译表。

------

**15. Qt 的资源系统？**

- **简答：** `.qrc` 文件打包资源，使用 `:/path/file` 访问。
- **扩展：** 资源编译进可执行文件，避免外部依赖。

------

## 二、进阶篇 (16-35)

**16. Qt 多线程实现方式？**

- **简答：** QThread、Worker+moveToThread、QtConcurrent、QThreadPool。
- **扩展：** 推荐用 Worker+moveToThread，QThread 仅管理线程生命周期。

------

**17. 信号槽连接类型？**

- **简答：** Direct、Queued、BlockingQueued、Auto。
- **扩展：** 同线程默认 Direct，跨线程默认 Queued。BlockingQueued 用于阻塞调用。

------

**18. 跨线程更新 UI？**

- **简答：** 用信号槽（QueuedConnection）。
- **扩展：** 因为 QWidget 线程不安全，UI 必须在主线程更新。

------

**19. Model/View 架构？**

- **简答：** Model 提供数据，View 显示，Delegate 负责绘制和编辑。
- **扩展：** QAbstractItemModel 支持通用数据接口，QListView/QTableView/QTreeView 是典型实现。

------

**20. QStyledItemDelegate 作用？**

- **简答：** 自定义 item 绘制和编辑器。
- **扩展：** 常用于复杂表格单元格绘制（比如复选框、下拉框）。

------

**21. Qt MVC 与 Qt Quick 的区别？**

- **简答：** Widgets 用 MVC，QML 用数据绑定。
- **扩展：** QML 更适合动画和移动端，Widgets 更适合桌面应用。

------

**22. 隐式共享原理？**

- **简答：** 引用计数 + COW（copy-on-write）。
- **扩展：** 写操作时复制，避免无谓的拷贝开销，典型类：QString、QByteArray。

------

**23. Qt 动画框架？**

- **简答：** QPropertyAnimation 基于属性系统。
- **扩展：** 支持并行动画、序列动画、缓动曲线。

------

**24. Qt 状态机框架？**

- **简答：** QStateMachine 管理复杂状态转换。
- **扩展：** 支持分层状态、并行状态、事件驱动。

------

**25. QTimer 作用？**

- **简答：** 定时器，基于事件循环。
- **扩展：** 精度受限于事件循环和系统调度。

------

**26. Qt 网络模块？**

- **简答：** QTcpSocket、QTcpServer、QUdpSocket、QNetworkAccessManager。
- **扩展：** QNAM 封装 HTTP/HTTPS，支持异步请求。

------

**27. Qt 序列化方式？**

- **简答：** QDataStream（二进制）、QJsonDocument（JSON）、QXmlStreamReader（XML）。
- **扩展：** QDataStream 更高效，JSON/XML 更适合跨平台和网络。

------

**28. QPixmap 与 QImage 区别？**

- **简答：** QPixmap 优化显示，QImage 适合像素操作。
- **扩展：** QPixmap 存储在 GPU，QImage 在 CPU 内存。

------

**29. QPainter 绘制流程？**

- **简答：** 创建 QPainter → 绑定 QPaintDevice → 绘制 → end()。
- **扩展：** 绘制只能在 paintEvent 内部，避免闪烁可使用双缓冲。

------

**30. QEventLoop 作用？**

- **简答：** 管理事件循环，可嵌套使用。
- **扩展：** 常用于 QDialog::exec，阻塞当前逻辑，进入局部事件循环。

------

**31. Qt 属性系统？**

- **简答：** Q_PROPERTY 定义属性，支持绑定和动画。
- **扩展：** 在 QML 中常用，C++ 中也可通过 metaObject 动态访问。

------

**32. Qt 插件机制？**

- **简答：** QPluginLoader 动态加载插件库。
- **扩展：** 使用 Q_INTERFACES 声明接口，插件需导出 Q_PLUGIN_METADATA。

------

**33. Qt OpenGL 支持？**

- **简答：** QOpenGLWidget、QOpenGLWindow。
- **扩展：** QPainter 可与 OpenGL 混合，Qt6 通过 RHI 抽象支持 Vulkan/Metal。

------

**34. Qt 大文件 I/O？**

- **简答：** QFile+QIODevice 分块读写。
- **扩展：** 可结合 QDataStream 处理结构化数据。

------

**35. 信号槽性能？**

- **简答：** 比函数调用慢，但可忽略。
- **扩展：** 多线程频繁通信时需优化，可用 direct connection 或 lambda。

------

## 三、源码与底层 (36-50)

**36. Qt 为什么需要 MOC？**

- **简答：** C++ 不支持反射，MOC 生成元对象。
- **扩展：** 解决信号槽、动态属性、运行时类型信息。

------

**37. 事件循环实现？**

- **简答：** 基于平台消息循环。
- **扩展：** Windows: GetMessage，Linux: epoll，macOS: CFRunLoop。

------

**38. 内存泄漏避免？**

- **简答：** 父子对象树 + QPointer + 智能指针。
- **扩展：** 结合工具 Valgrind、ASan、VLD 检测。

------

**39. QObject 为什么不能拷贝？**

- **简答：** 信号槽和父子树关系会丢失。
- **扩展：** 只能用指针传递。

------

**40. QCoreApplication 与 QApplication 区别？**

- **简答：** QCoreApplication 不带 GUI，QApplication 带 GUI 支持。
- **扩展：** QGuiApplication 介于两者之间（仅 GUI，不带 QWidget）。

------

**41. Qt 跨平台 UI 技术？**

- **简答：** QPA 抽象。
- **扩展：** 提供插件接口，封装窗口、输入、绘图。

------

**42. 元对象系统？**

- **简答：** 提供 RTTI、信号槽、属性反射。
- **扩展：** QMetaObject 存储类名、信号槽列表、属性表。

------

**43. 内存池机制？**

- **简答：** Qt 容器减少内存分配次数。
- **扩展：** 常用于 QString、QList，提升性能。

------

**44. Qt 反射机制？**

- **简答：** QMetaObject+QMetaProperty+QMetaMethod。
- **扩展：** 支持 `invokeMethod()` 动态调用函数。

------

**45. Qt 是否有垃圾回收？**

- **简答：** 没有 GC，靠父子树管理。
- **扩展：** 辅助工具：QPointer、智能指针。

------

**46. QWidget 为什么线程不安全？**

- **简答：** GUI 必须在主线程执行。
- **扩展：** 底层平台 API 不支持跨线程 UI 更新。

------

**47. Qt5 与 Qt6 区别？**

- **简答：** Qt6 基于 C++17，全面采用 RHI 渲染层。
- **扩展：** 移除了 QtScript，推荐 QJSEngine/QML，增强多媒体和图形性能。

------

**48. Qt 热更新实现？**

- **简答：** QML 动态加载或插件机制。
- **扩展：** QML 支持运行时重新加载 QML 文件，常用于 OTA 更新。

------

**49. Qt 多语言支持？**

- **简答：** tr() → TS → QM → QTranslator。
- **扩展：** 动态切换语言时需调用 retranslateUi()。

------

**50. Qt Web 交互？**

- **简答：** QWebEngineView、QNetworkAccessManager。
- **扩展：** QWebEngineView 基于 Chromium，可与 JS 交互。
