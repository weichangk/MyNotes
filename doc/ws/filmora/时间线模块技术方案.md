### 0.5 UI ↔ 数据 双向绑定关系图

```
数据层（VBL）                     UI层（QGraphicsScene）
─────────────────────────────────────────────────────────

IFFTimeline ◄──────────────────► FGraphicsTimelineScene
    │  Observer 回调                    │  操作调用
    │                                   │
IFFTrack  ◄────────────────────► FGraphicsTimelineTrackItem
    │  QMap<IFFTrack*, TrackItem*>       │
    │                                   │
IFFClip   ◄────────────────────► FGraphicsClipItem
          QMap<IFFClip*, ClipItem*>

数据变更方向：
  用户操作 → FTimelinePresenter → IFFTimeline::moveClip()
           → VBL 内部更新 → Observer::onClipChanged()
           → FGraphicsClipItem::setPos() → QGraphicsScene::update()

UI刷新方向（Observer 回调）：
  ADD    → 创建新 ClipItem 插入 Scene
  REMOVE → 从 Scene 删除对应 ClipItem
  UPDATE → 调用 ClipItem->prepareGeometryChange() + update()
  MOVE   → 调用 ClipItem->setPos(frameToScenePos(newFrame))
```

### 1.2 核心设计原则

| 原则 | 实践方式 |
|------|---------|
| 数据与视图分离 | IFF* 接口持有数据，FGraphics* 持有显示 |
| 单向数据流 | 用户操作 → Presenter → IFF* 数据 → 回调 → 图形刷新 |
| 接口隔离 | 30+ 个 `IFF*` 纯虚接口，替换底层渲染库只需重新实现接口 |
| Pimpl 隐藏实现 | `Q_DECLARE_PRIVATE` + `QScopedPointer<Private> d_ptr` |

### 3.2 坐标系设计

时间线使用两套坐标，以及一个"内容区偏移"（轨道头宽度 = 104px）：

```
屏幕像素                 Scene 坐标               帧坐标
──────────────────────────────────────────────────────────
View X = 0              Scene X = 0              (标尺起点)
View X = 104            Scene X = 104            (内容区左边界 contentLeftMargin)
View X = 104 + W        Scene X = 104 + W        Frame = pixelToDuration(W)

── 坐标转换 ──
Frame N  →  sceneX = contentLeftMargin + durationToPixel(N)
sceneX   →  Frame  = pixelToDuration(sceneX - contentLeftMargin)
```

转换方法汇总：

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `frameToScenePos(frame)` | 帧号 | Scene X | 帧 → 绝对场景坐标 |
| `scenePosToFrame(sceneX)` | Scene X | 帧号 | 场景坐标 → 帧 |
| `durationToPixel(frames)` | 帧数量 | 像素宽度 | 时长 → 像素（相对） |
| `pixelToDuration(pixels)` | 像素宽度 | 帧数量 | 像素 → 时长（相对） |
| `mapToContentPos(sceneX)` | Scene X | 内容区 X | 减去 leftMargin |
| `mapFromContentPos(contentX)` | 内容区 X | Scene X | 加上 leftMargin |

**缩放实现**：改变底层 `pixelsPerFrame` 比例因子（由 `FTimelinePresenter::setZoomLevel()` 驱动），所有图形项通过 `durationToPixel()` 重新计算宽度，实现时间线缩放。

```
缩放级别  pixelsPerFrame   1秒（30fps）宽度
─────────────────────────────────────────
最小缩放       0.5px/帧        15px
默认缩放       4px/帧         120px
最大缩放      80px/帧        2400px
```

关键转换方法：

```cpp
// 帧 → 场景像素
qreal frameToScenePos(int frameNo) const;

// 场景像素 → 帧
int scenePosToFrame(qreal sceneX) const;

// 持续时间（帧数） → 像素宽度
qreal durationToPixel(int durationFrame) const;

// 像素宽度 → 持续时间（帧数）
int pixelToDuration(qreal pixels) const;
```

**缩放实现**：改变 `pixelsPerFrame` 比例因子，所有图形项根据此值动态计算宽度，实现时间线缩放效果。

## 四、Substitute（替身/幽灵）拖拽模式 ⭐（核心设计）

这是时间线模块最核心的技术设计，解决了"拖拽预览不修改数据"的难题。

### 4.1 问题背景

用户拖拽 Clip 时，需要：
1. **实时视觉反馈**：显示 Clip 将要放置的目标位置
2. **数据零污染**：拖拽过程不触发数据层变更（避免大量回调）
3. **松手后原子提交**：鼠标释放时一次性提交最终位置

如果直接修改 `IFFClip` 数据，会触发 VBL 层回调链，导致：
- 每帧拖动触发一次数据变更 → UI 重建 → 严重卡顿
- 无法轻松实现"取消（ESC 键）"功能

### 4.2 Substitute 设计方案

#### 状态机图

```
                    ┌─────────────────────────────────────────────────────┐
                    │             Substitute 拖拽状态机                    │
                    └─────────────────────────────────────────────────────┘

   ┌──────────┐  鼠标按下(>阈值)   ┌──────────────────┐
   │  IDLE    │ ──────────────────► │   SUBSTITUTING   │
   │（静止）  │                     │   （拖拽中）      │
   └──────────┘                     └──────────────────┘
        ▲                              │           │
        │         ESC键 / 鼠标移出     │           │  鼠标释放
        │◄─────────────────────────────┘           │
        │           CANCEL                         ▼
        │                                   ┌──────────────┐
        │◄──────────────────────────────────│   COMMITTING │
        │           COMMITTED               │  （提交中）   │
        └──────────────────────────────────►└──────────────┘
```

#### 详细流程图

```
鼠标按下（mousePress on ClipItem）
    │
    ▼

    │  判断拖拽意图（移动 or 裁剪）
    ▼

    │
    ├─► FGraphicsItemSubstituteBuilder
    │       .setSourceClipItem(clipItem)
    │       .setAllowCrossTrack(true)
    │       .setSnapEnabled(true)
    │       .build()
    │       ─► 创建 FGraphicsClipItemSubstitute（Ghost Item）
    │              - 复制原 ClipItem 的外观（缩略图/名称/颜色）
    │              - 原 ClipItem 设为半透明（alpha=128）
    │              - 在 Scene 中置于最顶层（setZValue）
    │
    ├─► 构建吸附目标帧列表（snapFrameMap）
    │       遍历所有 ClipItem 的入点/出点
    │       + 播放头位置 + 标记点位置
    │
    └─► 启动 FGraphicsClipItemSubstituteMoveHandler

鼠标移动（mouseMove）
    │
    ▼
Handler::onMouseMove(scenePos)
    │
    ├─► 计算目标帧：
    │       rawFrame = scenePosToFrame(scenePos.x - grabOffset)
    │
    ├─► 磁性吸附修正：
    │       SnapService::calcAutoSnapOffset(snapFrameMap, maxOffset, Δframe)
    │       targetFrame = rawFrame + Δframe
    │
    ├─► 垂直轨道联动（如有）：
    │       VTrackLinkHandler::syncLinkedSubstitutes(Δframe)
    │
    └─► 更新 Ghost Item 位置：

    │
    ▼

    │
    ├─► 读取 substitute->currentRange()
    ├─► IFFTimeline::moveClip(clip, targetTrack, targetFrame)
    │       ─► VBL 内部更新数据
    │       ─► Observer 回调 onClipChanged(MOVE)
    │       ─► ClipItem 位置更新（原 ClipItem 恢复不透明）
    │
    ├─► IFFUndoRedoService 记录操作（支持 Ctrl+Z 撤销）
    │
    └─► 销毁 substitute，清空 snapFrameMap，重置 Handler

ESC 键（cancel）
    │
    ▼
Handler::cancel()
    ├─► 销毁 substitute（不提交任何数据操作）
    └─► 原 ClipItem 恢复不透明（回到原位）
```

## 五、Action Layer 事件分层系统

### 5.1 设计动机

时间线场景中，鼠标事件需要被多个系统同时处理（选中、拖拽、框选、关键帧操作等），如果写在 `QGraphicsScene::mousePressEvent()` 中，代码会极度膨胀。

**解决方案**：责任链模式（Chain of Responsibility）——事件按优先级依次经过各 Layer，由最合适的 Layer 处理。

### 5.2 事件路由全流程图

```
用户鼠标点击/移动
       │
       ▼
Qt 事件系统
       │
       ▼
FGraphicsTimelineView::viewportEvent()
       │  重写 QGraphicsView 事件入口
       ▼
FGraphicsTimelineScene::mousePressEvent()
  (继承 QGraphicsScene，重写事件方法)
       │
       ▼
FGraphicsTimelineSceneActionLayerGroup::dispatchMousePressEvent(event)
       │
       │  按 Priority 从小到大依次询问：
       │
       ├──[P10] FGraphicsClipMoveLayer::mousePressEvent(event)
       │           │  命中 ClipItem 且可拖拽？
       │           ├── YES → 处理，return true ──→ 事件消费，停止传递
       │           └── NO  → return false ──────→ 继续向下
       │
       ├──[P20] FGraphicsClipSelectionLayer::mousePressEvent(event)
       │           │  点击 ClipItem 选中？
       │           ├── YES → 处理，return true
       │           └── NO  → return false
       │
       ├──[P30] FGraphicsClipTrimLayer::mousePressEvent(event)
       │           │  命中 TrimIn/TrimOut 热区？
       │           ├── YES → 处理，return true
       │           └── NO  → return false
       │
       ├──[P40] FGraphicsKeyFrameMoveLayer::mousePressEvent(event)
       │           │  命中关键帧菱形？
       │           └── ...
       │
       ├──[P50..P100] 其他 Layer...
       │
       └──[NONE] 所有 Layer 都返回 false
                   → 触发框选（RectSelectService）
                   → 取消所有选中状态
```

## 六、磁性吸附（Snap）服务

### 6.1 吸附原理

拖拽 Clip 时，当 Clip 的边缘接近其他 Clip 的边缘、播放头位置或标记点时，自动"吸附"对齐。

### 6.2 核心算法

```cpp
class FGraphicsTimelineSceneSnapService {
    // 核心方法：计算吸附偏移量
    // snapFrameMap: 当前所有吸附目标帧（key=帧号, value=吸附强度）
    // maxOffset: 最大吸附距离（帧数，对应像素距离 ÷ pixelsPerFrame）
    // offset: 当前拖拽偏移量（输出：修正后的偏移量）
    bool calcAutoSnapOffset(
        const QMap<int, int>& snapFrameMap,
        int maxOffset,
        int& offset
    );

    // 拖拽过程中实时更新吸附线（供视觉显示）
    void update(const QList<int>& snapPosList);

    // 清空吸附线（拖拽结束后调用）
    void clear();
};
```

### 6.3 吸附示意图

```
时间线（帧坐标）：
────────────────────────────────────────────────────────────────────
 0        30        60        90        120       150
 │        │         │         │         │         │
 [  ClipA  ]        [  ClipB  ]         [  ClipC  ]
            ↑out=30           ↑out=90   ↑in=120

正在拖拽 ClipX（宽度=40帧），鼠标释放时 rawTargetFrame = 85：

 rawFrame=85（ClipX 的入点）
 snapFrameMap 中最近的目标：90（ClipB 的出点），距离 = 5帧
 maxOffset = 8帧（约 15px，小于阈值）
 → calcAutoSnapOffset 返回 Δ = +5
 → targetFrame = 85 + 5 = 90   ← 自动吸附到 ClipB 出点！

 结果：
 [  ClipA  ]        [  ClipB  ][ClipX]  [  ClipC  ]
            ↑30               ↑90↑130   ↑120
                               ✨吸附对齐

吸附可视化（黄色竖线）：
 ─────────────────[│]──────────────────────────────
                   90  ← SnapService::update([90]) 在此位置绘制竖线
```

### 6.4 吸附目标来源

拖拽时，以下位置会成为吸附目标：
1. **其他 Clip 的入点/出点**（最常用）
2. **播放头位置**
3. **标记点（Marker）位置**
4. **序列入点/出点**

```cpp
// 构建吸附目标帧列表
void buildSnapFrameMap(QMap<int, int>& snapFrameMap) {
    // 1. 遍历所有 TrackItem 上的 ClipItem
    for (auto trackItem : allTrackItems) {
        for (auto clipItem : trackItem->getAllClipItem()) {
            if (clipItem == m_pDraggingClip) continue; // 跳过正在拖拽的
            snapFrameMap[clipItem->startFrame()] = DEFAULT_SNAP_STRENGTH;
            snapFrameMap[clipItem->endFrame()]   = DEFAULT_SNAP_STRENGTH;
        }
    }
    // 2. 播放头
    snapFrameMap[scene->playheadFrame()] = HIGH_SNAP_STRENGTH;
    // 3. 标记点
    for (auto marker : timeline->markers()) {
        snapFrameMap[marker->frameNo()] = DEFAULT_SNAP_STRENGTH;
    }
}
```

## 十一、数据与视图同步机制

### 11.1 数据变更 → 视图更新

VBL 层数据变更时，通过 Observer 模式通知 `FGraphicsTimelineScene`：

```
IFFTimeline（数据变更）
    │ Observer 回调
    ▼
FTimelinePresenter::onClipChanged(IFFClip*, FFDataChangedMode mode)
    │
    ├── mode = ADD    → 创建新的 FGraphicsClipItem 并加入 Scene
    ├── mode = REMOVE → 从 Scene 移除对应 ClipItem
    ├── mode = UPDATE → 调用 ClipItem->update()，触发重绘
    └── mode = MOVE   → 更新 ClipItem 的场景坐标（setPos）
```

### 11.2 FFDataChangedMode 枚举

```cpp
enum FFDataChangedMode {
    ADD,        // 新增
    REMOVE,     // 删除
    UPDATE,     // 属性变化（如持续时间、位置）
    MOVE,       // 位置变化
    BATCH,      // 批量变化（暂停事件后的统一通知）
};
```

### 11.3 暂停事件机制（批量更新优化）

多选操作（如删除 10 个 Clip）时：
```cpp
// 1. 暂停 VBL 事件广播（所有变更积压）
FF_DECLARE_PAUSE_EVENT_COMMANDER(undoRedoService);

// 2. 执行多个数据操作
for (auto clip : selectedClips) {
    timeline->removeClip(clip);  // 不触发回调
}

// 3. 作用域结束，恢复广播
//    一次性发出 BATCH 事件 → Scene 一次性重建所有变化
```

**效果**：N 个 Clip 操作只触发 1 次 UI 重建，而非 N 次。

## 十四、面试高频问题 Q&A

### Q1：时间线模块为什么用 QGraphicsView 而不是自定义 Widget？

**答**：QGraphicsView 框架提供了：
1. **层级管理**：`QGraphicsItemGroup` 自然映射"轨道-Clip"层次，无需手动管理 Z 轴
2. **局部重绘**：`QGraphicsScene::update(rect)` 只重绘脏区域，比 `QWidget::update()` 精确
3. **碰撞检测**：`QGraphicsScene::items(rect)` 快速查找指定区域的图形项（用于框选功能）
4. **坐标变换**：ViewportTransformation 简化了缩放/平移的实现
5. **鼠标事件分发**：QGraphicsScene 自动将鼠标事件转发给最上层的 Item，减少手写碰撞检测

**代价**：QGraphicsView 默认的 `boundingRect` 机制在大量 Item 时有性能开销，需要通过 `setFlag(ItemDoesntPropagateOpacityToChildren)` 等手段优化。

---

### Q2：Substitute（替身）拖拽模式的核心价值是什么？

**答**：核心价值有三点：

1. **数据零污染**：拖拽过程不修改 `IFFClip` 数据，VBL 层不触发任何回调，避免 UI 重建循环。

2. **支持取消操作**：用户按 ESC 键时，只需销毁 SubstituteItem，原 ClipItem 恢复显示，无需执行任何撤销操作。

3. **跨轨道预览**：Substitute 可以在任意轨道显示，而原 Clip 仍在原轨道，让用户直观看到"目标位置"的效果。

对比方案（直接修改数据）：
- 每帧拖动触发数据变更回调 → Observer 通知 → 图形项 setPos → 场景重布局 → 性能灾难
- 无法轻松取消（需要实现拖拽专用的撤销逻辑）

---

### Q3：如何保证多线程安全（缩略图异步加载 + UI 更新）？

**答**：
1. **缩略图生成在后台线程**：`IFFClipVideoThumbnailService` 使用 `QThreadPool` 异步解码帧图像
2. **结果回调到主线程**：通过 Qt 的跨线程信号（`Qt::QueuedConnection`）将解码结果投递到主线程
3. **主线程更新 ClipItem**：收到信号后，调用 `clipItem->updateThumbnail(QPixmap)` 并触发 `update()`
4. **生命周期安全**：使用 `QPointer<FGraphicsClipItem>` 持有 ClipItem 弱引用，回调时检测 `!ptr.isNull()` 防止野指针

```cpp
// 跨线程安全示例
connect(thumbnailService, &IFFClipVideoThumbnailService::sigThumbnailReady,
        this, [this, weakClipItem = QPointer<FGraphicsClipItem>(clipItem)](QPixmap px) {
    if (!weakClipItem) return;  // ClipItem 已销毁，安全跳过
    weakClipItem->setThumbnail(px);
    weakClipItem->update();
}, Qt::QueuedConnection);  // 确保在主线程执行
```

### Q4：IFF* 接口设计有什么好处？

**答**：`IFF*` 是纯虚接口（类似 COM 接口），好处：

1. **跨平台**：Windows 用 DirectX 渲染，macOS 用 Metal 渲染，上层 UI 代码完全一致，只有 VBL 实现层不同。

2. **可测试性**：单元测试中可以 Mock `IFFTimeline`，不需要真实渲染环境。

3. **版本兼容**：VBL 库升级时，只要接口不变，上层代码零修改。

4. **依赖倒置**：`FTimelinePresenter` 依赖 `IFFTimeline` 抽象，而非具体实现类，符合 DIP 原则。

**缺点**：
- 纯虚接口调用有虚表开销（在时间线高频路径中需注意）
- 调试时调用栈复杂（需穿透多层接口）

### Q5：FTimelineModulePresenter 为什么管理多个 FTimelinePresenter？

**答**：Filmora 中存在多个"时间线上下文"：

| 上下文 | 时间线类型 | 说明 |
|--------|-----------|------|
| 主编辑 | 主时间线 | 完整功能，多轨道 |
| AI封面 | 封面时间线 | 简化版，仅选帧 |
| 字幕编辑 | 字幕时间线 | 只有字幕轨 |
| 模板编辑 | 模板时间线 | 固定结构，有限编辑 |

每个上下文需要独立的 `FTimelinePresenter`（独立状态、独立撤销历史、独立 Scene）。`FTimelineModulePresenter` 负责：
- 切换当前活跃的 Presenter（`switchPresenter(type)`）
- 协调多 Presenter 间的数据共享（如素材库引用）

### Q6：轨道高度变化（展开关键帧面板）如何高效处理？

**答**：使用"缓存 + 失效"策略：

1. **初始状态**：`FGraphicsTrackHeightCache` 存储轨道高度，有效。
2. **触发变化**：用户展开关键帧面板 → `FGraphicsKeyFramePanelItem` 展开 → 调用 `m_pHeightCache->invalidate()`。
3. **惰性重算**：下次访问 `m_pHeightCache->height()` 时才重新计算（包含基础高度 + 关键帧面板高度）。
4. **布局传播**：轨道高度变化后，所有轨道的 Y 坐标需要重新排布，Scene 调用 `relayoutTracks()` 统一处理。

Pimpl 模式用于隐藏缓存实现细节：
```cpp
class FGraphicsTrackHeightCache : public QObject {
    int  height() const;
    void invalidate();
private:
    QScopedPointer<FGraphicsTrackHeightCachePrivate> d;  // Pimpl
};
```

### Q7：框选多个 Clip 的实现思路？

**答**：由 `FGraphicsTimelineSceneRectSelectService` 实现：

1. **鼠标按下（空白区域）**：记录起始点，创建半透明矩形遮罩（`QGraphicsRectItem`）
2. **鼠标移动**：更新矩形大小，调用 `QGraphicsScene::items(rect, Qt::IntersectsItemShape)` 获取矩形内所有图形项
3. **过滤**：只保留 `FGraphicsAbstractClipItem` 类型（Clip/Transition），过滤轨道头、播放头等
4. **高亮选中**：调用 `clipItem->setChecked(true)` 高亮
5. **鼠标释放**：销毁矩形遮罩，将选中结果通过信号通知 `FTimelinePresenter`，进而更新属性面板
