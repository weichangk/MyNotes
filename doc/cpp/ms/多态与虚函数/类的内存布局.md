C++ 类的内存布局（Class Memory Layout）本质上取决于 **成员变量、继承关系、虚函数机制以及对齐规则**。面试中通常重点考这几块。

------

# 一、普通类的内存布局（无继承 / 无虚函数）

```
class A {
    char a;
    int b;
    double c;
};
```

### 内存特点

1. **成员按声明顺序存储**
2. **存在字节对齐（alignment）**
3. **可能有 padding（填充字节）**

### 示例布局（典型 64-bit）：

```
| a (1B) |
| padding (3B) |
| b (4B) |
| c (8B) |
```

👉 `sizeof(A)` 通常是 16 或 24（取决于对齐策略）

------

# 二、有虚函数的类（核心重点）

```
class A {
public:
    virtual void func();
    int x;
};
```

### 内存布局：

对象中会多一个 **虚表指针 vptr**

```
| vptr (8B) |   -> 指向 vtable
| x    (4B) |
| padding   |
```

### 关键点

- 每个对象都有一个隐藏的 `vptr`
- `vptr` 指向类的 **vtable（虚函数表）**
- 虚函数通过 vtable 间接调用

------

# 三、继承关系的内存布局

## 1. 单继承（无虚函数）

```
class A { int a; };
class B : public A { int b; };
```

布局：

```
| A::a |
| B::b |
```

👉 子类 = 父类 + 自己的成员

------

## 2. 单继承 + 虚函数

```
class A {
    virtual void f();
    int a;
};

class B : public A {
    int b;
};
```

布局：

```
| vptr |
| A::a |
| B::b |
```

👉 vptr 只会存在一份（在最顶层对象）

------

## 3. 多继承（重点难点）

```
class A { virtual void fa(); int a; };
class B { virtual void fb(); int b; };

class C : public A, public B {
    int c;
};
```

布局：

```
| A subobject |
| vptr_A      |
| a           |

| B subobject |
| vptr_B      |
| b           |

| c           |
```

### 关键点：

- C 对象里有 **多个 vptr**
- 每个基类子对象都有自己的布局
- this 指针在不同基类视角会发生偏移

------

# 四、虚继承（最复杂）

```
class A { int a; };
class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};
```

### 特点：

为了解决菱形继承问题：

👉 A 只保留 **一份共享实例**

布局大致：

```
| B subobject |
| C subobject |
| D own data  |
| shared A    |  <-- 虚基类表定位
```

### 引入：

- vbptr（virtual base pointer）
- vbtable（虚基类表）

👉 访问虚基类需要运行时定位

------

# 五、对象内存布局总结（面试一句话版）

可以这样回答：

> C++ 类的内存布局由成员变量、对齐规则、继承结构以及是否存在虚函数决定。
>  有虚函数时对象中会增加 vptr 指向 vtable；单继承是线性布局；多继承会出现多个子对象和多个 vptr；虚继承通过 vbptr 和 vbtable 解决菱形继承的共享问题。

# 六、面试高频追问点

## 1. 为什么要对齐？

- CPU 访问效率
- 硬件要求

## 2. vptr 什么时候初始化？

- 构造函数执行时

## 3. 虚函数表在哪里？

- 编译期生成，存放在只读数据区

## 4. 空类大小？

```
class A {};
sizeof(A) == 1;
```

👉 保证对象唯一地址
