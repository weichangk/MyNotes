在 C++ 里，**父类（基类）和子类（派生类）的构造与析构顺序是固定且对称的**，这是面试高频点。

------

# 一、构造函数调用顺序（创建对象时）

当创建一个派生类对象时：

## 👉 执行顺序

**1️⃣ 先构造父类（基类）**
 **2️⃣ 再构造子类（派生类）**

### 示例

```
#include <iostream>
using namespace std;

class Base {
public:
    Base() {
        cout << "Base constructor" << endl;
    }
};

class Derived : public Base {
public:
    Derived() {
        cout << "Derived constructor" << endl;
    }
};
```

### 创建对象

```
Derived d;
```

### 输出

```
Base constructor
Derived constructor
```

------

## 👉 为什么必须先构造父类？

因为：

- 子类可能依赖父类成员
- 如果父类没初始化，子类使用父类数据会出错
- C++ 保证对象“从底层到上层”逐步构建完整结构

------

# 二、析构函数调用顺序（销毁对象时）

当对象生命周期结束：

## 👉 执行顺序（与构造相反）

**1️⃣ 先析构子类（派生类）**
 **2️⃣ 再析构父类（基类）**

### 示例

```
class Base {
public:
    ~Base() {
        cout << "Base destructor" << endl;
    }
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Derived destructor" << endl;
    }
};
```

### 输出

```
Derived destructor
Base destructor
```

------

## 👉 为什么析构是反过来的？

因为：

- 子类可能使用父类资源
- 必须先释放子类自己的资源
- 再释放父类资源，避免“悬空依赖”

------

# 三、完整生命周期顺序总结

```
构造：
Base -> Derived

析构：
Derived -> Base
```

------

# 四、重点面试加分点（非常重要）

## 1️⃣ 如果基类析构函数不是 virtual 会怎样？

```
Base* p = new Derived();
delete p;
```

### ❌ 可能问题

- 只调用 Base 析构
- Derived 资源泄漏
- 未定义行为（UB）

### ✔ 正确写法

```
virtual ~Base() {}
```

------

## 2️⃣ 构造函数不能是虚函数

原因：

- 虚函数依赖 vtable
- 而 vtable 在构造完成后才完整建立
- 构造阶段对象类型不完整

------

## 3️⃣ 构造初始化列表顺序（常考坑点）

```
class Derived : public Base {
    Member a;
public:
    Derived() : a(), Base() {}  // ❌ 写顺序没用
};
```

### 真正顺序是固定的：

```
Base（基类）
Member（成员变量按声明顺序）
Derived（自身构造函数体）
```

👉 **与初始化列表书写顺序无关！**

------

# 五、一句话面试回答版本（可背）

> C++ 中构造顺序是先构造基类，再构造派生类；析构顺序相反，先析构派生类再析构基类。这样设计是为了保证资源依赖关系的正确性。如果基类用于多态，析构函数必须是 virtual，否则可能导致派生类资源无法正确释放。
