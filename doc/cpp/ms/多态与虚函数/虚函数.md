# 什么是虚函数（Virtual Function）

虚函数是 C++ 用于实现**运行时多态（Runtime Polymorphism）**的机制。

在基类中使用 `virtual` 修饰函数后，子类可以重写（override）该函数，通过**基类指针/引用调用时，会动态决定真正执行哪个函数。**

# 底层原理：虚表（vtable）

当类中存在虚函数时，编译器会自动生成：

## 1. 虚函数表（vtable）

本质：

```
函数指针数组
```

里面存放该类所有虚函数的地址。

## 2. 虚表指针（vptr）

每个对象内部会隐藏一个：

```
vfptr / vptr
```

指向当前类对应的虚表。

对象内存结构类似：

```
class Dog
{
    vptr ----> Dog vtable
}
```

# 虚函数调用过程（重点）

代码：

```
Animal* a = new Dog();
a->speak();
```

实际过程：

```
a
 ↓
对象中的 vptr
 ↓
Dog vtable
 ↓
找到 Dog::speak 地址
 ↓
调用函数
```

因此：

## 虚函数调用是运行时决定的

这就是：

```
动态绑定（Dynamic Binding）
```

# 为什么需要虚函数？

核心目的：

## 实现多态

例如 Qt 中：

```
QWidget* w = new QPushButton;
w->show();
```

不同控件调用不同实现。

## 1. 构造函数不能是虚函数

因为对象构造期间：

```
vptr 还没完全指向子类虚表
```

多态机制不完整。

## 2. 析构函数通常必须是虚函数（高频面试点）

错误：

```
class Base {
public:
    ~Base() {}
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "release" << endl;
    }
};

Base* p = new Derived;
delete p; // 未定义行为
```

只调用 Base 析构。

正确：

```
virtual ~Base() {}
```

> **为了保证“通过父类指针删除子类对象”时，子类析构函数能够正确执行。**

## 3. static 函数不能是虚函数

因为：

```
static 属于类
virtual 属于对象
```

static 没有 this 指针。

## 4. 内联函数可以是虚函数

但：

```
virtual 调用通常无法内联
```

因为运行时才知道调用哪个函数。

# 纯虚函数（Pure Virtual）

```
virtual void func() = 0;
```

表示：

```
基类不给实现
强制子类实现
```

含有纯虚函数的类：

```
抽象类（Abstract Class）
```

不能实例化。

# 多继承中的虚表（高级）

多继承时：

```
class C : public A, public B
```

可能存在：

```
多个 vptr
多个 vtable
```

因为需要支持不同父类的多态。

# Qt 中的虚函数应用（面试加分）

Qt 大量使用虚函数实现框架扩展：

例如：

```
paintEvent()
mousePressEvent()
event()
```

本质：

```
框架调用父类接口
子类重写实现行为
```

这是典型的：

```
模板方法模式 + 多态
```

# 面试简答版

虚函数是 C++ 实现运行时多态的机制。
 类中存在 virtual 函数时，编译器会生成虚函数表 vtable，对象内部保存虚表指针 vptr。

通过基类指针调用虚函数时，会先通过 vptr 找到虚表，再找到真正函数地址，因此函数调用在运行时决定，这就是动态绑定。

虚函数主要用于接口抽象和框架扩展，例如 Qt 的事件系统。
