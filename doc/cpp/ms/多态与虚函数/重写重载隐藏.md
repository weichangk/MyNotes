在 C++ 里，“重写（override）”、“重载（overload）”和“隐藏（hide）”是三种非常容易混淆但面试高频的概念，本质分别对应**多态、同名函数决议、作用域遮蔽**。

------

# 1. 重载（Overload）——同一作用域内“同名不同参”

## ✅ 定义

在**同一个作用域**中，函数名相同，但**参数列表不同（参数个数/类型/顺序）**的一组函数。

## ❗ 特点

- 发生在**同一个类/作用域**
- 与返回值无关
- 编译期确定（静态多态）

## 📌 示例

```
class A {
public:
    void func(int a) {}
    void func(double a) {}
    void func(int a, int b) {}
};
```

## ✔ 本质

👉 编译器根据参数选择函数（函数名修饰 name mangling）

------

# 2. 重写（Override）——子类改写父类虚函数

## ✅ 定义

子类中定义一个与父类**完全一致签名的虚函数**，用于实现运行时多态。

## ❗ 条件（必须满足）

- 父类函数必须是 `virtual`
- 函数签名必须完全一致
- 发生在父子类之间

## 📌 示例

```
class Base {
public:
    virtual void func(int a) {
        cout << "Base\n";
    }
};

class Derived : public Base {
public:
    void func(int a) override {
        cout << "Derived\n";
    }
};
```

## ✔ 特点

- 运行时多态（动态绑定）
- 通过虚函数表（vtable）实现
- 可用 `override` 关键字增强可读性

------

# 3. 隐藏（Hide / Name Hiding）——同名函数“屏蔽父类版本”

## ✅ 定义

子类中出现与父类**同名的函数（不要求参数一致）**，会导致父类所有同名函数被隐藏。

## ❗ 关键点（面试重点）

👉 只要“名字相同”，父类所有重载版本都会被隐藏！

------

## 📌 示例（最经典陷阱）

```
class Base {
public:
    void func(int a) {}
    void func(double a) {}
};

class Derived : public Base {
public:
    void func(int a) {}  // 隐藏 Base::func
};
```

### 调用结果：

```
Derived d;
d.func(1);     // 调用 Derived::func
d.func(1.0);   // ❌ 仍然不会调用 Base::func（被隐藏）
```

------

## ✔ 如何访问父类函数？

```
d.Base::func(1.0);
```

------

## ✔ 如何“恢复重载集”？

```
class Derived : public Base {
public:
    using Base::func;  // 引入父类所有重载
    void func(int a) {}
};
```

------

# 4. 三者核心区别（面试必背）

| 项目             | 重载 overload | 重写 override    | 隐藏 hide            |
| ---------------- | ------------- | ---------------- | -------------------- |
| 作用域           | 同一作用域    | 父子类           | 父子类               |
| 参数要求         | 必须不同      | 必须相同         | 不要求               |
| virtual要求      | 不需要        | 必须             | 不需要               |
| 绑定时间         | 编译期        | 运行期           | 编译期               |
| 多态性           | 无            | 有（运行时多态） | 无                   |
| 是否影响父类函数 | 不影响        | 替换实现         | **屏蔽父类同名函数** |

------

# 5. 一句话总结（面试杀手级）

- **重载**：同名不同参，编译期选择
- **重写**：子类改父类 virtual，运行期多态
- **隐藏**：子类同名函数，直接把父类同名函数“盖住”
