在 C/C++ 面试中，“**inline 函数 vs 宏（macro）**”是高频基础题，本质是在对比：**预处理替换 vs 编译期优化函数**。

------

# 一、宏（#define）

## 1. 本质

宏是**预处理器做的文本替换**，发生在编译之前。

```
#define SQUARE(x) x * x
```

调用：

```
int a = SQUARE(3 + 2);
```

预处理展开：

```
int a = 3 + 2 * 3 + 2;  // ❌ 结果错误
```

👉 因为只是“字符串替换”，没有类型/语法保护。

------

## 2. 宏的特点

### 优点

- 无函数调用开销
- 可用于条件编译（#ifdef）
- 可跨类型使用

### 缺点（重点面试）

- ❌ 无类型检查
- ❌ 容易产生副作用
- ❌ 运算优先级问题
- ❌ 调试困难（断点进不去）

------

# 二、inline 函数

## 1. 本质

inline 是**建议编译器在调用点展开函数代码**（注意：只是“建议”）。

```
inline int square(int x) {
    return x * x;
}
```

------

## 2. 使用效果

```
int a = square(3 + 2);
```

展开后类似：

```
int a = (3 + 2) * (3 + 2);  // 正确
```

👉 保留函数语义 + 避免调用开销

------

## 3. inline 特点

### 优点

- ✔ 有类型检查
- ✔ 作用域安全（属于函数）
- ✔ 调试友好
- ✔ 无宏副作用问题
- ✔ 编译器可优化为内联

### 缺点

- ❌ 不保证一定内联（只是建议）
- ❌ 代码膨胀（频繁展开）

------

# 三、核心对比（面试重点）

| 对比项       | 宏（#define） | inline 函数 |
| ------------ | ------------- | ----------- |
| 处理阶段     | 预处理        | 编译阶段    |
| 是否类型安全 | ❌ 否          | ✔ 是        |
| 是否调试友好 | ❌ 否          | ✔ 是        |
| 是否函数语义 | ❌ 否          | ✔ 是        |
| 参数求值     | 多次展开      | 一次计算    |
| 是否一定内联 | 强制替换      | 可能        |

------

# 四、最经典坑（必会）

## 宏的副作用

```
#define ADD(x) x + x
int a = ADD(3 * 2);
```

展开：

```
3 * 2 + 3 * 2 = 12 ❌（可能误解）
```

------

## inline 正确行为

```
inline int add(int x) {
    return x + x;
}

int a = add(3 * 2);  // 12 ✔
```

------

# 五、inline 也可能不内联

编译器可能拒绝 inline：

- 函数过大
- 存在递归
- 虚函数调用
- 编译优化策略不同（O0 / O2）

👉 inline = “优化建议”，不是强制

------

# 六、现代 C++ 建议

## ❌ 不推荐宏做函数

```
#define MAX(a,b) ((a) > (b) ? (a) : (b))
```

## ✔ 推荐

```
inline int max(int a, int b) {
    return a > b ? a : b;
}
```

或

```
template<typename T>
inline T max(T a, T b) {
    return a > b ? a : b;
}
```

------

# 七、面试一句话总结（重点背）

> 宏是预处理文本替换，没有类型检查，容易产生副作用；inline 是编译期函数优化，保留函数语义并具备类型安全，是否内联由编译器决定。
