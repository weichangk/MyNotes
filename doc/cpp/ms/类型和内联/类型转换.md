C++ 中的**类型转换（Type Conversion）\**主要分为\**隐式转换**和**显式转换（强制类型转换）**两大类，是面试高频点。

------

# 一、隐式类型转换（Implicit Conversion）

由编译器自动完成，不需要写任何转换语句。

## 1. 常见情况

### （1）基本类型提升

```
int a = 10;
double b = a;   // int → double
```

### （2）算术转换

```
int a = 10;
double b = 3.14;
double c = a + b; // a 自动转 double
```

### （3）派生类 → 基类（向上转型，安全）

```
class Base {};
class Derived : public Base {};

Derived d;
Base* b = &d;   // 自动转换（安全）
```

### （4）数组 → 指针

```
int arr[10];
int* p = arr; // 自动退化
```

------

## 2. 隐式转换的风险

- 可能丢精度（int → float）
- 可能产生意外行为（unsigned / signed 混算）
- 函数重载匹配复杂化

------

# 二、显式类型转换（C++ 强制类型转换）

C++ 提供 4 种标准 cast，比 C 风格更安全、更可控。

------

# 1. static_cast（最常用）

## 特点：

- 编译期转换
- 用于“逻辑上可转换”的类型
- 无运行时检查

## 用途：

### 基本类型转换

```
double d = 3.14;
int i = static_cast<int>(d);
```

### 基类/派生类转换（向上安全，向下不安全）

```
Base* b = new Derived();
Derived* d = static_cast<Derived*>(b); // 不安全
```

------

# 2. dynamic_cast（运行时类型检查）

## 特点：

- 依赖 RTTI
- 只能用于**多态类型（含虚函数）**
- 安全的向下转型

## 示例：

```
class Base {
public:
    virtual ~Base() {}
};

class Derived : public Base {};

Base* b = new Derived();

Derived* d = dynamic_cast<Derived*>(b);
```

## 失败返回：

- 指针：返回 nullptr
- 引用：抛异常

------

## 使用场景：

- 基类指针转派生类
- 插件系统 / Qt QObject体系中常见

------

# 3. const_cast（去/加 const）

## 特点：

- 只能改变 const 属性
- 不能改变真实类型

## 示例：

```
const int a = 10;

int* p = const_cast<int*>(&a);
```

## ⚠️危险点：

- 如果原对象本身是 const，修改它是未定义行为

------

## 使用场景：

- 接口兼容
- 老代码适配

------

# 4. reinterpret_cast（最危险）

## 特点：

- 纯“按位重新解释”
- 不做任何类型检查
- 可跨类型乱转

## 示例：

```
int a = 10;
char* p = reinterpret_cast<char*>(&a);
```

------

## 使用场景：

- 底层网络协议解析
- 内存映射
- 句柄转换

------

## ⚠️风险：

- 可移植性差
- 非常容易出 bug

------

# 三、C 风格转换（不推荐）

```
(int)3.14
int i = (int)3.14;
```

## 问题：

- 不明确（可能是 static / const / reinterpret 混合行为）
- 不利于代码审查
- 容易误用

------

# 四、四种 C++ cast 对比（面试重点）

| 类型             | 安全性 | 用途         | 是否运行时检查 |
| ---------------- | ------ | ------------ | -------------- |
| static_cast      | 中等   | 常规转换     | 否             |
| dynamic_cast     | 高     | 多态向下转型 | 是             |
| const_cast       | 低     | const 修饰   | 否             |
| reinterpret_cast | 很低   | 底层重解释   | 否             |

------

# 五、面试加分总结

可以这样总结一句话：

> C++ 类型转换分为隐式转换和显式转换。显式转换推荐使用 C++ 的四种 cast，其中 static_cast 用于常规类型转换，dynamic_cast 用于多态安全向下转型，const_cast 用于修改常量性，reinterpret_cast 用于底层二进制重解释。
