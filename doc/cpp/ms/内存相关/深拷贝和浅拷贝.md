在 C++（以及 Qt 面试）里，“深拷贝 vs 浅拷贝”本质是在回答一个问题：**拷贝对象时，要不要连“指针指向的资源”一起复制一份？**

------

# 一、浅拷贝（Shallow Copy）

## 1. 定义

浅拷贝只是**按位复制成员变量**，包括指针本身，但**不复制指针指向的内容**。

👉 结果：两个对象共享同一块内存资源。

------

## 2. 示例

```
class A {
public:
    int* data;

    A(int value) {
        data = new int(value);
    }

    // 默认拷贝构造（浅拷贝）
    // A(const A& other) = default;
};
```

使用：

```
A a1(10);
A a2 = a1;   // 浅拷贝

*a2.data = 20;
```

### 结果：

```
a1.data 和 a2.data 指向同一块内存
```

------

## 3. 问题

### ❌ 1. 双重释放（double free）

```
~A() {
    delete data;
}
```

a1 和 a2 析构时都会 `delete data` → 崩溃

------

### ❌ 2. 资源共享导致互相影响

修改一个对象影响另一个对象

------

# 二、深拷贝（Deep Copy）

## 1. 定义

深拷贝不仅复制对象本身，还会**重新分配内存，并复制指针指向的数据**。

👉 结果：两个对象拥有**独立资源**

------

## 2. 示例

```
class A {
public:
    int* data;

    A(int value) {
        data = new int(value);
    }

    // 深拷贝构造函数
    A(const A& other) {
        data = new int(*other.data);
    }

    A& operator=(const A& other) {
        if (this != &other) {
            delete data;
            data = new int(*other.data);
        }
        return *this;
    }

    ~A() {
        delete data;
    }
};
```

------

## 3. 结果

```
A a1(10);
A a2 = a1;

*a2.data = 20;
a1.data = 10
a2.data = 20
```

👉 完全独立

------

# 三、对比总结（面试重点）

| 对比点               | 浅拷贝     | 深拷贝             |
| -------------------- | ---------- | ------------------ |
| 是否复制指针指向内容 | ❌ 否       | ✅ 是               |
| 内存是否共享         | ✅ 是       | ❌ 否               |
| 是否安全             | ❌ 容易崩溃 | ✅ 安全             |
| 性能                 | ✅ 快       | ❌ 慢（有分配开销） |
| 默认行为             | C++默认    | 需手写             |

------

# 四、什么时候用？

## ✔ 浅拷贝适合：

- 不管理资源（纯数据结构）
- Qt 隐式共享类（如 QString / QVector）

👉 Qt 的很多类是：

> **copy-on-write（写时拷贝）= 浅拷贝 + 隐式深拷贝优化**

------

## ✔ 深拷贝适合：

- 类中有 `new / malloc` 管理资源
- 文件句柄、socket、数据库连接等
- 必须“对象独立”的场景

------

# 五、面试加分点（非常重要）

## 1. 三/五法则（Rule of Three / Five）

如果类管理资源：

必须考虑：

- 析构函数
- 拷贝构造函数
- 拷贝赋值函数

（C++11后还包括 move constructor / move assignment）

------

## 2. Qt 里的典型坑

### QObject 不能拷贝

```
QObject a;
QObject b = a; // ❌ 禁止
```

原因：

- QObject 有 parent-child 树结构
- 有 signal/slot 元对象系统
- 拷贝会破坏对象树一致性

👉 Qt 直接删除拷贝构造/赋值

------

## 3. 更现代的写法（推荐）

用智能指针避免手写深拷贝：

```
class A {
public:
    std::unique_ptr<int> data;

    A(int v) : data(std::make_unique<int>(v)) {}
};
```

👉 自动避免 double free

------

# 六、一句话总结（面试用）

> 浅拷贝只是复制对象的指针或成员值，多个对象共享同一资源；深拷贝会重新分配资源并复制内容，使每个对象拥有独立资源，从而避免悬空指针和双重释放问题。
