在 C++ 里“拷贝一段数据”通常有几种方式，取决于数据类型（普通数组 / std::vector / 自定义对象）。

------

## 1. 拷贝原始内存（C 风格）

适用于 **连续内存块（数组、结构体）**

```
#include <cstring>

int src[5] = {1, 2, 3, 4, 5};
int dst[5];

std::memcpy(dst, src, sizeof(src));
```

### 特点

- 按字节拷贝
- **速度快**
- 适用于 POD（普通数据类型）
- ❌ 不适用于含指针/构造函数的对象（可能出错）

------

## 2. C++ 标准算法拷贝（推荐）

```
#include <algorithm>

int src[5] = {1, 2, 3, 4, 5};
int dst[5];

std::copy(src, src + 5, dst);
```

### 特点

- 类型安全
- 支持迭代器（vector、list 等）
- 更符合 C++ 风格

------

## 3. vector 拷贝（最常见）

```
#include <vector>

std::vector<int> a = {1, 2, 3, 4, 5};
std::vector<int> b = a;   // 直接拷贝
```

或者：

```
std::vector<int> b(a.begin(), a.end());
```

------

## 4. 深拷贝 vs 浅拷贝（面试重点）

### 浅拷贝（默认指针复制）

```
class A {
public:
    int* data;
};
```

👉 只复制地址，会导致：

- 两个对象指向同一块内存
- 释放时可能 double free

------

### 深拷贝（手动拷贝数据）

```
class A {
public:
    int* data;

    A(const A& other) {
        data = new int(*other.data);
    }
};
```

------

## 一句话总结（面试用）

- `memcpy`：按字节拷贝，快但危险
- `std::copy`：类型安全，推荐
- `vector/对象赋值`：默认做深拷贝（取决于实现）
- 自定义类要注意 **深拷贝 vs 浅拷贝**
