**对象池（Object Pool）\**是一种常见的\**资源复用设计模式**，核心思想是：

> 提前创建一批对象放入“池子”，需要时直接取，用完再归还，而不是频繁 new/delete。

------

# 一、为什么需要对象池？

在 C++ / Qt / 游戏 / 高并发场景中，频繁创建和销毁对象会带来：

### ❌ 问题

- `new/delete` 开销大（涉及系统调用、内存分配器）
- 产生内存碎片
- 影响性能稳定性（抖动）
- 高频对象（如连接、消息、粒子）成本极高

------

# 二、对象池的核心思想

```
初始化：创建一批对象 → 放入池中

获取：从池中取一个可用对象

归还：使用完 → 放回池中复用
```

------

# 三、对象池结构（经典模型）

```
         +------------------+
         |   Object Pool    |
         +------------------+
         | idle list (空闲) |
         | used list (使用) |
         +------------------+
```

------

# 四、简单 C++ 实现示例

```
#include <vector>
#include <stack>
#include <memory>

class Object {
public:
    void reset() {
        // 重置状态
    }
};

class ObjectPool {
private:
    std::stack<Object*> pool;

public:
    ObjectPool(size_t size = 10) {
        for (size_t i = 0; i < size; i++) {
            pool.push(new Object());
        }
    }

    Object* acquire() {
        if (pool.empty()) {
            return new Object(); // 扩容策略
        }
        Object* obj = pool.top();
        pool.pop();
        return obj;
    }

    void release(Object* obj) {
        obj->reset();
        pool.push(obj);
    }

    ~ObjectPool() {
        while (!pool.empty()) {
            delete pool.top();
            pool.pop();
        }
    }
};
```

# 五、现代 C++ 优化版本（推荐思路）

可以用 `unique_ptr` + 自定义 deleter：

```
using ObjectPtr = std::unique_ptr<Object, std::function<void(Object*)>>;

class ObjectPool {
private:
    std::vector<std::unique_ptr<Object>> storage;
    std::stack<Object*> freeList;

public:
    ObjectPool(int n) {
        for (int i = 0; i < n; i++) {
            storage.emplace_back(std::make_unique<Object>());
            freeList.push(storage.back().get());
        }
    }

    ObjectPtr acquire() {
        Object* obj = freeList.top();
        freeList.pop();

        return ObjectPtr(obj, [this](Object* p) {
            p->reset();
            freeList.push(p);
        });
    }
};
```

# 六、对象池适用场景

✔ 高频创建/销毁对象
 ✔ 生命周期短但数量大
 ✔ 性能敏感系统

典型例子：

- 游戏：子弹、粒子、特效
- 网络：连接对象、请求包
- GUI：临时控件对象
- 日志系统 buffer

------

# 七、优点

- 🚀 提升性能（减少 new/delete）
- 🧠 降低 GC/内存管理压力（C++减少碎片）
- 📉 减少系统调用
- 🔁 提高复用率

------

# 八、缺点 / 面试高频坑

### ❌ 内存占用更高

提前预分配

### ❌ 复杂度上升

需要管理生命周期

### ❌ 可能“对象污染”

必须 reset 状态

### ❌ 不适合：

- 生命周期不稳定对象
- 数量极小对象

------

# 九、面试一句话总结（重点）

> 对象池是一种通过预分配对象并复用来减少频繁创建销毁开销的设计模式，适用于高频短生命周期对象场景，可以显著降低内存分配和释放的性能损耗。
