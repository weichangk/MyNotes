`delete[]` 是 C++ 中用于**释放动态数组内存**的操作符，它和 `new[]` 必须成对使用。

------

## 一、基本规则

```
int* p = new int[10];
delete[] p;
```

### 核心原则

- `new` ↔ `delete`
- `new[]` ↔ `delete[]`

**必须严格匹配，否则是未定义行为（UB）**

------

## 二、为什么要有 delete[]

因为数组和单对象的内存释放方式不同：

### 1️⃣ 单对象

```
T* p = new T;
delete p;
```

只析构一个对象。

------

### 2️⃣ 数组对象

```
T* p = new T[3];
delete[] p;
```

会做两件事：

1. **逐个调用析构函数**
2. 释放整块连续内存

------

## 三、如果写错会怎样？

### ❌ new[] + delete（错误）

```
T* p = new T[10];
delete p;   // 错误
```

后果：

- 只析构第一个对象（甚至行为依赖实现）
- 其余对象析构未调用
- 内存释放方式不匹配 → UB

------

### ❌ new + delete[]（错误）

```
T* p = new T;
delete[] p;  // 错误
```

后果：

- 会尝试“按数组方式析构”
- 可能读取错误的 metadata（数组长度信息）
- 导致崩溃

------

## 四、底层机制（面试重点）

### new[] 通常会做：

```
[隐藏头部记录 N] + [N个对象内存]
```

编译器会在内存前面存：

- 数组长度
- 或构造信息

------

### delete[] 做的事：

1. 读取隐藏的 N

2. 循环调用：

   ```
   p[i].~T()
   ```

3. 释放整块内存

------

## 五、与普通 delete 的区别

| 对比         | delete | delete[]     |
| ------------ | ------ | ------------ |
| 对象数量     | 1      | N            |
| 析构调用     | 1次    | N次          |
| 内存结构     | 单对象 | 带隐藏数组头 |
| 是否安全混用 | ❌      | ❌            |

------

## 六、常见面试追问

### 1️⃣ delete[] 为什么能知道数组长度？

👉 编译器在 `new[]` 时在内存前加了 header（实现细节）

------

### 2️⃣ 基本类型也必须 delete[] 吗？

```
int* p = new int[10];
delete[] p;
```

是的，虽然 int 没析构，但：

- 内存分配方式不同
- 必须匹配释放方式

------

### 3️⃣ vector 为什么比 new[] 安全？

```
std::vector<int>
```

原因：

- 自动管理内存
- 自动析构
- 不需要手动 delete[]

------

## 七、总结一句话（面试用）

> `delete[]` 用于释放 `new[]` 分配的数组内存，它会逐个调用对象析构函数，并释放整块内存；必须与 `new[]` 配对使用，否则会导致未定义行为。
