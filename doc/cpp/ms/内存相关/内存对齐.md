内存对齐（Memory Alignment）是 C/C++ 面试中的高频基础题，本质是在讲：**数据在内存中的起始地址必须按一定“字节边界”对齐存放**。

------

## 一、什么是内存对齐

在 Computer Architecture 中，CPU 访问内存通常不是“按字节随意访问”，而是按**特定字长（如 4/8/16 字节）对齐访问**。

> 内存对齐 = 变量的起始地址必须是其对齐系数的整数倍

例如：

- `int`（4字节）通常要求 4 字节对齐
- `double`（8字节）通常要求 8 字节对齐

------

## 二、为什么需要内存对齐

### 1. 提升访问效率（核心原因）

CPU 访问内存通常是“按块读取”的：

- 对齐访问：1 次读取
- 不对齐访问：可能需要 2~3 次读取 + 拼接

👉 结果：对齐能显著提升性能

------

### 2. 硬件限制（部分架构）

某些 CPU（尤其早期或嵌入式）：

- 不支持非对齐访问
- 或非对齐访问会触发异常

------

### 3. cache line 友好

对齐可以减少跨 cache line 访问，提高缓存命中率

------

## 三、结构体内存对齐规则（重点）

编译器通常遵循：

### 规则1：成员对齐

每个成员的起始地址必须是其自身对齐值的整数倍

### 规则2：整体对齐

结构体大小必须是**最大成员对齐值的整数倍**

------

## 四、经典例子

```
struct A {
    char a;   // 1字节
    int b;    // 4字节
    short c;  // 2字节
};
```

### 内存布局：

```
a (1字节)
padding(3字节)   -> 为了让 b 对齐
b (4字节)
c (2字节)
padding(2字节)   -> 结构体整体对齐
```

👉 sizeof(A) 通常 = 12

------

## 五、对齐带来的问题（面试加分点）

### 1. 内存浪费

大量 padding

### 2. cache inefficiency

结构体过大导致 cache miss

------

## 六、如何控制内存对齐

### 1. alignas（C++11 推荐）

```
struct alignas(16) MyData {
    int a;
};
```

------

### 2. #pragma pack（不推荐滥用）

```
#pragma pack(1)
struct A { ... };
#pragma pack()
```

⚠️ 影响性能，可能导致非对齐访问

------

### 3. 手动调整成员顺序（最优解）

```
struct A {
    int b;
    short c;
    char a;
};
```

👉 减少 padding（非常重要的优化手段）

------

## 七、对齐与 new/delete

- `new` 返回的地址通常是满足最大对齐要求的
- C++17 引入：
  - `std::aligned_alloc`
  - aligned new

------

## 八、一句话总结（面试版）

> 内存对齐是为了让 CPU 以更高效率访问内存，通过在变量之间插入 padding，使数据地址满足特定字节边界，从而提升访问速度并兼容硬件访问要求。
