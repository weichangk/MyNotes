`constexpr` 是 C++11 引入的一个关键字，它的作用是在**编译期求值**，即让变量或函数的值**在编译阶段就确定下来**，从而实现更高效、更安全的代码优化。它是对 `const` 的补充和加强，它可以用于变量、函数、构造函数等。



## ✅ `constexpr` 和 `const` 有什么区别？

| 特性                         | `const`              | `constexpr`                    |
| ---------------------------- | -------------------- | ------------------------------ |
| 含义                         | 表示变量**不可修改** | 表示变量的**值在编译期可确定** |
| 是否一定是编译期常量         | ❌ 不一定             | ✅ 一定是                       |
| 能否用于数组大小、模板参数等 | ⛔️ 可能报错           | ✅ 适合用                       |
| 可用于函数返回值             | ❌ 不可               | ✅ 可，函数结果在编译期计算     |

## ✅ 举例说明

### 1. `const` 不一定是编译期常量：

```cpp
const int a = rand(); // ✅ 合法：a 是 const，但值在运行时决定
```

------

### 2. `constexpr` 一定是编译期常量：

```cpp
constexpr int a = 10; // ✅ 编译期可知
const int b = rand();
int arr[b];           // ❌ 错误：数组大小必须是编译期常量
constexpr int c = b;  // ❌ 错误：b 不是编译期常量
```

------

### 3. 用于数组大小或模板参数：

```cpp
const int len = 10;
int arr1[len]; // ⚠️ 不一定通过，len 是否是编译期常量 取决于标准和编译器

constexpr int size = 8;
int arr2[size]; // ✅ 编译期确定，合法
```

------

### 4. `constexpr` 函数示例：

```cpp
constexpr int square(int x) {
    return x * x;
}

int arr[square(5)]; // ✅ OK，编译期求值
```

------

## ✅ 可以用于类的构造函数（C++11+）

```
struct Point {
    int x, y;
    constexpr Point(int x, int y) : x(x), y(y) {}
};

constexpr Point p(1, 2); // ✅ 编译期创建对象
```

------

## ✅ 现代 C++ 推荐的使用方式

- 用 `constexpr` 替代需要在编译期确定的 `const`
- 用 `constexpr` 编写轻量函数用于模板、数组、条件编译等场景

------

## ✅ 总结

| 项目         | `const`                | `constexpr`                    |
| ------------ | ---------------------- | ------------------------------ |
| 修饰变量     | 只读，不一定编译期确定 | 编译期确定且只读               |
| 修饰函数     | ❌ 不支持               | ✅ 支持编译期计算               |
| 编译器优化   | 有可能不优化           | 编译器**必须**优化为常量表达式 |
| 是否分配内存 | 可能                   | 通常不会                       |
