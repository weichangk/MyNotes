## 1. 基础概念

- **符号表**：编译器生成的存储变量、函数等符号信息的数据结构，用于链接和调试。
- **链接属性（Linkage）**：
  - **Internal linkage（内部链接）**：符号仅在当前翻译单元（源文件）可见。
  - **External linkage（外部链接）**：符号对所有翻译单元可见，可跨文件访问。

## 2. 全局 `const` 变量的默认链接属性

- 全局作用域下的 `const` 变量默认是 **internal linkage**，即 **只在本文件有效**。

  ```cpp
  // a.cpp
  const int a = 10;  // internal linkage，默认不可跨文件访问
  ```

- 如果想让 `const` 变量拥有外部链接，需要加 `extern` 显式声明：

  ```cpp
  // a.h
  extern const int a;
  
  // a.cpp
  const int a = 10;  // external linkage
  ```

## 3. `const` 变量与符号表的关系

1 编译器优化和“内联”

- 当 `const` 变量**未被取地址**且只在当前翻译单元中使用时，编译器通常会：

  - 将使用它的地方直接替换为字面常量（例如 `10`），即“**内联**”常量值；
  - 不必生成对应的符号和内存空间。

  ```cpp
  const int a = 10;
  
  int foo() {
      return a + 5;  // 编译时直接替换成 10 + 5 = 15
  }
  ```

- 这样做减少了内存访问，提升性能。

2 生成符号表的条件

- **当 `const` 变量被取地址时，必须在符号表生成符号和对应内存**，因为需要提供地址。

  ```cpp
  const int a = 10;
  const int* p = &a;  // 必须生成符号，a 需存储在内存中
  ```

- **当 `const` 变量有 external linkage（通过 `extern` 声明跨文件访问）时，也必须生成符号**。

## 4. 总结表格

| 场景                                             | 链接类型         | 是否生成符号表 | 是否内联（替换为常量） |
| ------------------------------------------------ | ---------------- | -------------- | ---------------------- |
| 全局 `const int a = 10;`，未取地址，仅本文件使用 | internal linkage | 否             | 是                     |
| 取地址 `&a`                                      | internal linkage | 是             | 否                     |
| 跨翻译单元访问，使用 `extern const int a;`       | external linkage | 是             | 否                     |
| 局部 `const` 变量                                | 无链接（local）  | 否             | 是                     |



------

## 5. 说明

- 不同编译器和优化等级对符号生成策略可能有细微差别。
- 可以通过工具 `nm`、`objdump`、查看汇编代码等确认符号是否存在。



## 6. 举例验证（Linux/macOS）

```cpp
g++ -c test.cpp -o test.o
nm test.o | grep a
```

- 如果没有输出 `a`，说明没生成符号，变量被内联替换了。
- 如果输出有 `a` 符号，则生成了符号表。
