在 C++ 中，const 和 #define 都可以用于定义常量，但它们在语法、类型安全、作用域、调试支持等方面有很大的区别。下面是详细对比：

### 🔶 一、基本语法对比

| 特性     | `#define`            | `const`                                |
| -------- | -------------------- | -------------------------------------- |
| 定义方式 | `#define PI 3.14`    | `const double PI = 3.14;`              |
| 预处理   | 是（预处理器宏）     | 否（由编译器处理）                     |
| 类型安全 | 否（没有类型）       | 是（具有具体类型）                     |
| 作用域   | 全局（文件作用域）   | 有作用域限制（可以是局部或命名空间内） |
| 调试支持 | 无法调试或查看       | 可以调试和查看                         |
| 命名空间 | 无                   | 可以与 `namespace` 配合使用            |
| 编译检查 | 无类型检查，容易出错 | 有类型检查，安全                       |

### 🔶 二、示例对比

使用 `#define`

```cpp
#define PI 3.14

double area(double r) {
    return PI * r * r;  // PI 被替换成 3.14（文本替换）
}
```

使用 `const`

```cpp
const double PI = 3.14;

double area(double r) {
    return PI * r * r;  // 编译器识别为 double 类型
}
```

### 🔶 三、优势与劣势对比

| 比较点       | `#define` 的问题                       | `const` 的优势                   |
| ------------ | -------------------------------------- | -------------------------------- |
| 类型安全     | 不进行类型检查，容易出错               | 编译器能进行类型检查             |
| 命名冲突     | 不在命名空间中，容易引发冲突           | 可以放入类或命名空间中，避免冲突 |
| 调试         | 宏在预处理阶段被替换，调试器看不到原名 | 保留变量名，调试器可识别         |
| 作用域控制   | 全局有效，不可限定作用域               | 可设为局部或类内成员             |
| 编译错误提示 | 宏替换后出错难以定位                   | 编译器会提示具体位置和类型       |

### ❌ 四、 宏替换副作用问题

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int x = 3;
int y = 4;
int result = MAX(x++, y++);  // 会展开成 ((x++) > (y++) ? (x++) : (y++))
```

这会导致：

- `x` 和 `y` 的自增调用次数不明确（多次执行，未定义行为）
- 极容易引入 **难以察觉的 bug**

✅ 正确做法

```cpp
template<typename T>
inline T max(T a, T b) {
    return a > b ? a : b;
}
```

这样就只会执行一次 `x++` 或 `y++`，安全可靠。

### 🔶 五、推荐做法总结

| 情况                         | 推荐使用方式                    |
| ---------------------------- | ------------------------------- |
| 简单常量                     | `const` 或 `constexpr`          |
| 表达式或宏函数               | `inline` 或 `constexpr`函数     |
| 类内常量                     | `static const` 或 `constexpr`   |
| 枚举值                       | `enum` 或 `enum class`          |
| 调试友好、类型安全要求较高时 | 一律使用 `const` 或 `constexpr` |

### ✅ 总结

> 在 C++ 中，**尽量避免使用 `#define` 来定义常量**，优先使用 `const`、`constexpr` 、`inline` 或 `enum`，以获得更好的类型安全、作用域控制和调试支持。



### ✅ 推荐使用宏的典型场景

1️⃣ 条件编译（最常用）

```cpp
#ifdef _WIN32
    // Windows 特有代码
#else
    // Unix/Linux/MacOS 特有代码
#endif
```

- 跨平台开发（Windows/Linux/macOS）
- 不同编译环境启用不同功能（如调试/发布）

2️⃣ 头文件防重复包含（include guard）

```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H

// 头文件内容

#endif
```

- 尽管可以用 `#pragma once`，但 `#ifndef` 是通用写法，兼容所有编译器。

3️⃣ 编译器属性/内建函数封装

```cpp
#define DEPRECATED [[deprecated]]
DEPRECATED void oldFunction();
```

- 用宏来封装属性、平台差异，保持代码整洁
