C++ 中的**内联函数（inline function）**是一种通过在函数调用处展开函数体，来减少函数调用开销的机制。

### 什么是内联函数？

**内联函数（inline function）是使用 `inline` 关键字声明的函数，编译器在调用处将函数体直接插入**，而不是像普通函数那样通过调用栈跳转执行。

```cpp
inline int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(1, 2); // 实际可能被展开为 int result = 1 + 2;
}
```

### 优点

1. **省去函数调用开销**
   - 没有函数调用压栈/出栈，速度更快。
2. **利于编译器优化**
   - 如常量传播、循环展开等。
3. **防止 ODR（One Definition Rule）冲突**
   - 在头文件中使用**`inline` 关键字**定义的内联函数可以防止多个 `.cpp` 引用该头文件后会造成重复定义。

### 缺点

1. **可能增大可执行文件体积（代码膨胀）**
2. **不能用于函数体过大的函数**
3. **改变函数逻辑后，需要重新编译所有用到它的代码**
4. **并不是强制展开，仅是建议，编译器可以忽略**

### 使用条件和限制

编译器能内联的函数通常具有以下特征：

| 条件               | 说明                           |
| ------------------ | ------------------------------ |
| 函数体小           | 逻辑简单                       |
| 没有递归           | 递归函数不能内联               |
| 在头文件中定义     | 编译器必须在调用处“看到”函数体 |
| 不包含复杂控制结构 | 如长循环、大量分支等           |

示例：

```cpp
// 建议将 inline 函数写在头文件中
// math_utils.h
inline int max(int a, int b) {
    return (a > b) ? a : b;
}
```

### 编译器不会内联的情况

- 包含递归调用
- 函数体太大
- 虚函数（除非在编译时能确定类型，如使用 `final`）
- `switch` / `try-catch` 语句过多

### 与宏的区别

| 区别     | 内联函数            | 宏函数               |
| -------- | ------------------- | -------------------- |
| 类型检查 | 有                  | 无                   |
| 安全性   | 高                  | 低（可能有副作用）   |
| 调试支持 | 支持断点等调试      | 不支持               |
| 作用域   | 遵循 C++ 作用域规则 | 纯文本替换，无作用域 |

### 小结

- `inline` 是一种**优化建议**，不是强制行为，是否内联由编译器决定。
- 适用于**短小且调用频繁**的函数。
- 函数体需要写在头文件中，以便被调用处“看到”。
- 与宏函数不同，内联函数是**类型安全**、支持调试的。
