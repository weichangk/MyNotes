## 栈（Stack）

### 栈的定义

- **栈**是由编译器自动分配和释放的一块内存区域。
- 用于存放**局部变量、函数参数、返回地址、临时变量等**。
- 后进先出（LIFO）结构，生命周期随作用域结束自动销毁。

### 栈的特点

- 分配速度快，效率高（基本是指针移动）。
- 不需要手动释放，自动回收。
- 空间较小（通常几 MB），存在**栈溢出（stack overflow）**风险。

### 示例

```cpp
void func() {
    int a = 10;      // a 存在栈中
    int arr[100];    // arr 也在栈上
}
```

## 堆（Heap）

### 堆的定义

- **堆**是由程序员手动管理的内存区域（通过 `new/delete` 或 `malloc/free`）。
- 用于分配**动态内存**，其生命周期可以跨越多个函数调用。

### 堆的特点

- 内存空间大，但分配和释放较慢（涉及系统调用）。
- 必须手动释放，否则会导致 **内存泄漏**。

### 示例

```cpp
void func() {
    int* p = new int(10);    // 在堆上分配
    delete p;                // 必须释放
}
```

## C++内存区域划分（以程序运行时角度）

| 内存区          | 说明                                 |
| --------------- | ------------------------------------ |
| **栈区**        | 局部变量、函数调用记录               |
| **堆区**        | 动态分配的内存（`new`/`malloc`）     |
| **全局/静态区** | 全局变量、static变量，程序结束时释放 |
| **文字常量区**  | 字符串常量等只读数据                 |
| **代码区**      | 函数的二进制机器指令                 |

## C++ 内存管理相关关键点

### new/delete 与 malloc/free 区别

| 特性         | new/delete          | malloc/free        |
| ------------ | ------------------- | ------------------ |
| 分配对象类型 | 会调用构造/析构函数 | 不会调用           |
| 返回类型     | 返回具体类型指针    | 返回 `void*`       |
| 安全性       | 类型安全            | 需强制转换         |
| 使用场景     | C++ 面向对象        | C 语言或底层兼容性 |

### 内存泄漏

未释放动态内存，导致内存无法重用。

```cpp
void leak() {
    int* p = new int[100];  // 没有 delete[]
}
```

### 野指针

指针指向已经释放或未初始化的内存区域。

```cpp
int* p = nullptr;
*p = 10;  // 段错误
```

## 现代 C++ 内存管理机制

### 智能指针（C++11 引入）

- `std::unique_ptr`：独占所有权，自动释放。
- `std::shared_ptr`：引用计数，最后一个释放自动 delete。
- `std::weak_ptr`：解决循环引用问题。

### RAII（资源获取即初始化）

- 利用构造/析构自动管理资源：

```cpp
class MyFile {
    FILE* fp;
public:
    MyFile(const char* filename) { fp = fopen(filename, "r"); }
    ~MyFile() { if (fp) fclose(fp); }
};
```

## 相关知识点

1. `new[]` 和 `delete[]` 是否要成对？
   - 是，混用会出现未定义行为。
2. 为何栈空间小？
   - 由于每线程会预分配固定大小，防止递归或大数组导致系统崩溃。
3. `delete` 后指针该怎么办？
   - 置为 `nullptr`。
