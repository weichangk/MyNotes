`std::list` 的底层实现本质上是 **双向链表（Doubly Linked List）**。

------

## 一、底层结构

每个元素不是连续存储的，而是一个 **独立的节点（node）**：

```
struct Node {
    T data;
    Node* prev;
    Node* next;
};
```

整个 `list` 通过头尾指针串起来：

```
head <-> node1 <-> node2 <-> node3 <-> tail
```

通常还会有一个 **哨兵节点（sentinel / dummy node）** 来简化边界操作（有些实现如 libc++ / libstdc++ 会用）。

------

## 二、核心特点（面试重点）

### 1. 非连续内存

- 每个节点单独分配
- 节点通过指针连接
- 不支持随机访问

👉 所以：

- ❌ `list[i]` 不支持
- ❌ cache 命中率差（CPU不友好）

------

### 2. 插入/删除高效

因为只需要改指针：

```
A <-> B <-> C
```

删除 B：

```
A.next = C
C.prev = A
```

👉 时间复杂度：

| 操作              | 复杂度 |
| ----------------- | ------ |
| 插入/删除已知位置 | O(1)   |
| 查找              | O(n)   |
| 访问第 k 个元素   | O(n)   |

------

### 3. 不会导致迭代器失效（很重要）

- 插入新节点：不影响其他节点迭代器
- 删除某节点：只失效当前节点迭代器

👉 对比 `vector`（扩容全失效）是常见面试点。

------

## 三、内存特点

### 优点

- 动态分配，按需增长
- 插入删除稳定

### 缺点

- 每个节点都有额外指针开销（prev + next）
- 内存碎片严重
- cache locality 差（性能慢）

------

## 四、和 vector 的核心区别（面试高频）

| 特性      | list   | vector |
| --------- | ------ | ------ |
| 内存      | 非连续 | 连续   |
| 随机访问  | ❌ O(n) | ✅ O(1) |
| 插入删除  | O(1)   | O(n)   |
| cache性能 | 差     | 很好   |
| 扩容问题  | 无     | 有     |

------

## 五、什么时候用 list？

一般只有这些情况才考虑：

- 频繁在中间插入/删除
- 不关心随机访问
- 需要稳定迭代器
- 节点频繁移动（例如任务队列、调度器）

👉 现代 C++ 中很多场景其实更偏向：

- `vector + erase/insert`
- 或 `deque`

------

## 六、一句话面试总结

> `std::list` 是基于**双向链表**实现的容器，通过指针连接节点，实现 O(1) 插入删除，但牺牲了随机访问能力和缓存局部性。
