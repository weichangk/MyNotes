# 四、核心面试题整理

# 1. C++基础

## vector 和 list 区别

### 回答重点

vector：

- 连续内存
- 缓存友好
- 随机访问快
- 中间插入删除慢

list：

- 双向链表
- 插入删除快
- 不连续内存
- 缓存不友好
- 随机访问慢

工业软件中：

一般更偏向 vector。

---

## shared_ptr 原理

### 回答重点

- 引用计数
- 控制块
- 原子操作
- 循环引用问题
- weak_ptr解决循环引用

---

## map 和 unordered_map 区别

### 回答重点

map：

- 红黑树
- 有序
- O(logN)

unordered_map：

- 哈希表
- 无序
- 平均O(1)
- 哈希冲突

---

# 2. Windows基础

## 什么是消息循环

### 推荐回答

Windows GUI程序本质上是消息驱动模型。

系统会把键盘、鼠标、窗口重绘等事件封装成消息，放入线程消息队列。

主线程通过：

```cpp
GetMessage
TranslateMessage
DispatchMessage
```

不断从消息队列取消息并分发到窗口过程函数处理。

---

## SendMessage 和 PostMessage 区别

### 推荐回答

SendMessage：

- 同步
- 调用线程会阻塞
- 直到消息处理完成

PostMessage：

- 异步
- 只投递消息
- 立即返回

跨线程场景通常更推荐 PostMessage。

---

## HWND是什么

### 推荐回答

HWND 本质上是窗口句柄。

它不是窗口对象本身，而是 Windows 内核用于标识窗口对象的一个引用。

应用程序通过 HWND 操作窗口。

---

# 3. 多线程

## 如何避免UI卡顿

### 推荐回答

核心原则：

```text
耗时任务不要运行在UI线程
```

常见方案：

- 工作线程
- 线程池
- 异步任务
- 消息队列
- 生产者消费者模型

Qt中通常会：

- moveToThread
- QtConcurrent
- 信号槽异步通信

---

## QueuedConnection 为什么线程安全

### 推荐回答

QueuedConnection 本质上是：

- 把槽调用封装成事件
- 投递到接收线程事件队列
- 由接收线程事件循环执行

因此槽函数始终运行在接收者线程。

避免了跨线程直接调用对象。

---

## 死锁产生条件

### 四个条件

- 互斥
- 占有并等待
- 不可抢占
- 循环等待

### 如何避免

- 锁顺序统一
- 减少锁粒度
- 使用RAII锁
- 避免嵌套锁

---

# 4. Qt相关

## QObject线程亲和性

### 推荐回答

QObject 会绑定创建它的线程。

事件处理、槽函数默认运行在所属线程。

moveToThread 可以修改线程亲和性。

UI对象必须运行在主线程。

---

## repaint 和 update 区别

### 推荐回答

repaint：

- 立即重绘
- 同步
- 可能导致频繁刷新

update：

- 异步
- 合并重绘区域
- 性能更好

通常推荐使用 update。

---

# 5. OpenCV高频

## Mat 为什么拷贝快

### 推荐回答

Mat 默认使用浅拷贝。

多个 Mat 共享同一块图像数据。

内部通过引用计数管理内存。

真正修改时才可能触发深拷贝。

---

## ROI 原理

### 推荐回答

ROI 本质上不是复制图像。

而是：

- 修改数据指针偏移
- 共享原图内存

因此速度快。

# 十、深入面试知识点准备

# 1. C++ 深入面试准备

# 1.1 vector 为什么性能好

## 底层原因

vector 使用连续内存。

CPU 访问数据时：

```text
会利用缓存预取（cache line）
```

连续内存非常适合 CPU cache。

因此：

- 遍历速度快
- cache miss 少
- SIMD优化友好

这也是为什么：

```text
工业软件 / 图像处理 / 高频数据处理
更偏向 vector
```

而不是 list。

---

## vector 扩容机制

典型会问：

```text
vector push_back 为什么有时候会卡？
```

因为：

- 空间不足时会重新申请更大的内存
- 搬迁旧数据
- 释放旧内存

复杂度：

```text
均摊 O(1)
```

但扩容瞬间可能很重。

因此工业软件里：

建议：

```cpp
vec.reserve(10000);
```

提前预分配。

---

# 1.2 shared_ptr 深入

## 底层结构

shared_ptr 本身：

```text
对象指针 + 控制块指针
```

控制块里：

- 引用计数
- weak计数
- 删除器
- allocator

---

## 为什么线程安全

引用计数使用：

```text
原子操作
```

比如：

```cpp
std::atomic<long>
```

因此：

引用计数增减线程安全。

但：

```text
对象本身不线程安全
```

这是高频陷阱题。

---

## 循环引用

经典场景：

```text
A 持有 B
B 持有 A
```

引用计数永远不为0。

解决：

```cpp
weak_ptr
```

---

# 1.3 RAII

工业软件非常喜欢问。

## 本质

```text
资源生命周期绑定对象生命周期
```

构造获取资源。

析构释放资源。

---

## 应用场景

- 锁
- 文件
- socket
- 内存
- 数据库连接

---

## lock_guard 为什么安全

因为作用域结束自动析构。

即使异常：

也会自动释放锁。

避免死锁。


# 5.3 clone 和 copyTo 区别

# 5.4 ROI 为什么快

ROI：

```text

```

只是：

```text
数据指针偏移
```

不真正复制图像。

# 5.5 OpenCV 多线程注意点

高频。

Mat 本身：

```text
引用计数线程安全
```

但：

```text
像素数据操作不线程安全
```

因此：

多线程写同一 Mat：

危险。
