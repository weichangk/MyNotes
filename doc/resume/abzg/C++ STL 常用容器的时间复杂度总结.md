下面是 **C++ STL 常用容器的时间复杂度总结（面试高频版）**，按容器分类整理，重点是 `增删查` 和关键操作复杂度。

------

# 一、顺序容器（Sequence Containers）

## 1. vector（动态数组）

- 访问：O(1)
- 尾部插入 push_back：**均摊 O(1)**
- 中间插入/删除：O(n)
- 查找：O(n)
- 扩容：O(n)（触发拷贝）

👉 特点：**连续内存 + cache友好 + 最常用**

------

## 2. deque（双端队列）

- 头尾插入/删除：O(1)
- 随机访问：O(1)
- 中间操作：O(n)

👉 特点：**分段连续内存，不需要整体搬迁**

------

## 3. list（双向链表）

- 任意位置插入/删除（已知位置）：O(1)
- 查找：O(n)
- 不支持随机访问

👉 特点：**插入删除快，但遍历慢（cache不友好）**

------

## 4. forward_list（单向链表）

- 插入/删除（已知前驱）：O(1)
- 查找：O(n)

👉 更轻量的链表

------

# 二、关联容器（有序）

底层：红黑树（RB Tree）

## 5. set / map

- 插入：O(log n)
- 删除：O(log n)
- 查找：O(log n)
- 有序遍历：O(n)

👉 map：key-value
 👉 set：key

------

## 6. multiset / multimap

- 同 set/map
- 允许重复 key

------

# 三、无序容器（哈希表）

底层：hash table

## 7. unordered_set / unordered_map

- 平均插入：**O(1)**
- 平均查找：**O(1)**
- 平均删除：**O(1)**
- 最坏情况：O(n)（hash冲突严重）

👉 面试关键点：**“平均 O(1)，最坏 O(n)”**

------

## 8. unordered_multiset / unordered_multimap

- 同上，允许重复 key

------

# 四、容器复杂度总结表（面试速记）

| 容器              | 查找     | 插入                    | 删除     | 特点   |
| ----------------- | -------- | ----------------------- | -------- | ------ |
| vector            | O(n)     | 均摊O(1)尾部 / O(n)中间 | O(n)     | 最常用 |
| deque             | O(n)     | O(1)头尾                | O(1)头尾 | 双端   |
| list              | O(n)     | O(1)                    | O(1)     | 链表   |
| set/map           | O(log n) | O(log n)                | O(log n) | 有序   |
| unordered_map/set | O(1)平均 | O(1)平均                | O(1)平均 | 哈希   |

# 五、面试高频追问点（很重要）

## 1. 为什么 unordered_map 平均 O(1)？

- hash 函数 + bucket
- 冲突少时链表很短

------

## 2. 为什么最坏 O(n)？

- 所有 key hash 到同一个 bucket
- 退化成链表/红黑树（实现相关）

------

## 3. vector 为什么插入是均摊 O(1)？

- 扩容是 2倍增长
- 扩容频率低 → amortized analysis

------

## 4. set/map 为什么不用 hash？

- 因为要 **有序性（inorder traversal）**
- 红黑树支持范围查询（lower_bound/upper_bound）

O(1) O(n) O(log n) 是什么？

这些是 **算法时间复杂度（Time Complexity）** 的表示方式，用来描述：**随着数据规模 n 增大，程序执行时间增长的趋势**。

------

# 一、O(1)、O(n)、O(log n) 是什么？

“**O**” 叫做 **大 O 记号（Big-O notation）**，表示“上界增长趋势”。

可以理解为：

> 👉 不关心具体时间，只关心“增长速度快不快”

------

# 二、O(1) —— 常数时间

## 含义

无论数据有多少，执行时间都差不多。

## 例子

```
int x = arr[0];  // 直接访问
```

## 特点

- 永远一步完成
- 不随 n 增长变化

## 直觉

👉 “取快递柜第 3 号箱子”

------

# 三、O(n) —— 线性时间

## 含义

数据量增加多少，时间就增加多少。

## 例子

```
for (int i = 0; i < n; i++) {
    cout << arr[i];
}
```

## 特点

- 数据翻倍 → 时间翻倍

## 直觉

👉 “挨个检查 100 个箱子”

------

# 四、O(log n) —— 对数时间（很快）

## 含义

每次操作都“砍一半”数据。

## 例子（经典：二分查找）

```
while (l <= r) {
    mid = (l + r) / 2;
}
```

## 特点

- n = 1,000,000 → 约 20 次操作就够

## 直觉

👉 “每次把书架一半丢掉找书”

------

# 五、三者对比（非常重要）

| 复杂度   | 增长速度 | 例子     | 性能  |
| -------- | -------- | -------- | ----- |
| O(1)     | 不变     | 数组访问 | ⭐⭐⭐⭐⭐ |
| O(log n) | 很慢增长 | 二分查找 | ⭐⭐⭐⭐  |
| O(n)     | 线性增长 | 遍历数组 | ⭐⭐⭐   |

------

# 六、一个非常重要的理解方式

假设 n = 1,000,000：

| 复杂度   | 操作次数  |
| -------- | --------- |
| O(1)     | 1         |
| O(log n) | ~20       |
| O(n)     | 1,000,000 |

👉 差距是“数量级碾压”

------

# 七、面试常问点

## 1. 为什么 log n 很快？

因为：

> 每次减少一半数据

------

## 2. log 是多少？

一般默认：

> log₂(n)

------

## 3. STL 里的对应关系

- vector 随机访问 → O(1)
- map/set 查找 → O(log n)
- unordered_map 查找 → O(1) 平均

------

# 八、一句话总结（面试用）

> O(1) 是常数时间，O(n) 是线性增长，O(log n) 是每次折半的对数级增长，表示算法随数据规模扩大的效率变化趋势。
