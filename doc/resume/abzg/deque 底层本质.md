`std::deque`（双端队列）的底层实现和 `vector` 很不一样，是面试高频点。

------

# 一、deque 底层本质（核心一句话）

👉 **deque = 分段连续内存 + 中央控制表（map）**

它不是一整块连续内存，而是：

- 多个“小连续数组”（buffer / block）
- 一个“指针数组”（map）管理这些块

------

# 二、结构长什么样（重点理解）

可以想象成：

```
map（指针数组）
  ↓     ↓     ↓     ↓
 [ ]   [ ]   [ ]   [ ]
  ↓     ↓     ↓     ↓
块1    块2    块3    块4   （每块是连续内存）
```

实际存储：

```
deque:
   block0   block1   block2   block3
+--------+ +--------+ +--------+ +--------+
|  ...   | |  ...   | |  ...   | |  ...   |
+--------+ +--------+ +--------+ +--------+

        ↑ map（中控数组，存每个 block 指针）
```

------

# 三、为什么要这样设计？

## 1. 解决 vector 的问题

vector：

- ❌ 扩容要整体搬家（realloc）
- ❌ 头部插入 O(n)

deque：

- ✔ 头尾插入 O(1)
- ✔ 不需要整体搬迁

------

## 2. 为什么不用链表？

链表：

- ❌ cache 不友好（指针跳转）
- ❌ 内存碎片严重

deque：

- ✔ 每块是连续内存（cache 友好）
- ✔ 又能动态扩展

------

# 四、deque 的关键操作复杂度

| 操作       | 复杂度    | 原因              |
| ---------- | --------- | ----------------- |
| push_back  | O(1) 均摊 | 当前 block 未满   |
| push_front | O(1) 均摊 | 前面 block 可扩展 |
| 随机访问   | O(1)      | 通过 map + offset |
| 中间插入   | O(n)      | 需要移动元素      |

------

# 五、随机访问是怎么做到 O(1) 的？

例如访问 `deque[i]`：

```
block_index = i / block_size
inner_index = i % block_size
```

然后：

```
map[block_index][inner_index]
```

👉 两次寻址即可

------

# 六、deque 和 vector 本质区别（面试必问）

| 对比      | vector       | deque        |
| --------- | ------------ | ------------ |
| 内存      | 一整块连续   | 多块连续     |
| 扩容      | 可能整体搬家 | 不搬整体     |
| 随机访问  | O(1)         | O(1)（稍慢） |
| 头插      | O(n)         | O(1)         |
| cache友好 | 很好         | 较好         |

------

# 七、常见面试追问

## 1. deque 是完全连续的吗？

❌ 不是
 ✔ 是“分段连续”

------

## 2. deque 为什么比 list 快？

因为：

- list：纯指针跳转（cache miss严重）
- deque：block 内连续内存

------

## 3. deque 会不会扩容？

会，但扩的是：
 👉 **map（中控指针数组）**
 而不是整体数据搬迁

------

## 4. STL 实现差异（重要补充）

不同实现略有差异：

- GCC STL：`_Deque_base`
- MSVC STL：block + map 结构类似
- block size 通常 512B ~ 4KB（按元素类型计算）

------

# 八、总结一句面试答案

如果面试官让你一句话说 deque：

> deque 底层是由多个固定大小的连续内存块组成，并通过一个指针数组（map）进行管理，从而实现了头尾插入 O(1) 和随机访问 O(1)，但整体并不是连续内存。
