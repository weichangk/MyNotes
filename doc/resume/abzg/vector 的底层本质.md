`std::vector` 的底层本质可以一句话概括：

> **连续内存的动态数组（dynamic array），由 allocator 管理的一块可扩容的堆内存。**

下面从面试角度把“底层实现”讲清楚。

------

## 1. 内存结构（核心）

`vector` 内部通常维护三个指针（或等价结构）：

- `start`：指向已分配内存的起始
- `finish`：指向当前最后一个元素之后
- `end_of_storage`：指向整块容量的末尾

可以理解为：

```
[start ........ finish ........ end_of_storage]
 |---- size ----|------ capacity ------|
```

### 含义：

- `size = finish - start`（已有元素数量）
- `capacity = end_of_storage - start`（总容量）

------

## 2. 为什么说是“连续数组”

`vector` 的元素在内存中是**连续存储**的：

```
std::vector<int> v = {1,2,3};
```

内存类似：

```
[1][2][3][ ][ ][ ]
```

👉 这也是它支持：

- O(1) 随机访问 `v[i]`
- CPU cache 友好（性能好）

------

## 3. 扩容机制（重点面试点）

当 `size == capacity` 时，再 `push_back` 会触发：

### 步骤：

1. 申请一块更大的新内存（通常 **2倍扩容** 或 1.5倍）
2. 将旧数据 **拷贝/移动** 到新内存
3. 释放旧内存
4. 更新指针

```
旧： [1 2 3]
新： [1 2 3 _ _ _ _ _]
```

------

## 4. 扩容代价（为什么 push_back 有时慢）

单次 `push_back`：

- 平均 O(1)
- 但扩容时是 O(n)

👉 所以是 **均摊 O(1)**（amortized O(1)）

------

## 5. allocator（内存管理）

`vector` 不直接 new/delete，而是通过：

```
std::allocator<T>
```

作用：

- 分配原始内存（allocate）
- 构造对象（construct）
- 销毁对象（destroy）

👉 分离“内存”和“对象生命周期”

------

## 6. iterator 失效问题（高频面试）

由于扩容会整体搬家：

### 会失效：

- 所有 `iterator`
- 所有 `pointer`
- 所有 `reference`

```
auto it = v.begin();
v.push_back(100); // 可能触发扩容
// it 失效
```

------

## 7. 和 malloc 数组区别

| 特性      | vector | malloc数组 |
| --------- | ------ | ---------- |
| 自动扩容  | ✔      | ✘          |
| RAII管理  | ✔      | ✘          |
| 类型安全  | ✔      | ✘          |
| 构造/析构 | ✔      | ✘          |

------

## 8. 性能特点总结

### 优点：

- 连续内存（cache友好）
- O(1) 随机访问
- 自动扩容
- STL标准兼容

### 缺点：

- 中间插入/删除 O(n)
- 扩容有开销
- iterator 容易失效

------

## 9. 面试一句话总结（推荐背）

> vector 底层是基于连续内存的动态数组，通过 capacity 管理扩容，当空间不足时重新申请更大内存并搬移数据，因此支持高效随机访问，但在扩容和中间插入删除时存在性能开销，同时会导致迭代器失效。
