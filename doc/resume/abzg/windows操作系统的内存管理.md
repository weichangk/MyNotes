Windows 的内存管理，本质上是在“进程如何安全、高效使用虚拟内存 + 操作系统如何调度物理内存”。

可以从 4 个核心点理解：

------

## 1️⃣ 虚拟内存机制（核心中的核心）

Windows 给每个进程提供一个**独立的虚拟地址空间**，看起来像各自拥有 4GB/128TB 内存（取决于 32/64 位）。

- 程序看到的是：虚拟地址
- 实际用的是：物理内存 + 磁盘（pagefile）

👉 好处：

- 进程隔离（A 崩溃不影响 B）
- 内存可以“看起来很大”

------

## 2️⃣ 页机制（Paging）

Windows 把内存切成固定大小的块：

- 一般是 **4KB 页**
- 虚拟内存 ↔ 物理内存 通过“页表”映射

当访问的页不在内存中：
 ➡️ 发生 **缺页中断（Page Fault）**
 ➡️ OS 从磁盘（pagefile 或文件）加载回来

------

## 3️⃣ 工作集（Working Set）

每个进程在物理内存中实际占用的页集合叫：

👉 Working Set

Windows 会动态调整：

- 内存紧张 → 回收不活跃页
- 进程活跃 → 扩大工作集

------

## 4️⃣ 内存回收与分页文件（Pagefile）

当 RAM 不够时：

Windows 会把“冷数据”换出到磁盘：

- 干净页（clean page）：可以直接丢
- 脏页（dirty page）：写入 pagefile 再换出

👉 pagefile.sys 就是“磁盘上的扩展内存”

------

## ⭐ 一句话总结

Windows 内存管理 =
 👉 **虚拟内存 + 分页机制 + 工作集调度 + 磁盘交换（pagefile）**
