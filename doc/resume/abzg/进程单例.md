“进程单例”（Process Singleton）本质目标是：**同一时刻只允许一个进程实例运行，其它启动的实例要检测到并退出或唤醒已有实例**。

常见实现方式取决于操作系统，核心思路都是：**利用系统级“唯一资源”做占用判断**。

------

## 一、最常见面试答案（推荐说法）

一句话版本：

> 通过操作系统提供的“互斥机制”（文件锁 / 命名互斥量 / socket / pid文件）确保同一时间只有一个进程能成功占用资源。

------

## 二、几种主流实现方式

### 1️⃣ 文件锁（跨平台，最常用）

#### 原理

- 创建一个锁文件
- 对该文件加**独占锁**
- 获取锁失败 → 说明已有实例运行

#### Linux 示例（flock）

```
int fd = open("/tmp/my_app.lock", O_CREAT | O_RDWR, 0666);
if (flock(fd, LOCK_EX | LOCK_NB) == -1) {
    // 已有实例
    exit(0);
}
```

#### 优点

- 简单可靠
- 支持崩溃自动释放（进程结束锁释放）

------

## 2️⃣ Windows 命名互斥量（最标准）

```
HANDLE hMutex = CreateMutex(NULL, TRUE, L"MyUniqueAppMutex");
if (GetLastError() == ERROR_ALREADY_EXISTS) {
    // 已存在实例
    exit(0);
}
```

#### 优点

- 系统级保证
- 非常稳定
- Windows 面试高频答案

------

## 3️⃣ PID 文件（Linux常见）

#### 原理

- 启动时写入 `/var/run/app.pid`
- 检查该 PID 是否仍在运行

```
ps -p <pid>
```

#### 问题

- 进程崩溃可能留下“脏文件”
- 需要额外清理机制

------

## 4️⃣ 本地端口占用（Socket方式）

#### 原理

- 绑定一个固定端口
- bind失败说明已有实例

```
socket->bind(12345);
```

#### 优点

- 还能顺便做 IPC（进程间通信）
- 很多 GUI 应用用这个（例如 Electron）

------

## 5️⃣ Linux Unix Domain Socket（更高级）

- 创建 `/tmp/app.sock`
- bind失败 → 单例已存在
- 还能通信

------

## 三、工业级推荐方案（面试加分点）

一般推荐：

> Windows：命名 Mutex
>  Linux：文件锁（flock）或 Unix Domain Socket

并且会加一句：

- 防止“僵尸锁”：启动时检查 PID 是否存活
- 或用 `try-lock` + 自动释放机制

------

## 四、面试追问点（重点）

### ❓1：进程崩溃后锁会不会残留？

- 文件锁：不会（进程结束自动释放）
- PID 文件：可能残留（需要清理）
- socket：通常自动释放

------

### ❓2：如何做到“已有进程唤醒新请求”？

👉 升级版单例：

- 新进程检测失败后
- 通过 socket / pipe / shared memory
- 通知已有进程打开窗口或处理参数

------

### ❓3：多用户环境怎么处理？

- 锁文件路径要区分 user id
- Windows mutex 可以带 Global/Session 前缀

------

## 五、一句话总结（面试收尾）

> 进程单例的本质是利用操作系统提供的全局互斥资源（文件锁、命名互斥量或端口绑定）来实现“全局唯一占用”，从而保证同一时间只有一个实例运行。

如何做到进程已运行时激活窗口

这是一个**进程单例 + 窗口唤醒（bring to front）**的经典面试组合题，本质是两件事：

> ✔ 检测已有进程
>  ✔ 通知已有进程“把主窗口激活并置前”

------

# 一、整体思路（面试标准答案）

当新进程启动时：

1. 发现已有实例（mutex / socket / pid）
2. **不退出静默结束**
3. 改为：向“已运行进程”发送消息
4. 已运行进程收到消息后：
   - restore window（如果最小化）
   - set foreground（置顶）

------

# 二、Windows 平台标准做法（最常考）

## 方案1：FindWindow + SendMessage（简单版）

### 1️⃣ 找到主窗口

```
HWND hWnd = FindWindow(L"MyWindowClass", NULL);
```

### 2️⃣ 发送自定义消息

```
#define WM_ACTIVATE_SINGLETON (WM_USER + 1)

SendMessage(hWnd, WM_ACTIVATE_SINGLETON, 0, 0);
```

### 3️⃣ 主进程处理消息

```
LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam)
{
    if (msg == WM_ACTIVATE_SINGLETON)
    {
        ShowWindow(hwnd, SW_RESTORE);
        SetForegroundWindow(hwnd);
        return 0;
    }
    return DefWindowProc(hwnd, msg, wParam, lParam);
}
```

------

## ⚠️ 注意（面试加分点）

Windows 有“前台锁限制”，直接 SetForegroundWindow 可能失败：

👉 推荐组合：

```
ShowWindow(hwnd, SW_RESTORE);
SetForegroundWindow(hwnd);
SetActiveWindow(hwnd);
```

或者：

```
AttachThreadInput(...)
```

------

# 三、更工程化方案（推荐说法）

## 方案2：Windows 消息投递（WM_COPYDATA）

适合传参 + 唤醒

```
COPYDATASTRUCT cds;
cds.dwData = 1;
cds.lpData = (void*)"activate";
cds.cbData = strlen("activate") + 1;

SendMessage(hWnd, WM_COPYDATA, 0, (LPARAM)&cds);
```

优点：

- 可以传启动参数（文件路径等）
- 稳定

------

# 四、Qt / QML 常见做法（工业UI常考）

Qt 有现成机制：

## 方案3：QLocalSocket / QSharedMemory

### 新进程：

```
QLocalSocket socket;
socket.connectToServer("MyAppServer");

if (socket.waitForConnected()) {
    socket.write("activate");
    socket.flush();
    return 0;
}
```

### 主进程：

```
connect(server, &QLocalServer::newConnection, this, [](){
    QLocalSocket *client = server->nextPendingConnection();

    // 激活窗口
    window->showNormal();
    window->raise();
    window->activateWindow();
});
```

👉 Qt面试最推荐说这一套

# 五、Linux 常见方案

## 方案4：Unix Domain Socket

- 新进程 connect socket
- 主进程收到消息
- 执行：

```
XRaiseWindow()
XMapRaised()
```

或 Wayland 对应接口

# 六、为什么不能只“激活窗口”？

面试高频追问：

### ❓Q：为什么新进程不能直接操作旧窗口？

👉 答：

- Windows 权限隔离
- UI线程归属不同进程
- 前台切换受系统限制
- 必须“跨进程通信 + 主进程自己操作窗口”

| 场景           | 推荐方案                     |
| -------------- | ---------------------------- |
| Windows 原生   | Mutex + SendMessage          |
| Qt 工业软件    | QLocalServer / QSharedMemory |
| Linux 工业软件 | Unix Domain Socket           |
| 需要传参数     | WM_COPYDATA / socket         |

# 八、面试一句话总结（必背）

> 进程已运行时激活窗口的本质是：新实例通过 IPC（如窗口消息或本地 socket）通知已有实例，由已有实例调用系统 API（ShowWindow + SetForegroundWindow）将主窗口恢复并置前。
