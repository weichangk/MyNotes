动态库（Dynamic Library）和静态库（Static Library）是 C/C++ 中两种常见的代码复用方式，本质区别在于**链接发生的时间点不同**。

------

# 一、核心区别（一句话）

- **静态库：编译链接时“拷贝进程序”**
- **动态库：运行时“加载使用”**

------

# 二、静态库（Static Library）

## 1. 形式

- Windows：`.lib`
- Linux：`.a`

## 2. 工作方式

编译链接阶段：

- 编译器把静态库里的代码**直接复制到可执行文件中**

👉 结果：生成的 exe / elf 文件已经包含库代码

------

## 3. 优点

✔ 运行时不依赖外部库
 ✔ 部署简单（一个可执行文件即可运行）
 ✔ 启动速度略快（无需加载外部库）

------

## 4. 缺点

❌ 文件体积大（每个程序都拷贝一份库代码）
 ❌ 更新麻烦（库更新 → 所有程序必须重新编译）
 ❌ 内存浪费（多个进程会各自保存一份）

------

# 三、动态库（Dynamic Library）

## 1. 形式

- Windows：`.dll`
- Linux：`.so`

------

## 2. 工作方式

分为两个阶段：

### （1）编译链接时

- 只记录“依赖关系”
- 不把代码复制进程序

### （2）运行时

- 操作系统加载 `.dll / .so`
- 多个程序共享同一份库代码

------

## 3. 优点

✔ 节省磁盘空间（多个程序共享库）
 ✔ 节省内存（共享一份代码段）
 ✔ 易于升级（替换库文件即可，无需重新编译主程序）
 ✔ 支持插件化设计（热插拔架构）

------

## 4. 缺点

❌ 运行时依赖库文件（可能“缺 DLL”错误）
 ❌ 版本兼容问题（DLL Hell）
 ❌ 启动时需要加载，略慢
 ❌ 调试复杂一些

------

# 四、对比总结（面试高频）

| 对比点       | 静态库     | 动态库       |
| ------------ | ---------- | ------------ |
| 链接时间     | 编译期     | 运行期       |
| 文件后缀     | .a / .lib  | .so / .dll   |
| 是否拷贝代码 | 是         | 否           |
| 依赖性       | 无外部依赖 | 依赖运行环境 |
| 内存占用     | 高         | 低（共享）   |
| 更新难度     | 高         | 低           |
| 扩展性       | 差         | 强           |

------

# 五、底层本质（面试加分点）

## 静态库：

👉 本质是“目标文件的集合”

- `.o/.obj` 打包成 `.a/.lib`
- 链接器直接合并进可执行文件

------

## 动态库：

👉 本质是“可重定位的共享代码”

- 通过 **GOT / PLT（Linux）**
- 或 **导入表（Windows PE结构）**
- 实现运行时符号绑定

------

# 六、典型使用场景

## 静态库适合：

- 嵌入式开发（无运行环境）
- 单机工具程序
- 对依赖要求严格的系统

------

## 动态库适合：

- 大型系统（Qt / 游戏 / 服务端）
- 插件架构（如 Photoshop 插件）
- SDK / API 分发

------

# 七、面试追问（高频）

### Q1：动态库一定比静态库好吗？

❌ 不一定
 → 小程序 / 嵌入式更适合静态库

------

### Q2：为什么动态库可以多个进程共享？

👉 因为代码段是只读 + 内存映射（mmap）

------

### Q3：动态库加载过程？

👉

1. ELF/PE 解析
2. 加载到内存
3. 重定位
4. 符号绑定（lazy / eager）

------

### Q4：静态库为什么不需要运行时加载？

👉 因为链接器已经把代码“写死”进可执行文件

------

# 八、如果你要面试奥比中光（加分总结）

可以这样一句话收尾：

> 静态库在编译期完成代码融合，优点是独立性强；动态库在运行期加载共享，优点是节省资源并支持模块化扩展，本质区别是“代码是否在链接阶段被拷贝进可执行文件”。
