### 基础概念类

#### Q1: 什么是依赖属性？为什么WPF要使用依赖属性而不是普通CLR属性？

**答案**:
依赖属性是WPF属性系统的核心，继承自DependencyObject。

**为什么使用依赖属性**:
1. **内存效率** - 只在赋值时占用内存，共享默认值
   - 示例：1000个Button，只有10个设置Background
   - CLR属性：1000个内存槽位
   - 依赖属性：10个实际值 + 默认值引用

2. **功能丰富**:
   - 数据绑定支持
   - 样式和模板支持
   - 动画支持
   - 属性值继承（如FontSize）
   - 属性变更通知
   - 值强制和验证

3. **值优先级系统**:
   ```
   优先级（从高到低）：
   1. 动画值
   2. 本地值（代码设置）
   3. 触发器
   4. 样式
   5. 属性值继承
   6. 默认值
   ```

**代码示例**:
```csharp
public static readonly DependencyProperty TitleProperty =
    DependencyProperty.Register(
        "Title",
        typeof(string),
        typeof(MyControl),
        new PropertyMetadata(
            defaultValue: "默认标题",
            propertyChangedCallback: OnTitleChanged
        )
    );

public string Title
{
    get => (string)GetValue(TitleProperty);
    set => SetValue(TitleProperty, value);
}

private static void OnTitleChanged(DependencyObject d, DependencyPropertyChangedEventArgs e)
{
    var control = (MyControl)d;
    // 处理属性变更
}
```

#### Q2: 解释WPF的数据绑定机制，绑定模式有哪些？

**答案**:

**数据绑定机制**:
WPF通过Binding对象连接目标属性（DependencyProperty）和源属性（实现INotifyPropertyChanged）。

**绑定模式**:
```csharp
public enum BindingMode
{
    TwoWay,         // 双向：控件 ↔ 数据源
    OneWay,         // 单向：数据源 → 控件
    OneTime,        // 一次性：仅初始化
    OneWayToSource, // 反向：控件 → 数据源
    Default         // 由控件决定
}
```

**默认模式**:
- TextBox.Text → TwoWay
- TextBlock.Text → OneWay
- Button.Command → OneWay

**工作原理**:
```
数据源（ViewModel）
  └── 属性变更
       └── PropertyChanged事件
            └── Binding监听
                 └── 更新目标属性
                      └── UI刷新
```

**性能优化**:
1. 只读数据用OneWay而非TwoWay
2. 静态数据用OneTime
3. 避免复杂的绑定路径
4. 使用UpdateSourceTrigger控制更新时机

```xml
<!-- 实时更新 -->
<TextBox Text="{Binding SearchText, UpdateSourceTrigger=PropertyChanged}" />

<!-- 失去焦点时更新（默认） -->
<TextBox Text="{Binding Name, UpdateSourceTrigger=LostFocus}" />
```

#### Q4: MVVM模式的核心思想是什么？与MVC、MVP有什么区别？

**答案**:

**MVVM核心思想**:
通过数据绑定和命令模式实现View和ViewModel的解耦，View只负责展示，ViewModel负责逻辑。

**架构对比**:

| 模式 | View | 中间层 | Model | 数据流 | 平台 |
|------|------|--------|-------|--------|------|
| **MVC** | 被动视图 | Controller | 业务逻辑 | Controller主导 | Web后端 |
| **MVP** | 被动视图 | Presenter | 业务逻辑 | Presenter操作View | Android/iOS |
| **MVVM** | 主动视图 | ViewModel | 业务逻辑 | 双向绑定 | WPF/Xamarin |

**MVVM架构**:
```
┌─────────────────┐
│      View       │ XAML定义UI
│   (MainView)    │ 数据绑定到ViewModel
└────────┬────────┘
         │ DataBinding
         │ Command
┌────────▼────────┐
│   ViewModel     │ 视图逻辑
│(MainViewModel)  │ INotifyPropertyChanged
│                 │ ICommand
└────────┬────────┘
         │ 调用
┌────────▼────────┐
│     Model       │ 业务逻辑
│  (DataModel)    │ 数据访问
└─────────────────┘
```

**MVVM优势**:
1. **可测试性** - ViewModel可独立单元测试
2. **解耦合** - View和ViewModel通过绑定解耦
3. **设计师友好** - 设计师可独立修改XAML
4. **代码复用** - ViewModel可复用到不同View

**实现示例**:
```csharp
// ViewModel
public class MainViewModel : INotifyPropertyChanged
{
    private string _name;
    public string Name
    {
        get => _name;
        set
        {
            _name = value;
            OnPropertyChanged(nameof(Name));
        }
    }

    public ICommand SaveCommand { get; }

    public MainViewModel()
    {
        SaveCommand = new RelayCommand(Save, CanSave);
    }

    private void Save()
    {
        // 保存逻辑
    }

    private bool CanSave()
    {
        return !string.IsNullOrEmpty(Name);
    }
}
```

```xml
<!-- View -->
<Window DataContext="{Binding MainViewModel}">
    <StackPanel>
        <TextBox Text="{Binding Name, Mode=TwoWay}" />
        <Button Content="Save" Command="{Binding SaveCommand}" />
    </StackPanel>
</Window>
```

#### Q6: Visual Layer是什么？在什么场景下使用？

**答案**:

**Visual Layer定义**:
WPF渲染架构的中间层，介于高级控件API和底层DirectX之间，提供轻量级的图形绘制能力。

**渲染层次**:
```
┌────────────────────────────┐
│  Controls (Button, etc)    │ ← 高级API，功能丰富但重量级
├────────────────────────────┤
│  Visual Layer              │ ← 轻量级，高性能
│  (DrawingVisual)           │
├────────────────────────────┤
│  Media Integration Layer   │
├────────────────────────────┤
│  DirectX / GPU             │ ← 硬件加速
└────────────────────────────┘
```

**使用场景**:

**1. 大量图形绘制**:
```csharp
// 绘制10000个点的图表

{

    public void DrawPoints(Point[] points)
    {
        using (DrawingContext dc = _visual.RenderOpen())

            {

        }

    protected override Visual GetVisualChild(int index) => _visual;

```

#### Q7: WPF中如何处理UI线程和后台线程的交互？

**答案**:

**核心原则**:
UI元素只能在创建它的线程（UI线程）上访问，这是WPF的线程亲和性（Thread Affinity）。

**解决方案**:

**1. Dispatcher调度**:
```csharp
// 检查是否在UI线程
if (Dispatcher.CheckAccess())
{
    // 在UI线程，直接更新
    UpdateUI();
}
else
{
    // 在后台线程，调度到UI线程
    Dispatcher.Invoke(() => UpdateUI());
}

// 异步调度（不阻塞后台线程）
Dispatcher.BeginInvoke(new Action(() =>
{
    UpdateUI();
}));

// 带优先级的调度
Dispatcher.BeginInvoke(
    DispatcherPriority.Background,
    new Action(() => UpdateUI())
);
```

**2. async/await模式（推荐）**:
```csharp
public async Task LoadDataAsync()
{
    // UI线程：显示加载指示
    LoadingIndicator.Visibility = Visibility.Visible;
    
    // 自动切换到后台线程
    var data = await Task.Run(() =>
    {
        // 后台线程：执行耗时操作
        return Database.LoadData();
    });
    
    // 自动切回UI线程：更新UI
    DataGrid.ItemsSource = data;
    LoadingIndicator.Visibility = Visibility.Collapsed;
}
```

**3. 线程安全集合**:
```csharp
// 问题：ObservableCollection不是线程安全的
public class UnsafeViewModel
{
    public ObservableCollection<Item> Items { get; } = new();
    
    public void AddFromBackground(Item item)
    {
        // ❌ 跨线程访问，会抛异常
        Items.Add(item);
    }
}

// 解决方案1：Dispatcher
public class SafeViewModel1
{
    public ObservableCollection<Item> Items { get; } = new();
    
    public void AddFromBackground(Item item)
    {
        Application.Current.Dispatcher.Invoke(() =>
        {
            Items.Add(item);
        });
    }
}

// 解决方案2：EnableCollectionSynchronization
public class SafeViewModel2
{
    private readonly object _lock = new object();
    public ObservableCollection<Item> Items { get; }
    
    public SafeViewModel2()
    {
        Items = new ObservableCollection<Item>();
        BindingOperations.EnableCollectionSynchronization(Items, _lock);
    }
    
    public void AddFromBackground(Item item)
    {
        lock (_lock)
        {
            Items.Add(item);
        }
    }
}
```

**4. 进度报告**:

2. **连接存储**:

```cpp
    QObject *sender;        // 发送者
    int signal_index;       // 信号索引
    QObject *receiver;      // 接收者
    int slot_index;         // 槽索引
```

```cpp
// 发射信号时

// 实际等价于
QMetaObject::activate(this, &staticMetaObject, signal_index, argv);

// activate内部：
// 1. 遍历该信号的所有连接
// 2. 检查连接类型（Direct/Queued/Auto）
// 3. 根据类型调用槽函数或发送事件
```

4. **跨线程处理**:
```cpp
// 自动连接（Qt::AutoConnection）
if (sender->thread() == receiver->thread()) {
    // 同线程：直接调用槽函数（DirectConnection）
    receiver->slot(args);
} else {
    // 跨线程：将调用封装成事件，投递到接收者线程（QueuedConnection）
    QCoreApplication::postEvent(receiver, new QMetaCallEvent(...));
}
```

### Q13: 如何实现一个高性能的Qt Model（支持10万+数据）？

**问题深度**: ⭐⭐⭐⭐⭐

**标准答案**:

**1. 延迟加载（Lazy Loading）**:

```cpp
class LazyTableModel : public QAbstractTableModel
{
    // 不存储所有数据，只存储索引信息
    DatabaseConnection *db;
    int totalRowCount;
    mutable QHash<int, QVector<QVariant>> rowCache;  // 行缓存
    const int cacheSize = 1000;  // 缓存1000行
    
public:
    int rowCount(const QModelIndex &parent) const override {
        return totalRowCount;  // 立即返回总数（从数据库元信息获取）
    }
    
    int columnCount(const QModelIndex &parent) const override {
        return 5;
    }
    
    QVariant data(const QModelIndex &index, int role) const override {
        if (!index.isValid() || role != Qt::DisplayRole)
            return QVariant();
        
        int row = index.row();
        int col = index.column();
        
        // 检查缓存
        if (rowCache.contains(row)) {
            return rowCache[row][col];
        }
        
        // 从数据库按需加载（预加载附近行）
        loadRowsRange(row, row + 100);
        
        // LRU缓存淘汰
        if (rowCache.size() > cacheSize) {
            evictOldestCacheEntries();
        }
        
        return rowCache[row][col];
    }
    
private:
    void loadRowsRange(int start, int end) const {
        QString sql = QString("SELECT * FROM table LIMIT %1 OFFSET %2")
                      .arg(end - start + 1).arg(start);
        QSqlQuery query = db->executeQuery(sql);
        
        int row = start;
        while (query.next()) {
            QVector<QVariant> rowData;
            for (int i = 0; i < columnCount(); ++i) {
                rowData.append(query.value(i));
            }
            rowCache[row++] = rowData;
        }
    }
};
```

```cpp
class HighPerformanceModel : public QAbstractTableModel
{
    Q_OBJECT
    
    DatabaseConnection *db;
    int totalRows;
    mutable QHash<int, QVector<QVariant>> cache;
    const int cacheSize = 2000;
    const int prefetchSize = 200;
    
    mutable QSet<int> loadingRows;  // 正在加载的行
    QThreadPool *threadPool;
    
public:
    HighPerformanceModel(DatabaseConnection *db, QObject *parent = nullptr)
        : QAbstractTableModel(parent), db(db)
    {
        // 快速获取总行数（使用COUNT缓存）
        totalRows = db->executeScalar("SELECT COUNT(*) FROM items").toInt();
        
        threadPool = new QThreadPool(this);
        threadPool->setMaxThreadCount(4);
    }
    
    int rowCount(const QModelIndex &parent) const override {
        return totalRows;
    }
    
    int columnCount(const QModelIndex &parent) const override {
        return 4;  // ID, Name, Price, Date
    }
    
    QVariant data(const QModelIndex &index, int role) const override {
        if (!index.isValid())
            return QVariant();
        
        if (role == Qt::DisplayRole || role == Qt::EditRole) {
            int row = index.row();
            int col = index.column();
            
            if (cache.contains(row)) {
                return cache[row][col];
            }
            
            // 触发异步加载
            if (!loadingRows.contains(row)) {
                const_cast<HighPerformanceModel*>(this)->loadRowsAsync(row);
            }
            
            return QVariant("Loading...");
        }
        
        return QVariant();
    }
    
private:
    void loadRowsAsync(int startRow) {
        loadingRows.insert(startRow);
        
        // 使用QRunnable进行异步加载
        class LoadTask : public QRunnable {
            HighPerformanceModel *model;
            int startRow;
        public:
            LoadTask(HighPerformanceModel *m, int row) : model(m), startRow(row) {}
            
            void run() override {
                model->loadRowsSync(startRow);
            }
        };
        
        threadPool->start(new LoadTask(this, startRow));
    }
    
    void loadRowsSync(int startRow) {
        int endRow = qMin(startRow + prefetchSize - 1, totalRows - 1);
        
        QString sql = QString("SELECT id, name, price, date FROM items "
                               "LIMIT %1 OFFSET %2")
                      .arg(endRow - startRow + 1).arg(startRow);
        
        QSqlQuery query = db->executeQuery(sql);
        
        QHash<int, QVector<QVariant>> newData;
        int row = startRow;
        while (query.next()) {
            QVector<QVariant> rowData = {
                query.value(0),  // ID
                query.value(1),  // Name
                query.value(2),  // Price
                query.value(3)   // Date
            };
            newData[row++] = rowData;
        }
        
        // 回到主线程更新
        QMetaObject::invokeMethod(this, "onDataLoaded", Qt::QueuedConnection,
                                   Q_ARG(int, startRow),
                                   Q_ARG(int, endRow),
                                   Q_ARG(QHash<int, QVector<QVariant>>, newData));
    }
    
private slots:
    void onDataLoaded(int startRow, int endRow, QHash<int, QVector<QVariant>> data) {
        // 合并到缓存
        cache.unite(data);
        
        // LRU淘汰
        while (cache.size() > cacheSize) {
            cache.remove(cache.keys().first());
        }
        
        loadingRows.remove(startRow);
        
        // 通知视图更新
        emit dataChanged(index(startRow, 0), index(endRow, columnCount() - 1));
    }
};
```

```cpp
// pluginmanager.h
class PluginManager : public QObject
{
    Q_OBJECT
    
    QList<IPlugin*> plugins;
    QString pluginsDir;
    
public:
    PluginManager(QObject *parent = nullptr) : QObject(parent) {
        // 插件目录
        pluginsDir = QCoreApplication::applicationDirPath() + "/plugins";
    }
    
    void loadPlugins() {
        QDir dir(pluginsDir);
        
        // 查找所有.dll/.so/.dylib文件
        QStringList filters;
#ifdef Q_OS_WIN
        filters << "*.dll";
#elif defined(Q_OS_MAC)
        filters << "*.dylib";
#else
        filters << "*.so";
#endif
        
        for (const QString &fileName : dir.entryList(filters, QDir::Files)) {
            QString filePath = dir.absoluteFilePath(fileName);
            loadPlugin(filePath);
        }
    }
    
    bool loadPlugin(const QString &filePath) {
        QPluginLoader loader(filePath);
        QObject *instance = loader.instance();
        
        if (!instance) {
            qWarning() << "Failed to load plugin:" << filePath;
            qWarning() << loader.errorString();
            return false;
        }
        
        IPlugin *plugin = qobject_cast<IPlugin*>(instance);
        if (!plugin) {
            qWarning() << "Not a valid plugin:" << filePath;
            delete instance;
            return false;
        }
        
        // 初始化插件
        if (!plugin->initialize()) {
            qWarning() << "Plugin initialization failed:" << plugin->name();
            delete instance;
            return false;
        }
        
        plugins.append(plugin);
        qDebug() << "Loaded plugin:" << plugin->name() << plugin->version();
        
        emit pluginLoaded(plugin);
        return true;
    }
    
    void unloadPlugins() {
        for (IPlugin *plugin : plugins) {
            plugin->shutdown();
            delete plugin;
        }
        plugins.clear();
    }
    
    QList<IPlugin*> getPlugins() const {
        return plugins;
    }
    
signals:
    void pluginLoaded(IPlugin *plugin);
    void pluginUnloaded(IPlugin *plugin);
};
```

```csharp
public interface IEventBus
{
    void Subscribe<T>(Action<T> handler);
    void Unsubscribe<T>(Action<T> handler);
    void Publish<T>(T @event);
}

public class EventBus : IEventBus
{
    private readonly Dictionary<Type, List<Delegate>> _subscribers = new();
    
    public void Subscribe<T>(Action<T> handler)
    {
        var eventType = typeof(T);
        if (!_subscribers.ContainsKey(eventType))
            _subscribers[eventType] = new List<Delegate>();
        
        _subscribers[eventType].Add(handler);
    }
    
    public void Unsubscribe<T>(Action<T> handler)
    {
        var eventType = typeof(T);
        if (_subscribers.ContainsKey(eventType))
            _subscribers[eventType].Remove(handler);
    }
    
    public void Publish<T>(T @event)
    {
        var eventType = typeof(T);
        if (!_subscribers.ContainsKey(eventType))
            return;
        
        foreach (var handler in _subscribers[eventType].Cast<Action<T>>())
        {
            handler(@event);
        }
    }
}
```

```csharp
public class ModelServiceServer
{
    public async Task StartAsync()
    {
        while (true)
        {
            using var server = new NamedPipeServerStream("ModelService",
                PipeDirection.InOut, NamedPipeServerStream.MaxAllowedServerInstances,
                PipeTransmissionMode.Byte, PipeOptions.Asynchronous);
            
            await server.WaitForConnectionAsync();
            
            // 处理请求
            _ = Task.Run(async () =>
            {
                try
                {
                    await ProcessRequestAsync(server);
                }
                finally
                {
                    server.Disconnect();
                }
            });
        }
    }

    {
        // 读取请求
        var request = await JsonSerializer.DeserializeAsync<Request>(stream);
        
        // 处理业务逻辑
        var response = await HandleRequestAsync(request);
        
        // 返回响应
        await JsonSerializer.SerializeAsync(stream, response);
    }
```

```csharp
// 配置重试策略
public class RetryPolicies
{
    // HTTP请求重试（指数退避）
    public static AsyncRetryPolicy<HttpResponseMessage> HttpRetryPolicy = 
        Policy<HttpResponseMessage>
            .Handle<HttpRequestException>()
            .Or<TaskCanceledException>()
            .OrResult(r => !r.IsSuccessStatusCode && r.StatusCode != HttpStatusCode.NotFound)
            .WaitAndRetryAsync(
                retryCount: 3,
                sleepDurationProvider: attempt => TimeSpan.FromSeconds(Math.Pow(2, attempt)),
                onRetry: (outcome, timespan, retryCount, context) =>
                {
                    Logger.Warn($"Retry {retryCount} after {timespan.TotalSeconds}s. " +
                                $"Reason: {outcome.Exception?.Message ?? outcome.Result.StatusCode.ToString()}");
                });
```

### Q20: 如何设计一个离线优先（Offline-First）的桌面应用？

**问题深度**: ⭐⭐⭐⭐⭐

**标准答案**:

**核心理念**: 应用应该在**无网络**的情况下也能正常工作，网络连接是**增强功能**而非必需。

**1. 数据同步策略**:

```csharp
// 本地优先数据模型
public class Model
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string Description { get; set; }
    
    // 同步元数据
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }
    public DateTime? LastSyncedAt { get; set; }
    public SyncStatus SyncStatus { get; set; }  // Local, Synced, Modified, Conflict
    public string Version { get; set; }  // 乐观锁
}

public enum SyncStatus
{
    Local,        // 仅本地，未同步
    Synced,       // 已同步
    Modified,     // 本地修改，待上传
    Conflict      // 冲突，需要用户解决
}

// 数据仓储（本地优先）
public class OfflineFirstModelRepository : IModelRepository
{
    private readonly ILocalDatabase _localDb;      // SQLite
    private readonly IApiClient _apiClient;        // 远程API
    private readonly ISyncQueue _syncQueue;        // 同步队列
    
    // 读取：优先本地
    public async Task<Model> GetByIdAsync(string id)
    {
        // 1. 从本地数据库读取
        var localModel = await _localDb.GetByIdAsync(id);
        if (localModel != null)
            return localModel;
        
        // 2. 如果本地没有且有网络，从远程获取
        if (NetworkHelper.IsOnline())
        {
            try
            {
                var remoteModel = await _apiClient.GetModelAsync(id);
                await _localDb.InsertAsync(remoteModel);
                return remoteModel;
            }
            catch (Exception ex)
            {
                Logger.Warn($"Failed to fetch model {id} from remote", ex);
            }
        }
        
        return null;
    }
    
    // 写入：立即写本地，异步同步远程
    public async Task SaveAsync(Model model)
    {
        model.UpdatedAt = DateTime.UtcNow;
        model.SyncStatus = SyncStatus.Modified;
        
        // 1. 立即写入本地数据库（快速响应）
        await _localDb.UpsertAsync(model);
        
        // 2. 加入同步队列（后台异步同步）
        await _syncQueue.EnqueueAsync(new SyncOperation
        {
            Type = SyncOperationType.Upsert,
            EntityType = typeof(Model),
            EntityId = model.Id,
            Data = model
        });
    }
    
    // 删除：软删除 + 同步队列
    public async Task DeleteAsync(string id)
    {
        // 1. 软删除（标记为已删除，不立即物理删除）
        await _localDb.SoftDeleteAsync(id);
        
        // 2. 加入同步队列
        await _syncQueue.EnqueueAsync(new SyncOperation
        {
            Type = SyncOperationType.Delete,
            EntityType = typeof(Model),
            EntityId = id
        });
    }
}
```

**2. 同步队列**:

```csharp
// 同步操作定义
public class SyncOperation
{
    public Guid Id { get; set; }
    public SyncOperationType Type { get; set; }
    public Type EntityType { get; set; }
    public string EntityId { get; set; }
    public object Data { get; set; }
    public int RetryCount { get; set; }
    public DateTime CreatedAt { get; set; }
}

public enum SyncOperationType
{
    Upsert,
    Delete
}

// 同步队列实现
public class SyncQueue : ISyncQueue
{
    private readonly ILocalDatabase _localDb;
    private readonly IApiClient _apiClient;
    private readonly Queue<SyncOperation> _queue = new();
    private bool _isSyncing;
    
    public async Task EnqueueAsync(SyncOperation operation)
    {
        // 持久化到本地数据库（防止应用崩溃丢失）
        await _localDb.InsertSyncOperationAsync(operation);
        
        _queue.Enqueue(operation);
        
        // 触发同步
        _ = Task.Run(() => ProcessQueueAsync());
    }
    
    private async Task ProcessQueueAsync()
    {
        if (_isSyncing) return;
        
        _isSyncing = true;
        
        try
        {
            while (_queue.TryDequeue(out var operation))
            {
                // 检查网络
                if (!NetworkHelper.IsOnline())
                {
                    Logger.Info("Offline, pausing sync");
                    _queue.Enqueue(operation);  // 放回队列
                    break;
                }
                
                // 执行同步
                bool success = await ExecuteSyncOperationAsync(operation);
                
                if (success)
                {
                    // 删除已同步的操作
                    await _localDb.DeleteSyncOperationAsync(operation.Id);
                    
                    // 更新实体同步状态
                    await UpdateEntitySyncStatusAsync(operation.EntityId, SyncStatus.Synced);
                }
                else
                {
                    // 重试
                    operation.RetryCount++;
                    if (operation.RetryCount < 5)
                    {
                        _queue.Enqueue(operation);
                        await Task.Delay(TimeSpan.FromSeconds(Math.Pow(2, operation.RetryCount)));
                    }
                    else
                    {
                        Logger.Error($"Sync operation failed after 5 retries: {operation.Id}");
                        await _localDb.MarkSyncOperationAsFailedAsync(operation.Id);
                    }
                }
            }
        }
        finally
        {
            _isSyncing = false;
        }
    }
    
    private async Task<bool> ExecuteSyncOperationAsync(SyncOperation operation)
    {
        try
        {
            switch (operation.Type)
            {
                case SyncOperationType.Upsert:
                    await _apiClient.UpsertModelAsync((Model)operation.Data);
                    break;
                    
                case SyncOperationType.Delete:
                    await _apiClient.DeleteModelAsync(operation.EntityId);
                    break;
            }
            
            return true;
        }
        catch (Exception ex)
        {
            Logger.Error($"Sync operation failed: {operation.Id}", ex);
            return false;
        }
    }
    
    // 应用启动时恢复未完成的同步操作
    public async Task RestoreQueueAsync()
    {
        var pendingOperations = await _localDb.GetPendingSyncOperationsAsync();
        foreach (var op in pendingOperations)
        {
            _queue.Enqueue(op);
        }
        
        _ = Task.Run(() => ProcessQueueAsync());
    }
}
```

```csharp
// 冲突检测
public class ConflictDetector
{
    public async Task<ConflictResolution> DetectConflictAsync(Model local, Model remote)
    {
        // 1. 如果版本号相同，无冲突
        if (local.Version == remote.Version)
            return ConflictResolution.NoConflict;
        
        // 2. 如果本地未修改，接受远程
        if (local.SyncStatus == SyncStatus.Synced)
            return ConflictResolution.AcceptRemote;
        
        // 3. 如果远程未修改，接受本地
        if (remote.UpdatedAt <= local.LastSyncedAt)
            return ConflictResolution.AcceptLocal;
        
        // 4. 双方都修改，标记为冲突
        return ConflictResolution.Conflict;
    }
}

// 冲突解决策略
public enum ConflictResolutionStrategy
{
    AcceptLocal,         // 使用本地版本
    AcceptRemote,        // 使用远程版本
    Merge,               // 合并两个版本
    Manual               // 手动解决
}

// 自动合并（针对非冲突字段）
public class ConflictResolver
{
    public Model AutoMerge(Model local, Model remote, Model base)
    {
        var merged = new Model { Id = local.Id };
        
        // 三路合并（three-way merge）
        merged.Name = MergeField(local.Name, remote.Name, base.Name);
        merged.Description = MergeField(local.Description, remote.Description, base.Description);
        
        // 更新元数据
        merged.UpdatedAt = DateTime.UtcNow;
        merged.Version = Guid.NewGuid().ToString();
        merged.SyncStatus = SyncStatus.Synced;
        
        return merged;
    }
    
    private string MergeField(string local, string remote, string base)
    {
        // 如果本地未改，使用远程
        if (local == base)
            return remote;
        
        // 如果远程未改，使用本地
        if (remote == base)
            return local;
        
        // 如果双方都改且相同，使用任意一个
        if (local == remote)
            return local;
        
        // 双方都改且不同，返回null标记为冲突
        return null;
    }
}
```

```csharp
public class NetworkMonitor
{
    private bool _isOnline;
    
    public bool IsOnline
    {
        get => _isOnline;
        private set
        {
            if (_isOnline != value)
            {
                _isOnline = value;
                OnlineStatusChanged?.Invoke(this, value);
                
                if (value)
                {
                    Logger.Info("Network available, resuming sync");
                    _ = _syncQueue.ProcessQueueAsync();
                }
                else
                {
                    Logger.Warn("Network unavailable, entering offline mode");
                }
            }
        }
    }
    
    public event EventHandler<bool> OnlineStatusChanged;
    
    public void StartMonitoring()
    {
        // 监听网络变化
        NetworkChange.NetworkAvailabilityChanged += (s, e) =>
        {
            IsOnline = e.IsAvailable && CanReachServer();
        };
        
        // 定期检测（防止误判）
        var timer = new System.Timers.Timer(30000);  // 30秒
        timer.Elapsed += (s, e) =>
        {
            IsOnline = NetworkInterface.GetIsNetworkAvailable() && CanReachServer();
        };
        timer.Start();
    }
    
    private bool CanReachServer()
    {
        try
        {
            using var client = new HttpClient { Timeout = TimeSpan.FromSeconds(5) };
            var response = await client.GetAsync("https://api.example.com/health");
            return response.IsSuccessStatusCode;
        }
        catch
        {
            return false;
        }
    }
}
```
