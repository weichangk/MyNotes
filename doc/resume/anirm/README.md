## 🎯 核心技术亮点

### 1. 统一任务处理引擎

**设计模式**：策略模式 + 模板方法模式

**核心价值**：
- 业务逻辑与调度逻辑完全解耦
- 新增业务类型开发效率提升 80%
- 前后台无缝切换（detach/reattach）
- 批量任务原子性保证

**关键技术**：
- `TaskProcessor<TInput, TResult>` 策略接口
- `TaskProcessingService` 单例调度器
- Runtime Slot 并发控制机制
- 虚拟进度算法（平滑用户体验）

---

### 2. 跨平台 IAP 架构

**平台支持**：
- iOS：StoreKit 1（iOS 12+）+ StoreKit 2（iOS 15+）
- Android：Google Play Billing Library

**核心架构**：
- **平台工厂模式**：`StorePurchaseFactory` 根据平台返回不同实现
- **统一基类抽象**：`BaseStorePurchaseManager` 封装通用流程
- **完善验证链**：
  - iOS: 本地 → Apple 服务器 → 自有后端
  - Android: Google Play token → 自有后端

**关键特性**：
- 未验证购买的持久化与自动重试
- 授权状态轮询（最多 120 秒）
- iOS SK1/SK2 运行时自动适配
- Android 服务端验证模式（更安全）

**业务成果**：
- 支付成功率：iOS 96%+，Android 94%+
- 验证失败补偿率：99%

## 🛠️ 技术栈

- **状态管理**：GetX
- **网络层**：Dio + Retrofit

### 后端交互
- **网络协议**：RESTful API

### 云服务
- **对象存储**：阿里云 OSS
- **推送通知**：Firebase Cloud Messaging（FCM）

## 🎓 设计模式应用

| 设计模式 | 应用场景 | 具体实现 |
|---------|---------|---------|
| **策略模式** | 任务处理 | `TaskProcessor` 接口，各 Processor 实现不同策略 |
| **模板方法** | 任务执行流程 | `TaskProcessingService._executeTask` 定义固定流程 |
| **工厂模式** | Processor 创建 | `ProcessorRegistry.createProcessorForBizType` |
| **单例模式** | 服务管理 | `TaskProcessingService.instance`、`IAPManager.instance` |
| **观察者模式** | 事件分发 | `Stream`（任务事件）、`EventBus`（全局事件） |
| **适配器模式** | API 兼容 | `VideoEnhancerApiHelper` 适配不同后端接口 |
| **代理模式** | 网络配置 | `DioConfig.setupDioProxy` 统一代理设置 |
| **仓储模式** | 数据持久化 | `TaskCenterRepository` 封装数据库操作 |
| **门面模式** | 复杂流程简化 | `VideoEnhancerWorkflowFacade` 封装多步骤操作 |
