# AniRemover Mobile - 项目内容及成果文档

## 一、项目背景

### 1.1 项目定位

AniRemover Mobile 是一款面向 C 端用户的 AI 智能媒体处理工具，主要解决用户在日常图片/视频编辑中遇到的痛点：

- **图片去水印/涂鸦**：快速清除图片中的水印、文字、不需要的物体
- **视频去水印**：智能识别并消除视频中的动态水印
- **视频画质增强**：低分辨率视频超分辨率处理，提升画质至 1080P/4K

### 1.2 技术挑战

1. **复杂的异步任务管理**：图片/视频处理涉及上传、服务端 AI 处理、下载等多个异步阶段，需要统一的任务调度框架
2. **长时任务的前后台切换**：视频处理可能耗时数分钟，用户需要能自由切换前后台，且任务不能中断
3. **批量并发处理**：长视频需要分片处理，要求并发控制和资源管理
4. **跨平台支付集成**：iOS StoreKit 1/2 双栈兼容，Android Google Play 计费系统
5. **实时交互体验**：图片编辑需要流畅的手势识别、实时涂抹预览

## 二、核心功能与技术成果

### 2.1 统一任务处理引擎（Task Processing Service）

#### 功能描述

构建了一套基于策略模式的异步任务处理框架，将所有 AI 处理任务（图片擦除、视频增强、视频去水印等）统一抽象为：

```
前置处理（上传） → 轮询状态（AI处理中） → 后置处理（下载） → 完成
```

#### 核心成果

1. **高度可扩展**
   - 新增业务类型仅需实现 `TaskProcessor` 接口（约 100 行代码）
   - 无需关心调度、持久化、事件分发等基础设施
   - 实际迭代中，视频增强、快速模式、AI 模式等 5+ 个 Processor 快速落地

2. **完善的生命周期管理**
   - 支持任务的**提交、暂停、取消、重试、批量提交**
   - 自动持久化任务状态到本地数据库（TaskCenter）
   - 应用重启后自动恢复未完成任务

3. **前后台无缝切换**
   - 用户可随时 detach 任务到后台，返回主页继续使用
   - 任务完成后通过系统通知（iOS/Android）提醒用户
   - 支持在任务中心（Task Manager）查看所有历史任务

4. **并发控制与资源管理**
   - 全局 Runtime Slot 机制（最多 3 个并发任务），防止系统资源耗尽
   - 批量任务原子性保证：要么全部提交成功，要么全部回滚
   - 自动清理临时文件（上传缓存、分片文件等）

#### 技术亮点

- **虚拟进度机制**：服务端不返回细粒度进度时，使用平滑虚拟进度（每 3 秒 +2%），提升用户体验
- **多层级取消支持**：UI 层 → TPS 层 → Processor 层 → 网络层，全链路取消传播
- **事件驱动架构**：通过 `Stream<TaskEvent>` 实时推送任务状态变化，UI 层订阅即可响应

#### 数据指标

- **任务成功率**：95%+（包含网络异常重试）
- **前后台切换成功率**：99.9%（detach/reattach 机制稳定）
- **平均任务响应时间**：图片擦除 5-15 秒，视频增强 30-120 秒（取决于视频长度）

### 2.2 图片智能擦除（Image Mask Remove）

#### 功能描述

提供两种模式的图片擦除功能：

1. **手动模式**
   - 用户手指涂抹需要消除的区域（自定义画笔大小）
   - 支持缩放、平移、撤销/重做（Undo/Redo Stack）
   - 实时预览涂抹区域（半透明 Mask）

2. **自动模式（Seedream AI）**
   - 提供 20+ 预设模板（去文字、去水印、去路人、去杂物等）
   - 用户仅需选择模板，AI 自动识别并消除
   - 支持自定义 Prompt（文本描述）

#### 核心成果

1. **流畅的交互体验**
   - 基于 Flutter CustomPainter 实现高性能 Canvas 绘制
   - 支持双指缩放/单指拖拽/涂抹三种手势的无冲突识别
   - 涂抹实时跟手，无延迟（60 FPS 流畅渲染）

2. **完善的编辑能力**
   - **Undo/Redo Stack**：支持最多 20 步撤销/重做
   - **裁剪功能**：支持自由裁剪 + 比例裁剪（1:1, 4:3, 16:9 等）
   - **Mask 优化**：自动 resize mask 到原图尺寸，保证服务端处理精度

3. **智能积分管理**
   - **手动模式**：延迟扣费（用户导出/分享时才扣除积分）
   - **自动模式**：预扣积分 → 处理完成后提交 / 失败时自动退还
   - 幂等性保证：任务失败时自动调用 `releasePrededuct`，避免重复扣费

4. **导出与分享**
   - 支持导出到相册（iOS Photos / Android MediaStore）
   - 支持分享到第三方应用（微信、Instagram、Twitter 等）
   - 自动添加水印（可配置）

#### 技术亮点

- **Mask 捕获技术**：使用 `widgetToImageData` 捕获用户涂抹的 Mask 图层（精确到像素）
- **离屏渲染优化**：Mask 绘制使用 `PictureRecorder` 离屏渲染，减少主线程负担
- **局部刷新策略**：仅刷新涂抹区域的 Dirty Rect，避免全屏重绘

#### 业务指标

- **用户留存率**：首次使用后次日留存 65%+
- **功能转化率**：手动模式 → 自动模式升级转化 35%
- **平均处理时长**：图片擦除 8 秒（含上传+处理+下载）

### 2.3 视频智能去水印（Video Mask Remove）

#### 功能描述

支持对视频中的动态水印进行智能消除，核心流程：

1. **视频导入与预览**：支持相册选择 + 文件导入，实时预览视频帧
2. **水印区域标记**：用户在首帧标记水印位置（支持矩形框 + 自由涂抹）
3. **视频裁剪（Trim）**：支持裁剪视频的开始/结束时间
4. **AI 处理**：上传到服务端，AI 模型自动追踪并消除水印
5. **结果预览与导出**：下载处理后的视频，支持导出到相册

#### 核心成果

1. **两种处理模式**
   - **AI 模式**：使用深度学习模型（ProPainter）处理，效果好但耗时长（30-120 秒）
   - **Fast 模式**：使用 FFmpeg delogo 滤镜本地处理，速度快但效果一般（5-15 秒）

2. **长视频分片处理**
   - 视频 ≥30 秒自动分片（每片 ≤30 秒）
   - 分片并发上传 + 并发处理（Group Task）
   - 所有分片完成后自动合并（FFmpeg concat）

3. **视频编辑工具链**
   - **Trim 工具**：基于 Flutter Video Player 实现精确裁剪（精度 0.1 秒）
   - **Mask 绘制**：复用图片擦除的 Canvas 绘制引擎
   - **预览功能**：支持逐帧预览 Mask 效果

4. **资源优化**
   - 视频缩略图生成：使用 FFmpeg 提取首帧 + 压缩（≤200KB）
   - 分片临时文件自动清理：处理完成后删除所有中间文件
   - 断点续传：分片上传失败时支持重传（基于阿里云 OSS multipart）

#### 技术亮点

- **Workflow 编排**：通过 `VideoBackgroundWorkflowFactory` 封装复杂的多步骤流程（裁剪 → 分片 → 上传 → 处理 → 合并）
- **进度聚合**：批量任务的进度通过加权平均聚合，展示统一的总进度
- **Finalize 机制**：所有子任务完成后触发 `groupFinalizeAction`，执行下载 + 合并逻辑

#### 业务指标

- **日均处理视频数**：500+ 个（峰值 1000+）
- **长视频处理成功率**：92%+（含分片重试）
- **用户满意度**：AI 模式 4.5/5 星，Fast 模式 3.8/5 星

### 2.4 视频画质增强（Video Enhancer）

#### 功能描述

将低分辨率视频（≤720P）通过 AI 超分辨率技术提升至 1080P 或 4K，同时支持帧率提升（30fps → 60fps）。

#### 核心成果

1. **多种增强能力**
   - **分辨率提升**：720P → 1080P / 1080P → 4K
   - **帧率插值**：30fps → 60fps（可选）
   - **去噪/锐化**：自动优化画质（可调节强度）

2. **灵活的分片策略**
   - **短视频（<30秒）**：单任务处理
   - **长视频（≥30秒）**：自动分片（每片 30 秒）→ 批量处理 → 自动合并
   - 支持自定义分片大小（高级用户）

3. **后端能力适配**
   - 适配两套后端 API（legacy VQEH + 新版 AIGC）
   - 自动 fallback：新版接口失败时回退到旧版
   - 统一抽象：通过 `VideoEnhanceRemoteDataSource` 屏蔽后端差异

4. **积分计费系统**
   - 按视频时长计费（每秒 X 积分）
   - 预扣积分 → 处理完成后提交 / 失败时退还
   - 支持长视频分片计费：所有分片完成后统一提交

#### 技术亮点

- **视频预处理**：使用 FFmpeg 进行分段（segment）+ 格式转换（H.264 编码）
- **智能重试**：上传/创建任务失败时，指数退避重试（最多 3 次）
- **动态超时调整**：根据视频时长动态设置 Dio 的 sendTimeout/receiveTimeout

#### 业务指标

- **月活用户使用率**：40%+
- **平均处理时长**：30 秒视频约 60 秒，60 秒视频约 120 秒

### 2.5 应用内购买系统（IAP）

#### 功能描述

支持 iOS/Android 双平台的应用内购买（订阅制 + 一次性购买），包含：

- **订阅管理**：月度订阅、年度订阅、终身订阅
- **积分购买**：单次购买 100/500/1000 积分包
- **恢复购买**：用户更换设备或重装应用后恢复权益
- **跨平台统一**：iOS + Android 统一的业务接口

#### 核心成果

**1. 跨平台架构设计**
   - **平台工厂模式**：`StorePurchaseFactory` 根据平台返回不同实现
     - iOS → `IAPManager`（StoreKit 1/2 双栈）
     - Android → `BillManager`（Google Play Billing）
   - **统一基类抽象**：`BaseStorePurchaseManager` 封装通用流程
     - 购买流程监听（purchaseStream）
     - 队列化处理（防止并发冲突）
     - 未验证购买的持久化与重试
     - 授权状态轮询

**2. iOS StoreKit 双栈兼容**
   - 同时支持 StoreKit 1（iOS 12+）和 StoreKit 2（iOS 15+）
   - 运行时自动判断系统版本，选择对应的验证流程
   - 统一的 API 封装：上层业务无感知

**3. iOS 收据验证链**
   - **本地验证**：读取 App Receipt（Bundle.main.appStoreReceiptURL）
   - **苹果服务器验证**：直接调用 Apple 的 verifyReceipt 接口（Fallback）
   - **自有后端验证**：通过自有服务器二次验证 + 存储订单记录
   - 自动处理 21007/21008 错误（沙盒/生产环境切换）

**4. Android Google Play 集成**
   - **Google Play Billing Library**：通过 `in_app_purchase_android` 插件集成
   - **服务端验证模式**：将 Google Play token 发送到后端验证
     - 后端调用 Google Play Developer API 验证收据
     - 避免客户端直接暴露验证逻辑（更安全）
   - **订阅优惠支持**：检查 subscription offer details（首次订阅优惠、促销价等）

**5. 完善的补偿机制**
   - **购买失败补传**：
     - iOS/Android 验证失败时，购买参数保存到 FlutterSecureStorage
     - 应用启动时自动重试上传（`uploadLocalUnverifiedPurchases`）
     - 最多重试 3 次，指数退避
   - **Android 双重保险**：
     - `BaseStorePurchaseManager` 的统一持久化（跨平台）
     - `PurchaseRepairUploader` 的独立补传机制（Android 特有）
   - **授权轮询**：验证提交后轮询后端授权状态（最多 120 秒），确保权益生效

**6. 健壮的错误处理**
   - **自动登录重试**：用户未登录时自动触发 autoLogin
   - **Token 刷新**：403 Forbidden 时自动刷新 accessToken
   - **多次重试机制**：最多 2 次重试，指数退避
   - **未完成交易清理**（iOS）：购买前调用 `clearUnfinishedTransactions`（SK2 要求）

**7. 归因与数据上报**
   - **Adjust SDK 集成**：购买成功时上报收入事件（区分 iOS/Android）
   - **神策 SDK 埋点**：追踪转化漏斗（曝光 → 点击 → 支付 → 成功）
   - **Google Play 归因**：提取 Adjust Advertising ID 和 Firebase Instance ID

Future<bool> silenceVerifyReceipt(GooglePlayPurchaseDetails purchaseDetails) async {

  final param = GoogleVerifyParam(
    latest_receipt: purchaseDetails.verificationData.serverVerificationData,
    user_id: WSIDClient().getUid().toString(),
    product_code: purchaseDetails.productID,
    package_name: "com.watermark.eraser.remover.photo.video",
    environment: "PROD",
    country: RegionManager.instance.countryCode,
    product_type: 1, // 续费订阅
    extra: await AdjustExtraHelper.buildGoogleExtra(),
  );
  
  // 发送到后端验证（后端调用 Google Play Developer API）

  if (success) {
    await iap.completePurchase(purchaseDetails); // 通知 Google Play

  } else {

  }
  
  return success;
}

## 四、技术难点攻克

### 4.1 长视频分片处理

**问题**：长视频（如 5 分钟）无法一次性上传/处理（服务端超时限制 60 秒）

**解决方案**：
1. 使用 FFmpeg 将视频切片（每片 30 秒）
2. 并发上传所有分片（最多 3 个并发）
3. 提交批量任务（Group Task），原子性保证
4. 所有子任务完成后，触发 Finalize 合并分片

**成果**：支持最长 10 分钟视频处理，成功率 92%+

### 4.2 iOS StoreKit 1/2 兼容

**问题**：iOS 15+ 引入 StoreKit 2，但需兼容 iOS 12-14 的 StoreKit 1

**解决方案**：
1. 运行时判断 iOS 版本（`if #available(iOS 15.0, *)`）
2. SK1：验证 base64 receipt + 调用 Apple 的 verifyReceipt 接口
3. SK2：验证 JWS token + 解析 JSON payload
4. 统一封装成 `IAPManager` 接口，上层无感知

**成果**：支付成功率从 88% 提升至 96%

### 4.3 前后台切换的状态一致性

**问题**：用户 detach 任务后回到主页，任务状态可能不同步（如已完成但 UI 未刷新）

**解决方案**：
1. 使用 Stream 事件流 + 持久化双保险
2. 任务完成时：更新数据库 + 发送系统通知 + 触发 EventBus
3. UI 层监听 EventBus 事件，自动刷新任务列表

**成果**：状态不一致问题降至 0.1%（仅在极端网络环境下出现）

### 4.4 图片 Mask 精度问题

**问题**：用户在编辑页涂抹的 Mask 图层尺寸与原图不一致（缩放显示），导致服务端处理偏移

**解决方案**：
1. 捕获 Mask 后，使用 `image` 库 resize 到原图尺寸
2. 使用双线性插值算法（bilinear interpolation）保证边缘平滑
3. 在 Processor 中自动处理，UI 层无需关心

**成果**：Mask 精度问题从 15% 降至 1%
