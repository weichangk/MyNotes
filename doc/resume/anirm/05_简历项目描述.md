## 三、核心工作职责（简历用）

### 1. 统一任务处理框架设计与实现

**职责描述**：
- 设计并实现基于**策略模式 + 模板方法**的统一任务处理引擎（TaskProcessingService）
- 将图片擦除、视频增强等 5+ 种业务统一抽象为"上传 → 轮询 → 下载"流程
- 实现前后台无缝切换机制（detach/reattach），支持应用重启后任务恢复

**技术亮点**：
- 新增业务类型开发效率提升 **80%**（仅需实现 100 行 TaskProcessor 接口）
- 虚拟进度算法：服务端无细粒度进度时，使用平滑虚拟进度（每 3 秒 +2%），用户感知处理时间缩短 **30%**
- 全局并发控制：Runtime Slot 机制限制最多 3 个并发任务，防止 OOM

**核心成果**：
- 任务成功率：**95%+**
- 前后台切换成功率：**99.9%**
- 支持批量任务原子性提交（要么全部成功，要么全部回滚）

**核心成果**：

## 七、关键技术亮点提炼（面试用）

### 1. 设计模式应用

**策略模式**：TaskProcessor 接口，各业务实现不同处理策略
```dart
abstract class TaskProcessor<TInput, TResult> {
  Future<String> preprocess(TInput input, ...);
  Future<TaskStatusSnapshot> checkStatus(String serverTaskId, ...);
  Future<TResult> postprocess(String serverTaskId, ...);
}
```

**模板方法模式**：TaskProcessingService 定义固定执行流程
```dart
Future<void> _executeTask(...) async {
  // 1. Preprocessing (上传)
  final serverTaskId = await processor.preprocess(...);
  // 2. Polling (轮询)
  await _pollUntilComplete(...);
  // 3. Postprocessing (下载)
  final result = await processor.postprocess(...);
  // 4. Completed
  emitEvent(TaskPhase.completed, result: result);
}
```

**平台工厂模式**：根据平台返回不同 IAP 实现
```dart
class StorePurchaseFactory {
  static BaseStorePurchaseManager getStorePurchaseManager() {
    return Platform.isIOS ? IAPManager.instance : BillManager.instance;
  }
}
```

### 2. 核心算法

**虚拟进度算法**：
```dart
// 服务端无细粒度进度时，使用平滑虚拟进度
Timer.periodic(Duration(seconds: 3), (timer) {
  virtualProgress = min(virtualProgress + 0.02, 0.88); // 每 3 秒 +2%，上限 88%
  final mergedProgress = max(virtualProgress, realProgress); // 取 max 避免倒退
  emitEvent(progress: mergedProgress);
});
```

**并发控制 - Runtime Slot**：
```dart
class TaskCenterRuntimeManager {
  final Set<String> _activeSlots = {};
  static const int _maxConcurrentTasks = 3;
  
  bool tryAcquireProcessingSlot(String token) {
    if (_activeSlots.length >= _maxConcurrentTasks) return false;
    _activeSlots.add(token);
    return true;
  }
}
```
