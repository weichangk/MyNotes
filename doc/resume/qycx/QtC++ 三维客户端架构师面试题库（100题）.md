# 1. Qt 的事件循环机制是什么？

------

# 标准回答

Qt 的事件循环本质是：

```
消息队列 + 事件分发
```

应用启动后：

```
QApplication::exec()
```

进入主事件循环。

Qt 会不断：

```
取消息
→ 转换成QEvent
→ 分发给QObject
→ 调用event()
```

底层依赖：

- Windows Message Loop
- epoll/select
- 平台事件系统

------

# 核心流程

```
OS消息
↓
QAbstractEventDispatcher
↓
QEventLoop
↓
QObject::event()
↓
mousePressEvent()
paintEvent()
timerEvent()
```

------

# 加分点

## deleteLater 为什么安全？

因为：

```
deleteLater 会发送 DeferredDelete 事件
```

对象不会立刻销毁。

而是在：

```
事件循环安全点
```

统一回收。

避免：

- 野指针
- 事件处理中析构

------

# 面试陷阱

很多人会只回答：

```
事件循环就是 while(true)
```

太浅。

必须讲：

- Dispatcher
- 消息队列
- QEvent
- 平台消息系统
