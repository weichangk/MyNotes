# UniConverter — Windows 综合多媒体处理软件

**技术栈：** C#、WinForms、.NET Framework、AgileTrans 转码引擎、OpenVINO、Inno Setup

---

## 项目介绍

UniConverter（原名 VideoConverter Ultimate）是万兴科技旗下面向 Windows 平台内容创作者的旗舰级综合多媒体处理软件，覆盖视频格式转换、压缩、下载、录制、编辑以及 AI 智能工具（水印去除、视频增强、图像增强、智能总结、语音转文字、人声提取、噪声消除等）全套能力，持续迭代至 16.x 版本。

项目使用 **C# + WinForms** 开发，采用分层架构（View + Controller + Model + 公共服务层 + 原生引擎层）：
- UI 层基于公司自研的 **WUL（Wondershare UI Library）** 皮肤控件框架，所有功能页面继承 `BasePage` 基类，工具子模块采用 **MVC-ish 分层**（Controller / Model / View / Component 四级目录）
- 底层通过 **DllImport / COM** 对接 AgileTrans 转码引擎、ffmpeg 等原生 C/C++ 库
- AI 功能支持**本地 SDK（OpenVINO 推理）与云端 API 双路执行**策略，按 GPU 能力动态调度
- 项目规模约 **80+ 子工程**，通过 CommonModule 以源码链接方式实现公共能力复用，支持国际版 / 国内版（`#if ZH` 编译宏）多变体构建

## 项目内容

### 一、云端 AI 任务处理框架（核心基础架构）

设计并实现贯穿多个 AI 功能模块的通用云端任务处理链路，作为全产品 AI 功能的统一执行基础设施。

- **抽象化通用流程**：封装「上传阿里云 OSS → 创建 AI 任务 → 轮询状态（排队/处理中/完成/失败）→ 下载结果」完整流程；以 `AIProcess` 抽象基类 + `IAPITaskParams` 参数接口为扩展点，各 AI 业务模块通过继承覆写 API 端点、请求参数、响应解析实现差异化接入，无需重复实现通用链路
- **统一状态语义**：轮询状态码语义（1=排队 / 2=处理中 / 3=成功 / 4~6=失败/取消/超时）在各模块间保持一致，便于统一的错误处理与任务上报；`AliOssHelper` 统一管理凭证获取与重试策略
- **多模型串联优化**：多模型串联时通过传递上一模型返回的 OSS 路径直接作为下一模型输入，省去重复上传开销，显著提升多阶段 AI 处理效率
- **效果**：显著降低 AI 功能接入成本，保障多模块任务调度的一致性与可扩展性

---

### 二、视频增强模块

基于 MVC 分层架构，支持 18 种 AI 增强模型，覆盖去频闪、视频稳像、智能调色、超分辨率、人像/动漫增强、降噪、帧插值等多类画质优化场景。

- **本地 / 云端双路执行**：通过检测 GPU SM 架构与驱动状态，在运行时动态分流——有 GPU 时优先走本地 SDK 加速，云端专属模型（去频闪、超分辨率、智能调色等）及无 GPU 时的人脸/动漫增强切换至云端路径
- **三阶段流水线编排**：云端处理由 Controller 编排三阶段流水线（视频切片 → 增强处理 → 合并输出），三阶段进度按 10%/80%/10% 加权呈现，用户感知连续平滑进度
- **多模型串联零上传开销**：多模型串联时复用云端 AI 任务处理链路，通过 OSS 路径透传省去中间结果的下载再上传开销
- **效果**：显著提升低质量素材的可修复能力，拓展内容创作者与专业用户的使用深度

---

### 三、图片增强模块

基于 MVC 分层架构，支持自动增强、AI 超分（2x/4x/8x）、旧照片修复/上色、人脸增强、色彩校正、NanaBanana 系列去模糊/光线修复等 **12 种 AI 模型**，全程云端处理。

- **批量并发处理**：复用云端 AI 任务处理链路，支持批量并发处理，由统一进度管理器协调多任务状态
- **预览缓存机制**：引入免费预览缓存机制，相同参数不重复提交请求，降低重复请求开销，提升用户预览体验
- **多模型串联**：多模型串联时上一模型返回的 OSS 路径直接作为下一模型输入，省去中间结果传输开销
- **效果**：覆盖修复、上色、降噪、超分等多类图像优化场景，满足内容创作者对老旧照片翻新与图像质量提升的核心诉求

---

### 四、智能总结模块

基于视频下载和 STT 基础，结合 DeepSeek 扩展 AI 总结能力，构建「音视频转录 → 字幕编辑 → AI 总结 → 导出」完整闭环。

- **端到端处理链路**：转录阶段音频分离后复用云端 AI 任务处理链路提交 STT 服务，字幕结果由总结模块进行分段、说话人标注编辑并支持多格式导出，覆盖学习与轻办公场景
- **结构化内容输出**：将碎片化音视频内容转化为可检索、可编辑的结构化摘要，有效降低用户信息提取成本
- **效果**：实现从碎片视频到高价值内容摘要的一键转化，提升用户信息处理效率

---

### 五、水印去除模块

支持图片与视频两种类型，提供手动与自动两种去除模式，本地与云端双路并行。

- **双模式设计**：手动模式由用户画笔框选生成 mask 文件（离线处理）；自动模式接入 **Seedream 4.5**，细分为文字水印、Logo 标记、人物三类场景，两种模式均复用云端 AI 任务处理链路
- **热更新 Prompt 策略**：AI prompt 策略支持热更新，通过代码默认值 → 本地配置文件 → 远端下发三级优先级动态覆盖，无需发版即可调整各场景的去除效果
- **本地 SDK 按需安装**：本地处理 SDK 在首次使用时由 SDK 管理器按需自动下载安装，供手动模式离线处理使用
- **效果**：覆盖文字、Logo、人物等多类水印场景，结合本地与云端双路策略，满足不同网络环境与隐私需求

---

### 六、录制模块

基于 MVC 分层架构对录制模块进行全面重构，涵盖屏幕录制、摄像头录制、音频录制三种模式。

- **COM 工厂接口封装**：通过 COM 工厂对象对接底层采集引擎接口，统一封装各录制模式的生命周期管理与参数控制
- **能力扩展**：新增游戏应用录制模式、摄像头镜像、鼠标样式与点击动效、麦克风实时降噪等专业录制能力，提升录制场景覆盖广度与专业度
- **效果：活跃用户数显著增长 16.28%，转化率大幅提升 37.48%**

---

### 七、任务调度系统（TaskManager）

设计并实现视频转换核心任务调度模块，负责编码任务的队列管理、并发控制与状态驱动。

- **四层继承架构**：接口（`ITaskItem`）→ 抽象基类（`TaskItem`）→ 逻辑主体（`MediaTaskItem`）→ 执行载体（`EncoderTaskItem`），职责分层清晰
- **工厂方法 + 策略模式**：`MediaTaskItem` 构造时按媒体类型动态选择 `EncoderTaskItem` 子类（DRM 解密 / 多文件合并 / iTunes M4V 进程通信），新增媒体类型只需扩展子类，不修改调度逻辑
- **状态机驱动调度**：`TaskItemState` 七态状态机（None → Waiting → Running → Paused / Finished / Failed / Cancelled），100ms WinForms Timer 轮询调度（UI 线程天然串行，无需额外加锁），最大并行任务数为 2
- **进度回调链**：`MediaBatchEncoder.OnEncodeProgress → EncoderTaskItem → MediaTaskItem → UI` 四层事件链，引擎层变更不影响 UI 刷新逻辑
- **线程安全保护**：`runningTasksCount`（多线程写）通过 `lock(runningTaskCountObj)` 保护；`drmRuningTaskCount`（静态跨实例共享）独立锁保护；调度逻辑利用 UI 线程 Timer Tick 天然互斥，避免过度加锁

---

### 八、安装包优化（安装速度专项）

主导安装包性能专项优化，针对 Inno Setup 5 单线程文件提取瓶颈设计并实施 **7z 多线程 SubPack 方案**。

- **问题根因分析**：通过对比测试排除 lzma 压缩级别调整（节省 25s，非主要瓶颈）与升级 Inno Setup 6（反而慢 2min5s）等方向，确认 IS5 运行时逐文件串行 I/O 是核心瓶颈
- **SubPack 分包架构**：将所有安装文件预先打成两个 `.7z` 压缩包（主程序包 SubPack1 → `{app}` / 公共资源包 SubPack2 → `{commonappdata}\Wondershare`），IS 安装脚本仅负责释放 7z.exe 和 SubPack 文件（4 个文件），完全绕过 IS5 单线程文件提取逻辑
- **多线程解压**：安装时通过 `AfterInstall` 回调调用 `7z.exe x -mmt=on` 多线程解压，充分利用多核 CPU 和高速 SSD I/O
- **构建流程集成**：`Build_7z_SubPacks.bat` 构建脚本支持 CBS / Full 双模式，分两个 Pass 采集文件后一次性压缩（`-mx=1 -mmt=on`），与 `WsVideoConvertUltimateBuild.bat` 主构建流程自动集成
- **效果**：安装速度显著提升，解决长期存在的安装体验短板

---

### 九、工程化与稳定性保障

- **多变体构建体系**：维护国际版 / 国内版（`#if ZH`）、CBS / Full / Steam / AI 等多构建配置，通过编译宏统一管理产品名称、ProductId、注册密钥、追踪 ID、支付渠道等核心常量差异
- **崩溃分析与线上质量**：基于 BugSplat 崩溃上报（注册 `Application.ThreadException` + `AppDomain.UnhandledException` 双钩子），日常收集崩溃日志定位多线程竞争、内存问题等复杂缺陷，持续响应用户反馈，保障线上稳定性
- **进程间通信**：通过 `WM_COPYDATA` 实现 Mutex 防多开激活已有窗口，多进程协作（录制进程、下载服务进程、iTunes 转换进程等）通过 `Process + 标准输入/输出流 / 自定义消息` 实现跨进程通信与生命周期管理
- **多语言 / 皮肤系统**：WUL 框架统一管理皮肤与多语言（`ML.GetText(key, defaultValue)`），支持运行时语言切换，国际版支持 10+ 语言
