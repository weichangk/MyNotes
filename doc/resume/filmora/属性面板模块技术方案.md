## 一、整体架构设计

### 1.1 模块职责

属性面板（`FPropertyPanelView`）是 Filmora 中专门用于编辑时间线 Clip 属性的右侧面板，承担以下职责：

- **属性展示**：根据选中的 Clip 类型，动态显示对应的属性组（变换、颜色、特效、动画、音频等）
- **属性编辑**：提供滑条、颜色选择器、下拉框、开关等丰富控件
- **关键帧管理**：在每个属性行旁边显示关键帧菱形按钮，支持添加/删除/跳转关键帧
- **批量编辑**：多选 Clip 时，展示聚合值（多值提示"—"），写入时批量更新
- **Undo/Redo 集成**：属性写入自动包装为 UndoRedo 操作，支持 Ctrl+Z 撤销

### 1.2 核心类一览

| 类名 | 职责 |
|------|------|
| `FPropertyView` | 属性面板 UI 根节点，管理 Tab 切换和属性组列表 |
| `FPropertyViewPrivate` | Pimpl 数据类，持有所有成员 |
| `FPropertyControlManager` | 事件总线监听 + 各 Control 生命周期管理 |
| `FPropertyControl` | 单个属性页的控制器（视频/颜色/字幕等） |
| `FPropertySelector` | 时间线 Clip 选择状态的二次封装 |
| `FPropertyWidgetFactory` | Widget 建造者工厂，注册/查找 Builder |
| `FPropertyBaseWidgetBuilder` | 工厂 Builder 基类，描述 Widget 元数据 |
| `FPropertyGroup` | 可折叠的属性组容器（含标题 + Body） |
| `FPropertyGroupHeader` | 属性组头部（开关/标题/关键帧/重置） |
| `FPropertyBaseWidget` | 属性体 Widget 基类，实现 Observer 接口 |
| `FPropertyComponent` | 单行属性组件（标签 + 关键帧按钮 + 输入控件） |
| `IFPropertyPresenter` | Presenter 抽象基类，控制 Component 的读写 |

| `FPropertyBatchSetter<T>` | 批量写入模板，包装 Undo/Pause 宏 |

## 二、控制器层（Control + Manager）

### 2.1 FPropertyControlManager — 事件总线监听者

`FPropertyControlManager` 是整个属性面板的核心调度器，同时实现了 5 个 Observer 接口：

```cpp
class FPropertyControlManager : public QObject,
    public IFFTimelineEventObserver,      // 时间线数据变化（Clip 增删、属性修改）
    public IFFMediaEditorEventObserver,   // 视觉编辑器状态变化（选中点移动）
    public IFFPlayheadEventObserver,      // 播放头位置变化
    public IFFUndoRedoServiceEventObserver, // Undo/Redo 操作完成
    public IFFProjectManagerEventObserver   // 工程切换/关闭
{
    // 收到时间线变化事件 → 更新当前 Clip → 通知各 Control 刷新 UI
    void onTimelineChangedEvent(const FFTimelineChangedDataList& dataList) override;

    // 收到播放头移动事件 → 通知 Control 刷新关键帧选中状态
    void onPlayheadPropertyChangedEvent(IFFPlayhead*, FFPlayheadPropertyType) override;

    // 收到 Undo/Redo 完成事件 → 全量刷新属性面板
    void onUndoRedoChangedEvent(FFVBLMODEL::FFDataChangedMode mode) override;
};
```

**Control 注册机制**（静态注册宏，类似 Command 模式）：

```
FPropertyControl（基类）
```

```cpp
class FPropertyControl : public QObject {
```

## 五、属性体（FPropertyBaseWidget + FPropertyComponent）

### 5.1 FPropertyBaseWidget — 属性体基类

每个具体属性面板（如变换、颜色调整、绿幕等）均继承 `FPropertyBaseWidget`，同时实现 `IFPropertyControlObserver` 接口：

```cpp
class FPropertyBaseWidget : public QWidget, public IFPropertyControlObserver {
public:
    // ── 初始化（Builder 调用）──
    virtual void init(const FPropertyInitData& data);

    // ── Observer 回调（由 FPropertyControl 驱动）──
    void onPropertyUpdateDataEvent(UpdateReasons reasons) final; // 派发给 refreshData()
    void onPlayheadChangedEvent() override;   // 播放头移动 → 更新关键帧选中态
    void onKeyFrameUpdateUIEvent() override;  // 关键帧增减 → 更新关键帧图标
    void resetData(bool bHasSwitch) override; // 重置所有属性到默认值

    // ── 子类实现 ──
    virtual void refreshData(UpdateReasons reasons);  // 从 VBL 读取最新值刷新 UI
    virtual void restoreDefaultValue();               // 还原各控件的默认值
    virtual void initPropertyInfos();                 // 注册关键帧相关的 EffectInfo
    virtual bool isModify() const;                    // 是否有属性被修改（影响重置按钮可用性）
    virtual FPropertyUsed getSwitchState() const;     // 返回开关状态（开/关/部分）

protected:
    FFPropertyType          m_type;          // 所属 Tab 类型（视频/颜色/音频/...）
    FFPropertyGroupType     m_groupType;     // 所属属性组类型
    FPropertyControl*       m_helper;        // 关联的 Control（属性读写入口）
    FFVBLObjectRefPtr<IFFEffect> m_pEffect;  // 关联的特效对象（特效 Group 专用）
    QList<FEffectPropertyBaseInfo> m_oPropertyInfoList;  // 开关关联的 Info 列表
    QList<FEffectPropertyBaseInfo> m_oKeyFrameInfoList;  // 关键帧关联的 Info 列表
    FPropertySelector* m_selector;          // 选中 Clip 的引用
};
```

### 5.2 UpdateReasons — 刷新原因位标志

`refreshData()` 接收 `UpdateReasons` 位标志，子类按需过滤，避免不必要的 VBL 调用：

```cpp
enum {
    URNoReasons       = 0x0000,
    URFirstRefresh    = 1 << 0,  // 首次初始化
    URClipChange      = 1 << 1,  // 选中 Clip 变化（必须全量刷新）
    UREnableChange    = 1 << 2,  // 特效启用/禁用变化
    URRefreshAll      = URClipChange | UREnableChange | URVisibleChange,

    URPlayHeadMoved   = 1 << 4,  // 播放头移动（需刷新关键帧当前值）
    URPropertyChanged = 1 << 5,  // 属性值改变（通常只需更新单个控件）
    URKeyFrameAdded   = 1 << 6,  // 关键帧添加
    URKeyFrameRemoved = 1 << 7,  // 关键帧删除

    URRippleBuildDone = 1 << 8,  // 音频涟漪构建完成
    UREffectEnableChange = 1 << 9, // 特效开关变化
    URUndoRedo        = 1 << 13, // Undo/Redo 完成

    URAllReason       = 0xffffffff
};
```

## 六、Presenter 层（MVP 模式）

### 6.1 Presenter 接口体系

属性面板的 MVP 层中，`FPropertyComponent` 是 View，`IFPropertyPresenter` 系列是 Presenter，数据读写通过 `FPropertyControl` 转发到 VBL：

```
Presenter 接口继承树：

IPresenterBase（QObject 基类）
  └── IFPropertyPresenter（基础 Presenter）
```

## 七、批量编辑系统

### 7.1 多选聚合读取

多选 Clip 时，属性面板显示聚合值：若所有 Clip 值相同则显示具体值，否则显示"—"：

```cpp
// FMultipleValue<T>：多值封装
template<typename T>
struct FMultipleValue {
    T    value;       // 代表值（取第一个 Clip 的值）
    bool isMultiple;  // true = 各 Clip 值不同，UI 显示"—"
    bool isValid;     // false = 无有效 Clip
};

// FPropertyBatchGetter：通用批量读取模板
template<typename Target, typename Value>
struct FPropertyBatchGetter {
    static FMultipleValue<Value> get(
        const QList<Target>& targets,
        const std::function<Value(Target)>& readFunc)
    {
        auto value = readFunc(targets.first());
        for (int i = 1; i < targets.size(); i++) {
            auto cur = readFunc(targets[i]);
            if (value != cur)
                return FMultipleValue<Value>(value, /*isMultiple=*/true);
        }
        return FMultipleValue<Value>(value, false); // 所有 Clip 值一致
    }
};
```

浮点数比较使用 `qFuzzyCompare` 避免精度问题：

```cpp
if constexpr (std::is_floating_point<Value>::value) {
    if (!qFuzzyCompare(static_cast<float>(currentValue),
                       static_cast<float>(value)))
        return FMultipleValue<Value>(value, true);
}
```

### 7.2 批量写入与 Undo 合并

批量写入通过 `FPropertyBatchSetter` 模板，自动包装 Undo 和 Pause 宏，将 N 次写入合并为 1 个 Undo 操作：

```cpp
template<typename Target>
struct FPropertyBatchSetter {
    static bool set(
        IFFUndoRedoService* undoRedoService,
        const QList<Target>& targets,
        const std::function<void(Target)>& setFunc,
        const QString& debugDescription = "")
    {
        // 创建 UndoRedo 操作边界（作用域结束时自动提交）
        FF_DECLARE_UNDO_COMMANDER(undoRedoService, debugDescription);
        // 暂停 VBL Observer 通知（作用域结束时发一次 BATCH 通知）
        FF_DECLARE_PAUSE_EVENT_COMMANDER(undoRedoService);

        for (auto target : targets) {
            setFunc(target);  // 逐一写入各 Clip
        }
        return true;
    }
};
```

**与时间线批量更新的协同**：`FF_DECLARE_PAUSE_EVENT_COMMANDER` 与时间线中同名宏共用同一套机制，确保无论哪层触发批量更新，底层 VBL 只发一次 BATCH 事件通知。

### 7.3 CF_MultEdit 标志

Builder 在 `PropertyFlag` 中声明 `prMultEdit` 表示该属性组支持多选编辑；`FPropertyBaseWidget` 通过 `isMultSelect()` 判断当前是否为多选模式并据此调整 UI 行为（如禁用某些高级选项）。

## 十六、面试高频技术点

### 16.1 Presenter 与接口隔离

**问题**：属性面板中滑条、颜色选择器、下拉框等控件如何与 VBL 数据解耦？

**答**：通过 `IFPropertyPresenter` 系列纯虚接口层实现。`FPropertyComponent` 只依赖 `IFPropertySliderPresenter`，不感知底层是修改位置、速度还是特效参数。同一种 Component（如 `FPropertySliderComponent`）可复用于数十种属性，`Presenter` 负责具体的 VBL 读写。

### 16.2 多选批量编辑

**问题**：多选 10 个 Clip 后拖动亮度滑条，如何保证 10 次写入合并为 1 个 Undo？

**答**：`FPropertyBatchSetter` 模板用 `FF_DECLARE_UNDO_COMMANDER`（作用域结束时合并提交 Undo）+ `FF_DECLARE_PAUSE_EVENT_COMMANDER`（暂停中间 VBL 通知）双 RAII 宏包装，所有写入完成后发一次 BATCH 事件。

### 16.3 工厂注册模式

**问题**：100+ 个属性面板 Widget 如何管理？`FPropertyView` 如何知道该显示哪些面板？

**答**：`FPropertyWidgetFactories` 全局工厂 + `ExecuteBeforeMain` 静态注册宏。每个 Widget 对应一个 Builder，在 `.cpp` 中通过 `RegisterPropertyBuilderByType(MyBuilder, pptNormal)` 完成静态注册。`FPropertyView` 只查询工厂，完全不依赖具体 Widget 类，符合开闭原则。

### 16.4 刷新原因过滤（性能优化）

**问题**：时间线数据变化频繁，属性面板如何避免不必要的 VBL 调用？

**答**：`refreshData(UpdateReasons)` 接收位标志参数。子类仅在关心的 flag 被设置时才执行对应刷新逻辑，例如：
- 仅 `URPlayHeadMoved`：只更新关键帧选中状态，不读属性值
- 仅 `URKeyFrameAdded`：只刷新关键帧菱形按钮状态
- `URClipChange`（Clip 切换）：必须全量重读所有属性值

### 16.5 三段式 Presenter（拖动预览）

**问题**：滑条拖动时如何实现实时预览但只产生 1 个 Undo 记录？

**答**：`IFPropertySliderPresenter` 定义三个生命周期方法：
1. `setValueBegin(value)` — 鼠标按下，记录起始值
2. `setValue(value)` — 鼠标拖动，直接写 VBL（不进 Undo 栈），触发实时预览
3. `setValueEnd()` — 鼠标松开，将起始值 → 最终值包装为一个 Undo 操作提交

### 16.6 `FPropertyControl::blockUpdateVBLData`

**问题**：属性面板复杂初始化时（如首次打开 Clip），如何避免初始化期间的 UI 回调堆积？

**答**：`blockUpdateVBLData(true)` 设置 `m_bCanUpdateVBLData = false`，配合 `F_CONTROL_BlOCK_GUARD(control)` RAII 宏，在作用域结束时自动解除 Block。Block 期间的 VBL 写入被忽略，解除后统一触发一次全量刷新。
